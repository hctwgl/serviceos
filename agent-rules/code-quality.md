# 代码质量、错误处理与兼容规则

在编写复杂业务、事务、并发、SQL、重试、安全逻辑，或引入兼容/清理路径时读取。

## 唯一权威路径

ServiceOS 是新系统。默认直接修正模型、契约、迁移、测试和调用方，不为假设中的旧客户端或脏数据增加永久兼容。

禁止：

- 长期保留新旧字段、状态、接口、模型或持久化双轨；
- 缺值、非法值或未知状态时静默补默认值或猜测语义；
- 吞异常后返回空对象、空集合或伪造成功；
- 无真实兼容对象的 `legacy`、`compat`、`fallback`、`temporary` 分支；
- 宽松解析、忽略未知字段或生产测试兜底；
- “先兼容以后再删”但没有期限、负责人和退出证据。

真实外部系统、已发布客户端或生产数据确需兼容时，必须有 ADR、明确对象、边界适配层、负责人、截止日期、删除里程碑、迁移/监控/回退方案和自动化证据。

## 结构与表达

- 优先用清晰结构表达意图，不用注释掩盖超长方法或错误分层；
- Controller 只做协议适配，Application Service 编排用例和事务，Domain 维护规则与不变量，Infrastructure 隔离技术实现；
- 公共 API/SPI、领域命令/事件、复杂 Repository 和重要配置使用中文 Javadoc 或等价说明；
- getter、简单赋值和自解释代码不要求机械注释；
- TODO/FIXME 写明原因、责任范围和删除条件；
- 修改逻辑时同步修正失效注释和错误提示。

## 中文注释和提示

复杂业务规则、事务、并发、状态迁移、SQL、重试、安全和跨模块协作必须说明：

- 为什么这样实现；
- 保护什么不变量；
- 幂等键、锁顺序和事务边界；
- 失败会产生什么结果；
- 如何重试、恢复或人工接管。

不受外部协议约束的日志、异常、校验、审计和操作提示优先使用准确、可检索、可定位的中文。错误码、Schema、事件类型、指标名、Trace 属性、数据库对象名和第三方协议保持稳定英文标识。

## 失败关闭

- 非法输入、未知状态、配置零命中/多命中、权限不足和并发冲突必须返回明确失败；
- 不把 UNKNOWN 当成功；
- 不因通知、审计或事件失败而伪造业务完成；
- 自动任务达到重试上限后进入人工接管，不无限重试；
- 高风险人工操作必须记录原因和审批依据。

## 清理要求

- 新权威路径生效后删除被替代字段、分支、适配器、配置和测试；
- 一次性迁移完成后删除临时默认值、双写和回退查询；
- 不混入与当前目标无关的大规模格式化、依赖升级或重命名；
- 最终报告明确兼容路径、清理结果和仍保留的临时结构。
