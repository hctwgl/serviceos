# 契约、安全与文件规则

仅在任务涉及 HTTP API、事件、外部接入、身份授权、敏感数据、文件或公开兼容时读取。

## HTTP 与事件契约

- OpenAPI 是 HTTP 契约事实源，JSON Schema 是事件契约事实源；
- 修改 Controller、DTO、公开命令或事件时同步机器契约和契约测试；
- 已发布契约不得原地破坏字段类型、必填性或语义；破坏性纠正必须获得明确批准并版本化；
- 事件只包含稳定标识和必要快照，不暴露实体对象图、内部表结构或车企 DTO；
- 生产者与消费者必须定义重复、迟到、乱序和未知版本语义；
- 业务写入与 Outbox 同事务，消费者通过 Inbox 或等价机制幂等。

## 身份与授权

- 身份来自 OIDC/JWT，授权基于 Capability 与 Tenant/Project Scope；
- 不信任客户端提供的 tenant、project、operator、角色或权限结果；
- 前端隐藏按钮不能替代服务端授权；
- 高风险命令必须记录操作者、原因、审批引用、影响对象和拒绝审计；
- 跨租户、跨项目、越权下载或绕过状态机均为阻断问题。

## 外部车企接入

- 必须验签、校验时间窗、防重放并定义业务幂等；
- transport replay 与业务键冲突分别处理；
- 车企 DTO 只能存在于 Anti-Corruption Layer；
- 外部网络调用不在持锁长事务中；
- UNKNOWN 不得自动当成功、自动重发或静默降级；
- 凭据不得进入仓库、镜像、日志、Trace、测试快照或普通查询 API。

## 文件与敏感数据

- 文件生命周期保持 `Begin -> 受限上传 -> Finalize -> 隔离 -> 扫描 -> 授权下载`；
- 禁止未扫描下载、公共暴露私有文件、长期保存大文件正文或永久对象存储 URL；
- 原始外部报文、签名、密钥、身份证、手机号、地址等必须最小化暴露并脱敏；
- 日志、Trace、指标、异常和审计不得泄漏 Secret 或不必要的个人信息；
- 下载必须重新校验实时授权和 tenant/project scope。

## 最低验证

- Controller/OpenAPI：MVC/安全测试 + OpenAPI 校验；
- Event Schema/消费者：Schema/兼容门禁 + Inbox 幂等测试；
- 授权/Scope：允许与拒绝路径 + 拒绝审计；
- 文件：上传、扫描、下载授权和越权负例；
- 外部集成：验签、防重放、幂等、超时/UNKNOWN、重试恢复和敏感输出门禁；
- 修改公开模块边界时运行 `ArchitectureTest`。
