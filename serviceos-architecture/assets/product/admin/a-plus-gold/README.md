## Admin 视觉合同

| 项目 | 金标口径 |
|---|---|
| 侧栏 | 白色，约 176px；细分隔线；选中项使用淡蓝底和企业蓝文字 |
| 顶栏 | 白色，约 54px；项目范围、全局搜索、通知、待办、帮助和用户菜单 |
| 工作区 | 极浅蓝灰背景，桌面页面边距约 20px |
| 主色 | 企业蓝；只用于选中、主操作和关键链接 |
| 卡片 | 白色，1px 浅灰边框，6～8px 圆角，默认不用重阴影 |
| 正文 | 12～14px；页面标题约 23px；高密度但保持明确分组 |
| 工单摘要 | 六列业务摘要；不得显示 UUID、英文枚举、原始命令或 JSON |
| 工单主体 | 主内容区 + 约 276px 右侧上下文栏 |
| 进度 | 单行阶段进度；完成、当前和未开始状态清晰区分 |
| 当前任务 | 左侧任务事实，右侧允许操作；动作来自服务端权限与状态 |
| 业务页签 | 基本信息、任务、预约上门、表单资料、审核整改、外部回传、操作日志 |
| 派单 | 约 520px 右侧抽屉；只展示服务端确认的合格候选及推荐依据 |

## 验收规则

每个黄金链路页面必须使用真实后端和统一产品场景数据，在 Chrome 中人工完成正常、空、错误、
无权限、只读和冲突状态。1440 × 900 与金标并排检查，1280 验证响应式。截图仅附在 PR 审查中，
不再保存为仓库测试基线，也不建立 Playwright 或视觉 E2E。
