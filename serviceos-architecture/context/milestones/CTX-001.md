# CTX-001 Agent 上下文路由与检索成本优化

## 目标

- 将 Agent 启动入口收敛到轻量基线、当前 Context Pack 和直接模块卡片；
- 根据里程碑和 Git diff 输出最小阅读集合、影响模块和验证提示；
- 把根 `AGENTS.md` 的详细规则拆为按需加载文件；
- 建立本地会话状态缓存，避免跨会话重复分析未变化文件；
- 在 CI 中验证上下文工具和引用完整性；
- 修复 CI 新 Runner 暴露的纳秒系统时间与 PostgreSQL 微秒精度不一致，保证首次结果与幂等重放完全相等。

## 明确非目标

- 不修改业务领域模型、状态机、数据库结构或公开契约；
- 不删除历史里程碑文档和实施证据；
- 不引入向量数据库、外部搜索服务或新的运行时依赖；
- 不自动决定业务下一里程碑；
- 不降低 PostgreSQL、契约、安全或 Spring Modulith 门禁。

- M1～M60 的逐里程碑实现正文。

## 已冻结决策

- Context Pack 只做路由，不复制业务语义；
- 当前轻量快照不替代实施状态和工程证据；
- 历史文档保留，但不作为每轮任务的默认全文输入；
- 影响分析只给出最低风险建议，安全、状态机、事务和契约变化仍需人工升级；
- 本地会话缓存必须被 Git 忽略，不得成为共享事实源；
- 脚本只依赖 Bash、Git、awk、sed 和 grep；
- 业务时间在进入 PostgreSQL 事务前统一到微秒刻度，不通过放宽重放断言掩盖精度差异。
