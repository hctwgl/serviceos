# example 模块卡片

## 事实所有权

- 列出本模块唯一拥有的聚合、运行时事实或技术职责。

## 公开边界

- 公开 API/SPI/事件路径；
- 禁止其他模块直接访问的内部包和表。

## 必读事实源

- 权威架构文档；
- Accepted ADR；
- OpenAPI、事件 Schema 或 Flyway 入口。

## 代码入口

- 生产代码根路径；
- 数据库迁移目录；
- 关键消费者或适配器。

## 核心测试

- 精准单元测试模式；
- PostgreSQL IT 模式；
- 必要的契约、安全或 ArchitectureTest。

## 相邻模块

- 上游；
- 下游；
- 只有哪些变化才需要展开阅读相邻模块。

## 不变量

- 只列路由所需的稳定不变量摘要；
- 完整语义仍引用权威文档，不在卡片中复制。

## 扩大检索触发条件

- 公共 API 或事件变化；
- 数据所有权变化；
- 状态机、事务、授权或兼容语义变化；
- 当前文档与代码证据冲突。
