# ADR-006：流程引擎只负责编排，不拥有业务事实

- 状态：Proposed
- 日期：2026-07-12

## 背景

长周期履约需要等待、定时器、并行、重试和恢复能力，但把资料、审核、预约和金额全部存入流程变量会造成事实重复、查询困难和引擎锁定。

## 决策

流程引擎负责编排执行位置、网关、等待事件和技术作业；业务模块拥有工单、任务、资料、审核、派单、履约事实和结算数据。二者通过稳定业务标识、适配器和领域事件协作。

## 约束

- 流程引擎不得直接更新业务表；
- 业务代码不得读取引擎内部表；
- 流程变量只保存标识和小型路由数据；
- 业务事件通过幂等关联推进流程；
- 业务自动任务重试由任务模块唯一调度，流程引擎只重试自身技术作业；
- 引擎替换不得改变领域 API 和事件语义。

## 后果

增加适配器和最终一致性处理，但降低引擎耦合，使业务事实可查询、可审计并支持后续替换流程技术。
