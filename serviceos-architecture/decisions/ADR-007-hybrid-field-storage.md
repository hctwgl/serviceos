# ADR-007：标准领域字段与动态提交采用混合存储

- 状态：Proposed
- 日期：2026-07-12

## 背景

平台预计存在约 500 个跨车企字段。全部建成固定列会频繁改表，全部使用 EAV/JSON 又会削弱类型、查询、权限和结算可靠性。

## 决策

业务身份、关系、生命周期、高频查询和核心规则字段使用明确类型化领域模型；项目扩展字段保存为通过发布 Schema 校验的不可变 FormSubmission。需要报表、派单或结算的扩展值提取为受控投影或 FulfillmentFact。

## 约束

- 标准字段目录提供稳定语义和类型；
- JSON/JSONB 不允许无 Schema 自由写入；
- 核心事实不能只存在动态表单；
- 价格规则只读取履约事实；
- 不使用一张通用 EAV 表承载全部业务数据；
- 不为单个车企建立专属核心宽表。

## 后果

需要维护字段目录、映射和提取过程，但兼顾配置扩展、类型安全、查询性能和历史可追溯。
