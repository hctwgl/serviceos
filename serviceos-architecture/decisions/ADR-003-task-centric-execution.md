# ADR-003：采用任务中心的履约执行模型

- 状态：Proposed
- 日期：2026-07-12

## 背景

单一工单状态无法准确表达勘测、安装、资料、审核、回传、整改和结算的并行、等待与异常，也无法稳定承载责任人、SLA 和动作权限。

## 决策

工单作为履约容器；阶段用于业务汇总；任务作为实际执行、责任和 SLA 单元；领域事件记录已发生事实。工单可保留面向列表查询的状态投影，但该投影必须可由任务和阶段重建。

## 后果

- 可以统一支持预约、审核、整改、回传和人工异常处理；
- 可明确到任务级的责任、超时和操作权限；
- 页面和报表必须理解“工单汇总投影”和“任务真实状态”的差别；
- 需要防止把每一个按钮都过度建模成独立任务。
