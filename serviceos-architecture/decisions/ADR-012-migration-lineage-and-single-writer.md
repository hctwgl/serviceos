# ADR-012：迁移保留血缘且每张工单单一写入权威

- 状态：Proposed
- 日期：2026-07-13

## 背景

旧系统数据存在覆盖历史、脏字段和文件缺失。在双轨期间，如果同一工单可以被两个系统修改和产生副作用，将无法判断最终状态并可能重复派单、通知、回传或结算。

## 决策

迁移以不可变 SourceSnapshot、版本化转换、IdMapping 和 MigrationLineage 记录来源；每个工单/创建路由键通过 CutoverCohort 获得唯一 authoritySystem。影子系统只计算和比较，SideEffectFence 阻止真实副作用。

## 约束

- 迁移重跑幂等；
- 不伪造旧系统不存在的领域事件或证据；
- 在途只迁移到批准恢复点；
- 已锁定历史金额不重算覆盖；
- cohort 改变默认只影响新工单；
- 回退先同步权威增量，不能简单恢复旧备份；
- 无法安全反向同步时禁止回退并向前修复。

## 后果

需要暂存区、血缘、路由和副作用 fence，但获得可审计迁移、确定权威和可控灰度/回退。
