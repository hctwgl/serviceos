# M383 项目履约配置产品审查

## 1. 审查结论

M378～M382 已形成有价值的纵向业务主链路：项目履约 Profile、不可变 Revision、校验、预览、发布、建单解析、工单冻结与配置快照均已有真实后端能力和前端入口。

但当前前端状态只能标记为：

- 技术：`RUNTIME_CONNECTED`；
- 前端：`FRONTEND_CONNECTED`；
- 产品：`PRODUCT_REJECTED`；
- 质量：功能与 a11y 部分通过，视觉未批准。

不得标记 `PRODUCT_ACCEPTED` 或“项目工单履约配置产品化完成”。

产品负责人已批准项目履约配置、工作流设计器和任务模板中心采用 **方案 A｜经典专业风**。批准目标详见 [经典专业风视觉与页面基线](12-classic-professional-visual-baseline.md)。这表示目标产品设计已明确，但当前实现仍需按本审查整改后重新验收。

## 2. 保留的实现

- ProjectFulfillmentProfile / Revision / Manifest / Resolver 领域能力；
- 草稿、校验、发布和生效时间；
- 新旧工单配置版本冻结；
- 工单配置快照 API；
- blockedActions 与表单/资料阻塞原因；
- 项目详情进入项目履约配置中心的产品路径；
- 独立列表、详情、编辑、预览、发布和快照路由；
- 生成的 `@serviceos/core-client` 接入；
- 专用发布流程页面而非普通 Modal。

## 3. 已批准目标页面结构

### 3.1 项目履约配置中心

```text
PageHeader
→ SummaryStrip：项目 / 已发布版本 / 草稿 / 工单类型数 / 最近发布 / 使用中工单
→ 左侧二级配置导航
→ 配置包概览
→ 工单类型配置表
→ 右侧配置说明 / 影响分析 / 版本时间线
```

左侧二级导航至少包含：基础信息、工单类型、工作流、任务模板、表单模板、资料要求、SLA、派单、通知、集成映射和发布记录。

### 3.2 工作流设计器

```text
左栏：流程目录 + 节点组件
中间：可视化 Workflow 画布 + 校验结果
右栏：节点配置
```

普通模式通过中文业务对象配置 Task、角色、分配、表单、资料、SLA、前置条件、完成动作和异常出口。

### 3.3 任务模板中心

```text
SummaryStrip
→ 分类树
→ 模板表格
→ 右侧模板详情
```

必须展示执行角色、分配策略、表单、资料、SLA、允许动作、升级策略、版本和 Workflow 引用关系。

## 4. P0 产品问题

### 4.1 新增工单类型不是产品流程 — **M385 已关闭**

已提供独立 FormPage 向导：选择工单类型、名称/说明、标准/复制/空白起始方式。

### 4.2 发布流程暴露 Manifest JSON — **M385 已关闭**

发布第二步渲染服务端 Runbook；`manifestJson` 仅进入开发诊断抽屉。

### 4.3 运行预览仍由前端解析 Manifest — **M385 已关闭**

`compile-preview` 返回产品化 `runbook`；前端只渲染 DTO。

### 4.4 编辑器仍以技术文档结构为中心 — **仍开放（M385b）**

编辑页仍读写 `documentJson`。M385 仅做动作门禁与诊断隔离；结构化 Draft 留后续切片。

### 4.5 发布“影响分析”不是实际差异 — **M385 已关闭**

`GET .../compare-impact` 返回真实差异与影响；配置中心右轨与发布第三步消费该 DTO。

### 4.6 操作没有统一使用 allowed-actions — **M385 已关闭（基础）**

详情/编辑/发布主操作按 Profile `allowedActions` 显隐。阻塞原因结构化仍可增强。

### 4.7 缺少 Task 模板产品模型 — **仍开放（M387）**

## 5. P1 产品问题

- 配置详情通过解析草稿 JSON 展示阶段，而不是使用结构化只读 DTO；
- 责任类型、目标阶段、SLA 引用仍可能显示英文或内部引用；
- 发布版本列表缺少版本状态、版本说明、差异入口和当前生效标识；
- 工单配置快照只显示数量，不能解释“为什么需要某项资料、为什么动作被阻塞”；
- 工单概览新增独立“配置来源”卡片，但应并入摘要/配置来源 Tab，避免继续堆卡片；
- 错误反馈缺少统一问题编号；
- 缺少只读、无权限、冲突恢复、已暂停、已停用等完整产品状态；
- 缺少经典专业风 1440×1024 与 1280px 的真实实现视觉审查证据；
- 左侧导航、页面头、概览条、表格和右侧说明区尚未形成统一视觉层级；
- 当前前端仍可能把配置对象拆成多个技术页面，缺少项目级统一配置工作区。
