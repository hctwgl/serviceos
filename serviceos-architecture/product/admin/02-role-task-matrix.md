# Admin 角色与任务矩阵

角色用于定义默认首页、业务语言和常见任务，不直接替代 Capability、Scope 或 allowed-actions。

## 1. 角色总览

| 角色 | 默认入口 | 主要目标 | 高频对象 | 不应默认获得 |
|---|---|---|---|---|
| 平台超级管理员 | 系统管理概览 | 管理平台基础设置和全局治理 | 用户、组织、角色、菜单、权限 | 日常业务强制处理权 |
| 运营管理员 | 运营工作台 | 协调全局履约、风险和异常 | 工单、SLA、异常、项目 | 技术配置直接编辑权 |
| 工单运营 | 我的待办/工单中心 | 初审、补充、分配、暂停、取消、跟踪 | 工单、客户、项目、网点 | 跨范围数据和配置发布权 |
| 调度人员 | 调度工作台 | 分网点、分师傅、改派、预约协调 | 任务、网点、师傅、产能 | 审核强制通过权 |
| 审核人员 | 审核队列 | 审核勘测/安装资料、整改和复审 | 审核单、资料、整改 | 修改现场事实权 |
| 项目配置管理员 | 项目管理 | 维护项目、工单类型和履约方案 | 项目、流程、表单、资料、SLA | 系统全局权限治理权 |
| 权限管理员 | 用户与权限 | 配置用户、角色、菜单和数据范围 | 用户、组织、角色、授权 | 业务工单强制操作权 |
| 审计人员 | 审计与监控 | 查询操作、配置、登录和版本追溯 | 日志、事件、配置版本 | 编辑和业务执行权 |

## 3. 角色首页

### 3.1 平台超级管理员

优先展示：安全风险、未配置管理员、权限变更、登录异常、系统服务状态。不要把日常工单待办作为默认中心。

### 3.2 运营管理员

优先展示：待处理总量、即将超时、已超时、重大异常、跨项目风险和最近处理。

### 3.3 工单运营

优先展示：待初审、待分网点、缺少信息、暂停待处理、本人最近处理。

### 3.4 调度人员

优先展示：待分网点、待分师傅、改派申请、预约冲突、网点/师傅负载风险。

### 3.5 审核人员

优先展示：待审核、即将超时、整改复审、资料机器校验失败和本人已处理。

### 3.6 项目配置管理员

优先展示：项目配置健康、草稿、校验失败、待发布、即将生效和最近配置变更。

### 3.7 权限管理员

优先展示：待处理授权、即将失效委托、无角色用户、高风险权限和最近权限变更。

### 3.8 审计人员

优先展示：高风险操作、配置发布、登录异常、数据导出和诊断事件。

## 4. 角色切换与权限表达

- 同一账号拥有多个职责时，可以切换工作视角，但不得切换成新的授权事实。
- 页面不可见、只读和动作不可执行必须区分。
- 动作被阻塞时优先显示业务原因；权限原因使用“当前账号无权执行此操作”，不显示 raw capability。
- 用户、角色、菜单、权限和数据范围页面面向权限管理员提供中文业务名称，技术编码作为次级信息或复制项。
