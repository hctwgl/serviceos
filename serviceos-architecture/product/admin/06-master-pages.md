# Admin 六张母版页面与配置子母版

六张母版用于冻结整个 Admin 的产品语言、信息层级、布局密度和交互模式。母版通过产品负责人截图验收前，不应以其为视觉金标批量迁移其他页面。

Admin 的视觉和页面结构必须遵守 [经典专业风视觉与页面基线](12-classic-professional-visual-baseline.md)。其中项目履约配置中心、工作流设计器、任务模板中心和工单详情的概念方向已经由产品负责人批准，但实际实现仍需逐页验收。

## 1. 母版一：运营工作台

### 页面目标

回答：我今天必须处理什么、什么即将或已经超时、哪里出现重大异常、最近处理了什么。

### 页面结构

```text
PageHeader：运营工作台 + 当前范围 + 更新时间
风险摘要：仅 P0/P1 或明确 SLA 风险
主区：我的待办（占主要宽度）
侧区：即将超时 / 已超时 / 重大异常
底部：最近处理 / 关注项目
```

### 核心内容

- 我的待办：对象、任务、项目、当前责任、截止时间、风险、操作；
- 即将超时：剩余时间和责任人；
- 已超时：超时时长、当前阻塞和升级状态；
- 重大异常：严重度、对象、发生时间、当前处理人；
- 最近处理：返回上次页面并恢复来源上下文。

### 禁止

- 大量同宽 KPI 卡片；
- 累计总量与当前待办混为同一语义；
- 技术队列名称；
- 无法下钻的数字。

### 验收

运营人员在 10 秒内能找到最紧急的三件事并打开处理页面。

## 2. 母版二：用户管理

### 页面目标

完成用户查询、新建、编辑、启停、组织归属、角色分配和安全状态查看。

### 列表页

筛选：姓名/账号关键词、所属组织、用户状态、角色。

列：姓名、登录账号、所属组织、角色摘要、状态、最近登录、更新时间、操作。

页面操作：新建用户。

行操作：查看、编辑；更多中放启用/停用、分配角色、查看登录记录。

### 用户详情

Tabs：基本信息、组织归属、角色与权限、授权与委托、登录与安全、变更记录。

主要文案使用业务名称；principalId、角色 ID、capability code 只作为次级诊断信息。

### 新建/编辑

字段：姓名、登录账号、手机号/邮箱（按实际契约）、所属组织、初始角色、账号状态。角色使用实体选择器，不输入 ID。

### 验收

权限管理员不阅读任何技术 ID，即可创建用户、归属组织、分配角色并确认最终数据范围摘要。

## 3. 母版三：项目管理

### 页面目标

管理车企项目，查看项目服务范围、合作网点、履约方案和版本。

### 项目列表

筛选：关键词、所属车企、项目状态、生效状态。

列：项目名称、项目编码、所属车企、状态、生效日期、服务区域、合作网点、已发布方案、草稿方案、更新时间、操作。

页面操作：新建项目。

创建表单进入独立 FormPage，不常驻列表上方。

### 项目详情

摘要：项目名称、编码、车企、状态、生效区间、服务区域数量、合作网点数量、工单类型数量、配置健康、最近发布。

Tabs：项目概览、工单类型与履约配置、服务范围、合作网点、车企接口与映射、版本与变更。

“工单类型与履约配置”必须直接进入项目履约配置中心，不得保留“请使用配置设计器”之类空壳。

### 验收

项目管理员进入项目后能明确看到项目支持哪些工单类型、每种类型使用什么方案、是否存在草稿或校验错误。

## 4. 母版四：工单中心

### 页面目标

统一查询授权范围内工单，快速识别当前阶段、责任、SLA 和风险。

### 筛选

默认：工单状态、车企、项目、关键词、更多筛选。

更多筛选：服务区域、网点、师傅、阶段、任务状态、SLA 风险、审核/整改状态、创建时间。

关键词支持工单编号、客户、手机号后四位或地址（以真实查询能力为准）。

### 表格

列：工单编号、状态、当前阶段、当前任务、车企/项目、客户（脱敏）、服务区域、当前责任人、网点/师傅、SLA、异常摘要、更新时间、操作。

主标识可点击；行操作以“查看详情”为主，低风险快捷动作按 allowed-actions 显示。

### 保存视图

待初审、待分配、待预约、上门中、待审核、整改中、已完成等作为服务端受控预设视图；用户保存视图放工具栏。

### 验收

运营人员不打开详情即可判断工单当前由谁负责、当前 Task 是什么、是否超时、下一步大致是什么。

## 5. 母版五：工单详情

### 页面目标

围绕一张工单完成理解、协作和业务决策，将 WorkOrder、Workflow、Task、预约、资料、审核整改、SLA 和外部回传组织在一个业务工作区中。

### 页面头

使用经典专业风的对象头和主操作区，显示：

- 工单业务编号；
- 客户和脱敏联系方式；
- 项目；
- 服务产品；
- 当前状态；
- 当前阶段；
- 当前任务；
- SLA 剩余时间；
- 当前责任人；
- 来源系统。

右上主操作按 allowed-actions 展示：分配网点、改派师傅、发起整改、提交车企审核和更多操作。完整 UUID 不显示为主内容。

### 履约进度

使用真实冻结流程展示阶段进度，例如：

```text
接单成功
→ 受理协调
→ 网点分配
→ 上门勘测
→ 上门安装（当前）
→ 资料审核
→ 车企回传
→ 完成
```

生命周期、阶段和任务分别表达，不合并为一个模糊状态。当前阶段下必须展示当前 Task、责任和预计完成时间。

### 当前任务卡片

展示：

- 任务名称；
- 责任网点；
- 责任师傅；
- 预约时间；
- 任务说明；
- 服务地址；
- 允许动作；
- 阻塞原因。

### Tabs

- 基本信息；
- 任务记录；
- 预约与上门；
- 表单资料；
- 审核整改；
- 外部回传；
- 操作日志；
- 履约依据（履约方案版本与匹配说明）；
- 技术诊断（仅开发或真实诊断能力）。

### 履约依据

工单详情不得只显示“流程版本”。必须展示工单绑定的履约方案与版本及其匹配来源：

- 履约方案名称与方案版本号；
- 匹配方式（自动 / 人工指定 / 异常处置 / 重新匹配）与匹配时间；
- 匹配条件摘要（品牌、业务类型、设备类型、故障等级、服务区域等命中项）；
- 人工指定时的操作人与原因；
- 读取的配置来自绑定版本，不随新版本发布变化。

### 右侧上下文

- 风险与提醒；
- 当前责任链；
- 外部集成信息；
- 最近关键时间线。

### 表单资料

展示表单完成度、必填/选填、资料缩略图、资料版本、缺失项、最近审核意见和原图入口。

### 验收

任何业务争议都能回答：当前为什么在这个阶段、当前 Task 为什么出现、为什么要求这些资料、为什么某动作不可执行、当前冻结的是哪个配置版本、最近一次外部回传发生了什么。

## 6. 母版六：项目履约配置中心

### 页面目标

让项目配置管理员不编辑 JSON，即可查看和配置某项目的工单类型、Workflow、Task、表单、资料、SLA、派单、通知和集成，并安全发布不可变配置版本。

### 标准页面结构

```text
PageHeader：项目履约配置中心
SummaryStrip：项目 / 已发布版本 / 草稿 / 工单类型数 / 最近发布 / 使用中工单
LeftSubNavigation
Main：配置包概览 + 工单类型配置表
RightRail：配置说明 / 影响分析 + 版本时间线
```

### SummaryStrip

必须展示：

- 当前项目；
- 当前发布版本；
- 草稿版本；
- 关联工单类型数；
- 最近发布时间；
- 使用中的工单数。

### 左侧二级导航

- 基础信息；
- 工单类型；
- 工作流；
- 任务模板；
- 表单模板；
- 资料要求；
- SLA 规则；
- 派单策略；
- 通知策略；
- 集成映射；
- 发布记录。

### 配置包概览

展示：配置包名称、客户/品牌、适用区域、当前生效版本、草稿版本、创建人、创建时间、发布说明和影响范围。

主操作：编辑草稿、版本对比、发布、查看运行快照。

### 工单类型配置表

列：工单类型、流程模板、表单模板、资料模板、SLA、派单策略、通知、状态、最后更新人、最后更新时间、操作。

### 右侧影响区

必须明确：

- 发布后哪些新工单使用新版本；
- 哪些存量工单继续使用冻结版本；
- 与当前发布版相比有哪些真实差异；
- 主要风险和缺口；
- 版本发布时间线。

### 发布流程

四步：完整性校验 → 运行说明预览 → 版本差异/影响分析 → 生效时间与发布。

发布后能查看不可变运行说明书和运行快照。

### 禁止

- Manifest JSON 作为普通用户主内容；
- 前端自行解析运行 Manifest；
- 用固定文案伪造差异和影响；
- 把草稿、发布版和运行快照混为一个对象；
- 让用户直接输入 stageCode、taskType、assetId、slaRef。

### 验收

项目配置管理员能够完成一套标准“先勘测后安装”方案；校验错误可定位到具体配置资产和字段；发布前看到真实差异和影响；发布后看到不可变运行说明书和版本时间线。

## 7. 配置子母版一：工作流设计器

### 页面目标

可视化定义履约路径、节点、条件分支、等待事件、异常出口和 Task 模板引用。

### 三栏结构

#### 左栏

- 流程目录；
- 草稿/已发布版本；
- 节点组件：开始、人工任务、系统任务、条件网关、等待事件、结束。

#### 中间画布

- 节点和连线；
- 条件标签；
- 汇合与分支；
- 当前选中节点；
- 缩放、适配、锁定和自动排版；
- 撤销、重做、校验、版本对比和发布；
- 底部校验结果和风险提示。

#### 右栏

节点配置：名称、编码、类型、任务模板、执行角色、分配策略、表单、资料、SLA、前置条件、完成动作、异常出口和说明。

### 验收

产品负责人不阅读 JSON，即可理解流程如何进入、分支、汇合、产生 Task、失败和结束；校验问题可从结果直接定位到画布节点。

## 8. 配置子母版二：任务模板中心

### 页面目标

统一管理 Task 的执行角色、分配策略、表单、资料、SLA、动作、升级和 Workflow 引用。

### 页面结构

```text
SummaryStrip
Toolbar
LeftCategoryTree
TemplateTable
RightDetailPanel
```

### 任务分类

受理类、派单类、预约类、勘测类、安装类、审核类、整改类、回访类和系统任务。

### 表格

列：模板名称、任务编码、所属分类、执行角色、分配策略、表单、资料要求、SLA、状态、引用流程数、最后更新人、最后更新时间、操作。

### 详情面板

展示：

- 基础信息；
- 执行规则；
- 表单与资料；
- 允许动作；
- 提醒与升级；
- 版本说明；
- 被以下流程引用。

### 验收

运营配置人员不阅读 taskType code 即可判断该任务由谁执行、如何分配、需要什么输入、能执行什么动作、何时超时、会升级给谁以及被哪些流程使用。

## 9. 母版评审要求

每张母版和配置子母版至少提供：

- 1440×1024 正常状态；
- 1280px 可操作状态；
- 空、错误、只读、无权限或冲突状态之一；
- 使用真实或脱敏的业务样例数据；
- 页面目标、主操作和关键路径说明；
- 与经典专业风视觉基线的对照；
- 可访问性和键盘操作证据。

产品负责人明确批准后，才能标记 `PRODUCT_ACCEPTED` 和 `VISUAL_APPROVED`，并建立视觉金标。

## 10. 配置子母版：高德省市服务覆盖

服务覆盖页面必须读取 [高德省市行政区域服务覆盖决策与批准视觉基线](13-service-coverage-amap-visual-baseline.md) 及仓库四张批准视觉图。

### 10.1 项目覆盖总览/分析

```text
PageHeader + Project/ServiceType/AsOf Context
→ MainMap（全国/省级下钻）
→ CoverageSummary / Risk / Ranking
→ ProvinceOrCityCoverageTable
→ SelectedRegionDetail
```

地图必须是主工作区，表格和风险面板负责解释地图，不能退化为“表格页面上方放一张装饰地图”。

### 10.2 网点服务区域配置

```text
ObjectHeader：项目 + 网点 + 业务类型
→ Left：省市树与搜索
→ Center：高德行政区地图
→ Right：已选区域 + 影响 + 校验
→ Footer：取消 / 保存 / 查看变更影响 / 确认生效
```

区域树与地图使用同一 `amapAdcode`；选择省级后城市显示继承，不支持省内排除。

### 10.3 变更审阅

必须在确认生效前展示：

- 当前有效与待生效区域；
- 新增、保留和移除区域；
- 重叠覆盖城市；
- 对后续新派单的影响；
- 不会自动改变的存量 `ServiceAssignment`；
- 修改原因、版本和审计信息。

### 10.4 视觉门禁

实现 PR 必须把真实页面与四张批准参考图并排展示。核心布局未经产品负责人批准不得变更；缺少视觉对照只能进入 `IMPLEMENTED_PENDING_VISUAL_REVIEW`。
