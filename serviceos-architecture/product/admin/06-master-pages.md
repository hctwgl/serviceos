# Admin 六张母版页面

六张母版用于冻结整个 Admin 的产品语言、信息层级、布局密度和交互模式。母版通过产品负责人截图验收前，不应以其为视觉金标批量迁移其他页面。

## 1. 母版一：运营工作台

### 页面目标

回答：我今天必须处理什么、什么即将或已经超时、哪里出现重大异常、最近处理了什么。

### 页面结构

```text
PageHeader：运营工作台 + 当前范围 + 更新时间
风险摘要：仅 P0/P1 或明确 SLA 风险
主区：我的待办（占主要宽度）
侧区：即将超时 / 已超时 / 重大异常
底部：最近处理 / 关注项目
```

### 核心内容

- 我的待办：对象、任务、项目、当前责任、截止时间、风险、操作；
- 即将超时：剩余时间和责任人；
- 已超时：超时时长、当前阻塞和升级状态；
- 重大异常：严重度、对象、发生时间、当前处理人；
- 最近处理：返回上次页面并恢复来源上下文。

### 禁止

- 大量同宽 KPI 卡片；
- 累计总量与当前待办混为同一语义；
- 技术队列名称；
- 无法下钻的数字。

### 验收

运营人员在 10 秒内能找到最紧急的三件事并打开处理页面。

## 2. 母版二：用户管理

### 页面目标

完成用户查询、新建、编辑、启停、组织归属、角色分配和安全状态查看。

### 列表页

筛选：姓名/账号关键词、所属组织、用户状态、角色。

列：姓名、登录账号、所属组织、角色摘要、状态、最近登录、更新时间、操作。

页面操作：新建用户。

行操作：查看、编辑；更多中放启用/停用、分配角色、查看登录记录。

### 用户详情

Tabs：基本信息、组织归属、角色与权限、授权与委托、登录与安全、变更记录。

主要文案使用业务名称；principalId、角色 ID、capability code 只作为次级诊断信息。

### 新建/编辑

字段：姓名、登录账号、手机号/邮箱（按实际契约）、所属组织、初始角色、账号状态。角色使用实体选择器，不输入 ID。

### 验收

权限管理员不阅读任何技术 ID，即可创建用户、归属组织、分配角色并确认最终数据范围摘要。

## 3. 母版三：项目管理

### 页面目标

管理车企项目，查看项目服务范围、合作网点、工单类型履约方案和配置版本。

### 项目列表

筛选：关键词、所属车企、项目状态、生效状态。

列：项目名称、项目编码、所属车企、状态、生效日期、服务区域、合作网点、已发布方案、草稿方案、更新时间、操作。

页面操作：新建项目。

创建表单进入独立 FormPage，不常驻列表上方。

### 项目详情

摘要：项目名称、编码、车企、状态、生效区间、服务区域数量、合作网点数量、工单类型数量、配置健康、最近发布。

Tabs：项目概览、工单类型与履约配置、服务范围、合作网点、车企接口与映射、版本与变更。

不得保留“履约配置请使用配置设计器”之类空壳。

### 验收

项目管理员进入项目后能明确看到项目支持哪些工单类型、每种类型使用什么方案、是否存在草稿或校验错误。

## 4. 母版四：工单中心

### 页面目标

统一查询授权范围内工单，快速识别当前阶段、责任、SLA 和风险。

### 筛选

默认：工单状态、车企、项目、关键词、更多筛选。

更多筛选：服务区域、网点、师傅、阶段、任务状态、SLA 风险、审核/整改状态、创建时间。

关键词支持工单编号、客户、手机号后四位或地址（以真实查询能力为准）。

### 表格

列：工单编号、状态、当前阶段、车企/项目、客户（脱敏）、服务区域、当前责任人、网点/师傅、SLA、异常摘要、更新时间、操作。

主标识可点击；行操作以“查看详情”为主，低风险快捷动作按 allowed-actions 显示。

### 保存视图

待初审、待分配、待预约、暂停、已完成等作为服务端受控预设视图；用户保存视图放工具栏。

### 验收

运营人员不打开详情即可判断工单当前由谁负责、是否超时、下一步大致是什么。

## 5. 母版五：工单详情

### 页面目标

围绕一张工单完成理解、协作和业务决策。

### 页面头

显示工单业务编号、状态、车企项目、风险、SLA、当前责任和主 allowed-action。完整 UUID 不显示为主内容。

### 履约进度

使用真实冻结流程展示阶段进度；生命周期、阶段和任务分别表达，不合并为一个模糊状态。

### Tabs

- 概览：当前任务、责任、SLA、风险、下一步、最近活动；
- 履约任务：按阶段组织的任务时间线；
- 表单与资料：按勘测、安装、整改分组；
- 审核与整改：审核决定、问题和整改状态；
- 客户与预约：联系方式（脱敏）、地址、联系和预约记录；
- SLA 与异常：时效实例、暂停、超时和异常；
- 活动记录：业务事件；
- 配置来源：项目、工单类型、冻结履约版本和快照；
- 技术诊断：仅开发或真实诊断能力。

### 操作

页面只呈现业务动作：分配网点、分配师傅、确认预约、提交审核、暂停、取消等。复杂操作进入 Drawer 或 Dedicated Flow，不能出现 principalIds/sourceId/If-Match 等输入。

### 验收

任何业务争议都能回答：当前为什么在这个阶段、为什么要求这些资料、为什么某动作不可执行、当前冻结的是哪个配置版本。

## 6. 母版六：项目履约配置

### 页面目标

让项目配置管理员不编辑 JSON，即可配置某项目某工单类型的流程、阶段、表单、资料、动作、责任、审核、SLA 和异常路径，并安全发布不可变版本。

### 配置列表

列：工单类型、履约方案、流程、阶段数、表单数、资料数、SLA、版本、状态、生效时间、操作。

主操作：新增工单类型。

### 编辑工作区

三栏：

- 左侧：阶段导航、顺序、错误数量；
- 中间：阶段规则、表单、资料要求、动作与流转、责任与派单、审核、SLA、异常路径；
- 右侧：草稿版本、当前发布版本、修改摘要、校验结果、预览和发布入口。

普通模式使用中文业务字段；高级模式才显示 stageCode、fieldKey、binding、表达式和原始引用。

### 运行说明书

服务端编译后展示：阶段、责任人、表单、必传资料、可执行动作、下一阶段、异常路径、SLA 和客户端支持。

### 发布流程

四步：完整性校验 → 运行说明预览 → 版本差异/影响分析 → 生效时间与发布。

明确显示：存量工单继续使用创建时冻结版本，新工单按生效时间使用新版本。

### 验收

项目配置管理员能够完成一套标准“先勘测后安装”方案；校验错误可定位到具体阶段和字段；发布后能查看不可变运行说明书和版本差异。

## 7. 母版评审要求

每张母版至少提供：

- 1440×1024 正常状态；
- 1280px 可操作状态；
- 空、错误或只读状态之一；
- 使用真实或脱敏的业务样例数据；
- 页面目标、主操作和关键路径说明。

产品负责人明确批准后，才能标记 `PRODUCT_ACCEPTED` 并建立视觉金标。
