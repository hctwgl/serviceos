# Admin 产品化实施路线图

## 1. 当前原则

- 暂停新的零散页面开发；
- 已批准视觉方向统一为 [经典专业风视觉与页面基线](12-classic-professional-visual-baseline.md)；
- 当前“项目工单类型与履约配置”纵向功能继续，但前端必须重构为项目履约配置中心；
- Workflow 和 Task 必须成为可见、可配置、可追踪的一等产品页面；
- 不把概念效果图或现有页面截图直接设为产品金标；
- 不以“已接入 Ant Design Vue”或“样式接近效果图”代替产品验收；
- 文档蓝图、经典专业风和核心页面母版先于全量迁移。

## 2. 阶段 A：蓝图与视觉方向冻结

交付：

- 产品定位；
- 角色任务矩阵；
- 一级和二级菜单；
- 目标页面清单；
- 页面模式；
- 技术到产品映射；
- 验收门禁；
- 实施路线图；
- 经典专业风视觉与页面基线。

验收：产品负责人确认范围、角色、菜单、视觉方向、六张母版和配置子母版方向。

当前结果：方案 A｜经典专业风和以下四页概念方向已获批准：

1. 项目履约配置中心；
2. 工作流设计器；
3. 任务模板中心；
4. 工单详情。

概念方向批准只标记 `PRODUCT_DESIGNED`，不代表真实页面已经接受。

## 3. 阶段 B：现有系统产品审计

扫描：

- Vue Router；
- Page Registry；
- 服务端 Navigation；
- `src/pages` 全部页面；
- 深链页面；
- 当前 API；
- 当前实施状态和里程碑；
- 当前 Design Token、AppShell 和组件；
- Workflow、Task、Profile 和配置资产当前路由。

产出 `admin-current-to-target-matrix.md`：

| 当前 Page ID/路由 | 当前页面 | 目标页面 | 处理方式 | 技术状态 | 产品状态 | 数据缺口 | 视觉缺口 |
|---|---|---|---|---|---|---|---|

处理方式只能是：保留并重构、合并为预设视图、重命名、迁入配置中心、迁入审计监控、删除正式入口。

## 4. 阶段 C：批准方向的真实静态页面

使用真实契约形状和脱敏样例，优先完成：

1. 项目履约配置中心；
2. 工作流设计器；
3. 任务模板中心；
4. 工单详情。

每张页面提供：

- 1440×1024 正常状态；
- 1280px 正常状态；
- 至少一个空/错/只读/无权限/冲突状态；
- 真实或脱敏业务样例；
- 页面结构与经典专业风对照；
- 产品负责人逐页审查。

静态页面必须使用可复用真实组件和路由骨架，不能只做不可运行的图片稿；但在结构批准前不接入新的高风险业务命令。

## 5. 阶段 D：四个核心页面真实接口接入

### D1 项目履约配置中心

接入：

- 项目与配置包概览；
- 工单类型配置；
- 已发布版、草稿和运行快照；
- Revision Compare/Impact；
- allowed-actions；
- 发布记录和运行说明书。

### D2 工作流设计器

接入：

- Workflow 目录和版本；
- 结构化 Workflow Draft DTO；
- 节点和连线编辑；
- Task、表单、资料、SLA 和异常引用；
- 校验、试运行、比较和发布；
- 只读已发布版本。

### D3 任务模板中心

接入：

- Task 模板目录；
- 执行角色和分配策略；
- 表单、资料、SLA、动作和升级；
- Workflow 引用；
- 草稿、比较、复制和发布。

### D4 工单详情

接入：

- 工单摘要；
- 冻结 Workflow 进度；
- 当前 Task 和责任；
- 预约、表单资料、审核整改；
- SLA、责任链和最近时间线；
- 外部回传；
- allowed-actions。

接入时保持服务端 Navigation、Capability、Scope、allowed-actions、OIDC 和生成 Client，不用静态 Mock 代替正式链路。

## 6. 阶段 E：其余母版

依次完成：

1. 运营工作台；
2. 用户管理；
3. 项目管理；
4. 工单中心。

已完成的工单详情和项目履约配置继续作为母版五、六。

每张页面必须经过独立产品验收，不得等全部页面完成后一次性走查。

## 7. 阶段 F：模块化迁移

### F1 系统管理

用户、组织、角色、菜单、权限、数据范围、授权与委托、登录安全。

### F2 客户与资源

车企、项目、企业组织、合作组织、网点、师傅、技能资质、服务区域。

### F3 工单运营与履约

工单、任务、预约、派单、调度、勘测、安装。

### F4 审核与整改

审核队列、审核工作区、整改跟踪、整改复审、审核记录。

### F5 配置中心

履约方案、Workflow、Task、表单、资料、SLA、日历、派单规则、通知、集成和版本发布。

### F6 审计与监控

操作、登录、配置、接口、任务失败和系统诊断。

每个模块完成后按页面逐项产品验收。

## 8. 阶段 G：Navigation 与权限产品化

- 服务端菜单按本蓝图重组；
- 预设视图复用相同页面；
- 技术页面移入配置中心或审计监控；
- 角色仅影响默认首页和排序；
- 权限页面提供中文业务名称和数据范围摘要；
- 深链仍受服务端鉴权；
- 前端使用服务端 section/pageId/route/order，不通过正则推断菜单归属；
- Workflow、Task 和配置入口按 Capability 与项目上下文生成。

## 9. 阶段 H：质量与金标

完成：

- 产品 UI 静态扫描；
- Design Token 禁用值扫描；
- TypeScript 和生产构建；
- 单元/组件测试；
- Playwright 功能；
- Playwright 视觉；
- 可访问性自动和人工检查；
- 真实 OIDC + 后端冒烟；
- 全页面截图索引；
- 实施状态重新标记；
- 1440×1024 和 1280px 人工视觉走查。

只有 `PRODUCT_ACCEPTED + VISUAL_APPROVED` 页面可以成为视觉金标。

## 10. 建议交付切片

实际编号必须读取届时最新里程碑后顺延。

- 切片 1：经典专业风 Token、AppShell 和基础页面组件；
- 切片 2：项目履约配置中心静态重构；
- 切片 3：项目履约配置真实 DTO、差异、影响和发布；
- 切片 4：工作流设计器静态结构与组件；
- 切片 5：Workflow Draft、校验、比较和发布接入；
- 切片 6：任务模板中心与引用关系；
- 切片 7：工单详情统一业务工作区；
- 切片 8：其余母版；
- 切片 9：系统管理、组织资源和审核整改迁移；
- 切片 10：全量质量、视觉和状态收口。

## 11. 近期具体动作

1. 合并经典专业风视觉和页面基线；
2. 建立四个批准方向页面的路由/API/数据缺口矩阵；
3. 先统一 AdminAppShell、PageHeader、SummaryStrip、RightContextRail 和表格 Token；
4. 按批准效果实现项目履约配置中心；
5. 再实现工作流设计器和任务模板中心；
6. 将工单详情重构为统一履约工作区；
7. 逐页产品负责人验收；
8. 通过后再迁移其他页面并建立视觉回归。

## 12. 变更控制

以下决定需要产品负责人明确批准：

- 一级菜单增删；
- 核心角色变化；
- 母版页面信息架构变化；
- 改变方案 A｜经典专业风；
- 将技术诊断移回普通运营页；
- 以新的基础 UI 框架替换 Ant Design Vue；
- 把项目履约配置、Workflow 或 Task 拆成平行产品；
- 降低产品验收门槛。

二级菜单顺序、列字段细化和普通文案可在不改变用户任务的前提下由产品设计负责人维护。
