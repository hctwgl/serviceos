# Admin 产品化实施路线图

## 1. 当前原则

- 暂停新的零散页面开发；
- 正在进行的“项目工单类型与履约配置”纵向功能继续，但其前端必须落入母版六；
- 不把现有页面截图直接设为新的产品金标；
- 不以“已接入 Ant Design Vue”代替产品验收；
- 文档蓝图与六张母版先于全量迁移。

## 2. 阶段 A：蓝图冻结

交付：

- 产品定位；
- 角色任务矩阵；
- 一级和二级菜单；
- 目标页面清单；
- 页面模式；
- 技术到产品映射；
- 验收门禁；
- 实施路线图。

验收：产品负责人确认范围、角色、菜单和六张母版方向。

## 3. 阶段 B：现有系统产品审计

扫描：

- Vue Router；
- Page Registry；
- 服务端 Navigation；
- `src/pages` 全部页面；
- 深链页面；
- 当前 API；
- 当前实施状态和里程碑。

产出 `admin-current-to-target-matrix.md`：

| 当前 Page ID/路由 | 当前页面 | 目标页面 | 处理方式 | 技术状态 | 产品状态 | 数据缺口 |
|---|---|---|---|---|---|---|

处理方式只能是：保留并重构、合并为预设视图、重命名、迁入配置中心、迁入审计监控、删除正式入口。

## 4. 阶段 C：六张母版静态设计

先不接入新的业务命令，使用真实契约形状和脱敏样例完成：

1. 运营工作台；
2. 用户管理；
3. 项目管理；
4. 工单中心；
5. 工单详情；
6. 项目履约配置。

每张页面提供 1440×1024、1280px 和至少一个异常状态。产品负责人逐张审查。未批准页面不得进入批量迁移。

## 5. 阶段 D：母版真实接口接入

按风险从低到高：

1. 用户管理；
2. 项目管理；
3. 工单中心；
4. 运营工作台；
5. 工单详情；
6. 项目履约配置。

接入时保持服务端 Navigation、Capability、Scope、allowed-actions、OIDC 和生成 Client，不用静态 Mock 代替正式链路。

## 6. 阶段 E：模块化迁移

### E1 系统管理

用户、组织、角色、菜单、权限、数据范围、授权与委托、登录安全。

### E2 客户与资源

车企、项目、企业组织、合作组织、网点、师傅、技能资质、服务区域。

### E3 工单运营与履约

工单、任务、预约、派单、调度、勘测、安装。

### E4 审核与整改

审核队列、审核工作区、整改跟踪、整改复审、审核记录。

### E5 配置中心

履约方案、流程、表单、资料、SLA、日历、派单规则、版本发布。

### E6 审计与监控

操作、登录、配置、接口、任务失败和系统诊断。

每个模块完成后按页面逐项产品验收，不允许等全部页面完成后一次性走查。

## 7. 阶段 F：Navigation 与权限产品化

- 服务端菜单按本蓝图重组；
- 预设视图复用相同页面；
- 技术页面移入配置中心或审计监控；
- 角色仅影响默认首页和排序；
- 权限页面提供中文业务名称和数据范围摘要；
- 深链仍受服务端鉴权。

## 8. 阶段 G：质量与金标

完成：

- 产品 UI 静态扫描；
- TypeScript 和生产构建；
- 单元/组件测试；
- Playwright 功能；
- Playwright 视觉；
- 可访问性自动和人工检查；
- 真实 OIDC + 后端冒烟；
- 全页面截图索引；
- 实施状态重新标记。

只有 `PRODUCT_ACCEPTED + VISUAL_APPROVED` 页面可以成为视觉金标。

## 9. 建议交付切片

实际编号必须读取届时最新里程碑后顺延。

- 切片 1：蓝图和当前→目标矩阵；
- 切片 2：六张母版静态实现；
- 切片 3：母版产品审查修订；
- 切片 4：用户与项目真实接入；
- 切片 5：工单中心与详情真实接入；
- 切片 6：工作台与履约配置真实接入；
- 切片 7：系统管理和组织资源迁移；
- 切片 8：履约、审核和整改迁移；
- 切片 9：配置中心与审计监控迁移；
- 切片 10：全量质量、视觉和状态收口。

## 10. 近期具体动作

1. 评审并合入本蓝图；
2. 等待当前项目履约配置 Agent 完成技术实现；
3. 对该交付只判断“技术能力/前端接入”，不自动标记产品完成；
4. 根据本蓝图创建当前→目标页面矩阵；
5. 先制作六张母版的真实可运行页面；
6. 产品负责人批准后再全量迁移。

## 11. 变更控制

以下决定需要产品负责人明确批准：

- 一级菜单增删；
- 核心角色变化；
- 母版页面信息架构变化；
- 将技术诊断移回普通运营页；
- 以新的基础 UI 框架替换 Ant Design Vue；
- 把项目履约配置拆成新的独立产品；
- 降低产品验收门槛。

二级菜单顺序、列字段细化和普通文案可在不改变用户任务的前提下由产品设计负责人维护。
