# 师傅移动端产品与离线交互规格

## 1. 目标

师傅 App 的首要目标是在弱网、断网、拍照/视频较多和现场时间紧张的情况下，可靠完成预约、到场、表单、资料、提交和整改。它不是后台工单详情的移动适配。

## 2. 体验原则

1. 当前任务和下一动作始终明确；
2. 拍摄、填写和上传分离，上传失败不丢本地成果；
3. 离线动作明确标记“本地已保存/服务器已确认”；
4. 服务端责任变更优先，旧师傅不能用离线数据覆盖；
5. 现场必填和资料条件尽可能在提交前本地预检，服务端仍权威校验；
6. GPS、水印、OCR 和照片质量结果可解释；
7. 不在非作业时间持续采集位置；
8. 不以一个“完成”按钮跨过预约、Visit、表单、资料和提交对象。

## 4. 任务 Feed

### 4.1 分组

- 待联系/预约；
- 今日上门；
- 进行中；
- 待上传/待同步；
- 被驳回待整改；
- 即将超时；
- 已完成（近期）。

每张卡片显示：任务类型、预约窗口、距离/区域摘要、用户脱敏名、SLA、离线包/同步状态和下一 allowed-action。

### 4.2 排序

优先级：进行中的 Visit → 已预约临近时间 → SLA 风险 → 整改 → 其他。排序规则来自服务端 feed；客户端离线时保持最后快照并显示 asOf。

### 4.3 责任变更

任务被改派后：

- 在线时立即从当前 Feed 移除并通知；
- 离线缓存标记“待确认”；
- 恢复网络后服务器拒绝未授权命令；
- 本地未上传文件不自动删除，进入只读隔离区供授权转交/清理；
- 不向旧师傅继续展示用户联系方式和下载新资料。

## 5. 任务详情

```text
任务头：类型、SLA、同步状态
下一动作：联系/预约/到场/继续填写/补传/提交
预约与地址：导航、联系（按权限）
现场清单：步骤、表单、资料完成度
异常入口：无法联系/无法施工/中断
历史：本任务预约、Visit、提交与整改摘要
```

“下一动作”来自 `/tasks/{id}/allowed-actions` 和工作包离线动作清单。客户端不得仅根据本地 Task 状态生成命令。

## 6. 工作包

### 6.1 下载触发

- 师傅接受/获得当前 TaskAssignment；
- 用户手工“离线准备”；
- 即将进入预约窗口的后台预取；
- 整改 Task 创建。

### 6.2 Manifest

```text
packageId / issuedAt / expiresAt
task/workOrder/assignment/authority versions
configurationBundleId/digest
allowedOfflineActions
form versions + UI/data schema digests
evidence slots + capture policies
prefill values + versions
address/appointment snapshot
required reference media thumbnails
server time offset
```

工作包只包含完成任务所需最小数据，不复制完整工单、其他任务、内部价格或其他网点信息。

### 6.3 有效性

进入现场前校验包未过期、assignment/authority 未变化。离线时允许按包内政策继续采集，但最终服务端逐命令复核。

## 7. 联系与预约

点击电话使用系统拨号并在返回 App 后提示记录结果；不得未经合规自动录音。结果使用标准 reasonCode，允许语音转文字作为草稿但提交前确认。

预约页面：

- 显示用户偏好/已有 revision；
- 提供时间窗口而非单一时间点；
- 展示与师傅日程冲突；
- 改约必选原因；
- 用户确认渠道和确认方明确；
- 离线预约草稿不显示“已预约”，只有服务器确认后进入 CONFIRMED。

## 8. 到场

Check-in 页面在用户主动点击时采集：当前时间、GPS、精度和设备。UI 展示定位结果：范围内、范围外、精度低、不可用，以及项目策略（允许、警告、需审批、阻止）。

- 不后台持续定位；
- capturedAt 与 receivedAt 分开；
- 离线到场显示“本地已记录，待服务器确认”；
- 服务器发现 assignment/预约变化时进入冲突，不创建虚假 Visit；
- 范围外允许继续时必须说明/证据并审计。

## 9. 现场步骤

任务步骤由 TaskDefinition/表单/资料配置组合，示例：

```text
1. 安全确认
2. 现场基础信息
3. 施工参数
4. 资料拍摄
5. 用户确认/签字
6. 提交前检查
7. 离场
```

步骤条只是导航和完成度投影，不是另一套流程状态。用户可以在允许范围返回修改草稿；已提交版本只能通过 Correction 动作产生新版本。

## 10. 动态表单

### 10.1 渲染

- 使用发布 FormVersion 的 data/UI schema；
- 本地执行同版本可移植校验规则；
- 复杂远程数据源明确要求联网并提供缓存/手工兜底；
- 条件显隐时不静默删除已有值，按规则保留/清除并提示；
- 单位、数字精度、枚举和默认值来自 FieldDefinition；
- 自动保存本地草稿，显示最后保存时间。

### 10.2 冲突

prefill 或服务器表单版本变化时，不做逐字段“最后写入胜出”。冲突页并列展示本地值、服务器值、字段来源和允许动作：采用服务器、保留本地重新校验、转人工。

## 11. 资料采集

每个 EvidenceSlot 卡片显示：名称、为什么需要、示例、拍摄/相册/GPS/水印/数量要求、机器校验和当前 revision。

### 11.1 相机

- 资料项要求现场拍摄时不提供相册入口；
- 拍摄前显示构图/安全提示；
- 捕获原图后计算本地 checksum；
- 水印作为派生版本，原始 capture metadata 单独保存；
- 不允许用户通过修改系统相册文件冒充原始拍摄；
- 视频显示时长/大小预算和压缩进度。

### 11.2 本地预检

可执行：模糊/过暗/反光提示、条码/SN OCR、数量和基本构图。结果是提示或按策略阻断，并保存模型/规则版本；本地通过不等于服务器审核通过。

### 11.3 安全

涉及配电操作的拍摄提示不能鼓励师傅在带电危险状态下操作手机。需要电气测量证据时，步骤和安全说明由业务安全负责人确认。

## 12. 上传队列

每个文件状态：

```text
LOCAL_ONLY
→ SESSION_PENDING
→ UPLOADING (part progress)
→ FINALIZING
→ SERVER_VALIDATING
→ AVAILABLE
或 RETRYABLE/CONFLICT/QUARANTINED/FAILED_FINAL
```

功能：Wi-Fi/蜂窝策略、暂停/继续、失败原因、逐项/全部重试、剩余大小、磁盘空间和本地保留。

- App 重启后队列恢复；
- 分片重试不创建新 EvidenceRevision；
- checksum 相同/deviceCommandId 相同返回既有结果；
- 服务器 AVAILABLE 并完成本地策略确认后才清理原文件；
- 退出账号前必须处理未同步资料，不能直接无提示删除。

## 13. 提交前检查

按区块展示：

- 表单错误/警告；
- 必需资料缺失；
- 文件仍上传/校验中；
- OCR/SN/VIN 不一致；
- 预约/Visit/assignment/authority 是否仍有效；
- 用户签字/协议；
- 异常结果和说明；
- 将要创建的 FormSubmission/EvidenceSet/FieldOperationSubmission。

点击提交生成稳定 deviceCommandIds。若离线允许排队，页面显示“待同步”，不显示“平台已提交”。服务器全部必要对象成功后，任务才进入可完成状态。

## 14. 无法施工与中断

异常入口不能只是自由文本。选择 reasonCode 后动态展示必需说明、资料、责任、是否再次预约和 SLA 影响。提交后创建协调/异常 Task；移动端不自行关闭工单。

紧急安全问题优先提示停止作业和联系渠道，不能要求先完成冗长表单。

## 15. 整改

整改卡片按被驳回对象显示：原版本、驳回原因/补充说明、正确示例、截止时间和最新补传。

- 只要求处理被驳回项；
- 补传创建新 revision；
- 已通过项只读保留；
- 车企驳回经客服协调后才形成师傅可见 Task；
- 师傅不能修改 ReviewDecision；
- 再提交展示轮次和精确新版本。

## 16. 同步中心

按任务显示：待发送、处理中、冲突、失败、完成。每个 OfflineCommand 有本地时间、依赖、attempt 和服务器结果。

冲突类型：

| 类型 | 处理 |
|---|---|
| Assignment changed | 禁止提交，隔离本地数据，联系网点/客服 |
| Aggregate version changed | 刷新并按字段/动作解决 |
| Form/config mismatch | 迁移草稿或使用旧版受控提交/转人工 |
| Evidence slot invalid | 不上传到错误槽位，选择新槽或转人工 |
| Authority changed | 禁止旧系统/旧版本写入 |
| Idempotent already accepted | 绑定服务器既有结果，清除重复待办 |

## 17. 日程与消息

日程以 AppointmentRevision 为准，显示未同步/冲突。消息深链到任务/整改/预约；打开前重新授权。通知“已发送”不等于用户已读，App 阅读回执按隐私策略处理。

## 18. 设备与本地数据

- 本地数据库加密，密钥使用系统安全存储；
- root/jailbreak 风险按策略警告或阻断高敏任务；
- 截图/录屏限制按平台能力和业务风险评估；
- 本地日志不含用户、地址、VIN、照片路径和 token；
- 远程注销/设备撤销后清理可清理数据，未同步业务资料进入受控恢复；
- 多账号不共享工作包和文件；
- 磁盘不足在拍摄前预警。

## 19. 性能与可用性

- Feed 首屏使用本地缓存立即显示并异步刷新；
- 工作包按需预取，参考图片压缩；
- 大视频上传不阻塞表单操作；
- 每个网络请求可取消/超时/重试，命令重试必须幂等；
- 重要动作提供触觉/视觉反馈但不依赖动画表达成功；
- 字体缩放、读屏标签、44px/等价触控区域和高对比度纳入验收。

## 20. 完成定义

1. 断网、杀进程和重启不丢表单/资料/命令；
2. 任务改派后旧师傅不能提交或读取新敏感数据；
3. 本地已保存、上传完成、服务器已确认三个状态不混淆；
4. 相同离线命令/分片重试不产生重复 Visit/Submission/Revision；
5. 条件表单和资料规则使用锁定版本；
6. 冲突不使用最后写入覆盖；
7. 整改只处理被驳回版本并保留历史；
8. GPS/相机/本地文件遵守最小采集和保留；
9. 关键路径满足可访问性和真实设备弱网测试；
10. 所有成功状态可关联服务器资源 ID/version。
