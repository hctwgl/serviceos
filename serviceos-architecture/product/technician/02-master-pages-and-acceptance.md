# Technician 核心页面母版与产品验收

## 1. 目的

本文件将已批准的师傅端方案 A 转化为 H5/iOS 页面母版、交互要求、离线状态和产品验收门禁。

核心原则：

> 师傅端围绕“当前任务、当前步骤、下一动作和同步可信度”组织，不围绕后台字段和模块组织。

## 2. 核心页面清单

第一阶段黄金闭环：

1. 今日任务；
2. 任务详情；
3. 联系与预约；
4. 到场签到；
5. 现场作业步骤；
6. 动态表单；
7. 资料拍摄与上传；
8. 上传队列；
9. 提交前检查；
10. 离场；
11. 整改任务；
12. 同步与冲突中心。

治理和辅助页面：

- 日程；
- 消息；
- 我的；
- 资质；
- 设备和本地数据；
- 登录与设备绑定。

# 母版一：今日任务

## 3. 页面目标

让师傅打开应用后立即知道今天有多少任务、哪些最紧急、何时上门、距离多远以及下一步操作。

## 4. 页面结构

```text
蓝色品牌栏 + 消息
→ 今日任务概览
→ 紧急任务
→ 今日任务
→ 底部导航
```

## 5. 今日概览

展示：

- 今日任务；
- 紧急任务；
- 待提交；
- 完成率。

数字必须来自服务端 Feed 或本地缓存快照，并显示截至时间。

## 6. 任务卡

展示：

- 预约时间窗口；
- 当前状态；
- 工单号；
- 任务标题；
- 客户脱敏姓名和电话；
- 地址摘要；
- 距离；
- SLA；
- 工作包/同步状态；
- 下一动作。

紧急任务使用红色标签和 SLA 倒计时，但不使用大面积红色背景造成持续压力。

## 7. 快捷动作

首页可以提供：

- 扫码签到；
- 继续进行中任务；
- 处理待同步；
- 查看整改。

动作由服务端或工作包 allowed-action 决定。

## 8. 今日任务验收

- 在线和离线快照状态清楚；
- 排序符合服务端优先级；
- 改派后任务及时移除或标记待确认；
- 敏感信息在失权后隐藏；
- 紧急和 SLA 信息不依赖颜色；
- 字体放大后仍可操作；
- 空任务、加载失败、离线、同步中和账号失效状态完整。

# 母版二：任务详情

## 9. 页面目标

集中展示完成当前任务所需的最小信息，并明确下一步。

## 10. 页面结构

```text
任务标题 + 紧急/SLA
→ 客户信息
→ 服务产品/设备/故障描述
→ 当前步骤
→ 下一步行动
→ 预约、优先级、同步状态
→ 固定底部操作
```

## 11. 客户与地址

展示：

- 客户脱敏姓名；
- 按权限解密的联系电话；
- 地址；
- 导航；
- 预约窗口；
- 客户要求。

电话和导航使用系统能力；返回应用后提示记录联系结果。

## 12. 当前步骤

展示任务步骤投影和当前步骤，例如：

```text
现场勘测/安装
```

同时说明：

- 当前要做什么；
- 为什么需要；
- 完成条件；
- 下一步；
- 异常入口。

## 13. 固定操作

常见组合：

- 联系客户 + 开始执行；
- 导航 + 到场签到；
- 继续填写；
- 处理整改；
- 查看冲突。

## 14. 任务详情验收

- 首屏看得到 SLA 和下一步；
- 不展示无关工单和总部信息；
- allowed-actions 与客户端按钮一致；
- 责任变化时立即失败关闭；
- 离线包过期、配置变化和数据冲突有明确提示；
- 联系和导航满足隐私策略。

# 母版三：联系与预约

## 15. 联系记录

操作：

```text
拨号
→ 返回 App
→ 选择结果
→ 填写必要说明
→ 本地保存/提交
```

联系结果使用标准原因：已联系、无人接听、拒绝、号码异常、稍后联系等。

## 16. 预约

展示：

- 当前预约 revision；
- 用户偏好；
- 可选时间窗口；
- 师傅日程冲突；
- 确认渠道和确认人；
- 改约原因。

离线保存的预约只能称“草稿”或“待同步”，不能显示已预约。

## 17. 联系预约验收

- 联系记录只追加；
- 不自动录音；
- 预约时间窗口而非单点；
- 并发 revision 冲突不覆盖；
- 通知失败不回滚预约事实；
- 用户拒绝和无法联系有业务化下一步。

# 母版四：到场签到

## 18. 页面结构

```text
工单摘要 + SLA
→ 客户和电话
→ 服务地址 + 地图 + 导航
→ 客户要求
→ 预约时间
→ 定位状态和服务范围
→ 到场签到
```

## 19. 定位状态

- 范围内；
- 范围外；
- 精度低；
- 定位不可用；
- 离线记录；
- 平台待确认。

页面必须显示项目策略：允许、警告、需要说明/证据、需要审批或阻止。

## 20. 签到验收

- 只在用户主动操作时采集定位；
- capturedAt 和 receivedAt 区分；
- 离线不显示平台确认；
- Assignment 变化时不创建虚假 Visit；
- 范围外受控放行记录原因和证据；
- 地图不可用时有文本地址和恢复方式。

---

# 母版五：现场作业步骤

## 21. 页面目标

通过明确步骤降低现场遗漏和返工。

## 22. 页面结构

```text
任务摘要
→ 步骤进度 2/5
→ 纵向步骤列表
→ 当前步骤说明
→ 操作指引/清单
→ 异常入口
→ 下一步按钮
```

## 23. 步骤状态

- 已完成：绿色对勾和完成时间；
- 当前：蓝色高亮；
- 待开始：灰色；
- 阻塞：红/橙色并说明；
- 待同步：状态图标和文字。

## 24. 现场步骤验收

- 步骤来自锁定 Task/Form/Evidence 配置；
- 不创建第二套流程状态机；
- 当前步骤可回到允许修改的草稿；
- 已提交版本只通过整改产生新版本；
- 异常入口不能只是自由文本；
- 安全风险优先提示停止作业和联系渠道。

---

# 母版六：动态表单

## 25. 页面结构

- 业务分组；
- 字段标题、说明和单位；
- 输入控件；
- 条件字段；
- 错误和警告；
- 本地自动保存状态；
- 上一步/下一步。

## 26. 字段体验

- 数字键盘和精度匹配；
- 枚举使用业务中文；
- 日期时间使用系统选择器；
- 长表单分段；
- 必填和选填清楚；
- 条件显隐有过渡但不依赖动画；
- 离线数据源说明缓存和限制。

## 27. 表单验收

- 使用冻结版本的 data/UI schema；
- 本地规则和服务端规则版本一致；
- 自动保存后显示时间和位置；
- 冲突并列展示本地值和服务端值；
- 不使用最后写入覆盖；
- 断网和杀进程不丢草稿。

---

# 母版七：资料拍摄与上传

## 28. 页面结构

```text
采集说明
→ 资料分组卡
→ 已拍/应拍数量
→ 缩略图网格
→ 拍照/受控相册入口
→ 本地/上传/校验状态
→ 提交资料
```

## 29. 资料卡

展示：

- 资料名称；
- 必填；
- 为什么需要；
- 示例；
- 数量；
- 拍摄策略；
- GPS/水印；
- Revision；
- 状态。

## 30. 图片状态

每张缩略图显示：

- 本地已保存；
- 待上传；
- 上传进度；
- 平台校验中；
- 已同步；
- 校验失败；
- 需要重拍。

## 31. 拍摄验收

- 现场拍摄要求时禁用相册；
- 本地计算 checksum；
- 原始元数据和水印派生分离；
- 图片质量提示可解释；
- 视频显示大小和压缩进度；
- 安全场景不鼓励危险拍摄；
- 上传失败不丢本地成果。

---

# 母版八：上传队列

## 32. 页面结构

按任务和状态分组：

- 已保存本地；
- 上传中；
- 已上传/平台校验；
- 已同步；
- 可重试失败；
- 冲突；
- 已隔离。

## 33. 队列项

展示：

- 缩略图；
- 文件名；
- 大小；
- 资料项；
- 进度；
- 网络策略；
- 尝试次数；
- 失败原因；
- 重试/查看/清理操作。

## 34. 上传验收

- App 重启恢复；
- 分片重试不重复创建 Revision；
- deviceCommandId/checksum 幂等；
- 平台确认前不删除原文件；
- 账号退出前处理未同步资料；
- 蜂窝、Wi-Fi、磁盘不足和后台限制有明确反馈。

---

# 母版九：提交前检查

## 35. 页面结构

清单分区：

- 表单完整性；
- 资料完整性；
- 文件上传与校验；
- 客户确认/签字；
- 预约、Visit 和 Assignment；
- 设备 SN/VIN/OCR；
- 异常说明；
- 将提交的结果摘要。

## 36. 提交按钮

状态：

- 可以提交；
- 缺少必填；
- 文件上传中；
- 平台校验中；
- 需要联网；
- 可以离线排队；
- 权限/责任已变化；
- 冲突待处理。

## 37. 提交验收

- 使用稳定 deviceCommandId；
- 离线排队不显示平台完成；
- 服务端返回资源 ID/version 后才显示确认；
- 部分对象失败时说明哪些已建立；
- 重复点击不产生重复 Visit、Submission 或 Revision。

# 母版十：离场

## 38. 页面内容

- 当前 Visit；
- 作业结果摘要；
- 表单和资料状态；
- 客户确认；
- 未完成项；
- 定位策略；
- 离场时间；
- 签退操作。

## 39. 离场验收

- 未满足门禁时阻止或要求受控说明；
- 离线签退标记待同步；
- 不用签退替代任务提交；
- 签退与任务完成分别记录。

---

# 母版十一：整改任务

## 40. 页面结构

- 整改轮次和截止时间；
- 被驳回项；
- 原资料/表单版本；
- 驳回原因；
- 正确示例；
- 新补传；
- 上传/校验状态；
- 再提交。

## 41. 整改验收

- 只处理被驳回项；
- 已通过项只读；
- 补传形成新 Revision；
- 车企驳回经协调后才形成师傅 Task；
- 师傅不能修改 ReviewDecision；
- 每轮版本和时间可追溯。

---

# 母版十二：同步与冲突中心

## 42. 页面结构

按任务展示：

- 待发送；
- 处理中；
- 冲突；
- 失败；
- 已确认。

每个命令显示：

- 本地时间；
- 业务动作；
- 依赖；
- 尝试次数；
- 服务器结果；
- 恢复操作。

## 43. 冲突处理

- 任务已改派：隔离本地资料，联系网点；
- 数据版本变化：刷新并重新确认；
- 表单/配置变化：迁移草稿或转人工；
- 资料槽位失效：重新选择或转人工；
- 权限变化：停止提交；
- 已幂等接受：绑定既有结果并清理重复待办。

## 44. 同步验收

- 冲突有业务说明；
- 不自动最后写入覆盖；
- 本地成果不静默删除；
- 每个动作能关联服务器结果；
- 恢复后队列状态一致；
- 多账号数据隔离。

# 产品完成门禁

## 45. H5 完成条件

```text
真实在线 API 接入
+ 与方案 A 页面母版一致
+ allowed-actions 和版本校验正确
+ 正常/空/错/无权限/改派/冲突状态完整
+ 自动化回归通过
+ 人工视觉与可访问性审查通过
+ 产品负责人批准
```

H5 通过不代表 iOS 离线生产能力完成。

## 46. iOS 生产完成条件

除上述条件外，还必须：

- Keychain 和本地数据库加密；
- 原生相机/GPS；
- 后台上传；
- OfflineCommand；
- 断网/杀进程/重启恢复；
- 设备撤销；
- 真机弱网；
- TestFlight 试点；
- 隐私和本地保留策略验收。

## 47. 黄金链路验收

真实师傅必须在真机独立完成：

```text
查看今日任务
→ 联系并确认预约
→ 导航到现场
→ 到场签到
→ 按步骤作业
→ 填写表单
→ 拍摄资料
→ 弱网/离线保存
→ 恢复上传
→ 提交前检查
→ 服务端确认提交
→ 离场
→ 接收并完成一次整改
```

全过程不得要求查看后台、复制内部 ID 或使用接口工具。

## 48. 真机与视觉验收

至少覆盖：

- 当前主流 iPhone 尺寸；
- 小屏设备；
- 大字体；
- 浅色模式；
- 弱网、断网、切换网络；
- 相机权限拒绝；
- 定位权限拒绝；
- 磁盘不足；
- 上传中杀进程和重启；
- 任务改派；
- 表单/配置冲突；
- 设备撤销；
- 读屏和 44px 触控区域。

产品负责人批准真实页面后才能建立截图或快照金标。

## 49. UI_DATA_GAP

缺少以下数据时必须补服务端或工作包契约：

- Feed 排序和总数；
- SLA 基准时间；
- 下一 allowed-action；
- 当前步骤和完成条件；
- 表单和资料锁定版本；
- 上传/校验状态；
- Assignment/Authority 版本；
- 冲突解释和恢复动作；
- 提交结果资源 ID/version；
- 整改轮次和精确驳回对象。

客户端不得自行猜测、拼接或静默降级。
