# M2 配置与执行内核验收矩阵

## 1. 测试层级

| 层级 | 目标 | 示例 |
|---|---|---|
| 领域单元测试 | 聚合状态机和不变量 | 同一任务不能完成两次 |
| 配置契约测试 | Schema、引用和发布校验 | 流程引用未发布表单被拒绝 |
| 模块集成测试 | 数据库事务、Outbox、幂等 | 命令和事件原子提交 |
| 流程适配测试 | 事件关联和任务创建 | TaskCompleted 激活下一节点 |
| API 契约测试 | HTTP、错误、ETag、权限 | 旧 If-Match 返回 409 |
| 恢复测试 | 重启、重复消息、外部超时 | 重启后业务自动任务继续重试 |
| 样本回放 | 真实项目符合度 | 勘测驳回整改闭环 |

## 3. 故障注入

必须覆盖：

- 数据库提交前/后进程终止；
- Outbox 已写未发布、已发布未标记；
- 消费者处理成功但 inbox 未提交；
- 外部系统超时、5xx、业务拒绝和重复回调；
- 流程引擎短时不可用；
- 权限投影延迟；
- 配置发布并发竞争；
- 时钟、节假日日历和 SLA 暂停边界。

## 4. 退出条件

- 所有 `Milestone=M2` 的 P0/P1 场景自动化通过；M3/M4 行是提前锁定的跨阶段契约，不阻塞 M2 退出；
- 没有无法解释的重复任务、重复外部副作用或状态漂移；
- 关键命令、事件和 HTTP 契约具备兼容测试；
- 使用首个试点项目的脱敏样本完成正常与异常链路回放；
- 授权拒绝、高风险操作和敏感导出审计可完整检索；
- 故障恢复结果与无故障执行的业务结果一致。
