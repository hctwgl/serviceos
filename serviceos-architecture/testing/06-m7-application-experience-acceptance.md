# M7 多 Portal 应用与交互验收矩阵

## 1. 规则

- P0 是对应 Portal/页面进入生产 cohort 的阻断门禁；
- 每个用例同时验证 UI、服务端授权、领域结果和审计/事件，不以截图代替；
- 使用 pageId、actionCode、API、样本和服务端资源版本形成证据；
- 角色测试必须包含数据范围变化、改派和权限撤销；
- 真机弱网/离线测试不能全部由浏览器网络模拟替代。

## 10. M7 退出条件

- 三个 Portal 的 MVP pageId、路由、能力、查询和命令映射完成；
- 所有适用 M7 P0 自动化/真机/人工验证通过；
- Admin 完成收单到审核/回传/影子试算的首切片；
- Network 完成师傅分配、预约协同和授权代补；
- Technician 完成离线工作包、现场提交、上传和整改；
- 跨 Portal 改派、预约冲突、审核整改和集成失败闭环；
- 投影延迟、重建、失权和缓存隔离验证通过；
- 关键旅程可访问性、敏感数据和埋点审查通过；
- 未实现页面/动作明确标为二期或未支持，不使用空菜单冒充完成。
