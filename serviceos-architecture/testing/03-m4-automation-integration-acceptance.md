## 2. 确定性回放

派单策略发布必须携带候选样例：输入上下文、指标快照、过滤结果、评分、排名和选择。相同策略版本与输入快照重复执行结果一致。

SLA 日历发布必须携带：

- 工作日/周末/节假日边界；
- 跨日工作时段；
- 夏令时（如适用）和时区；
- 多次暂停；
- deadline 恰好命中停止事件；
- 迟到事件和重算。

## 3. 集成故障注入

- 连接建立前失败、发送中断、响应丢失；
- 2xx 但正文不合法；
- 429 + Retry-After；
- 401/403 凭据失效；
- 5xx 后恢复；
- 外部重复处理但返回冲突；
- 相互矛盾的业务回执；
- SFTP 写一半、文件重复、文件名相同内容不同；
- OutboundDelivery 已提交但进程在写 attempt 前/后退出；
- 人工重试与自动重试并发。

## 4. 负载与积压

以实际峰值和增长预测验证：

- 同区域批量工单的候选查询与容量竞争；
- 月度比例指标读取；
- 大量 SLA milestone 同一分钟到期；
- 车企接口停机造成 delivery 积压和恢复后的限流排空；
- 通知供应商故障后的重试风暴；
- 异常工作台分页和最老任务查询。

## 5. 退出条件

- 所有 P0 场景自动化通过；
- P1 场景通过或有明确延期记录；
- 派单选择、排除和人工覆盖均可解释；
- 并发和重试不产生重复 ServiceAssignment、外部回传或通知；
- SLA 在重启、暂停、迟到事件和日历边界下可重算；
- 自动失败均能恢复或生成有责任 Task 的人工闭环；
- 使用试点车企真实脱敏接口样本完成入站、回传、驳回和恢复演练；
- 运维能够从工单追踪到策略、attempt、异常、Task 和最终恢复证据。
