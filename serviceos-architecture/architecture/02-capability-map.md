## 2. 一级能力清单

| 能力 | MVP | 核心职责 | 首期不做 |
|---|---:|---|---|
| 项目与业务产品 | 是 | 管理品牌、合同周期、区域、业务类型和配置组合 | 通用商品商城 |
| 工单与任务 | 是 | 标准工单、阶段、任务、动作和时间线 | 任意案件管理平台 |
| 服务网络 | 是 | 网点准入、区域、能力、资质、停派和评分 | 完整供应商采购 |
| 派单 | 是 | 硬过滤、候选评分、指定派单、改派和人工兜底 | 实时路线优化 |
| 预约与现场作业 | 是 | 联系、预约、改约、到场、勘测、安装、整改 | 消费者自助排期市场 |
| 表单与字段 | 是 | 节点表单、字段映射、显隐、必填、校验和版本 | 通用页面搭建器 |
| 资料与审核 | 是 | 资料要求、采集约束、版本、单项审核和整改 | 首期全自动 AI 审核 |
| SLA | 是 | 节点时钟、日历、暂停、预警、超时和升级 | 复杂预测型 SLA |
| 外部集成 | 是 | 接入记录、幂等、映射、重试、回传和人工修复 | 通用 ESB 产品 |
| 运营异常与人工接管 | 是 | 汇集自动化失败、创建处理任务并验证恢复 | 另一套工单状态机 |
| 履约事实与计价 | 是 | 事实提取、规则命中、费用明细和可解释试算 | 总账、资金支付 |
| 数据迁移与切换 | 是 | 源快照、血缘、在途恢复点、cohort 和回退 | 原样复制旧数据库 |
| 发布门禁与可观测性 | 是 | 业务追踪、SLI/SLO、灰度门禁和 runbook | 只看服务器 CPU |
| 对上/对下结算 | 二期 | 对账、调整、冻结、红冲、账期 | 完整财务 ERP |
| 资产与物料 | 二期 | SN、库存位置、调拨、消耗、退料和旧件 | 独立大型 WMS |
| 投诉与业务案件 | 二期 | 投诉、责任认定、客服协调和业务闭环 | 全渠道客服平台 |
| AI 辅助 | 后续 | OCR、质量预检、审核辅助、配置草稿 | 无人监督自动决策 |

## 3. 能力依赖规则

- 业务应用可以组合领域能力，不得直接绕过领域能力写平台表；
- 领域能力通过明确接口使用流程、规则、表单和 SLA，不读取其内部存储；
- 平台能力不得知道“比亚迪安装”等具体项目名称；
- OCR、GIS、对象存储和消息中间件由适配层封装，避免领域绑定供应商；
- 报表读取事实表或投影，不直接拼接流程引擎内部表。

## 4. 能力成熟度

每项能力按四级管理：

1. `L1 固定实现`：支持单一验证场景；
2. `L2 参数可配`：关键参数和模板可配置；
3. `L3 策略可扩展`：支持版本、继承、决策表和扩展点；
4. `L4 产品化`：具备模拟、发布治理、监控、解释和跨项目复用。

MVP 的目标不是全部达到 L4。工单、任务、资料、审核、版本和集成需达到 L2；派单、SLA、表单和计价先达到受控的 L2/L3。
