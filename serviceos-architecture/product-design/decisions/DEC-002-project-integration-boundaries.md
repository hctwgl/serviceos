# DEC-002 项目集成规则、系统连接与运行记录分层

状态：Accepted  
适用范围：ServiceOS 外部工单接入、状态同步、结果回传、业务通知和异常处理

## 1. 决策背景

外部协同同时涉及项目业务规则、认证和网络连接、实际消息收发与失败诊断。如果把这些内容放在同一页面或同一版本对象中，会产生以下问题：

- 项目运营人员被迫理解密钥、协议、消息队列和技术报文；
- 修改认证或网络连接可能被误认为需要发布项目履约配置；
- 发布项目规则可能错误改写正在重试的回传内容；
- 技术运行记录被误当成可编辑业务配置；
- 普通用户可能接触认证秘密或敏感原始报文；
- 无法解释某次入站或回传使用了哪个项目规则版本。

## 2. 已接受决策

ServiceOS 将外部协同明确分为三层：

```text
项目履约配置
└── 业务协同规则
    ├── 入站业务范围
    ├── 字段和状态映射
    ├── 回传事件与内容
    ├── 敏感信息规则
    └── 失败业务处理

系统管理
└── 外部连接
    ├── 接口或消息连接
    ├── 认证、密钥和证书
    ├── 网络与安全
    ├── 技术协议和字段目录
    └── 连接健康状态

工单运行数据
└── 外部协同记录
    ├── 实际入站结果
    ├── 实际回传快照
    ├── 发送和确认记录
    ├── 失败与重试记录
    └── 人工处理记录
```

三层对象不得合并为一套配置，也不得互相覆盖。

## 3. 项目履约配置层

项目配置负责表达业务含义：

- 哪些已授权外部连接可以用于当前项目；
- 外部业务如何识别当前项目；
- 哪些外部业务允许创建、更新或请求取消工单；
- 外部字段如何映射为 ServiceOS 业务字段；
- 内外部状态如何映射；
- 哪些工单业务事件需要回传；
- 回传哪些业务字段和资料；
- 哪些敏感信息允许发送；
- 业务失败后如何告警、创建待办或转人工处理。

这些规则进入对应履约方案的草稿并随该方案版本原子发布。

项目页面不得保存认证秘密、完整接口地址、消息主题、技术消息头、队列配置或原始报文。

## 4. 系统连接层

系统管理负责：

- 创建和授权外部连接；
- 维护账号、密码、令牌、密钥和证书；
- 维护网络、白名单和安全策略；
- 定义底层协议和技术字段目录；
- 监测连接健康和技术延迟；
- 提供受控技术诊断。

项目只引用已授权连接的稳定标识和业务能力。连接维护不等于发布履约方案版本，但连接切换或授权变化必须经过明确影响确认。

连接失效时，不允许项目静默使用其他连接或旧凭据兜底。

## 5. 工单运行数据层

每次入站和回传都必须保存：

- 项目和工单；
- 外部协同方；
- 外部业务唯一标识；
- 使用的履约配置版本；
- 使用的字段和状态映射版本；
- 当次业务输入或输出快照；
- 处理结果；
- 对方确认结果；
- 失败和重试记录；
- 人工处理记录。

运行记录是已发生业务事实，不属于履约配置草稿。

已经生成的回传必须冻结当次业务快照和映射。新配置发布后，正在重试的回传仍发送原冻结内容，不按最新配置重新拼装。

## 6. 页面规则

项目履约配置使用以下业务语言：

- 对接概览；
- 入站工单；
- 字段与状态映射；
- 状态与结果回传；
- 场景预览。

普通运营页面不得把以下技术对象作为导航：

- Inbox；
- Outbox；
- Topic；
- Queue；
- Envelope；
- Dead Letter；
- HTTP 状态码列表；
- 异常堆栈。

工单页面可以展示业务协同记录和可解释失败原因。完整报文、内部跟踪标识和技术重试诊断只对具有受控技术权限的人员开放。

## 7. 安全规则

- 认证秘密不进入项目读模型；
- 普通用户不能查看完整原始报文；
- 场景预览只能使用脱敏示例数据；
- 敏感信息不得默认全部回传；
- 外部系统需要负责人信息时，使用工单项目岗位人员快照；
- 不回传项目区域分工内部结构；
- 人工重发不得编辑已冻结业务事实；
- 业务事实需要更正时，必须通过工单业务命令产生新事件。

## 8. 当前阶段不包含

- 通用 API 网关产品；
- 通用消息队列控制台；
- 在项目页面维护认证凭据；
- 任意脚本式报文转换；
- 公共集成模板市场；
- 发布新项目规则后重写历史收发记录；
- 重试时按最新配置重建旧回传内容。

## 9. 验收标准

- 项目业务规则、系统连接和运行记录具有独立责任边界；
- 项目发布不会覆盖认证连接和实际运行记录；
- 系统连接维护不会静默改变项目业务含义；
- 每次入站和回传都能追溯使用的配置和映射版本；
- 待重试回传内容不会因新配置发布而改变；
- 普通运营页面不暴露认证秘密和技术队列对象；
- 失败处理使用可理解的业务状态和明确人工出口。
