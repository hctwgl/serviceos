# DEC-018：ServiceOS 前端设计系统与页面架构决策 V1.0

状态：Accepted

版本：1.0.0

日期：2026-07-24

适用范围：

- ServiceOS Admin Web
- ServiceOS Network Web
- ServiceOS Technician Web
- Vue3 + Vben 前端体系
- ServiceOS Design System
- 页面信息架构
- 业务组件体系


---

# 1. 决策背景

ServiceOS 前端目标不是构建一个普通后台管理系统。

目标：

> 建设一个面向新能源现场履约业务的企业级 SaaS 操作平台。


过去页面容易出现：

- CRUD后台感
- 信息堆叠
- 大量解释文字
- 卡片泛滥
- 缺少任务导向
- 页面围绕数据库对象设计


本决策用于冻结 ServiceOS 前端产品体验标准。


---

# 2. 前端总体架构


采用三层架构：


```
Vben Framework

        ↓

ServiceOS Design System

        ↓

Domain Components

        ↓

Business Pages
```


---

# 3. Vben 定位


Vben 仅作为工程基础。


负责：

- 应用框架
- 路由
- 权限入口
- Layout
- 基础组件
- 构建体系


禁止：

直接依赖 Vben 默认页面风格作为最终产品。


---

# 4. ServiceOS Design System


ServiceOS 必须拥有自己的设计语言。


核心关键词：

- 专业
- 克制
- 高信息密度
- 任务优先
- 企业级
- 长时间使用友好


---

# 5. 视觉设计原则


## 5.1 经典专业任务型


采用：

A+ 经典专业任务型。


特点：

- 浅色工作区
- 清晰侧边导航
- 高密度信息布局
- 状态优先
- 操作明确


禁止：

- 大屏科技风
- 炫酷渐变
- 营销官网风格


---

# 6. 信息架构原则


页面围绕：

用户任务。


不是：

数据库对象。


错误：

```
项目表

方案表

工单表

任务表
```


正确：

```
项目管理

↓

项目工作区


履约方案设计

↓

蓝图设计器


工单执行

↓

履约 Workspace
```


---

# 7. 页面布局规范


标准桌面布局：

```
------------------------------------------------

Sidebar

Header

------------------------------------------------

Workspace


------------------------------------------------
```


支持：

- 1440×900
- 1280×800
- 1920×1080


---

# 8. 页面密度规范


企业软件优先：

信息效率。


禁止：

大量空白。

禁止：

每个字段一个卡片。


推荐：

- 表格
- 时间线
- 状态条
- 工作区
- 侧边栏
- 属性面板


---

# 9. 页面类型规范


ServiceOS 页面分为：


## Workspace 工作区


用于：

日常运营。


例如：

- 项目工作区
- 工单工作区
- 调度中心


特点：

状态 + 风险 + 下一动作。


---

## Designer 设计器


用于：

配置业务规则。


例如：

- Blueprint Designer
- 表单设计器


特点：

左导航 + 中画布 + 右属性。


---

## Management 管理页


用于：

资源管理。


例如：

- 项目列表
- 服务商列表


特点：

搜索 + 筛选 + 高密度列表。


---

# 10. 业务组件规范


禁止：

每个页面独立开发。


必须沉淀领域组件。


例如：


## 项目组件

- ProjectSummaryHeader
- ProjectStatusBadge
- ProjectMetricPanel


## 履约组件

- FulfillmentStageBar
- WorkflowCanvas
- VersionTimeline


## 工单组件

- WorkOrderTimeline
- TaskCard
- EvidenceGallery
- RiskPanel


## 调度组件

- ResourceCandidateCard
- DispatchTimeline


---

# 11. 状态设计规范


状态必须：

颜色 + 文本 + 图标。


例如：


完成：

绿色


进行中：

蓝色


风险：

橙色


异常：

红色


禁止：

只依赖颜色。


---

# 12. 中文业务语言规范


用户界面默认中文。


禁止：

直接暴露技术词。


错误：

- Blueprint Designer
- Workflow Node
- Task Template
- SLA Rule


正确：

- 履约方案设计器
- 流程节点
- 任务模板
- 时效规则


---

# 13. 表单设计规范


动态表单必须：

业务可理解。


禁止：

展示 JSON Schema。


正确：

字段库

↓

表单预览

↓

字段属性


---

# 14. 设计器规范


所有设计器统一：

三栏结构。


```
左：

目录


中：

设计区域


右：

属性配置
```


适用于：

- 履约方案
- 流程
- 表单
- 规则


---

# 15. 页面禁止事项


禁止：

1. 页面堆大量解释文字。

2. 使用大量卡片包装简单字段。

3. 把接口字段直接映射到页面。

4. 中英文混杂。

5. 页面只展示数据，没有下一步动作。

6. 创建没有业务价值的大屏。


---

# 16. Agent 前端开发约束


任何 Agent 开发前端必须遵守：


必须先回答：

- 页面用户是谁？
- 用户打开页面目的是什么？
- 用户下一步动作是什么？
- 页面核心信息是什么？


禁止：

先创建 Vue 文件，再决定产品结构。


---

# 17. 验收标准


页面通过标准：


用户打开页面：

5秒内知道：

- 当前状态
- 当前责任
- 当前风险
- 下一步动作


并可以直接操作。


---

# 18. 最终目标


ServiceOS 前端应该像：

- ServiceNow
- Salesforce
- Jira
- 飞书项目

这样的企业级操作系统。


而不是：

一个新能源行业 CRUD 后台。
