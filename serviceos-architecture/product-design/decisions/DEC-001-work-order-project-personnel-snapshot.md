# DEC-001 工单项目岗位人员快照作为统一人员来源

状态：Accepted  
适用范围：ServiceOS 工单流程、SLA、审核、整改、通知和页面展示

## 1. 决策背景

项目根据工单地址所属行政区域，匹配当前项目对应的：

- 客服经理；
- 项目经理；
- 项目助理。

匹配结果由“项目团队与区域分工”产生，并写入工单关联人员快照。

如果流程、SLA、审核和通知各自再次根据区域查询人员，会产生以下问题：

- 同一工单在不同模块中出现不同负责人；
- 区域分工变更后，历史工单的责任人被静默改变；
- 多套匹配算法难以解释和审计；
- 人员缺失时，不同模块使用不同默认兜底；
- 页面无法说明某个人为什么与工单关联。

## 2. 已接受决策

工单项目岗位人员快照是运行时业务模块获取客服经理、项目经理和项目助理的唯一来源。

```text
项目团队与区域分工
        ↓
按项目 + 标准行政区编码匹配
        ↓
工单项目岗位人员快照
        ↓
流程 / SLA / 审核整改 / 通知 / 页面展示
```

流程、SLA、审核、整改、通知和页面读模型不得自行重新执行行政区域人员匹配。

## 3. 快照内容

每个岗位快照至少记录：

- 岗位；
- 人员 ID；
- 人员显示名；
- 所属组织；
- 联系方式快照；
- 命中的行政区域；
- 匹配方式；
- 是否通过上级区域继承；
- 是否使用备份负责人；
- 分工记录 ID；
- 匹配时间；
- 调整原因；
- 当前状态。

页面只展示业务名称和可解释匹配依据，不展示内部主键。

## 4. 消费规则

### 4.1 流程

流程节点和流转动作可以引用：

- 当前工单客服经理；
- 当前工单项目经理；
- 当前工单项目助理。

运行时直接读取快照，不重新查询区域分工。

### 4.2 SLA

SLA 预警、超时升级、确认待办可以选择上述工单岗位人员作为通知或处理对象。

触发时读取工单当前有效的岗位人员快照。若已有工单通过明确操作调整了关联人员，调整后的后续预警使用新快照；历史通知和历史责任记录不被改写。

### 4.3 审核与整改

审核通知、整改责任协同和复核通知可以引用工单岗位人员快照。

具体审核人或现场整改人仍可来自流程节点、责任网点或已分配师傅，但不得把“项目经理”等岗位重新按区域计算。

### 4.4 通知与外部回传

业务通知可以选择工单岗位人员快照作为收件对象。

通知渠道和外部连接由“集成与回传”管理；人员来源仍为工单快照。外部系统需要接收负责人信息时，回传当次工单快照中的业务信息，不回传区域分工内部结构。

### 4.5 工单页面

工单工作区必须展示：

- 客服经理；
- 项目经理；
- 项目助理；
- 每个岗位的匹配状态；
- 实际命中的行政区域；
- 最近一次人工调整记录。

页面不得每次打开时实时重算人员。

## 5. 人员缺失规则

当业务规则需要某个岗位人员，但快照中没有有效人员时，必须：

1. 明确显示缺失岗位；
2. 将工单标记为“项目人员待确认”；
3. 创建待办给具有项目人员维护权限的用户；
4. 在需要该人员的流程节点、SLA 升级或审核动作中执行已配置的阻塞或人工处理方式。

禁止静默使用：

- 系统管理员；
- 项目创建人；
- 前一节点处理人；
- 最近工单负责人；
- 其他行政区域人员；
- 任意同岗位人员。

## 6. 变更规则

区域分工变化只影响后续新建或明确重新匹配的工单。

已有工单需要换人时，必须执行“调整项目关联人员”操作，并记录：

- 原人员；
- 新人员；
- 调整岗位；
- 调整原因；
- 操作人和操作时间；
- 是否通知相关人员；
- 对未完成任务和后续通知的影响。

不得通过修改区域分工静默改派已有任务。

## 7. 与履约方案版本的关系

- 项目团队与区域分工不是履约方案版本内容；
- 工单岗位人员快照不是履约方案版本内容；
- 流程、SLA、审核和通知规则可以在履约方案版本中引用岗位名称；
- 工单正式受理时匹配并绑定履约方案版本，同时生成当时的岗位人员快照；
- 新履约方案版本发布不修改已有工单人员；
- 新区域分工生效不修改已有工单绑定的履约方案版本。

## 8. 当前阶段不包含

- 项目自定义岗位；
- 通用责任矩阵；
- 动态脚本式候选人规则；
- 各模块独立维护区域人员映射；
- 静默人员兜底；
- 根据区域分工变化自动迁移历史工单。

## 9. 验收标准

- 同一工单在所有模块看到的三个项目岗位人员一致；
- 每个人员关联都能解释匹配区域和匹配方式；
- 历史工单不会因区域分工修改而静默换人；
- 人员缺失时所有模块使用统一的异常状态和处理方式；
- 流程、SLA、审核和通知中不存在第二套行政区域人员匹配实现；
- 人工调整具有完整审计记录。
