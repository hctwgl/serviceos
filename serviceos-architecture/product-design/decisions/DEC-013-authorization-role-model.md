# DEC-013：ServiceOS 权限与角色模型决策 V1.0

状态：Accepted

版本：1.0.0

日期：2026-07-24

## 1. 决策背景

ServiceOS 是多租户、多项目、多组织、多角色履约平台。

权限必须同时控制：

-   能做什么
-   能看什么
-   操作哪些项目
-   操作哪些数据

## 2. 权限模型

采用四层模型：

1.  功能权限
2.  数据权限
3.  领域对象权限
4.  字段权限

## 3. 核心角色

## 平台管理员

范围：

全平台。

权限：

-   创建租户
-   管理模板
-   管理系统配置

## 项目经理

范围：

所属项目。

权限：

-   管理项目
-   配置履约方案
-   查看工单
-   查看项目数据

## 运营人员

范围：

项目运营。

权限：

-   查看履约情况
-   处理异常
-   协调任务

## 服务商管理员

范围：

所属服务网络。

权限：

-   查看分配任务
-   管理工程师

## 调度员

范围：

授权项目和区域。

权限：

-   分配任务
-   调整预约
-   调整资源

## 工程师

范围：

个人任务。

权限：

-   查看自己的任务
-   执行现场服务
-   提交资料

## 审核人员

范围：

审核职责。

权限：

-   审核表单
-   审核证据
-   驳回整改

## 4. 数据权限模型

权限范围：

Tenant

↓

Project

↓

Region

↓

Network

↓

WorkOrder

↓

Task

## 5. 授权原则

后端负责最终授权。

前端权限只用于：

隐藏不可用入口。

禁止：

只依赖前端按钮控制权限。

## 6. Agent 开发约束

禁止：

-   写死角色判断
-   前端模拟权限
-   绕过后端授权

必须：

-   使用统一权限服务
-   遵循数据范围
-   记录关键操作审计

# End of Decision
