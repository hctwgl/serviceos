# DEC-016：ServiceOS 外部集成与回传模型决策 V1.0

状态：Accepted

版本：1.0.0

日期：2026-07-24

## 1. 决策背景

ServiceOS 需要连接车企、CRM、ERP、支付、消息等外部系统。

外部系统负责业务来源和结果接收。

ServiceOS 负责履约执行事实。

核心原则：

外部系统驱动履约，但不控制内部履约模型。

---

## 2. 集成边界

外部系统：

- 创建服务请求
- 提供客户和车辆信息
- 接收履约结果

ServiceOS：

- 匹配履约方案
- 创建工单
- 执行流程
- 生成任务
- 采集现场资料
- 返回结果

---

## 3. 核心链路

```
外部订单

↓

Service Request

↓

WorkOrder

↓

Fulfillment Version

↓

Workflow Runtime

↓

Execution Result

↓

回传外部系统
```

---

## 4. 外部入站模型

外部请求必须经过：

- 身份认证
- 幂等校验
- 数据校验
- 业务匹配

禁止：

外部接口直接创建内部任务。

---

## 5. 状态映射

外部状态：

- 已预约
- 已安装
- 已完成

内部状态：

由 Workflow 和 Task 决定。

必须通过映射层转换。

---

## 6. 回传模型

回传内容：

- 工单状态
- 完成时间
- 现场结果
- 验收结果
- 证据摘要

---

## 7. 可靠性要求

必须支持：

- 幂等
- 重试
- 失败补偿
- 消息追踪
- 人工恢复

---

## 8. Agent 开发约束

禁止：

- 外部状态直接驱动内部流程
- 为每个车企复制一套业务代码
- 集成逻辑侵入履约领域

必须：

- 使用 Connector 边界
- 使用事件驱动
- 保持内部模型稳定

# End of Decision
