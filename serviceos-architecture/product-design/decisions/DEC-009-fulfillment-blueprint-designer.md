# 1. 决策背景

ServiceOS
的核心竞争力不是工单管理，而是将新能源现场服务业务抽象成可配置、可版本化、可模拟执行的履约蓝图。

履约方案不是简单配置页面，而是一套面向履约专家的业务设计工具。

------------------------------------------------------------------------

# 2. Blueprint Designer 定义

Blueprint Designer：

履约蓝图设计器。

负责定义：

一个履约版本如何执行。

关系：

    Fulfillment Plan

    ↓

    Fulfillment Version

    ↓

    Fulfillment Blueprint

一个 Version 对应一个完整 Blueprint。

示例：

    比亚迪王朝勘测安装 V3

    流程：
    客户联系 → 预约 → 勘测 → 审核 → 安装 → 验收

    任务：
    6个

    表单：
    3个

    证据规则：
    5个

    SLA：
    8条

------------------------------------------------------------------------

# 3. 产品定位

Blueprint Designer 不是：

-   普通配置后台
-   低代码平台
-   流程画图工具

它是：

> 面向履约专家的业务设计工作台。

------------------------------------------------------------------------

# 4. 设计原则

## 4.1 业务语言优先

禁止展示：

-   Workflow Node
-   Task Instance
-   Schema JSON
-   Expression

用户语言：

-   流程节点
-   任务模板
-   表单字段
-   规则条件

------------------------------------------------------------------------

## 4.2 配置态与执行态分离

设计器负责定义规则。

工单负责执行规则。

禁止设计器展示大量运行数据。

------------------------------------------------------------------------

## 4.3 一个版本一个完整快照

发布后的版本不可修改。

修改必须创建新版本。

------------------------------------------------------------------------

# 10. Stage 阶段设计

Stage 用于业务理解。

例如：

    客户确认阶段

    现场勘测阶段

    施工安装阶段

    验收交付阶段

-   项目驾驶舱展示
-   工单进度展示
-   用户理解业务过程

# 11. Workflow Node 节点设计

每个节点必须包含：

-   节点名称
-   节点类型
-   责任角色
-   任务模板
-   进入条件
-   完成条件
-   SLA
-   表单
-   证据要求
-   异常策略

节点类型：

-   开始节点
-   任务节点
-   审核节点
-   条件节点
-   等待节点
-   结束节点

------------------------------------------------------------------------

# 12. Task Template 任务模板

任务模板定义：

谁负责什么。

包含：

-   任务名称
-   责任角色
-   执行主体
-   输入数据
-   输出结果
-   关联表单
-   关联证据
-   SLA
-   完成条件
-   异常策略

示例：

    现场勘测任务

    责任角色：
    工程师

    输入：
    客户地址

    输出：
    勘测结果

    关联：
    勘测表

    证据：
    现场照片

    SLA：
    24小时

------------------------------------------------------------------------

# 13. Form Template 表单设计器

定义现场采集数据结构。

布局：

    字段库 | 表单预览 | 字段属性

字段类型：

-   文本
-   数字
-   选择
-   日期
-   图片
-   附件
-   签名

字段属性：

-   字段名称
-   显示名称
-   类型
-   是否必填
-   校验规则
-   默认值
-   显示条件

------------------------------------------------------------------------

# 14. Evidence Rule 证据规则

定义任务完成证明。

示例：

安装任务：

必须：

-   设备铭牌照片
-   安装位置照片
-   施工照片
-   测试视频
-   客户确认

------------------------------------------------------------------------

# 15. SLA Policy 时效规则

定义时间约束。

示例：

安装资料提交：

目标：

1小时

预警：

45分钟

升级：

90分钟

支持：

-   目标时间
-   预警
-   升级
-   通知对象

------------------------------------------------------------------------

# 16. Settlement Rule 结算规则

定义商业计算规则。

示例：

    安装完成

    基础费用

    +

    区域补贴

    +

    质量奖励

    -

    超时扣减

一期支持配置模型。

------------------------------------------------------------------------

# 19. Blueprint 与 WorkOrder 边界

Blueprint：

回答：

"应该怎么执行？"

WorkOrder：

回答：

"这一次实际执行怎么样？"

------------------------------------------------------------------------

Blueprint 页面禁止展示：

-   当前工单列表
-   完成率
-   工程师实时状态
-   当前异常数量

这些属于：

工单管理和运营分析。

------------------------------------------------------------------------

# 20. Agent 开发约束

禁止：

1.  将 Blueprint 做成普通表单页面。
2.  将流程做成静态阶段列表。
3.  将任务做成字段展示。
4.  将版本做成修改日志。
5.  将运行数据混入配置页面。

必须：

1.  Blueprint 是设计工具。
2.  Version 是冻结快照。
3.  Workflow 是执行骨架。
4.  Task/Form/SLA/Evidence 是节点能力。
5.  发布后不可修改。

------------------------------------------------------------------------

# 21. 页面验收标准

打开履约方案设计器时：

用户应该感觉：

"我正在设计一个新能源服务流程。"

而不是：

"我正在编辑数据库配置。"

验收重点：

-   是否可以理解流程
-   是否可以配置任务
-   是否可以配置表单
-   是否可以配置 SLA
-   是否可以管理版本
-   是否可以模拟执行
