# 项目履约配置的信息架构与页面关系

状态：Accepted  
适用范围：ServiceOS Admin 项目管理与履约配置  
视觉基线：A+ 经典专业任务型

> 说明：文件名保留历史名称，但“配置中心”不再是正式产品概念，也不作为 Admin 一级菜单。

## 1. 已确认领域关系

ServiceOS 当前阶段采用以下关系：

```text
客户
└── 项目
    └── 履约配置
        └── 一个履约工作流
```

- 项目是实际履约的业务对象；
- 履约配置是项目的一部分，不是独立项目，也不是第二套项目主数据；
- 配置页面只是编辑项目履约规则的工作区，不拥有项目；
- 一个项目只有一份当前履约配置；
- 一个项目当前只支持一个履约工作流；
- 履约配置可以存在一个生效版本和一个编辑中的草稿版本；
- 工单在创建时绑定当时的项目履约配置版本。

禁止在当前阶段提前支持项目内多流程、流程市场或公共规则中心。

## 2. 取消“配置中心”一级菜单

Admin 主导航不设置“配置中心”。

项目履约配置的权威入口为：

```text
客户与项目
└── 项目列表
    └── 项目详情
        └── 履约配置
```

项目列表仍然是项目主数据的唯一列表。不得另建“项目配置列表”，避免用户在两套项目列表之间判断差异。

跨项目配置待办放入 Admin 工作台，例如：

- 待首次配置；
- 存在未发布草稿；
- 校验失败；
- 等待发布；
- 发布失败；
- 最近完成发布。

点击待办后直接进入对应项目详情的“履约配置”，不经过独立配置中心。

## 3. 项目详情信息架构

项目详情至少包含：

1. 项目概览；
2. 服务与合同；
3. 项目成员；
4. 履约配置；
5. 项目工单；
6. 操作记录。

“履约配置”是项目详情中的业务页签。页面顶部必须持续展示项目名称、客户、项目状态和当前查看的配置版本，确保用户始终知道自己正在配置哪个项目。

## 4. 履约配置工作区

进入项目详情的“履约配置”后，默认打开配置概览。

工作区左侧采用统一浅色导航，只保留以下项目内业务模块：

1. 配置概览；
2. 服务范围；
3. 流程设计；
4. 表单与资料；
5. SLA 与预约；
6. 角色与责任；
7. 网点与资源；
8. 审核与整改；
9. 集成与回传；
10. 版本与发布。

页面顶部固定展示：

- 项目名称；
- 项目状态；
- 当前生效配置版本；
- 当前草稿版本；
- 配置完整度；
- 校验状态；
- 最近保存时间；
- 当前首要动作。

用户不能在工作区内随意切换项目。切换项目应返回项目列表或使用明确的项目切换入口，并在离开前处理未保存修改。

## 5. 配置概览

配置概览负责回答：

- 当前项目能否开始接单和履约；
- 哪些配置尚未完成；
- 当前生效配置是什么；
- 是否存在未发布草稿；
- 哪些错误阻止发布；
- 发布后会影响什么；
- 用户下一步应处理什么。

必须展示：

- 配置完整度；
- 九个业务配置模块的完成状态；
- 生效版本摘要；
- 草稿变更摘要；
- 阻塞错误；
- 重要警告；
- 最近操作；
- 推荐下一步。

配置概览不直接承载复杂编辑。

## 6. 服务范围

服务范围定义项目提供什么服务、服务适用于什么区域和业务条件。

内部内容包括：

- 服务项目和业务类型；
- 服务区域；
- 适用条件；
- 范围校验；
- 变更影响。

服务区域采用已确认的行政区规则和地图交互，不向运营用户暴露行政区内部数据结构。

## 7. 流程设计

### 7.1 一个项目一个工作流

点击“流程设计”后，直接进入该项目唯一的履约流程设计工作区。

当前阶段不提供：

- 流程列表；
- 新建多个流程；
- 复制流程；
- 停用某个流程；
- 项目内流程切换；
- 独立流程发布。

当项目尚未配置流程时，工作区展示空状态并提供“开始设计流程”。

### 7.2 统一流程设计工作区

工作区包含：

1. 流程画布；
2. 节点配置；
3. 流转规则；
4. 关联表单和资料；
5. 权限与处理人；
6. SLA 规则；
7. 校验与影响提示。

点击节点后，在右侧属性面板编辑节点；点击连线后，在右侧属性面板编辑流转规则。它们不是独立菜单或独立首页。

### 7.3 节点配置

节点至少包含：

- 节点名称；
- 节点类型；
- 责任角色；
- 完成条件；
- 关联表单；
- 必要资料；
- SLA 或时限；
- 可执行动作；
- 跳过、撤回和回退规则；
- 业务说明。

### 7.4 流转规则

每条连线都必须有明确流转规则：

- 来源节点；
- 目标节点；
- 触发动作；
- 进入条件；
- 条件组合关系；
- 表单完整性要求；
- 资料完整性要求；
- 可执行角色；
- 状态变化；
- 自动分配；
- 通知动作；
- 超时处理；
- 是否允许撤回或回退；
- 多规则命中优先级；
- 无规则命中时的明确中文错误。

“流线规则”不是正式术语，统一使用“流转规则”。

## 8. 表单与资料

“表单与资料”管理当前项目履约过程中使用的表单和资料要求。

默认管理首页包含两个页签：

- 表单；
- 资料要求。

页面关系：

```text
表单列表
→ 表单详情（默认只读）
→ 新建表单或编辑草稿
→ 表单设计器
```

不得进入模块后直接打开字段编辑器，也不得把列表和编辑器挤在同一页面。

当前阶段的表单和资料要求归属于项目履约配置，不建设跨项目公共模板库。

## 9. SLA 与预约

内部内容包括：

- SLA 指标；
- 时间计算规则；
- 触发条件；
- 通知和升级；
- 预约规则；
- 校验与影响。

所有规则都归属当前项目履约配置草稿，不独立发布。

## 10. 角色与责任

本模块只管理项目履约责任，不替代系统级用户和 RBAC 管理。

内部内容包括：

- 项目参与角色；
- 责任矩阵；
- 节点处理人规则；
- 可执行动作；
- 数据可见范围；
- 责任冲突校验。

## 11. 网点与资源

内部内容包括：

- 项目服务覆盖；
- 网点准入条件；
- 师傅能力要求；
- 服务资源要求；
- 产能约束；
- 覆盖与资源校验；
- 变更影响。

## 12. 审核与整改

内部内容包括：

- 审核策略；
- 审核任务规则；
- 审核结论；
- 整改类型；
- 整改时限；
- 整改资料要求；
- 复核规则；
- 通知和升级。

审核和整改规则可被流程节点引用，但不在流程画布中重复维护完整规则内容。

## 13. 集成与回传

本模块使用业务协同语言，不把 Inbox、Outbox、Envelope 等技术对象作为普通导航。

内部内容包括：

- 入站工单规则；
- 外部字段映射；
- 结果回传规则；
- 状态同步；
- 失败处理；
- 对接校验；
- 最近协同记录。

技术诊断进入独立受控诊断工具，不进入项目履约配置页面。

## 14. 版本与发布

### 14.1 配置版本是整体版本

一个项目的履约配置版本整体包含：

- 服务范围；
- 唯一工作流；
- 表单与资料；
- SLA 与预约；
- 角色与责任；
- 网点与资源；
- 审核与整改；
- 集成与回传。

工作流、表单和 SLA 在当前阶段不独立发布。发布动作只发生在项目履约配置层级。

### 14.2 草稿与生效版本

每个项目最多存在：

- 一个当前生效版本；
- 一个正在编辑的草稿版本。

生效版本不可修改。新修改基于最新生效版本创建草稿，发布后形成新的不可变履约配置版本。

### 14.3 发布步骤

1. 保存草稿；
2. 执行跨模块完整性校验；
3. 预览履约结果；
4. 查看新旧版本业务差异；
5. 执行影响分析；
6. 填写发布说明；
7. 最终确认；
8. 发布；
9. 展示发布结果。

不得跳过校验和影响分析直接发布。

### 14.4 工单版本绑定

工单创建时绑定当时的项目履约配置版本。

项目发布新版本后：

- 新工单使用新版本；
- 进行中工单继续使用创建时绑定的版本；
- 当前阶段不提供自动迁移存量工单的兼容机制。

## 15. 页面层级规则

统一遵守：

- 项目列表是项目的唯一权威列表；
- 履约配置从项目详情进入；
- Admin 工作台只提供跨项目配置待办和深链接；
- 模块默认先展示概览或列表；
- 详情通过列表进入；
- 编辑器通过“新建”或“编辑草稿”进入；
- 节点和连线通过同一流程工作区的属性面板编辑；
- 版本对比、影响分析和发布采用明确步骤；
- 面包屑必须反映项目上下文；
- 返回时保留筛选、分页和滚动位置。

## 16. 当前阶段明确废弃

- Admin 一级菜单“配置中心”；
- 第二套“项目配置列表”；
- 项目内流程列表；
- 一个项目多个工作流；
- 独立流程版本和独立流程发布；
- 深色履约配置导航；
- `流程列表 / 流程版本 / 节点设计 / 流线规则` 同级菜单；
- 独立节点设计首页；
- 多张重复节点编辑页面；
- 履约配置首页直接展开表单编辑器；
- 表单列表与编辑器同时塞入一个首页；
- 技术 JSON、内部 ID 和英文枚举直接展示给运营人员。

此前生成的相关图片仅作为探索记录，不构成产品事实源。

## 17. 下一期候选：公共模板

公共模板不进入当前阶段范围。

下一期在出现明确跨项目复用需求后，可评估：

- 流程模板；
- 表单模板；
- 资料要求模板；
- SLA 模板；
- 通知模板；
- 公共业务字典。

公共模板应放在“系统管理 / 公共模板”一类明确入口中，不恢复含义模糊的“配置中心”。

4. 表单列表、表单详情和表单设计器；
5. 资料要求列表和详情；
6. SLA 与预约；
7. 角色与责任；
8. 网点与资源；

10. 集成与回传；

在页面规格和领域关系未确认前，不继续批量生成高保真图片。

## 19. 验收标准

- Admin 主导航中不存在“配置中心”；
- 项目只有一个权威列表；
- 履约配置明确归属于项目；
- 一个项目当前只支持一个工作流；
- 工作流不独立发布；
- 项目履约配置作为完整不可变版本发布；
- 工单绑定创建时的履约配置版本；
- 表单、资料和 SLA 当前归属项目，不依赖公共模板；
- 公共模板明确延期到下一期；
- 所有页面使用统一浅色导航和 A+ 产品语言；
- 后续规格、读模型和实现均以本文件为事实源。
