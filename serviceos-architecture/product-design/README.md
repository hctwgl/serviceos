# ServiceOS 产品设计基线

状态：Accepted

本目录是 ServiceOS 所有用户界面和交互设计的唯一事实源。

## 已确认方向

ServiceOS 采用 **A+：经典专业任务型** 产品设计方向：

- 以经典企业级视觉为基础；
- 以用户任务、责任、风险和下一步动作组织页面；
- 保持适合长期高频使用的中高信息密度；
- Admin、网点端和原生 iOS 师傅端使用统一产品语言，但不强行使用相同布局；
- 不以数据库表、接口或技术对象组织产品页面；
- 不显示原始 UUID、英文枚举、内部字段、调试数据和技术上下文；
- 不为旧页面、旧路由和旧交互保留兼容设计。

## 首批设计范围

1. Admin 工作台；
2. Admin 工单中心；
3. 工单工作区；
4. 派单与改派流程；
5. 项目履约配置中心；
6. 流程配置与版本发布。

## 目录说明

- `01-design-baseline.md`：A+ 设计原则和通用页面模式；
- `02-screen-inventory.md`：页面清单、设计等级和完成状态；
- `specifications/`：页面与流程的正式产品规格；
- `prototypes/`：经确认的高保真原型及说明；
- `decisions/`：产品设计过程中的已确认决策。

## 完成定义

页面只有同时满足以下条件，才可以进入开发：

- 明确目标角色和核心任务；
- 明确信息优先级和主要动作；
- 已定义正常、空、错误、无权限、只读和冲突状态；
- 已定义页面读模型和业务命令；
- 已完成高保真原型或明确引用已接受的页面模式；
- 已通过产品负责人明确确认；
- 不包含演示、调试或兼容性设计。
