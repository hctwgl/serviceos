# 通用管理页面模式

状态：Accepted

## 1. 文档目的

本文件定义 ServiceOS Admin 中 B 级和 C 级管理页面的统一产品模式。

它解决以下问题：

- 列表、详情、新建、编辑、状态变更和记录查询采用一致结构；
- 普通管理页面不再由开发人员根据接口字段临时拼装；
- 业务名称、中文状态、权限、错误和并发冲突有统一表达；
- 页面可以复用设计系统，但不同领域仍保留自己的业务规则和命令；
- B 级、C 级页面拥有明确完成门禁，不因“只是管理页”退化成调试控制台。

本文件不是通用 CRUD 引擎设计。用户、项目、网点、师傅、外部连接等对象拥有不同状态机、权限、审计和安全边界，不得通过一个动态元数据页面统一增删改查。

## 2. 适用范围

适用于：

- 项目目录和项目概览；
- 车企与品牌主数据；
- 用户目录和用户详情；
- 系统角色、授权与委托；
- 企业组织和合作组织；
- 服务网点目录和详情；
- 师傅档案目录和详情；
- 服务目录；
- 外部连接目录和安全摘要；
- 消息中心；
- 审计和操作记录。

不适用于：

- Admin 工作台；
- 工单中心和工单工作区；
- 派单与改派；
- 项目履约配置；
- 流程设计器；
- 审核与整改工作区；
- 网点履约工作台；
- 原生 iOS 现场作业；
- 其他已经定义为 A 级的复杂、高频或高风险工作区。

## 3. 基本原则

### 3.1 页面围绕管理任务，不围绕数据库表

页面必须回答：

- 用户正在管理什么业务对象；
- 当前最需要判断什么；
- 哪些状态会影响业务运行；
- 当前允许执行什么动作；
- 动作会产生什么业务影响；
- 发生冲突或失败后如何恢复。

不得把数据库字段、内部聚合、技术资源和接口逐一变成菜单。

### 3.2 一个对象只有一个权威入口

- 项目列表是项目主数据的唯一权威列表；
- 用户目录是人员账号治理的权威入口；
- 服务网点目录是网点主数据入口；
- 师傅档案是师傅能力与资格入口；
- 外部连接目录是连接安全和技术状态入口。

工作台、选择器、工单和项目详情只能深链接到权威页面，不能复制第二套可编辑主数据。

### 3.3 业务事实由服务端提供

前端可以格式化时间、金额和布局，但不能：

- 调用多个领域接口后自行 Join 业务对象；
- 根据 ID 猜测显示名；
- 根据枚举猜测中文状态；
- 根据页面状态推导权限；
- 根据当前时间自行判断业务状态；
- 把查询失败显示成空集合或零；
- 用旧缓存覆盖最新权威结果。

### 3.4 不做技术字段兜底

禁止：

```text
名称缺失 → 显示 UUID
中文标签缺失 → 显示英文枚举
组织缺失 → 显示组织编码
查询失败 → 显示 0
权限不足 → 显示空白
未知状态 → 显示“正常”
```

必要业务显示值缺失时，页面进入明确的“数据不完整”状态并提供问题位置，不泄露内部标识顶替。

## 4. 页面模式总览

```text
目录列表
    → 对象概览 / 对象详情
        → 新建或编辑页
        → 状态变更确认
        → 关系管理页或抽屉
        → 变更记录

选择器
    → 复用同一权威目录查询语义

审计或运行记录
    → 只读查询与详情
```

通用页面可以复用骨架，但每个业务规格仍必须定义自己的字段、筛选、状态、动作、权限和命令。

## 5. 目录列表模式

### 5.1 页面职责

目录列表负责：

- 查找对象；
- 按业务条件筛选；
- 判断状态和风险；
- 进入对象详情；
- 发起允许的创建或局部管理动作。

目录列表不负责：

- 展开完整对象详情；
- 在表格中编辑复杂关系；
- 在单元格中直接修改状态；
- 展示所有数据库字段；
- 承担跨领域业务工作流。

### 5.2 标准布局

```text
面包屑
→ 页面标题、业务说明和唯一主操作
→ 必要的状态摘要或风险提示
→ 已保存视图 / 常用业务视图
→ 搜索和筛选栏
→ 已选筛选条件
→ 批量操作区（仅业务允许时）
→ 数据表格
→ 分页、数据时间和刷新状态
```

没有业务价值的指标卡不得为了“看起来像工作台”强行加入目录页。

### 5.3 搜索

搜索框必须说明支持的业务字段，例如：

> 搜索项目名称或项目编码

不得用“请输入关键词”掩盖实际能力，也不得声称支持接口没有实现的模糊搜索。

搜索行为：

- 输入防抖但支持立即回车；
- 查询条件写入 URL；
- 返回详情后恢复原筛选、分页和滚动位置；
- 清空搜索恢复当前业务视图，不清除无关筛选；
- 查询失败保留用户输入并允许重试。

### 5.4 筛选

默认筛选只放最常用的三至五项。低频条件进入“更多筛选”。

筛选器使用业务实体选择器和中文状态，不允许输入：

- UUID；
- Tenant ID；
- Project Scope 表达式；
- Capability code；
- 数据库字段名；
- 原始枚举；
- 自由 SQL 或 JSON。

筛选条件必须由服务端查询契约支持。前端不能先加载一页结果后进行“全局筛选”。

### 5.5 表格

表格默认支持：

- 固定表头；
- 明确列宽和溢出策略；
- 业务名称完整查看；
- 中文状态和风险说明；
- 行点击进入详情；
- 独立操作列；
- 服务端分页；
- 数据生成时间；
- 返回位置恢复；
- 键盘导航和读屏标签。

表格不默认支持：

- 任意单元格内联编辑；
- 任意列排序；
- 任意字段导出；
- 全选整个数据集；
- 未经业务确认的批量状态变更。

### 5.6 行操作

每行只展示一个最常用操作和“更多”菜单。操作来自服务端 `allowedActions` 或等价权威能力，不根据状态文本自行拼装。

高风险操作必须进入详情或确认流程，不能在列表中一键执行。

## 6. 对象概览与详情模式

### 6.1 对象头部

对象头部至少包含：

- 业务名称；
- 业务编号或编码；
- 中文状态；
- 上级业务对象；
- 关键生效信息；
- 首要风险或数据完整性提示；
- 当前允许的主要动作；
- “更多”中的低频动作。

UUID、资源版本、内部类型和技术码只允许在受权限控制的诊断信息中显示，并支持复制；普通页面不显示。

### 6.2 详情信息层级

```text
对象概览
├── 基本信息
├── 业务关系
├── 当前状态与影响
├── 相关业务对象
├── 运行或使用摘要
└── 操作记录
```

只有真正需要独立导航的内容才使用页签。少量字段使用区块，不为每个接口创建一个页签。

### 6.3 详情页动作

- 主要动作放在对象头部；
- 关系调整在对应区块内发起；
- 编辑动作进入独立页面或业务抽屉；
- 状态变化显示影响和原因；
- 所有写操作使用并发版本、幂等和服务端授权；
- 操作成功后刷新权威对象，不在前端直接篡改最终状态。

## 7. 新建和编辑模式

### 7.1 独立页面与抽屉边界

使用独立页面：

- 字段分多个业务区块；
- 需要解释业务影响；
- 包含关系、范围、生效时间或安全配置；
- 提交前需要校验或预览；
- 用户可能停留较长时间；
- 成功后需要进入新对象详情。

使用抽屉或对话框：

- 只修改一个局部、低风险属性；
- 上下文必须与当前详情同时可见；
- 字段数量少且不包含复杂依赖；
- 失败后可以在原上下文直接恢复。

不根据固定字段数量机械决定形式。

### 7.2 表单原则

- 字段使用中文业务名称和完整说明；
- 必填、选填和只读明确；
- 实体关系通过选择器选择；
- 日期说明生效语义和时区；
- 编码说明生成、唯一性和修改规则；
- 敏感字段说明保存和展示范围；
- 服务端错误定位到具体字段或业务区块；
- 未保存离开提供明确确认；
- 保存失败保留用户输入；
- 不自动填入猜测值通过校验。

### 7.3 创建结果

创建命令必须幂等。成功后进入新对象详情并展示平台确认结果；响应丢失时查询原幂等结果，不能再次创建。

重复业务编码、失效关系、权限变化和并发冲突必须分别解释。

### 7.4 编辑并发

编辑页面基于对象版本或 ETag。提交时版本变化：

- 不使用最后写入覆盖；
- 展示平台已经变化的内容；
- 保留用户未提交输入；
- 要求刷新、重新确认或转业务冲突流程；
- 不允许前端静默重新提交。

## 8. 状态变更模式

状态变更不是普通布尔开关。

每个领域规格必须说明：

- 当前状态；
- 目标状态；
- 允许角色；
- 前置条件；
- 对现有业务的影响；
- 是否可恢复；
- 是否需要原因；
- 是否需要二次确认；
- 成功后的平台事实；
- 失败后的恢复方式。

通用页面不能自行定义“启用、停用、关闭、删除”的状态机。没有权威命令和已确认语义时，只展示状态，不生成操作按钮。

## 9. 关系管理模式

项目与区域、项目与网点、用户与组织、角色与成员、网点与师傅等关系必须通过明确业务命令调整。

关系调整页面必须：

- 显示当前有效关系；
- 使用业务实体选择器；
- 说明新增和移除的影响；
- 展示生效时间或当前版本；
- 使用整体修订或明确增删命令；
- 保存不可变变更记录；
- 并发变化时失败关闭；
- 不允许输入 ID 或批量粘贴未知编码绕过校验。

## 10. 选择器模式

选择器是权威目录的受限视图，不是第二套目录。

必须满足：

- 只返回当前用户、当前上下文和当前命令可选择的对象；
- 显示业务名称、必要上级关系和可选状态；
- 不可选项说明原因，或由服务端直接排除并返回范围说明；
- 支持服务端搜索与分页；
- 已停用对象不会因为旧缓存继续可选；
- 选择结果提交后服务端再次校验；
- 名称缺失时不显示 ID 兜底。

## 11. 记录与审计模式

变更记录和审计页面只读，至少显示：

- 发生时间；
- 操作者业务名称和身份类型；
- 业务动作；
- 目标对象；
- 结果；
- 变更摘要；
- 失败或拒绝原因；
- 关联业务对象；
- 可授权查看的技术诊断入口。

普通用户看到业务动作，不直接看到原始事件名、数据库表名、JSON、异常堆栈或技术 payload。

审计记录不能提供“编辑历史”或“删除记录”。敏感字段按查看权限脱敏；导出必须有独立能力和审计。

## 12. 页面状态

每个管理页面必须覆盖：

### 12.1 首次加载

显示稳定骨架，不闪现“暂无数据”。

### 12.2 业务空状态

说明为什么为空以及正常下一步，例如：

> 当前还没有项目。创建项目后，可以配置服务范围和履约规则。

只有拥有创建权限时才显示创建按钮。

### 12.3 筛选无结果

保留筛选条件，提供清除或调整筛选，不显示初始化演示数据。

### 12.4 查询失败

显示失败来源、影响范围和重试动作。部分区块失败时其他成功区块可以继续展示，但不得把失败值显示为零或空。

### 12.5 无权限

区分：

- 无权访问页面；
- 可查看但不可编辑；
- 可查看对象但无权查看敏感区块；
- 有页面权限但数据范围为空。

前端隐藏按钮不替代服务端授权。

### 12.6 数据不完整

必要业务名称、状态标签或关系缺失时，显示“页面数据不完整”及问题区块，不用技术字段补位。

### 12.7 并发冲突

保留用户输入，展示最新平台事实和恢复动作，不使用最后写入覆盖。

## 13. 数据获取与刷新

核心目录和详情应由页面级 Read Model 支撑。

列表读模型至少提供：

```text
ManagementDirectoryView
├── querySummary
├── items[]
├── nextCursor or pageSummary
├── filterOptions
├── dataAsOf
├── partialAvailability
└── allowedActions
```

详情读模型至少提供：

```text
ManagementObjectView
├── objectHeader
├── businessSections[]
├── relationSummaries[]
├── usageSummary
├── riskSummary
├── recentChanges
├── dataAsOf
└── allowedActions
```

这两个结构是产品语义模板，不是要求所有领域返回相同 JSON。

刷新规则：

- 页面明确数据生成时间；
- 用户写操作成功后刷新权威区块；
- 选项和列表使用相同业务名称来源；
- 授权变化后旧游标和旧选择失败关闭；
- 部分来源不可用时返回结构化可用性，不伪装完整结果；
- 不在前端持久化第二套主数据。

## 14. 权限与数据范围

每个页面必须同时定义：

- 页面访问能力；
- 对象数据范围；
- 详情敏感区块能力；
- 创建、编辑和状态命令能力；
- 导出能力；
- 审计查看能力。

列表、总数、筛选选项、选择器和导出必须使用同一授权边界。不能出现：

- 列表无权看到对象，但筛选项泄露名称；
- 页面不显示对象，但总数泄露数量；
- 用户无权编辑，却可以调用隐藏命令；
- 跨 Tenant 返回 403 泄露对象存在；
- Project、Region 或 Network Scope 只在前端过滤。

## 15. 批量操作

批量操作默认关闭。只有业务规格明确证明以下条件时才启用：

- 所有选中对象使用同一命令语义；
- 每个对象都单独通过服务端授权和状态校验；
- 页面能解释共同影响；
- 支持逐项结果和部分失败；
- 不为批量便利放宽领域规则；
- 不使用“全部数据”绕过分页和范围授权。

批量导出、启停、关系调整和权限变更都必须独立评审，不能因为表格支持多选就自动获得。

## 16. 导入与导出

导入、导出不属于所有目录的默认能力。

确需提供时必须定义：

- 业务用途；
- 字段和版本；
- 数据范围；
- 敏感信息处理；
- 文件安全扫描；
- 预校验和错误报告；
- 幂等与重复语义；
- 部分成功；
- 审计和保留期限。

不得直接导出页面背后的完整 DTO、内部 ID、权限码或技术状态。

## 17. 中文产品语言

普通页面必须使用统一中文业务表达：

```text
ACTIVE → 使用中的业务名称，例如“服务中”或“已启用”
SUSPENDED → 根据领域显示“已暂停”，并说明影响
CLOSED → 根据领域显示“已结束”或“已关闭”
Capability → 可执行操作
Scope → 数据范围或服务范围
Principal → 用户或服务身份
```

同一个英文枚举在不同领域可能具有不同中文业务含义，不能建立一个全局机械翻译表。领域规格必须定义最终标签和说明。

## 18. 视觉与密度

- 统一使用 A+ 浅色专业基线；
- 左侧菜单保持浅色，不使用深色主题；
- 页面背景为浅蓝灰，业务内容区为白色；
- 目录页采用中高密度，详情页采用中等密度；
- 主要容器优先使用边框，减少无意义阴影；
- 状态色只表达真实语义，不为每个标签分配颜色；
- 高风险红色只用于需要立即处理或不可逆动作；
- 表格和表单在 1280px 与 1440px 宽度下可用。

## 19. 无障碍与键盘

- 页面标题、筛选、表格和分页具有正确语义层级；
- 所有操作可通过键盘访问；
- 焦点顺序与视觉顺序一致；
- 表格行操作具有对象名称上下文；
- 状态和风险不只依赖颜色或图标；
- 动态字体或浏览器 200% 缩放后不遮挡主操作；
- 对话框关闭后焦点返回触发点；
- 查询更新和错误可以被读屏感知但不过度播报。

## 20. B 级页面规格模板

每个 B 级页面必须写明：

1. 页面定位和目标角色；
2. 权威入口与导航关系；
3. 业务对象和非目标；
4. 搜索、筛选和业务视图；
5. 列表字段与排序；
6. 对象头部和详情区块；
7. 新建与编辑字段；
8. 状态和允许动作；
9. 权限与数据范围；
10. 正常、空、错、无权限、数据不完整和并发冲突；
11. 页面级 Read Model；
12. 命令、幂等和版本；
13. 中文文案；
14. 验收标准；
15. 明确禁止实现。

缺少以上任一关键部分时，不能仅凭复用页面模板进入开发。

## 21. C 级页面规格模板

每个 C 级页面至少写明：

1. 使用角色和查询目的；
2. 数据来源和保留范围；
3. 筛选、列和详情；
4. 权限与敏感信息；
5. 空、错和不可用状态；
6. 导出或深链边界；
7. 基本验收标准。

C 级表示交互复杂度较低，不表示可以暴露技术字段或跳过授权、中文和可访问性。

## 22. 通用完成门禁

- [ ] 页面有明确业务对象和权威入口；
- [ ] 没有复制第二套可编辑主数据；
- [ ] 搜索、筛选、排序与真实服务端查询一致；
- [ ] 不显示 UUID、原始枚举、字段名和技术对象；
- [ ] 必要显示值缺失时不使用 ID 兜底；
- [ ] 操作来自服务端权限和状态判断；
- [ ] 写操作具有幂等、版本和并发冲突处理；
- [ ] 查询失败不显示为空或零；
- [ ] 业务空、筛选无结果、无权限、只读、错误和数据不完整状态完整；
- [ ] 批量、导入和导出只有经过业务确认才出现；
- [ ] 中文文案说明业务影响和恢复方式；
- [ ] 1280px、1440px、键盘和 200% 缩放可用；
- [ ] 产品规格与页面级 Read Model、命令契约没有语义冲突；
- [ ] 产品负责人完成业务审查。

## 23. 禁止实现

禁止：

- 建立通用动态 CRUD 页面替代领域设计；
- 把所有数据库表自动生成菜单；
- 列表加载多个目录接口后在前端 Join；
- 名称缺失时显示 ID；
- 中文状态缺失时显示英文枚举；
- 查询失败时显示空集合或零；
- 根据前端状态推导权限；
- 在单元格中直接编辑复杂关系或状态；
- 未经业务确认提供批量启停、批量删除或全量导出；
- 使用物理删除作为通用“删除”动作；
- 并发冲突时最后写入覆盖；
- 保存失败后清空用户输入；
- 无权查看对象时通过总数、筛选项或选择器泄露；
- 将技术日志、原始 JSON 和异常堆栈展示给普通业务用户；
- 为旧页面、旧路由、旧 DTO 或旧开发数据保留兼容模式；
- 因页面属于 B 级或 C 级而跳过产品审查。
