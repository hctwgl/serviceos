# AD-002 Admin 工单中心

状态：Draft for Product Review  
设计等级：A

## 1. 目标用户

- 平台运营人员；
- 项目运营人员；
- 审核及异常处理人员；
- 具有跨项目查询权限的管理人员。

## 2. 核心任务

- 快速找到需要处理的工单；
- 按责任、阶段、状态、风险和时间筛选；
- 保存常用工作视图；
- 批量执行允许的业务操作；
- 进入工单工作区继续处理。

## 3. 页面结构

### 3.1 业务视图

默认提供：

- 全部工单；
- 我的待办；
- SLA 风险；
- 异常阻塞；
- 等待外部处理；
- 无人负责；
- 已完成。

视图是业务语义，不是单个数据库字段筛选。

### 3.2 筛选区域

首行保留高频筛选：

- 项目；
- 客户；
- 工单阶段；
- 业务状态；
- 责任网点；
- 责任师傅；
- 预约时间；
- SLA 风险。

低频条件进入“更多筛选”。

### 3.3 保存视图

用户可保存：

- 筛选条件；
- 排序；
- 显示列；
- 列顺序；
- 分页大小。

系统默认视图不能被普通用户覆盖。

### 3.4 工单表格

默认列：

- 工单编号；
- 客户；
- 项目；
- 当前阶段；
- 责任网点；
- 责任师傅；
- 预约时间；
- SLA 风险；
- 业务状态；
- 操作。

操作列仅显示当前行允许执行的主要动作和“更多”。

### 3.5 批量操作

根据所选工单交集中的 `allowed-actions` 决定可执行动作。

必须支持：

- 全部成功；
- 部分成功；
- 全部失败；
- 处理中断；
- 刷新后重新计算允许动作。

## 4. 交互规则

- 点击工单编号进入工单工作区；
- 点击责任对象可打开只读摘要，不直接跳到技术详情页；
- 点击 SLA 风险显示计算依据和截止时间；
- 不允许执行的动作必须解释原因；
- 表格加载失败不得以空表替代。

## 5. 页面读模型

建议使用：`WorkOrderDirectoryView`

必须包含：

- `scope`；
- `views`；
- `activeView`；
- `filterOptions`；
- `items`；
- `pagination`；
- `columnDefinitions`；
- `allowedBatchActions`；
- `generatedAt`。

每个 `item` 至少包含：

- 工单标识和业务编号；
- 客户、项目、网点和师傅的名称；
- 当前阶段中文名称；
- 当前状态中文名称；
- 预约时间；
- SLA 等级、说明和截止时间；
- 行级允许动作。

前端不得额外请求项目、网点和师傅目录来拼装显示名称。

## 6. 状态

必须设计：

- 正常有数据；
- 当前视图无结果；
- 当前筛选无结果；
- 无查询权限；
- 部分筛选项不可用；
- 表格加载失败；
- 批量操作部分成功；
- 保存视图冲突；
- 数据已过期需要刷新。

## 7. 验收标准

- 用户可以在三次交互内进入目标工单；
- 表格不展示截断 UUID；
- 不使用英文枚举作为状态标签；
- 页面不在浏览器中连接多个领域接口拼表；
- 筛选条件具有业务含义；
- 批量操作结果可追踪；
- 1280 宽度下默认列仍可使用；
- 键盘可以完成筛选、选行和进入详情。
