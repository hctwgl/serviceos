# 项目履约配置中心状态矩阵

状态：Draft  
视觉方向：A+ 经典专业任务型

## 1. 目的

项目履约配置不是数据库字段编辑页，而是配置资产的设计、校验、影响分析、发布和历史审计工作区。所有高保真和实现必须覆盖完整配置生命周期。

## 2. 生命周期状态

| 状态 | 页面重点 | 主操作 | 必须说明 |
|---|---|---|---|
| 未配置 | 缺少的配置域、建议起点、项目基础信息 | 开始配置 | 哪些业务尚不能运行 |
| 草稿编辑中 | 当前草稿、自动保存状态、配置完整度 | 继续编辑 | 修改尚未影响运行中工单 |
| 存在校验错误 | 错误位置、业务影响、修复建议 | 修复错误 | 错误必须阻止发布 |
| 仅有警告 | 警告内容、潜在影响、确认责任 | 查看影响 | 警告是否允许带确认发布 |
| 校验通过 | 完整度、变更摘要、受影响范围 | 预览发布 | 尚未正式生效 |
| 待发布确认 | 新旧版本差异、影响分析、生效时间 | 确认发布 | 对新工单、存量工单和外部集成的影响 |
| 发布中 | 操作编号、当前阶段、开始时间 | 查看进度 | 不允许重复发布同一草稿 |
| 发布成功 | 新版本、生效时间、发布人、影响结果 | 查看生效版本 | 哪些工单采用新版本 |
| 发布失败 | 失败阶段、中文原因、是否可重试 | 修复后重试 | 不得假装部分成功或静默回滚 |
| 已有新草稿 | 生效版本与未发布草稿并列 | 继续草稿 | 清楚区分“当前生效”和“正在编辑” |
| 只读 | 只读原因、当前版本、历史 | 查看历史 | 不使用大量置灰控件冒充只读页面 |
| 无权限 | 可见项目摘要、权限边界 | 返回项目 | 不泄露内部 Capability 或授权代码 |
| 并发冲突 | 他人修改内容、最新版本、冲突范围 | 重新加载并确认 | 不自动覆盖他人草稿 |

## 3. 页面结构

### 顶部对象区

- 项目名称和客户；
- 当前生效版本；
- 草稿状态；
- 配置完整度；
- 最近发布人和发布时间；
- 主要操作：保存草稿、校验、预览、发布。

### 左侧配置导航

- 基本信息；
- 服务产品与业务类型；
- 履约流程；
- 角色与责任；
- 网点与服务覆盖；
- 预约规则；
- SLA 规则；
- 表单与资料；
- 审核与整改；
- 外部系统对接；
- 版本与发布。

### 中央工作区

只呈现当前配置域的业务编辑界面。禁止把整个 Manifest、JSON 或所有数据库字段同时暴露给运营人员。

### 右侧上下文区

- 当前配置域说明；
- 校验错误和警告；
- 引用关系；
- 影响对象；
- 变更摘要；
- 下一步建议。

## 4. 版本与影响规则

发布前必须展示：

- 新旧版本业务差异；
- 新增、修改和删除项；
- 受影响的项目、网点、师傅、流程、表单和外部对接；
- 对新建工单和存量工单的不同影响；
- 是否需要通知业务人员；
- 是否存在不可逆变化。

没有真实服务端差异和影响数据时，页面必须明确显示“暂时无法完成影响分析”，不得使用伪造统计或固定示例。

## 5. 高保真原型要求

至少生成并评审：

1. 配置概览与草稿状态；
2. 流程配置工作区；
3. SLA 规则配置；
4. 表单与资料配置；
5. 校验错误定位；
6. 版本差异与影响分析；
7. 发布确认；
8. 发布失败；
9. 并发冲突；
10. 只读历史版本。

每个画面必须说明当前生命周期状态、主要动作、允许动作来源和所需页面读模型。
