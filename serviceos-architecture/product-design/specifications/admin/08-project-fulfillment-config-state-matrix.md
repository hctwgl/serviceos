# 项目履约配置状态矩阵

状态：Draft for Product Review  
视觉方向：A+ 经典专业任务型

## 1. 目的

项目履约配置位于项目详情中，不是独立“配置中心”。本矩阵定义项目履约配置从首次创建、草稿编辑、校验、对比、影响分析、发布到历史恢复的完整生命周期状态。

所有页面都必须保持明确项目上下文，并遵守：

- 一个项目一份履约配置；
- 一个项目一个履约工作流；
- 每个项目最多一个当前生效版本；
- 每个项目最多一个活动草稿；
- 所有业务规则随项目履约配置整体原子发布；
- 工单绑定创建时使用的履约配置版本；
- 进行中工单不自动迁移；
- 当前阶段没有公共模板、单模块发布和旧配置兼容入口。

## 2. 履约配置生命周期状态

| 状态 | 页面重点 | 主操作 | 必须说明 |
|---|---|---|---|
| 项目未配置 | 缺少的配置模块、建议起点、项目基础信息 | 开始配置 | 项目尚不能正常接单或履约 |
| 首个草稿编辑中 | 当前草稿、保存状态、配置完整度 | 继续配置 | 尚无生效版本，修改未对任何工单生效 |
| 有生效版本且无草稿 | 当前生效配置、最近发布、使用中的工单 | 创建新草稿 | 生效版本只读 |
| 已有活动草稿 | 生效版本与草稿并列、变更摘要 | 继续草稿 | 清楚区分当前生效与正在编辑 |
| 草稿保存中 | 当前保存阶段、最近成功保存时间 | 等待保存 | 不允许用户误以为内容已保存 |
| 草稿保存失败 | 未保存范围、中文失败原因、本地内容状态 | 重试保存 | 不得丢失本地输入或静默忽略 |
| 草稿并发冲突 | 他人修改内容、最新版本、冲突范围 | 重新加载并确认 | 不自动覆盖他人草稿 |
| 草稿准备不足 | 未完成模块和推荐下一步 | 继续配置 | 尚不能进入最终发布确认 |
| 存在校验错误 | 错误位置、业务影响、修复建议 | 修复错误 | 错误阻止项目级发布 |
| 仅有警告 | 警告内容、潜在影响、确认责任 | 逐项确认 | 是否允许发布由明确业务规则决定 |
| 校验通过 | 完整度、校验时间、草稿版本 | 业务预览 | 尚未正式生效 |
| 校验已过期 | 草稿变更时间、旧校验依据 | 重新校验 | 旧结果不能用于发布 |
| 业务预览部分不可用 | 可预览范围、依赖失败原因 | 修复或稍后重试 | 不使用伪造数据假装预览通过 |
| 等待版本对比 | 草稿和基线版本 | 生成版本对比 | 需要看清中文业务差异 |
| 版本对比已过期 | 草稿已变化的范围 | 重新生成对比 | 旧差异不能作为发布依据 |
| 等待影响分析 | 变更模块、待分析对象 | 执行影响分析 | 尚不能发布 |
| 影响分析失败 | 无法分析的对象和原因 | 修复后重试 | 不使用固定数字或示例统计 |
| 影响分析已过期 | 草稿变更和旧分析时间 | 重新分析 | 发布按钮必须重新禁用 |
| 等待警告确认 | 警告项、确认人、潜在影响 | 确认警告 | 每个允许带入发布的警告都需记录 |
| 可以发布 | 有效校验、对比、影响分析和发布说明 | 进入最终确认 | 发布条件基于当前草稿版本 |
| 最终发布确认 | 高风险变化、警告、新旧工单规则 | 发布为新版本 | 明确新工单与进行中工单使用哪个版本 |
| 发布锁定中 | 草稿锁定状态、操作人 | 等待 | 发布开始后不能继续编辑同一草稿 |
| 发布处理中 | 操作追踪标识、开始时间、当前业务阶段 | 查看进度 | 不允许重复发布同一草稿 |
| 发布成功 | 新版本、生效时间、发布人、影响结果 | 查看生效版本 | 新工单开始使用新版本 |
| 发布失败 | 失败阶段、中文原因、是否可重试 | 修复后重试 | 当前生效版本保持不变，不存在部分成功 |
| 重复发布请求 | 已存在的发布操作和结果 | 查看原操作 | 不创建第二个生效版本 |
| 只读查看 | 只读原因、当前版本、历史记录 | 查看历史 | 不使用大量置灰控件冒充只读页面 |
| 无权限 | 可见项目摘要、权限边界 | 返回项目 | 不泄露内部权限代码 |
| 外部依赖不可用 | 受影响配置模块、不可执行动作 | 稍后重试 | 不使用伪造数据完成校验或分析 |

## 3. 版本历史与恢复状态

| 状态 | 页面重点 | 主操作 | 必须说明 |
|---|---|---|---|
| 当前生效版本 | 完整配置摘要、发布依据、使用中工单 | 查看详情 | 不可编辑 |
| 历史生效版本 | 与当前版本差异、历史工单使用情况 | 基于此版本创建恢复草稿 | 不直接切换生效指针 |
| 发布失败记录 | 失败阶段、草稿和生效版本状态 | 查看失败详情 | 不是可被工单绑定的版本 |
| 正在创建恢复草稿 | 来源历史版本、当前生效版本 | 等待创建 | 恢复仍需重新校验和发布 |
| 已存在活动草稿 | 当前草稿摘要、来源 | 处理现有草稿 | 不能并行创建恢复草稿 |
| 恢复草稿编辑中 | 来源历史版本、与当前生效版本差异 | 继续配置 | 发布后形成新的更高版本号 |

## 4. 页面结构

### 4.1 项目详情上下文

固定展示：

- 面包屑：客户与项目 / 项目名称 / 履约配置；
- 项目名称和客户；
- 项目状态；
- 当前生效版本；
- 草稿状态；
- 配置完整度；
- 最近保存人和时间；
- 最近发布人和时间；
- 当前主要操作。

不得提供第二套项目配置列表或含义模糊的配置中心入口。

### 4.2 履约配置导航

1. 配置概览；
2. 服务范围；
3. 流程设计；
4. 表单与资料；
5. SLA 与预约；
6. 网点与资源；
7. 审核与整改；
8. 集成与回传；
9. 版本与发布。

“项目团队与区域分工”位于项目详情，不属于履约配置导航。

### 4.3 版本与发布内部页签

- 发布准备；
- 版本对比；
- 影响分析；
- 发布历史。

最终确认、发布进度和发布结果是明确步骤或结果页，不设置为履约配置左侧菜单。

### 4.4 中央工作区

只呈现当前业务模块内容。禁止把完整配置包、Manifest、JSON 或数据库字段暴露给运营人员。

### 4.5 右侧上下文区

- 当前模块说明；
- 校验错误和警告；
- 引用关系；
- 影响对象；
- 变更摘要；
- 结果生成时间；
- 下一步建议。

## 5. 模块状态

七个项目业务规则模块为：

1. 服务范围；
2. 流程设计；
3. 表单与资料；
4. SLA 与预约；
5. 网点与资源；
6. 审核与整改；
7. 集成与回传。

每个模块显示：

- 未开始；
- 配置中；
- 已完成；
- 有警告；
- 有错误；
- 只读；
- 不适用于当前项目。

错误必须可以定位到具体页面、对象和属性。配置概览点击错误后直接进入问题位置。

## 6. 发布准备结果有效性

以下结果必须绑定项目、草稿和草稿并发版本：

- 完整性校验；
- 业务场景预览；
- 版本对比；
- 影响分析；
- 警告确认。

草稿发生任何成功保存后：

- 原校验结果标记为已过期；
- 原版本对比标记为已过期；
- 原影响分析标记为已过期；
- 原警告确认需要按新结果重新判断；
- 发布按钮禁用。

不得使用旧草稿的结果发布新内容。

## 7. 发布影响规则

履约配置整体版本包含：

- 服务范围；
- 唯一履约工作流；
- 表单与资料；
- SLA、预约和服务日历；
- 项目网点准入、覆盖和师傅能力规则；
- 审核与整改规则；
- 入站、字段状态映射、回传和通知业务规则。

发布前必须展示：

- 新旧版本业务差异；
- 新增、修改和删除项；
- 高风险变化；
- 对新工单的影响；
- 进行中工单继续使用原绑定版本；
- 已有人员、网点、师傅和预约不会被静默修改；
- 已创建审核整改任务不会被改写；
- 已生成或正在重试的回传继续使用原冻结内容；
- 是否需要通知业务人员和外部协同方；
- 是否存在阻止发布的问题。

没有真实服务端差异和影响数据时，页面必须显示“暂时无法完成影响分析”，不得使用伪造统计或固定示例。

## 8. 原子发布规则

发布业务语义必须是：

```text
完整新版本全部生效
或者
当前生效版本完全不变
```

禁止部分模块生效。

发布失败时：

- 当前生效版本不变；
- 草稿默认保留；
- 页面说明失败阶段；
- 不存在可被工单绑定的半成品版本；
- 同一发布请求重复执行不产生多个版本。

## 9. 流程设计特殊状态

由于一个项目只有一个履约工作流，流程设计页面必须覆盖：

- 尚未创建流程；
- 生效流程只读；
- 草稿流程编辑中；
- 节点选中；
- 连线和流转规则选中；
- 条件分支空缺；
- 条件分支冲突；
- 引用表单、资料、SLA 或审核规则失效；
- 处理对象缺失；
- 流程校验通过；
- 流程校验失败。

不得出现流程列表、流程切换和独立流程发布状态。

## 10. 需要评审的关键界面

优先通过页面规格和交互流程评审：

1. 项目详情中的履约配置入口；
2. 配置概览与草稿状态；
3. 项目唯一流程设计工作区；
4. 表单列表、详情与设计器关系；
5. 校验错误定位；
6. 发布准备；
7. 项目级版本差异；
8. 影响分析；
9. 发布确认；
10. 发布处理中；
11. 原子发布失败；
12. 并发冲突；
13. 只读历史版本；
14. 基于历史版本创建恢复草稿。

仅在复杂交互存在理解分歧时生成少量最终高保真原型，不再为每个状态批量生成图片。

## 11. 验收标准

- 所有状态均处于明确项目上下文；
- 不存在“配置中心”一级菜单；
- 不存在第二套项目配置列表；
- 一个项目只展示一个履约工作流；
- 七个业务模块整体发布；
- 错误与警告清晰区分；
- 草稿变化后旧校验、对比和影响分析立即失效；
- 发布全部成功或生效版本完全不变；
- 发布失败不造成部分生效；
- 并发冲突不覆盖他人修改；
- 重复发布不创建多个版本；
- 新工单与进行中工单的版本规则明确；
- 历史规则恢复形成新版本，不修改旧版本；
- 公共模板、定时发布、灰度发布和存量自动迁移不进入当前阶段。
