# 项目履约配置状态矩阵

状态：Draft  
视觉方向：A+ 经典专业任务型

## 1. 目的

项目履约配置位于项目详情中，不是独立“配置中心”。本矩阵定义项目履约配置从首次创建、草稿编辑、校验、发布到历史查看的完整生命周期状态。

所有页面都必须保持明确项目上下文，并遵守：

- 一个项目一份履约配置；
- 一个项目一个履约工作流；
- 工作流、表单、SLA 等内容随项目履约配置整体发布；
- 工单绑定创建时使用的履约配置版本；
- 当前阶段没有公共模板和旧配置兼容入口。

## 2. 生命周期状态

| 状态 | 页面重点 | 主操作 | 必须说明 |
|---|---|---|---|
| 项目未配置 | 缺少的配置域、建议起点、项目基础信息 | 开始配置 | 项目尚不能正常接单或履约 |
| 首个草稿编辑中 | 当前草稿、自动保存状态、配置完整度 | 继续配置 | 尚无生效版本，修改未对任何工单生效 |
| 有生效版本且无草稿 | 当前生效配置、最近发布、使用中的工单 | 创建新草稿 | 生效版本只读 |
| 已有新草稿 | 生效版本与草稿并列、变更摘要 | 继续草稿 | 清楚区分“当前生效”和“正在编辑” |
| 草稿保存中 | 当前保存阶段、最近成功保存时间 | 等待保存 | 不允许用户误以为内容已保存 |
| 保存失败 | 未保存范围、中文失败原因、本地内容状态 | 重试保存 | 不得丢失本地输入或静默忽略 |
| 存在校验错误 | 错误位置、业务影响、修复建议 | 修复错误 | 错误必须阻止项目级发布 |
| 仅有警告 | 警告内容、潜在影响、确认责任 | 查看影响 | 是否允许带确认发布由业务规则决定 |
| 校验通过 | 完整度、变更摘要、受影响范围 | 进入发布准备 | 尚未正式生效 |
| 待发布确认 | 新旧版本差异、影响分析、生效规则 | 确认发布 | 新工单和进行中工单使用哪个版本 |
| 发布中 | 操作编号、当前阶段、开始时间 | 查看进度 | 不允许重复发布同一草稿 |
| 发布成功 | 新版本、生效时间、发布人、影响结果 | 查看生效版本 | 新工单开始使用新版本 |
| 发布失败 | 失败阶段、中文原因、是否可重试 | 修复后重试 | 不得假装部分成功或静默回滚 |
| 只读查看 | 只读原因、当前版本、历史记录 | 查看历史 | 不使用大量置灰控件冒充只读页面 |
| 无权限 | 可见项目摘要、权限边界 | 返回项目 | 不泄露内部权限代码 |
| 并发冲突 | 他人修改内容、最新版本、冲突范围 | 重新加载并确认 | 不自动覆盖他人草稿 |
| 外部依赖不可用 | 受影响配置域、暂不可执行动作 | 稍后重试 | 不使用伪造数据完成校验或影响分析 |

## 3. 页面结构

### 3.1 项目详情上下文

固定展示：

- 面包屑：客户与项目 / 项目名称 / 履约配置；
- 项目名称和客户；
- 项目状态；
- 当前生效版本；
- 草稿状态；
- 配置完整度；
- 最近保存人和时间；
- 最近发布人和时间；
- 当前主要操作。

不得提供第二套项目配置列表或含义模糊的配置中心入口。

### 3.2 履约配置导航

- 配置概览；
- 服务范围；
- 流程设计；
- 表单与资料；
- SLA 与预约；
- 角色与责任；
- 网点与资源；
- 审核与整改；
- 集成与回传；
- 版本与发布。

### 3.3 中央工作区

只呈现当前配置域的业务内容。禁止把整个配置包、Manifest、JSON 或数据库字段同时暴露给运营人员。

### 3.4 右侧上下文区

- 当前配置域说明；
- 校验错误和警告；
- 引用关系；
- 影响对象；
- 变更摘要；
- 下一步建议。

## 4. 模块状态

每个配置域必须显示：

- 未开始；
- 配置中；
- 已完成；
- 有警告；
- 有错误；
- 只读；
- 不适用于当前项目。

错误必须可以定位到具体页面、对象和属性。配置概览点击错误后直接进入问题位置。

## 5. 版本与影响规则

履约配置整体版本包含：

- 服务范围；
- 唯一履约工作流；
- 表单与资料；
- SLA 与预约；
- 角色与责任；
- 网点与资源；
- 审核与整改；
- 集成与回传。

发布前必须展示：

- 新旧版本业务差异；
- 新增、修改和删除项；
- 对网点、师傅、表单、外部对接和新工单的影响；
- 进行中工单继续使用原绑定版本；
- 是否需要通知业务人员；
- 是否存在不可逆变化；
- 是否存在阻止发布的问题。

没有真实服务端差异和影响数据时，页面必须显示“暂时无法完成影响分析”，不得使用伪造统计或固定示例。

## 6. 流程设计特殊状态

由于一个项目只有一个履约工作流，流程设计页面必须覆盖：

- 尚未创建流程；
- 生效流程只读；
- 草稿流程编辑中；
- 节点选中；
- 连线和流转规则选中；
- 条件分支空缺；
- 条件分支冲突；
- 引用表单、资料、角色或 SLA 失效；
- 流程校验通过；
- 流程校验失败。

不得出现流程列表、流程切换和独立流程发布状态。

## 7. 需要评审的关键界面

优先通过页面规格和交互流程评审：

1. 项目详情中的履约配置入口；
2. 配置概览与草稿状态；
3. 项目唯一流程设计工作区；
4. 表单列表、详情与设计器关系；
5. 校验错误定位；
6. 项目级版本差异与影响分析；
7. 发布确认；
8. 发布失败；
9. 并发冲突；
10. 只读历史版本。

仅在复杂交互存在理解分歧时生成少量最终高保真原型，不再为每个状态批量生成图片。

## 8. 验收标准

- 所有状态均处于明确项目上下文；
- 不存在“配置中心”一级菜单；
- 不存在第二套项目配置列表；
- 一个项目只展示一个履约工作流；
- 工作流、表单和 SLA 不独立发布；
- 错误与警告清晰区分；
- 发布失败不造成部分生效；
- 并发冲突不覆盖他人修改；
- 新工单与进行中工单的版本规则明确；
- 公共模板不进入当前阶段。
