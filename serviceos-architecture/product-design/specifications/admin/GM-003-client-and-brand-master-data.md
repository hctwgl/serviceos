# GM-003 车企与品牌主数据

状态：Accepted
设计等级：B

## 1. 页面定位

“车企与品牌主数据”是 Admin 中维护合作车企及其品牌稳定身份、业务名称和可用状态的唯一权威入口。

正式位置：

```text
客户与项目
└── 车企与品牌
    ├── 车企目录
    ├── 车企详情
    │   ├── 基本信息
    │   ├── 品牌
    │   ├── 关联项目
    │   └── 操作记录
    └── 品牌详情
```

页面必须帮助用户回答：

1. 当前系统登记了哪些合作车企；
2. 每个车企有哪些品牌；
3. 哪些车企和品牌当前可以用于新项目、配置和业务选择；
4. 稳定编码和中文业务名称是什么；
5. 停用会影响哪些后续选择，但不会改写哪些历史事实；
6. 当前车企和品牌被哪些项目使用；
7. 谁在什么时候创建、改名、启用或停用。

## 2. 术语边界

本页面中的“车企”是与 ServiceOS 开展项目合作的企业客户主数据，对应稳定 `clientCode`。

它不同于工单中的终端客户和联系人：

```text
车企
    项目的合作方和外部系统身份

品牌
    归属于车企的业务品牌

项目
    归属于一个车企的运营项目

终端客户 / 联系人
    具体工单中的服务对象和联系人
```

页面不得把终端客户姓名、手机号、地址或工单联系人放入车企主数据，也不得用工单客户列表替代车企目录。

## 3. 已确认领域关系

```text
Tenant
└── 车企 1
    ├── 品牌 0..N
    └── 项目 0..N

项目
└── 所属车企 1

工单
├── 所属项目
├── clientCode
└── brandCode
```

边界：

- `clientCode` 是租户范围内稳定车企身份；
- `brandCode` 在所属车企范围内稳定；
- 品牌不能脱离车企独立存在；
- 项目必须选择一个已启用车企；
- 项目、配置、工单和外部连接只引用稳定编码，不复制可独立维护的车企或品牌主数据；
- 核心业务模块可以持有通用车企编码，但不能按具体车企编码编写协议分支；
- 服务产品属于 GM-009，不与品牌合并；
- 当前阶段不建设多级子品牌树。

## 4. 本规格非目标

GM-003 不负责：

- 管理终端客户和工单联系人；
- 管理车企合同、结算、发票或价格；
- 管理外部连接凭据和技术协议；
- 管理项目履约配置；
- 管理服务产品；
- 建设任意层级品牌树；
- 合并两个车企编码或迁移历史工单；
- 删除已经被业务引用的稳定身份；
- 使用备注、JSON 或扩展字段承载未建模业务；
- 兼容旧“主数据治理”页面路由或旧登记接口语义。

## 5. 权威入口与导航边界

车企与品牌业务主数据归“客户与项目”。行政区目录不是车企子功能，应由独立基础主数据入口管理或作为业务选择器消费。

目标产品只保留一个车企与品牌权威页面：

```text
客户与项目 / 车企与品牌
```

当前 `/master-data` 同时混合车企、品牌和行政区树时，后续重构应拆分业务归属并切换正式导航；不保留第二套可编辑车企目录、旧路由别名或双写页面。

项目创建、履约配置、工单筛选和外部连接页面使用本目录的受限选择器，但不能在选择器中直接创建或修改车企和品牌。

## 6. 页面结构

### 6.1 车企目录

```text
面包屑：客户与项目 / 车企与品牌
→ 页面标题、说明和“新建车企”
→ 搜索与状态筛选
→ 车企列表
→ 分页、数据时间和刷新状态
```

### 6.2 车企详情

```text
车企对象头部
→ 基本信息
→ 品牌列表
→ 关联项目摘要
→ 使用影响
→ 操作记录
```

品牌列表默认在车企详情中展示，不建立脱离车企上下文的全局品牌管理页。

## 7. 车企目录搜索与筛选

搜索框文案：

> 搜索车企名称或车企编码

默认筛选：

- 状态：全部、已启用、已停用；
- 是否存在服务中项目；
- 是否已登记品牌。

搜索和筛选必须由服务端执行。当前车企目录只支持状态筛选时，应新增名称、编码、项目使用和品牌使用查询；不得加载全部结果后在前端过滤。

状态筛选默认“全部”，因为治理页面必须能找到已停用记录。项目创建等业务选择器默认只返回已启用车企。

## 8. 车企列表字段

默认列：

1. 车企名称与编码；
2. 状态；
3. 品牌数量；
4. 项目使用摘要；
5. 服务中项目；
6. 最近更新时间；
7. 操作。

### 8.1 车企名称与编码

车企名称是主要文本，稳定编码是次级业务标识。

编码可以在治理详情中展示，但项目、工单和普通业务页面优先显示名称。名称缺失时不能用编码顶替，必须显示“车企主数据不完整”。

### 8.2 状态

| 稳定值 | 中文名称 | 业务含义 |
|---|---|---|
| `ACTIVE` | 已启用 | 可以出现在新项目和其他允许的业务选择器中 |
| `DISABLED` | 已停用 | 不再用于新的业务选择，已有引用和历史事实继续保留 |

停用不是删除，也不代表现有项目、工单、配置或外部记录失效。

### 8.3 品牌数量

分别展示已启用品牌和全部品牌，例如：

```text
3 个已启用 / 共 4 个
```

查询不可用或无权查看时不能显示 0。

### 8.4 项目使用摘要

展示：

- 项目总数；
- 服务中项目数量；
- 筹备中项目数量；
- 已暂停或结束项目数量（详情中展示）；
- 进入 GM-002 车企筛选视图的深链。

项目计数必须遵守项目数据范围。用户只能管理车企主数据但无权查看项目明细时，页面显示受控汇总或“无权查看”，不能泄露项目名称。

### 8.5 最近更新时间

展示车企名称或状态最近变化时间。当前目录只返回 `asOf` 时，不能用目录生成时间冒充单条车企更新时间。

## 9. 车企详情

对象头部展示：

- 车企名称；
- 车企编码；
- 中文状态；
- 品牌摘要；
- 项目使用摘要；
- 最近更新时间；
- 当前允许的主要动作。

详情区块：

### 9.1 基本信息

- 车企名称；
- 稳定编码；
- 状态；
- 创建时间；
- 最近更新时间；
- 创建人和最近修改人（有权限时）。

当前主数据模型没有统一社会信用代码、注册地址、商务联系人或合同信息时，页面不得自行新增这些字段或使用备注 JSON 代替领域建模。

### 9.2 品牌

展示该车企的品牌列表和“新建品牌”动作。

### 9.3 关联项目

展示项目摘要和最近项目，完整查询进入 GM-002：

```text
项目管理
筛选：当前车企
```

不在车企详情复制第二套项目目录。

### 9.4 使用影响

在停用前展示：

- 是否有服务中项目；
- 是否有筹备中项目；
- 已启用品牌数量；
- 是否被项目创建选择器使用；
- 是否被外部连接或配置引用（只有安全摘要）；
- 停用后哪些新建或选择动作不再可用。

该摘要用于解释影响，不代表页面可以修改项目、配置或连接。

## 10. 新建车企

新建车企使用对话框或简洁独立表单，字段：

- 车企名称；
- 车企编码。

编码规则：

- 在当前 Tenant 内唯一；
- 用于项目、工单、配置和外部协同的稳定身份；
- 创建后不可修改；
- 不使用数据库 UUID 作为业务编码；
- 不允许通过大小写或空格创建语义重复编码；
- 不直接使用中文名称作为稳定编码；
- 格式由机器契约统一校验。

创建成功后默认状态为“已启用”，进入车企详情。创建命令使用幂等键。

重复编码必须返回明确冲突，不能把“新建”静默解释成改名或重新启用。

## 11. 编辑车企名称

车企编码不可编辑。名称修改使用独立命令并满足：

- 需要主数据维护权限；
- 使用对象版本或 ETag；
- 展示当前名称和新名称；
- 说明不会改变稳定编码；
- 保存不可变操作记录；
- 并发变化失败关闭；
- 修改成功后刷新项目选择器和页面显示；
- 不改写历史外部报文和已经冻结的业务快照。

名称修改不是新建，也不能隐式改变启停状态。

## 12. 启用与停用车企

启用和停用使用明确的状态命令，不能使用行内即时开关。

### 12.1 停用

停用确认必须展示使用影响，并要求填写原因。

停用成功后：

- 项目创建等新业务选择器不再返回该车企；
- 现有项目继续保留所属车企；
- 已有工单、配置版本、审计和外部记录不被改写；
- 车企详情和治理目录继续可查询；
- 品牌记录不发生级联状态写入；
- 因父车企停用，所属品牌不能用于新的业务选择；
- 外部入站引用停用车企时，按已接受的集成规则明确拒绝或进入人工处理，不能使用默认车企兜底。

### 12.2 启用

启用确认必须展示：

- 车企名称和编码；
- 仍有效的品牌状态；
- 关联项目摘要；
- 启用后将恢复哪些选择器可用性。

重新启用车企不会自动启用已停用品牌，也不会自动恢复已暂停或已结束项目。

### 12.3 状态幂等

重复提交同一目标状态返回相同业务结果；同一幂等键改变目标状态必须失败。

## 13. 品牌列表

品牌列表位于车企详情中。

搜索：

> 搜索品牌名称或品牌编码

筛选：

- 全部；
- 已启用；
- 已停用。

默认列：

1. 品牌名称与编码；
2. 状态；
3. 展示顺序；
4. 项目或配置使用摘要；
5. 最近更新时间；
6. 操作。

父车企已停用时，品牌行必须明确显示“所属车企已停用”，不能仅根据品牌自身 `ACTIVE` 显示为可用。

## 14. 新建品牌

字段：

- 品牌名称；
- 品牌编码；
- 展示顺序。

规则：

- 品牌必须归属于当前车企；
- 品牌编码在当前车企范围内稳定且唯一；
- 创建后不能改编码或移动到另一车企；
- 默认状态为“已启用”；
- 展示顺序只影响治理和选择器展示，不影响业务优先级、派单或流程；
- 创建使用幂等键；
- 重复编码不能静默变成改名或重新启用。

当前阶段只支持车企下一级品牌，不提供父品牌、子品牌和任意树结构。

## 15. 编辑品牌

允许通过独立命令修改：

- 品牌名称；
- 展示顺序。

不允许修改：

- 品牌编码；
- 所属车企；
- 历史项目、工单和配置引用。

名称与展示顺序修改不改变品牌启停状态。

## 16. 启用与停用品牌

品牌停用后：

- 不再出现在新的品牌选择器；
- 现有项目配置和历史工单继续保留编码引用；
- 不删除配置、工单或外部记录；
- 不自动停用车企或其他品牌；
- 新入站引用该品牌时按集成规则明确处理，不能改成默认品牌。

品牌重新启用后恢复新的选择器可用性，但不会改写历史业务事实。

父车企停用时，不需要对所有品牌执行级联写入；页面和服务端选择器综合父、子状态判断是否可用于新业务。

## 17. 项目、工单和配置中的引用

### 17.1 项目

项目创建选择一个已启用车企。项目列表和详情显示车企当前业务名称，但稳定关联使用车企编码或权威身份。

停用车企不会静默改变项目所属关系，也不会自动暂停项目。

### 17.2 工单

工单保存稳定 `clientCode` 和 `brandCode`。普通工单页面显示中文车企和品牌名称，名称解析失败时进入数据不完整，不显示编码兜底。

外部业务唯一键仍按接受的集成规则使用稳定编码，显示名修改不能改变幂等和定位语义。

### 17.3 履约配置

履约配置只引用稳定车企和品牌身份及随版本冻结的选择维度，不复制第二套可编辑名称或生命周期。

品牌停用不会改写已经生效的配置版本。新草稿是否允许继续引用已停用品牌，必须由配置校验明确阻止或要求处理，不能前端静默替换。

### 17.4 外部连接

外部连接可以绑定车企身份，但认证凭据、证书、Endpoint 和协议配置属于 GM-010。车企页面只显示安全使用摘要，不读取或展示秘密。

## 19. 当前契约缺口

当前契约已支持：

- 按状态查询车企；
- 登记车企；
- 启停车企；
- 查询、新增和启停品牌；
- 品牌展示顺序。

但不足以支撑长期主数据治理目标：

- 车企和品牌目录缺少名称、编码搜索和服务端分页；
- 缺少项目、配置和连接使用摘要；
- 缺少创建时间、更新时间、修改人和对象版本；
- 缺少行级 `allowedActions`；
- 车企与品牌写权限复用 `project.create`，没有区分项目创建和主数据治理；
- `registerProjectClient` 在编码已存在时会改名并自动置为 `ACTIVE`；
- `registerProjectClientBrand` 同样混合创建、编辑和重新启用；
- 品牌缺少明确编辑名称和展示顺序的独立命令；
- 状态命令缺少使用影响摘要和原因；
- `/master-data` 混合车企品牌与行政区治理，不符合目标业务导航。

实现 GM-003 前应直接重做为唯一的新契约：

- 创建车企；
- 修改车企名称；
- 启用车企；
- 停用车企；
- 创建品牌；
- 修改品牌名称和顺序；
- 启用品牌；
- 停用品牌；
- 查询页面级目录和详情。

不保留“登记或更新并自动启用”的兼容命令，也不让新 UI 在多个接口之间猜测结果。

## 20. 权限与数据范围

至少区分以下业务能力：

- 查看车企与品牌主数据；
- 创建和编辑车企；
- 启用和停用车企；
- 创建和编辑品牌；
- 启用和停用品牌；
- 查看项目使用摘要；
- 查看外部连接安全摘要；
- 查看主数据操作记录。

精确 Capability code 在安全设计和机器契约中确定。长期目标不能继续用“可以创建项目”自动推导“可以改车企和品牌”。

Tenant 隔离必须在服务端执行。跨租户编码相同不代表同一对象，跨租户请求不能泄露对象存在性。

## 21. 状态与错误

### 21.1 业务空状态

> 当前还没有车企主数据。登记车企后，才可以创建归属于该车企的项目。

只有具有创建权限时显示“新建车企”。

### 21.2 品牌空状态

> 当前车企还没有品牌。只有业务确实区分品牌时才需要创建。

不要求每个车企必须拥有品牌，也不创建“默认品牌”兜底。

### 21.3 查询失败

车企基础信息可以展示而项目或连接摘要不可用时，分别显示“暂时无法获取”。不得把项目数、品牌数或连接数显示为 0。

### 21.4 数据不完整

名称、状态或父车企关系缺失时显示“主数据不完整”，不使用编码、未知枚举或父级 ID 顶替。

### 21.5 并发冲突

保留用户输入，显示平台最新名称和状态。名称编辑、启停和品牌调整都不能最后写入覆盖。

### 21.6 使用影响变化

用户打开停用确认后项目或连接使用情况发生变化时，提交必须重新校验并返回最新影响，不能按旧摘要强制执行。

## 22. 批量、导入与删除

当前阶段不提供：

- 批量创建车企；
- 批量启停车企；
- 批量创建或启停品牌；
- Excel 全量导入；
- 合并车企或品牌编码；
- 移动车企品牌；
- 物理删除车企或品牌。

未来确需批量导入时，必须单独定义编码冲突、重复语义、预校验、部分成功、审计和回滚，不复用“登记即更新”接口。

## 24. 无障碍与键盘

- 车企名称和品牌名称链接带明确对象上下文；
- 状态不能只依赖颜色；
- 车企和品牌列表可键盘导航；
- 停用确认完整说明影响并支持读屏；
- 动态字体和 200% 缩放下保留名称、状态和主操作；
- 详情页品牌列表更新后保持焦点；
- 表单错误可以定位到名称或编码字段；
- 成功、冲突和部分不可用结果提供文本反馈。

## 25. 验收标准

### 领域与入口

- [ ] 车企、品牌、项目和工单联系人概念不混淆；
- [ ] 每个品牌归属于一个车企；
- [ ] 车企与品牌只有一个权威管理入口；
- [ ] 行政区治理不与车企品牌混成同一业务页面；
- [ ] 项目和配置只引用稳定身份，不复制主数据生命周期。

### 创建、编辑和状态

- [ ] 创建、改名、启用和停用使用不同命令；
- [ ] 重复编码创建明确失败，不静默改名或启用；
- [ ] 车企和品牌编码创建后不可修改；
- [ ] 品牌不能移动到另一车企；
- [ ] 停用不是删除，不改写现有项目、工单和配置；
- [ ] 重新启用车企不会自动启用品牌或恢复项目。

### 列表与详情

- [ ] 支持服务端名称、编码和状态搜索筛选；
- [ ] 治理目录可以找到已停用记录；
- [ ] 项目选择器只返回可用车企；
- [ ] 品牌显示父车企的有效状态影响；
- [ ] 项目、品牌和连接摘要区分零、无权和不可用；
- [ ] 名称缺失时不使用编码兜底。

### 权限与可靠性

- [ ] 主数据治理权限不由项目创建权限隐式扩大；
- [ ] Tenant 隔离在服务端执行；
- [ ] 写操作使用幂等键和对象版本；
- [ ] 使用影响变化时停用请求重新校验；
- [ ] 并发编辑不使用最后写入覆盖；
- [ ] 所有改名和状态变化具有操作记录。

### 产品质量

- [ ] 不显示原始枚举、技术字段、UUID 或 Capability code；
- [ ] 正常、空、筛选无结果、无权限、只读、部分失败、数据不完整和并发冲突状态完整；
- [ ] 1280px、1440px、键盘和 200% 缩放可用；
- [ ] 页面读模型由服务端提供，不在前端 Join 项目和连接；
- [ ] 旧“登记或更新并启用”接口在新实现中被删除而非适配。

## 26. 禁止实现

禁止：

- 把终端客户或工单联系人当作车企主数据；
- 把车企、品牌和行政区混成一个无业务层级的标签页集合；
- 新建车企时同编码存在就自动改名并启用；
- 新建品牌时同编码存在就自动改名并启用；
- 修改车企或品牌稳定编码；
- 把品牌移动到另一车企；
- 停用车企时级联改写所有品牌状态；
- 停用主数据时删除或改写现有项目、工单、配置和外部记录；
- 为停用品牌选择默认品牌兜底；
- 外部入站无法识别车企或品牌时使用最近项目或默认值；
- 使用 `project.create` 长期代替主数据治理权限；
- 用编码代替缺失名称；
- 查询失败时显示项目、品牌或连接数量为零；
- 在前端跨接口统计使用摘要；
- 通过自由 JSON 扩展车企和品牌业务字段；
- 物理删除、批量启停或 Excel 覆盖主数据；
- 为旧 `/master-data` 页面、旧登记接口或旧开发数据保留兼容层。
