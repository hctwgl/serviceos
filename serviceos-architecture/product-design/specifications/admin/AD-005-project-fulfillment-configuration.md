# AD-005 项目详情—履约配置

状态：Draft for Product Review  
设计等级：A

## 1. 产品位置

履约配置不是独立业务实体，也不属于单独的“配置中心”。

正式入口：

```text
客户与项目
└── 项目列表
    └── 项目详情
        └── 履约配置
```

Admin 工作台可以展示跨项目配置待办，但点击后必须直接进入对应项目详情的“履约配置”。不得建立第二套项目配置列表。

## 2. 已确认领域约束

- 一个项目只有一份当前履约配置；
- 一个项目当前只支持一个履约工作流；
- 履约配置最多存在一个生效版本和一个草稿版本；
- 工作流、表单、SLA、审核、整改和集成规则不独立发布；
- 发布形成项目级不可变履约配置版本；
- 工单创建时绑定当时的项目履约配置版本；
- 项目团队与区域分工不进入履约配置版本；
- 网点、师傅、排班、负载和实时产能不进入履约配置版本；
- 实际审核、整改、复核和外部协同记录不进入履约配置版本；
- 外部连接、认证凭据和网络安全配置不进入履约配置版本；
- 当前阶段不提供公共模板库，也不提供旧配置兼容或迁移入口。

## 3. 目标用户

- 项目管理员；
- 项目实施和交付人员；
- 履约规则配置人员；
- 具有配置审核或发布权限的管理人员；
- 只读查看项目配置和历史版本的审计人员。

## 4. 核心任务

用户需要在明确项目上下文中完成一份可执行、可校验、可预览、可发布和可追溯的履约配置。

页面必须帮助用户判断：

- 当前项目是否具备接单和履约条件；
- 当前生效配置是什么；
- 是否存在未发布草稿；
- 哪些配置尚未完成；
- 哪些错误阻止发布；
- 草稿与生效版本有何业务差异；
- 发布后哪些新工单、网点、师傅、审核任务和外部协同会受到影响；
- 下一步最需要处理什么。

## 5. 项目详情上下文

### 5.1 项目详情一级页签

项目详情至少包含：

1. 项目概览；
2. 服务与合同；
3. 项目团队与区域分工；
4. 履约配置；
5. 项目工单；
6. 操作记录。

### 5.2 固定项目头部

履约配置页面顶部必须展示：

- 项目名称；
- 客户名称；
- 项目状态；
- 服务周期；
- 当前生效配置版本；
- 当前草稿版本；
- 配置完整度；
- 校验状态；
- 最近保存人和时间；
- 最近发布人和时间；
- 当前首要动作。

面包屑必须明确显示：

```text
客户与项目 / 项目名称 / 履约配置 / 当前模块
```

用户不得在不知情的情况下切换项目。离开存在未保存修改的页面时必须明确提示。

## 6. 履约配置导航

工作区内部统一使用浅色导航，按业务结果组织：

1. 配置概览；
2. 服务范围；
3. 流程设计；
4. 表单与资料；
5. SLA 与预约；
6. 网点与资源；
7. 审核与整改；
8. 集成与回传；
9. 版本与发布。

禁止在导航中出现：

- 流程列表；
- 流程版本；
- 节点设计；
- 流转规则；
- 表单设计器；
- 校验结果；
- 影响分析；
- 发布确认；
- Inbox、Outbox、Topic、Queue 或死信队列。

这些内容属于模块内部工作区、详情页、属性面板、发布步骤或受控技术诊断工具。

## 7. 默认页面：配置概览

配置概览不直接承载复杂编辑，必须展示：

- 当前是否允许创建工单；
- 配置完整度；
- 七个业务规则模块的状态；
- 当前生效版本摘要；
- 草稿版本摘要；
- 草稿变更数量和变更范围；
- 阻塞错误；
- 重要警告；
- 最近操作记录；
- 推荐下一步；
- 当前用户允许执行的操作。

七个规则模块为：服务范围、流程设计、表单与资料、SLA 与预约、网点与资源、审核与整改、集成与回传。

点击模块状态卡进入对应模块；点击错误直接进入具体问题位置。

### 7.1 模块状态

每个业务模块具有以下状态之一：

- 未开始；
- 配置中；
- 已完成；
- 有警告；
- 有错误；
- 只读；
- 不适用于当前项目。

错误必须阻止发布。警告是否允许发布必须由明确业务规则决定，并要求发布人确认。

## 8. 草稿与生效版本

### 8.1 首次配置

项目尚无生效版本时，用户点击“开始配置”创建首个草稿。

在首个版本发布前：

- 项目不得被标记为可正常接单；
- 页面必须说明尚缺哪些必要配置；
- 不得以默认空规则或隐式回退规则代替未配置内容。

### 8.2 已有生效版本

- 生效版本始终只读；
- 用户点击“创建新草稿”后，基于最新生效版本编辑；
- 同一项目最多一个活动草稿；
- 页面必须明显区分“当前生效”和“正在编辑”；
- 不允许直接修改生效版本；
- 不允许静默覆盖他人草稿。

### 8.3 发布结果

发布成功后：

- 草稿成为新的不可变生效版本；
- 新工单使用新版本；
- 进行中工单继续使用创建时绑定的旧版本；
- 页面记录发布人、发布时间、发布说明和影响分析结果。

## 9. 规则、主数据、连接和运行数据

履约配置页面必须明确区分四类内容：

### 9.1 项目业务规则

随履约配置整体发布，包括：

- 服务范围；
- 唯一履约工作流；
- 表单与资料；
- SLA、预约和服务日历；
- 项目网点准入、覆盖和师傅能力规则；
- 审核与整改规则；
- 入站、字段状态映射、回传和通知业务规则。

### 9.2 项目运营主数据

不进入履约配置版本，包括：

- 项目成员；
- 项目区域人员分工；
- 项目岗位人员生效时间。

### 9.3 系统主数据和连接能力

不进入履约配置版本，包括：

- 网点和师傅主数据；
- 技能和资质真实状态；
- 外部连接；
- 认证、密钥、证书；
- 网络、安全和技术协议。

### 9.4 工单运行数据

不进入履约配置版本，包括：

- 工单项目岗位人员快照；
- 责任网点和责任师傅；
- 实时排班、负载和产能；
- 已确认预约；
- 实际审核、整改和复核记录；
- 实际入站、回传、确认、失败和重试记录。

新配置发布不得静默修改以上运行数据。

## 10. 工作区通用布局

- 顶部：项目上下文、版本和主要操作；
- 左侧：履约配置业务导航与完成状态；
- 中间：当前模块内容；
- 右侧：业务说明、校验结果、引用关系和影响提示；
- 页面底部或固定操作区：保存草稿、校验和返回概览。

每个配置项必须说明业务效果，不能只展示字段名或内部配置键。

## 11. 版本与发布

项目履约配置整体发布，完整版本包含：

- 服务范围；
- 唯一履约工作流；
- 表单与资料；
- SLA、预约和服务日历；
- 项目网点准入、覆盖和师傅能力规则；
- 审核与整改规则；
- 入站、映射、回传和通知业务规则。

发布流程：

1. 保存草稿；
2. 跨模块完整性校验；
3. 业务预览；
4. 新旧版本对比；
5. 影响分析；
6. 填写发布说明；
7. 最终确认；
8. 发布；
9. 展示发布结果。

不允许单独发布工作流、表单、SLA、审核、整改或集成规则，也不允许跳过校验和影响分析。

发布新配置不得：

- 修改已有工单人员；
- 改派责任网点或师傅；
- 改变已确认预约；
- 改写审核、整改或复核任务；
- 重新映射历史外部入站数据；
- 改变已经冻结并正在重试的回传内容。

## 12. 页面读模型

建议使用：`ProjectFulfillmentConfigurationView`

必须包含：

- `projectContext`：项目、客户、状态和服务周期；
- `activeVersion`：当前生效版本摘要；
- `draftVersion`：当前草稿摘要；
- `completionSummary`：整体完整度；
- `sectionStatuses`：各业务模块状态；
- `validationSummary`：错误和警告；
- `changeSummary`：草稿变更摘要；
- `impactSummary`：影响对象和结论；
- `publishReadiness`：发布准备状态；
- `recentActivities`：最近操作；
- `recommendedAction`：推荐下一步；
- `allowedActions`：当前用户可执行操作；
- `concurrencyVersion`：并发控制版本。

页面读模型必须直接提供中文业务名称，不允许浏览器通过多个领域接口自行拼装显示名称。

## 13. 业务命令

至少包含：

- 创建首个履约配置草稿；
- 基于生效版本创建新草稿；
- 保存某个配置模块；
- 放弃草稿；
- 执行完整性校验；
- 生成业务预览；
- 生成版本对比；
- 执行影响分析；
- 发布履约配置版本；
- 重新加载并处理并发冲突。

所有写命令必须：

- 校验项目状态和操作权限；
- 使用明确并发版本；
- 返回中文业务错误；
- 不使用静默回退；
- 不自动覆盖其他用户修改。

## 14. 权限

至少区分：

- 查看履约配置；
- 创建或编辑草稿；
- 执行校验；
- 查看影响分析；
- 发布版本；
- 放弃草稿；
- 查看历史版本和审计记录。

各模块仍需区分维护表单、SLA、网点规则、审核规则、敏感回传字段等具体权限。

查看外部连接认证秘密和技术诊断不属于项目履约配置权限。

只读用户看到专门的只读界面，不使用大面积禁用控件模拟只读状态。

## 15. 必须设计的页面状态

- 项目尚未配置；
- 首个草稿编辑中；
- 有生效版本且无草稿；
- 有未发布草稿；
- 草稿自动保存中；
- 保存失败；
- 校验失败；
- 校验有警告；
- 校验通过；
- 外部连接不可用；
- 等待发布确认；
- 发布处理中；
- 发布失败；
- 发布成功；
- 只读查看；
- 无权限；
- 草稿被他人更新；
- 外部依赖不可用。

## 16. 当前阶段不包含

- Admin 一级菜单“配置中心”；
- 第二套项目配置列表；
- 一个项目多个流程；
- 公共流程、表单、资料、SLA、审核整改或集成模板；
- 单模块独立发布；
- 存量工单自动迁移；
- 在项目页面维护外部连接认证秘密；
- 通用 API 网关或消息队列控制台；
- 旧配置转换、兼容模式或隐式默认规则。

公共模板作为下一期候选需求单独评估。

## 17. 验收标准

- 履约配置只能从明确项目上下文进入；
- 项目列表是唯一项目权威列表；
- 用户始终知道正在查看哪个项目和哪个版本；
- 一个项目只存在一个履约工作流；
- 用户可以看清配置完整度和阻塞发布的问题；
- 项目业务规则、运营主数据、系统连接和工单运行数据边界清楚；
- 配置页面不退化为数据库 CRUD 或技术控制台；
- 生效版本不可修改；
- 发布前完成跨模块校验和影响分析；
- 新工单使用新版本，进行中工单保留原绑定版本；
- 新配置不改变已冻结外部回传内容；
- 所有模块使用统一中文术语；
- 不展示内部 ID、技术字段、认证秘密或技术消息对象。
