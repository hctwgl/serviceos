# 工单工作区状态矩阵

状态：Draft  
视觉方向：A+ 经典专业任务型

## 1. 目的

工单工作区不能只设计“正常履约”截图。实现前必须覆盖以下状态，并明确页面首屏信息、允许动作、阻塞原因和恢复方式。

## 2. 状态矩阵

| 状态 | 首屏必须显示 | 主操作 | 禁止行为 |
|---|---|---|---|
| 待派单 | 当前责任网点、服务区域、预约要求、SLA | 派单 | 不显示不存在的师傅名称，不自动选择默认师傅 |
| 已派单待预约 | 责任师傅、客户联系方式、预约时限 | 确认预约 | 不允许绕过师傅资格或服务范围校验 |
| 已预约待上门 | 预约时间、地址、联系人、变更记录 | 调整预约 | 不把预约变化静默覆盖 |
| 履约进行中 | 当前步骤、已完成步骤、待提交资料、师傅在线状态 | 查看现场进度 | Admin 不代替师傅完成受设备能力约束的现场动作 |
| 待审核 | 提交人、提交时间、资料完整度、校验结果 | 进入审核 | 不把接口成功等同审核通过 |
| 整改中 | 驳回原因、责任人、截止时间、缺失资料 | 跟踪整改 | 不允许隐藏原驳回原因或重新创建平行整改单 |
| SLA 高风险 | 剩余时间、风险来源、当前阻塞、责任人 | 处理风险 | 不仅使用红色标签，必须给出下一步动作 |
| 已完成 | 完成时间、完成结果、关键资料、审计记录 | 查看结果 | 不继续显示可变更履约状态的动作 |
| 已取消 | 取消原因、操作人、影响范围 | 查看记录 | 不允许恢复为进行中，除非产品另有明确重开流程 |
| 无权限 | 用户可见的对象摘要和无权限原因 | 返回列表 | 不泄露敏感字段、内部权限代码或 UUID |
| 只读 | 只读原因、当前状态、可查看内容 | 无主操作 | 不以置灰按钮替代只读解释 |
| 并发冲突 | 已变化字段、最新操作人、最新时间 | 刷新并重新确认 | 不自动覆盖服务器最新结果 |
| 数据不完整 | 缺失的业务展示字段和追踪编号 | 返回或重试 | 不使用 ID、英文枚举或空字符串兜底 |
| 系统错误 | 中文错误标题、影响范围、traceId | 重试或返回 | 不展示堆栈、SQL、Java 类名或第三方原始错误 |

## 3. 页面共同结构

所有状态保持稳定的对象头部：

- 工单编号；
- 客户与项目；
- 当前阶段与中文状态；
- 当前责任网点和师傅；
- SLA 与风险；
- 主要允许动作。

主内容区根据状态突出当前任务，不把所有区块平均展示。右侧上下文栏固定承载联系人、地址、责任关系、风险和下一步建议。

## 4. 动作规则

所有动作来自服务端 `allowed-actions`。前端不得自行推导可操作性。

不可执行动作原则上不进入主操作区；当用户需要理解原因时，展示中文阻塞解释，包括缺失条件、当前责任人和建议处理方式。

## 5. 高保真原型要求

至少生成并评审以下独立画面：

1. 待派单正常状态；
2. 履约进行中；
3. SLA 高风险且存在阻塞；
4. 审核驳回与整改；
5. 无权限只读；
6. 并发冲突；
7. 数据不完整错误。

每张原型必须使用一致的 A+ 设计语言，并标注主操作、状态来源和后端读模型字段。
