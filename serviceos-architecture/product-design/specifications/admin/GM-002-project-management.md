# GM-002 项目管理

状态：Accepted
设计等级：B

## 1. 页面定位

“项目管理”是 Admin 中查找、创建和理解运营项目的唯一权威入口。

正式位置：

```text
客户与项目
└── 项目管理
    ├── 项目列表
    ├── 新建项目
    └── 项目详情
        ├── 项目概览
        ├── 服务与合同
        ├── 项目团队与区域分工
        ├── 履约配置
        ├── 项目工单
        └── 操作记录
```

项目管理必须帮助用户回答：

1. 当前有哪些项目；
2. 每个项目属于哪个客户；
3. 项目处于筹备、服务、暂停还是结束状态；
4. 服务周期是什么；
5. 项目团队与区域分工是否完整；
6. 履约配置是否存在生效版本、草稿或阻塞问题；
7. 当前项目是否已经产生工单和运营风险；
8. 用户下一步应该进入哪个正式业务页面。

## 2. 已确认领域关系

```text
客户（车企）
└── 项目
    ├── 项目基本信息
    ├── 服务与合同信息
    ├── 项目团队与区域分工
    ├── 一份项目履约配置
    │   └── 一个履约工作流
    ├── 项目工单
    └── 操作记录
```

边界：

- 项目是运营业务对象，不是配置容器别名；
- 履约配置是项目的一部分，不是平级对象；
- 项目列表是唯一项目权威列表，不建立“项目配置列表”；
- Admin 工作台只展示跨项目配置待办，点击后进入对应项目详情；
- 项目团队与区域分工不属于履约配置版本；
- 服务范围、工作流、表单资料、SLA、网点准入、审核整改和集成规则属于履约配置；
- 实际工单、派单、预约、审核、整改和回传属于运行数据；
- 公共项目模板、公共流程模板和公共配置模板不进入当前阶段。

## 3. 目标角色

- 项目管理员；
- 运营管理人员；
- 项目实施和交付人员；
- 履约配置人员；
- 具有项目只读权限的审核、客服和管理人员；
- 具有审计权限的审计人员。

能查看项目不代表能创建项目、维护团队、编辑履约配置、发布配置或查看全部项目工单。

## 4. 本规格非目标

GM-002 不负责：

- 设计项目状态迁移的业务前置条件；
- 在没有接受的命令契约时生成启用、暂停、关闭或重新开启按钮；
- 维护车企和品牌主数据；
- 维护项目区域人员分工的具体规则；
- 编辑履约流程、表单、SLA 或网点规则；
- 展开完整工单处理；
- 建设合同管理、结算或发票系统；
- 建设第二套配置中心；
- 兼容旧项目列表、旧复数方案模型或旧路由。

项目状态迁移涉及接单、存量工单、外部集成和配置生效，是核心业务决策。当前只展示机器契约已有状态；具体状态动作必须在后续状态机事实被接受后进入页面。

## 5. 项目列表页面结构

```text
面包屑：客户与项目 / 项目管理
→ 页面标题、说明和“新建项目”
→ 项目状态摘要（仅真实可用时）
→ 常用业务视图
→ 搜索与筛选
→ 已选条件
→ 项目列表
→ 分页、数据时间和刷新状态
```

项目列表不放：

- 新建项目常驻表单；
- 履约配置编辑器；
- 项目团队编辑表格；
- 工单明细；
- 调试接口、项目 UUID、Tenant ID 或配置内部引用；
- 初始化演示数据入口。

## 6. 常用业务视图

默认视图：

- 全部有权项目；
- 筹备中的项目；
- 服务中的项目；
- 已暂停项目；
- 已结束项目。

在页面级 Read Model 能提供真实配置状态后，可以增加：

- 尚无生效履约配置；
- 存在未发布草稿；
- 存在发布阻塞问题。

这些视图只是项目目录筛选，不复制 Admin 工作台“配置待办”的优先级、处理状态和任务投影。

## 7. 搜索与筛选

### 7.1 搜索

搜索框文案：

> 搜索项目名称或项目编码

搜索必须由服务端在当前授权范围内执行。当前目录契约未提供名称或编码搜索参数时，应新增页面级查询能力；不得加载当前一页后在浏览器过滤并声称完成搜索。

### 7.2 默认筛选

- 客户；
- 项目状态；
- 服务日期；
- 履约配置状态。

### 7.3 更多筛选

- 服务行政区域；
- 项目团队完整性；
- 是否存在进行中工单；
- 最近更新时间范围。

筛选选项必须使用客户名称、行政区名称和中文业务状态。禁止输入 `clientId`、region code、项目 UUID 或原始枚举。

### 7.4 筛选一致性

- URL 保存当前筛选；
- 返回列表时恢复筛选、分页和滚动位置；
- 授权变化后旧游标失败关闭；
- 客户、区域、状态和配置筛选由同一个页面查询支持；
- 无权查看的客户或项目不能通过筛选选项和计数泄露。

## 8. 项目状态中文表达

当前机器契约存在四种项目状态，本页面使用：

| 稳定值 | 中文名称 | 页面说明 |
|---|---|---|
| `DRAFT` | 筹备中 | 项目基础信息已建立，尚处于上线准备阶段 |
| `ACTIVE` | 服务中 | 项目处于正常运营服务期 |
| `SUSPENDED` | 已暂停 | 项目服务已暂停，具体影响由服务端状态规则说明 |
| `CLOSED` | 已结束 | 项目已经结束，历史事实继续保留 |

以上只定义用户可见名称，不定义允许迁移路径。页面不得根据枚举自行生成状态动作。

服务周期和项目状态必须分别展示：

- 服务周期是日期事实；
- 项目状态是业务决策；
- 到达开始日期不自动把项目状态显示为“服务中”；
- 到达结束日期不自动把项目状态显示为“已结束”；
- 客户端不得根据当前时间改写状态。

## 9. 列表字段

默认列：

1. 项目名称与项目编码；
2. 客户名称；
3. 项目状态；
4. 服务周期；
5. 服务范围摘要；
6. 项目团队状态；
7. 履约配置状态；
8. 进行中工单摘要；
9. 最近更新时间；
10. 操作。

### 9.1 项目名称与编码

项目名称是主要文本，项目编码是次级稳定业务标识。点击名称进入项目详情。

项目 UUID 不显示，也不能在名称缺失时作为兜底。

### 9.2 客户名称

展示车企业务名称。`clientId` 或 `clientCode` 只作为契约标识，不直接展示给普通用户。

客户名称不可用时，该行进入“项目数据不完整”，不能显示编码代替。

### 9.3 服务周期

展示开始日期和可选结束日期：

```text
2026-08-01 至 2027-07-31
```

没有结束日期时显示“长期”，前提是机器契约明确 `endsOn = null` 代表无固定结束日期。

### 9.4 服务范围摘要

服务范围来自当前生效履约配置，不读取项目旧关系字段拼装第二套范围。

示例：

```text
上海市等 4 个区域
```

无生效配置时显示“尚未发布服务范围”。无权查看履约配置时显示“无权查看”，不能显示零个区域。

### 9.5 项目团队状态

展示：

- 已完整；
- 存在未分配岗位；
- 存在区域分工缺口；
- 尚未配置；
- 无权查看；
- 暂时无法获取。

摘要来自 AD-009 的项目团队读模型，不由列表查询项目成员和区域接口后自行计算。

### 9.6 履约配置状态

展示项目唯一履约配置：

- 尚未配置；
- 有草稿，尚无生效版本；
- 当前生效版本；
- 有未发布草稿；
- 存在阻塞错误；
- 发布处理中；
- 发布失败；
- 无权查看；
- 暂时无法获取。

页面不展示“已发布方案数量”“草稿方案数量”，因为已确认模型是一项目一份履约配置、一个工作流。

### 9.7 进行中工单摘要

有 `workOrder.read` 和对应数据范围时展示真实计数；无权查看时显示“无权查看”；来源不可用时显示“暂时无法获取”。

不得把 `null`、权限不足或截断计数显示为 0。

### 9.8 最近更新时间

由项目页面读模型返回项目事实最近更新时间。当前基础 Project 只提供 `createdAt` 时，不能用创建时间冒充最近更新时间。

## 10. 默认排序

默认排序建议为：

```text
服务中的项目
→ 筹备中的项目
→ 已暂停项目
→ 已结束项目
```

同一状态内按业务名称和项目编码稳定排序。

具体排序必须由服务端执行并写入查询契约。前端不能在当前页重新排序后伪装成全量顺序。

如果现有契约仍固定按 `projectCode/projectId` 排序，应将目标排序登记为页面查询缺口；在新契约完成前诚实使用契约顺序，不在前端制造不一致分页。

## 11. 列表操作

页面主操作：

- 新建项目。

行内主要操作：

- 查看项目。

根据服务端 `allowedActions` 可以展示的深链操作：

- 继续团队与区域分工；
- 进入履约配置；
- 查看项目工单；
- 查看操作记录。

列表中不提供：

- 状态切换开关；
- 直接编辑服务范围；
- 直接绑定网点；
- 直接发布履约配置；
- 直接删除项目；
- 批量启停或批量关闭；
- 多项目批量发布。

## 12. 新建项目

新建项目使用独立页面，不使用列表顶部常驻表单。

### 12.1 当前核心字段

- 项目名称；
- 项目编码；
- 所属客户；
- 服务开始日期；
- 服务结束日期（可选）。

项目编码说明：

- 是租户范围内稳定业务编码；
- 创建时校验格式和唯一性；
- 普通用户不需要理解数据库主键；
- 创建后能否修改必须由接受的业务规则明确，当前页面不提供通用改码动作。

所属客户使用 GM-003 车企主数据选择器。被停用或无权使用的客户不可选，服务端提交时重新校验。

### 12.2 不在创建页配置

- 项目团队成员；
- 区域人员分工；
- 履约服务范围；
- 项目网点和师傅能力；
- 工作流；
- 表单与资料；
- SLA 与预约；
- 审核整改；
- 外部连接和字段映射；
- 初始工单；
- 演示数据。

项目创建成功后进入项目概览，并通过明确的上线准备步骤引导用户维护团队和履约配置。

当前 `CreateProjectRequest` 仍包含 `regionCodes` 和 `networkIds` 时，目标产品不使用这些字段在创建页复制履约配置。后续契约应直接修正并同步调用方，不为旧创建模型保留兼容表单。

### 12.3 创建结果

创建使用幂等键。响应丢失时查询同一幂等结果，不重复创建项目。

失败必须区分：

- 项目编码重复；
- 客户已停用或不可用；
- 日期范围无效；
- 权限不足；
- 幂等键载荷变化；
- 平台暂时不可用。

## 15. 项目概览

项目概览负责理解项目，不直接承载复杂编辑。

结构：

```text
上线与运营准备摘要
→ 基本信息
→ 服务与合同摘要
→ 项目团队与区域分工摘要
→ 履约配置摘要
→ 项目工单摘要
→ 风险与待办
→ 最近操作
```

### 15.1 上线与运营准备摘要

展示：

- 项目基本信息是否完整；
- 团队和区域分工是否完整；
- 是否存在生效履约配置；
- 当前配置是否有草稿或阻塞错误；
- 是否已经进入服务周期；
- 是否存在平台可见的关键风险。

这是服务端聚合摘要，不由前端把多个模块状态相加得到“完成度”。

### 15.2 基本信息

- 项目名称；
- 项目编码；
- 客户名称；
- 项目状态；
- 服务周期；
- 创建时间；
- 最近更新时间。

### 15.3 服务与合同摘要

GM-002 只展示已确认的客户和服务周期。合同编号、合同金额、结算规则和合同状态没有接受的领域模型时不得在页面中伪造字段或建立空壳表单。

后续形成真实合同模型后，“服务与合同”按独立业务规格扩展；当前没有数据时不展示无功能页签内容。

### 15.4 项目团队摘要

- 当前有效成员数量；
- 三个固定项目岗位是否配置；
- 覆盖区域数量；
- 区域分工缺口；
- 最近变更；
- 进入 AD-009 的深链。

### 15.5 履约配置摘要

- 当前生效版本；
- 当前草稿；
- 配置完整度；
- 阻塞错误和警告；
- 发布准备状态；
- 最近发布；
- 进入 AD-005 或具体问题的深链。

### 15.6 项目工单摘要

- 进行中工单；
- SLA 风险；
- 审核整改；
- 外部协同异常；
- 最近创建或完成趋势（只有真实聚合时展示）；
- 进入 AD-002 项目筛选视图的深链。

项目概览不复制工单列表。

## 16. 服务与合同边界

当前阶段“服务与合同”至少承载：

- 所属客户；
- 服务开始和结束日期；
- 项目基础合作说明（只有接受字段时）；
- 后续合同模型的权威入口。

不得把以下内容放入自由文本或通用 JSON：

- 计价规则；
- 结算规则；
- 开票规则；
- 外部回传协议；
- 服务范围；
- SLA；
- 审核规则。

这些内容分别属于明确领域。没有合同模型时不提供伪合同 CRUD。

## 17. 项目团队与区域分工入口

该页签直接进入 AD-009，管理：

- 项目成员；
- 客服经理、项目经理和项目助理；
- 行政区域人员分工；
- 匹配预览和变更记录。

项目概览只显示摘要，不允许在那里直接选择人员。

## 18. 履约配置入口

该页签直接进入 AD-005，并继续进入 AD-006 至 AD-013。

项目管理不得：

- 再建一套“配置方案列表”；
- 显示复数已发布方案数和草稿方案数；
- 允许工作流、表单或 SLA 独立发布；
- 根据当前代码中的多个 FulfillmentProfile 反向修改已确认产品模型；
- 在项目概览直接发布配置。

## 19. 项目工单入口

项目工单复用 AD-002 工单中心的项目筛选视图：

```text
工单中心
筛选：当前 projectId
```

项目详情可以展示摘要和最近工单，但不建立第二套可筛选、可批量、可处理的工单列表。

进入工单工作区后，返回项目详情时保留项目上下文。

## 20. 操作记录

操作记录只读展示：

- 项目创建；
- 基本信息变更；
- 状态变化（未来存在正式命令时）；
- 团队和区域分工修订；
- 履约配置草稿、校验和发布；
- 项目范围和关系变化；
- 权限拒绝和重要失败；
- 关联工单入口（必要时）。

普通业务用户看到中文动作和业务摘要，不展示原始事件名、表名、JSON、异常堆栈或内部版本对象。

## 21. 编辑项目基本信息

编辑使用独立页面或明确业务抽屉，提交使用项目版本或 ETag。

当前产品边界：

- 项目名称、客户、服务周期和项目编码是否可修改，必须由项目状态和已接受业务规则决定；
- 在这些规则形成机器契约前，页面只展示服务端明确返回为可编辑的字段；
- 不提供“全部字段可编辑”的通用表单；
- 不允许通过编辑基本信息绕过履约配置发布、区域分工或合同规则；
- 并发变化不使用最后写入覆盖。

此边界避免在未确认核心状态语义时擅自允许改客户、改编码或缩短服务周期。

## 22. 项目状态动作

页面可以展示 `DRAFT`、`ACTIVE`、`SUSPENDED`、`CLOSED` 的中文状态，但状态动作必须满足：

- 有已接受的状态迁移规则；
- 有明确 Capability；
- 服务端返回 `allowedActions`；
- 说明对新工单、进行中工单、履约配置、外部接入和通知的影响；
- 要求必要原因和二次确认；
- 使用幂等键和并发版本；
- 记录审计。

当前不存在这些完整事实时，不显示启用、暂停、结束或重新开启按钮，也不提供通用状态开关。

物理删除项目不属于本页面能力。

## 24. 当前契约缺口

现有项目目录契约可以提供项目编码、客户标识、名称、服务日期、基础范围关系、状态和版本，但尚不足以直接支撑本页面目标：

- 缺少项目名称或编码搜索；
- 缺少客户显示名；
- 缺少统一中文状态说明；
- 缺少项目团队完整性摘要；
- 缺少项目唯一履约配置状态和版本摘要；
- 仍返回复数 `publishedSchemeCount`、`draftSchemeCount`，与已确认产品模型冲突；
- 缺少进行中工单权限化摘要；
- 缺少最近更新时间和风险；
- 缺少行级 `allowedActions`；
- 固定 `projectCode/projectId` 排序不能表达目标业务优先级。

实现 GM-002 前应直接建立新的页面级 Read Model，并同步 OpenAPI 和调用方。禁止：

- 前端调用项目、客户、团队、配置和工单接口自行 Join；
- 将复数方案数量改名后继续显示；
- 名称缺失时显示 `clientId`；
- 查询失败时把团队、配置或工单显示为零；
- 为旧 Project DTO 保留兼容适配层。

## 25. 权限与数据范围

至少区分：

- `project.read` 或等价项目查看能力；
- 项目创建能力；
- 项目基本信息编辑能力；
- 项目团队查看和维护能力；
- 履约配置查看、编辑和发布能力；
- 工单查看能力及 Project/Region/Network 数据范围；
- 操作记录和敏感诊断查看能力。

目录、总数、筛选选项、项目详情、深链和导出必须使用同一 Tenant/Project/Region/Network 范围。

跨租户对象不得泄露存在性。用户能查看项目但无权查看配置或工单时，相应摘要显示“无权查看”，不能显示零或隐藏成“尚未配置”。

## 26. 页面状态

### 26.1 业务空状态

> 当前还没有项目。创建项目后，可以维护项目团队并配置履约规则。

只有拥有创建权限时显示“新建项目”。

### 26.2 筛选无结果

> 没有符合当前条件的项目。

保留筛选条件并提供调整或清除，不提供演示数据初始化。

### 26.3 部分数据不可用

项目基本信息可以展示，但团队、配置或工单摘要分别标记“暂时无法获取”。不把缺失模块显示为正常、零或已完成。

### 26.4 无权限

- 无权访问项目管理；
- 有项目查看权但数据范围为空；
- 可查看项目但无权查看团队、配置或工单；
- 可查看但不可创建或编辑。

### 26.5 数据不完整

客户名称、项目名称、中文状态或服务周期缺失时，显示项目数据不完整，并提供受权限控制的问题详情。不得显示 ID 或原始枚举顶替。

### 26.6 并发冲突

保留用户输入，展示平台最新项目事实，并要求刷新和重新确认。不能自动覆盖。

## 27. 批量、导入与导出

当前阶段不提供：

- 批量创建项目；
- 批量启用、暂停或结束；
- 批量修改客户和服务周期；
- 批量绑定区域、网点或团队；
- 多项目批量发布履约配置；
- 通过 Excel 导入完整项目模型。

项目目录导出只有在业务用途、字段、范围、敏感信息、审计和保留规则明确后才能单独设计，不因表格存在而默认提供。

## 30. 验收标准

### 权威入口与信息架构

- [ ] 项目列表是唯一项目权威列表；
- [ ] 不存在第二套项目配置列表；
- [ ] 项目详情固定包含概览、服务与合同、团队分工、履约配置、项目工单和操作记录；
- [ ] Admin 工作台配置待办深链到同一项目详情；
- [ ] 项目工单复用工单中心项目视图，不复制第二套列表。

### 项目目录

- [ ] 支持服务端项目名称和编码搜索；
- [ ] 客户、状态、日期、配置、区域和团队筛选在授权范围内执行；
- [ ] 列表展示客户业务名称和中文项目状态；
- [ ] 团队、配置和工单摘要区分零、无权和不可用；
- [ ] 服务范围来自生效履约配置，不读取旧关系字段拼装；
- [ ] 不展示复数方案数量。

### 创建与详情

- [ ] 新建项目使用独立页面和幂等命令；
- [ ] 创建页不提前配置团队、流程、网点和演示数据；
- [ ] 创建成功进入项目概览；
- [ ] 项目概览提供团队、履约配置和工单正式深链；
- [ ] 没有合同模型时不创建伪合同 CRUD；
- [ ] 必要显示值缺失时不使用 ID 兜底。

### 状态、权限和并发

- [ ] 项目状态与服务日期分别表达；
- [ ] 没有接受的状态迁移规则时不显示状态动作；
- [ ] 写操作使用服务端授权、幂等和并发版本；
- [ ] 跨租户对象不泄露存在性；
- [ ] 配置或工单无权查看时不显示为零；
- [ ] 并发变化不使用最后写入覆盖。

### 产品质量

- [ ] 不显示 UUID、原始枚举、内部字段或技术对象；
- [ ] 正常、空、筛选无结果、无权限、只读、部分失败、数据不完整和冲突状态完整；
- [ ] 1280px、1440px、键盘和 200% 缩放可用；
- [ ] 页面读模型由服务端组合，不在前端跨模块 Join；
- [ ] 当前契约缺口在实现前完成 OpenAPI 和调用方同步。

## 31. 禁止实现

禁止：

- 建立“项目配置列表”或恢复一级“配置中心”；
- 一个项目显示多个履约工作流或复数方案数量；
- 在项目创建页配置完整服务范围、网点、流程和团队；
- 用 `regionCodes`、`networkIds` 复制履约配置；
- 用创建时间冒充最近更新时间；
- 用客户编码、项目 UUID 或原始枚举作为显示兜底；
- 前端调用多个接口拼团队、配置和工单状态；
- 查询失败时显示团队完整、配置正常或工单为零；
- 根据服务日期自动改变项目状态；
- 没有状态机事实时生成启停、关闭或重新开启按钮；
- 使用通用开关改变项目状态；
- 物理删除项目；
- 列表单元格直接编辑客户、状态、区域、网点或配置；
- 多项目批量发布；
- 无合同领域模型时使用自由 JSON 或备注字段伪造合同管理；
- 为旧项目 DTO、复数 FulfillmentProfile 或旧路由保留兼容层；
- 根据当前实现反向覆盖已确认的一项目一份履约配置模型。
