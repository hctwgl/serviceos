# AD-012 项目集成与回传

状态：Draft for Product Review  
设计等级：A

## 1. 产品位置

集成与回传位于：

```text
客户与项目
└── 项目详情
    └── 履约配置
        └── 集成与回传
```

本模块定义当前项目如何接收外部工单、如何识别和转换外部业务数据、哪些工单事件需要回传，以及回传失败后如何处理。

它面向项目配置和运营人员，不是技术消息队列控制台，也不把 Inbox、Outbox、Webhook、Topic、Envelope 等内部技术对象作为产品导航。

## 2. 真实业务链路

### 2.1 入站工单

```text
客户或合作方系统提交工单
        ↓
识别外部协同方和当前项目
        ↓
校验外部单号、字段和业务范围
        ↓
执行项目字段映射
        ↓
检查重复提交和允许的业务动作
        ↓
创建工单 / 更新允许字段 / 进入人工确认
        ↓
记录入站结果并返回业务响应
```

### 2.2 状态与结果回传

```text
工单发生已配置的业务事件
        ↓
冻结当次回传业务快照
        ↓
按项目回传规则生成业务数据
        ↓
通过已启用的外部连接发送
        ↓
对方确认接收 / 拒绝 / 暂时失败
        ↓
完成 / 自动重试 / 创建人工处理待办
```

关键原则：

- 履约配置定义业务规则和字段映射；
- 系统管理维护外部连接、认证凭据、证书、网络和技术协议；
- 工单运行时保存实际入站、回传、确认、失败和重试记录；
- 发布新配置不能改写已经接收的工单或已经生成的回传快照；
- 运营页面使用业务语言解释结果，技术诊断进入受控工具。

## 3. 当前阶段边界

### 3.1 进入履约配置版本的内容

以下项目规则随履约配置整体发布：

- 当前项目允许使用哪些已启用外部连接；
- 外部业务如何识别当前项目；
- 哪些外部业务类型允许创建工单；
- 外部单号和 ServiceOS 工单的对应规则；
- 入站必填字段和业务校验；
- 外部字段与项目工单字段的映射；
- 重复提交时的处理方式；
- 外部更新和取消允许修改什么；
- 哪些工单业务事件需要回传；
- 每种回传事件包含哪些业务字段；
- 外部状态与内部业务状态的映射；
- 敏感信息是否允许回传以及如何脱敏；
- 业务通知对象和通知时机；
- 失败后的业务重试、告警和人工处理规则；
- 入站与回传场景的校验和预览规则。

### 3.2 不进入履约配置版本的内容

以下内容属于系统级连接配置或运行数据：

- 接口地址、消息代理地址和网络路由；
- 账号、密码、令牌、密钥和证书；
- IP 白名单和防火墙规则；
- 底层协议、序列化格式和技术消息头；
- 实际入站请求和原始报文；
- 实际回传任务和发送尝试；
- 对方响应和确认记录；
- 技术重试队列和死信记录；
- 连接健康状态和实时延迟；
- 技术跟踪标识和诊断日志。

项目配置只引用系统管理员已经建立并授权给当前项目使用的外部连接，不在项目页面保存或展示认证秘密。

## 4. 外部连接与项目规则的关系

一个项目可以关联一个或多个已启用的外部连接，例如客户工单系统、合作方平台或业务通知渠道。

```text
系统管理：外部连接
├── 连接方式
├── 认证与证书
├── 网络与安全
├── 可用业务能力
└── 技术健康状态

项目履约配置：集成与回传
├── 选择允许使用的连接
├── 入站业务规则
├── 字段和状态映射
├── 回传业务规则
└── 失败业务处理
```

项目页面必须展示连接的业务名称、可用能力、授权范围和当前可用性摘要，不显示密钥、完整地址或内部连接标识。

连接被系统管理员停用或失效时：

- 项目生效配置仍保留引用关系和历史解释；
- 新入站或回传按明确异常规则处理；
- 页面显示“外部连接不可用”；
- 不得静默切换到其他连接；
- 不得使用旧凭据或未授权连接兜底。

## 5. 页面结构

点击“集成与回传”后进入统一工作区，包含五个一级页签：

1. 对接概览；
2. 入站工单；
3. 字段与状态映射；
4. 状态与结果回传；
5. 场景预览。

默认打开“对接概览”。

页面顶部持续展示：

- 项目名称；
- 当前生效配置版本；
- 当前草稿版本；
- 已关联外部连接数量；
- 入站规则数量；
- 回传规则数量；
- 校验错误和警告数量；
- 最近保存状态；
- 返回配置概览。

运行记录和技术诊断不作为履约配置一级页签。用户从具体工单或异常待办进入对应运行记录；具有技术权限的人员再进入受控诊断工具。

## 6. 对接概览

对接概览负责回答：

- 当前项目与哪些外部系统协同；
- 哪个连接负责接收工单；
- 哪些业务事件需要回传；
- 字段和状态映射是否完整；
- 外部连接当前是否可用；
- 是否存在阻止发布的问题；
- 最近发生了哪些业务异常；
- 用户下一步应处理什么。

至少展示：

- 外部协同方和连接摘要；
- 入站业务范围；
- 回传事件摘要；
- 字段映射完整度；
- 状态映射完整度；
- 敏感信息回传摘要；
- 连接可用性摘要及更新时间；
- 阻塞错误；
- 重要警告；
- 推荐下一步。

连接实时健康信息只作为参考，不进入草稿保存和版本差异。

## 7. 入站工单规则

### 7.1 列表字段

至少展示：

- 规则名称；
- 外部协同方；
- 使用的外部连接；
- 外部业务类型；
- 项目识别方式；
- 允许的业务动作；
- 重复提交处理；
- 当前状态；
- 校验问题；
- 最近修改人和时间。

允许的业务动作当前包括：

- 创建新工单；
- 补充尚未确认的外部信息；
- 更新规则明确允许修改的字段；
- 请求取消工单；
- 查询接收结果。

外部系统不得通过通用更新接口任意修改工单流程状态、项目岗位人员、责任网点、责任师傅、审核结论或历史业务记录。

### 7.2 项目识别

入站规则必须明确外部业务如何识别当前项目，例如：

- 外部项目编码与当前项目固定对应；
- 外部合同编码与当前项目固定对应；
- 当前外部连接只允许向一个明确项目提交；
- 由受控业务映射表确定项目。

当前阶段禁止：

- 根据项目名称模糊匹配；
- 未识别项目时使用默认项目；
- 根据历史最近工单猜测项目；
- 由浏览器临时选择项目后绕过服务端校验。

无法识别项目时，必须拒绝或进入明确的“外部工单待确认”，不能创建到任意项目。

### 7.3 外部单号与幂等

每条入站规则必须定义外部业务唯一标识，至少由以下内容组成：

```text
外部协同方 + 外部业务类型 + 外部单号
```

系统必须使用该业务唯一标识防止重复创建工单。

重复提交时只能执行已配置的明确行为：

- 返回已存在工单的接收结果；
- 对比后忽略完全重复内容；
- 更新明确允许更新的字段；
- 发现冲突后进入人工确认；
- 拒绝并返回具体冲突原因。

禁止因重复请求创建第二张业务工单，也禁止覆盖已经发生的流程、派单、预约、审核和整改数据。

### 7.4 地址和业务范围

外部工单创建前必须校验：

- 服务地址是否足以解析为标准行政区域；
- 地址是否位于项目服务范围；
- 服务类型是否属于项目；
- 必要客户和联系人信息是否完整；
- 是否满足项目接单条件。

地址无法识别、超出服务范围或服务类型无效时，应返回明确业务错误或进入人工确认，不能静默修正为相邻区域或默认服务类型。

工单创建成功后，后续由项目团队与区域分工生成人员快照，并由网点与资源提供派单候选。集成模块不重新实现人员和网点匹配。

### 7.5 外部更新与取消

项目必须为每种外部动作明确允许范围。

外部更新可以允许修改的内容示例：

- 尚未确认的联系人信息；
- 尚未开始履约时的服务地址；
- 客户补充说明；
- 外部业务参考信息。

以下内容不得被外部更新直接覆盖：

- 已绑定的履约配置版本；
- 工单项目岗位人员快照；
- 已确认责任网点和师傅；
- 已确认预约；
- 已完成表单和资料快照；
- 审核结论和整改记录；
- 已发生的流程历史。

外部取消只能发起取消请求。是否允许取消、由谁确认以及取消后的业务影响由工单流程和允许动作决定，不能由外部请求直接删除或关闭工单。

## 8. 字段映射

### 8.1 映射原则

字段映射必须使用双方可理解的业务名称。

页面展示：

- 外部业务字段名称和说明；
- ServiceOS 业务字段名称和说明；
- 数据类型；
- 是否必填；
- 转换规则摘要；
- 默认值是否允许；
- 敏感信息标识；
- 校验问题。

普通项目配置人员不填写 JSON Path、XPath、SQL、脚本或内部数据库字段。

若某个外部协议需要技术字段路径或报文结构，应由系统管理员在外部连接的受控技术配置中注册为可选择的业务字段，再提供给项目映射页面使用。

### 8.2 转换规则

当前阶段允许的业务转换包括：

- 固定值；
- 选项值映射；
- 日期和时间格式转换；
- 金额和数字单位转换；
- 地址组成或拆分；
- 空值处理；
- 字符长度限制；
- 明确的文本拼接；
- 脱敏输出。

不允许项目配置人员编写任意脚本。

默认值只能用于业务明确允许的字段，并必须在页面显示其业务效果。项目、服务类型、行政区域、责任人员和责任网点等关键字段不得使用隐式默认值。

### 8.3 映射版本

每次入站和回传都必须记录使用的履约配置版本和映射版本。

发布新配置后：

- 新入站使用新映射；
- 已创建工单不重新映射历史入站数据；
- 已生成但尚在重试的回传继续使用生成时冻结的业务快照和映射；
- 不允许重试时按最新配置重新拼装不同内容。

## 9. 状态映射

内部工单状态和外部状态必须通过明确业务映射关联。

每条映射至少定义：

- ServiceOS 业务事件或状态；
- 外部状态名称；
- 是否需要附加原因；
- 是否需要时间；
- 是否需要责任方信息；
- 是否允许重复回传；
- 外部确认方式；
- 无对应外部状态时的处理。

不得把内部技术枚举直接作为外部状态发送。

多个内部状态映射到同一个外部状态时，页面必须显示信息损失和业务影响。缺少必要状态映射时阻止发布或明确标记为不回传，不能运行时自行猜测。

## 10. 状态与结果回传规则

### 10.1 回传触发事件

当前阶段可以选择明确的工单业务事件，例如：

- 工单已受理；
- 已分配责任网点；
- 已分配师傅；
- 预约已确认；
- 已到场；
- 作业已完成；
- 审核已通过；
- 已发起整改；
- 整改已通过；
- 工单已完成；
- 工单已取消；
- 工单进入异常处理。

回传规则引用流程和业务事件，不自行创建隐藏流程状态。

### 10.2 回传内容

每条回传规则至少定义：

- 规则名称；
- 外部协同方和连接；
- 触发业务事件；
- 外部业务类型；
- 外部单号；
- 外部状态；
- 回传字段集合；
- 是否包含表单结果摘要；
- 是否包含资料链接或资料摘要；
- 是否包含项目岗位人员信息；
- 是否包含责任网点或师傅信息；
- 敏感信息处理；
- 对方确认要求；
- 失败处理规则。

外部系统需要负责人信息时，只能使用工单当前项目岗位人员快照中的业务信息，不回传项目区域分工内部结构，也不重新按地址计算负责人。

### 10.3 表单和资料回传

回传表单和资料时必须明确：

- 回传哪些表单和字段；
- 回传当前提交还是最终通过版本；
- 是否只回传摘要；
- 文件采用安全链接还是受控传输；
- 链接有效期；
- 是否允许对方再次下载；
- 敏感资料是否允许回传；
- 审核未通过的资料是否允许回传。

不得默认回传全部字段和全部附件。

运行时回传使用工单绑定的表单、资料和审核结果快照，不因项目后来修改表单定义而改变历史业务结果。

## 11. 业务通知

项目可以为入站、回传和异常配置业务通知，例如：

- 外部工单接收失败；
- 外部工单进入人工确认；
- 回传被对方拒绝；
- 多次重试后仍未成功；
- 连接不可用；
- 关键状态超过回传时限；
- 敏感信息策略阻止回传。

工单已经创建时，通知对象可以引用：

- 当前工单客服经理；
- 当前工单项目经理；
- 当前工单项目助理；
- 当前责任网点；
- 当前节点处理人；
- 具有集成异常处理权限的人员。

三个项目岗位统一读取工单人员快照。

工单尚未创建时，不存在工单人员快照。此类入站异常只能通知明确的项目配置负责人、具有集成异常处理权限的人员或系统集成运营人员，不能根据不完整地址猜测项目岗位人员。

通知渠道由已启用外部连接或系统通知能力提供，本模块只选择业务渠道和通知对象。

## 12. 失败、重试与人工处理

### 12.1 业务状态

运营页面使用以下业务状态：

- 待发送；
- 发送中；
- 已送达；
- 对方已确认；
- 对方拒绝；
- 暂时失败，等待重试；
- 重试已用尽；
- 等待人工处理；
- 已人工解决；
- 已取消回传。

不得只显示 HTTP 状态码、异常类名或消息队列状态。

### 12.2 自动重试

项目规则可以定义业务层重试策略摘要：

- 哪些暂时性失败允许自动重试；
- 最长重试时间；
- 重试用尽后的通知对象；
- 是否创建人工处理待办；
- 是否阻止后续业务动作。

具体重试调度、退避算法和队列实现属于技术能力，不在项目页面配置。

以下情况不得无条件自动重试：

- 对方明确拒绝业务内容；
- 外部单号冲突；
- 字段映射错误；
- 敏感信息策略不允许发送；
- 项目或连接授权已失效；
- 对方确认该业务状态非法。

### 12.3 人工处理

人工处理页面必须展示：

- 工单和项目；
- 外部协同方；
- 业务事件；
- 业务失败原因；
- 已尝试次数和最近时间；
- 使用的配置版本；
- 已冻结的回传摘要；
- 允许执行的业务动作；
- 操作影响。

允许的业务动作可以包括：

- 修正明确允许修正的外部参考信息；
- 在不改变业务事实的前提下重新发送同一冻结快照；
- 标记对方已通过其他渠道确认；
- 取消不再需要的回传并填写原因；
- 转交系统集成运营人员；
- 创建业务异常记录。

不得允许运营人员直接编辑已经冻结的业务结果后伪装成原事件重新发送。确需更正业务事实时，必须通过对应工单业务操作产生新的业务事件和新回传。

## 13. 场景预览

配置人员可以使用脱敏示例数据预览：

- 外部工单能否识别到当前项目；
- 外部单号如何判重；
- 地址和服务类型是否通过校验；
- 外部字段如何映射为工单字段；
- 哪些字段缺失或转换失败；
- 指定业务事件会生成什么回传摘要；
- 内部状态会映射成什么外部状态；
- 哪些敏感信息会被排除或脱敏；
- 失败时会创建什么待办和通知。

预览不得：

- 创建真实工单；
- 占用外部单号；
- 发送真实回传；
- 发送真实通知；
- 暴露认证凭据；
- 使用未脱敏生产报文作为示例。

连接可用性测试属于系统级受控操作，项目预览只读取测试结果或执行不包含业务数据的授权检查。

## 14. 校验规则

发布前至少校验：

- 引用的外部连接是否存在且已授权给当前项目；
- 入站规则是否能够唯一识别当前项目；
- 外部业务类型是否映射到有效项目服务类型；
- 外部单号和幂等规则是否完整；
- 必填字段映射是否完整；
- 关键字段是否使用不允许的默认值；
- 状态映射是否覆盖所有已选择回传事件；
- 外部更新是否试图覆盖受保护工单数据；
- 外部取消是否存在明确流程处理；
- 回传规则是否引用有效业务事件；
- 回传字段是否引用有效表单、字段和资料定义；
- 敏感信息是否具有明确回传许可和脱敏规则；
- 项目岗位人员字段是否统一来自工单人员快照；
- 失败处理是否具有明确通知和人工出口；
- 已停用连接是否仍被规则引用；
- 多条规则是否会对同一事件产生冲突或重复回传；
- 场景预览是否存在无法解释的转换错误。

错误阻止发布。连接暂时不可用可以作为警告或错误，必须由明确项目规则决定，不能静默忽略。

## 15. 与其他模块的关系

### 15.1 与流程设计

流程设计定义业务事件和外部任务节点；集成与回传选择这些业务事件作为入站后续动作或回传触发点。

集成模块不得自行创建隐藏流程路径或直接改写流程状态。

### 15.2 与表单和资料

字段与结果回传引用项目表单、字段和资料要求。删除或停用被回传规则引用的内容时必须阻止，并列出引用位置。

### 15.3 与 SLA

项目可以为外部接收确认、关键状态回传和人工处理引用已定义的 SLA 指标。本模块不重新实现服务日历、时间计算和预警算法。

### 15.4 与项目人员快照

回传和通知需要客服经理、项目经理或项目助理时，统一读取工单项目岗位人员快照，不重新执行行政区域匹配。

### 15.5 与网点、审核和整改

责任网点、责任师傅、审核结论和整改结果来自工单运行记录。集成模块只选择是否以及如何回传这些业务结果，不重新执行派单、审核或整改逻辑。

## 16. 版本与运行规则

- 生效项目规则只读；
- 修改进入当前履约配置草稿；
- 入站、映射、回传和通知规则随履约配置整体发布；
- 新入站工单使用新发布规则；
- 已创建工单保留创建时绑定的履约配置版本；
- 每次入站和回传记录实际使用的配置版本；
- 已生成回传冻结当次业务快照和映射；
- 新配置发布不修改待重试回传内容；
- 已确认的外部结果和历史发送记录不可覆盖；
- 当前阶段不支持将进行中工单自动迁移到新集成规则。

外部连接的认证和网络配置可以在系统级维护，但不得因此改变项目规则的业务含义。连接切换必须显式完成授权、校验和影响确认。

## 17. 页面读模型

建议使用：

- `ProjectIntegrationAndCallbackView`；
- `ProjectIntegrationOverviewView`；
- `ProjectInboundRuleListView`；
- `ProjectInboundRuleDetailView`；
- `ProjectBusinessFieldMappingView`；
- `ProjectStatusMappingView`；
- `ProjectCallbackRuleListView`；
- `ProjectCallbackRuleDetailView`；
- `ProjectIntegrationScenarioPreviewView`；
- `ProjectIntegrationValidationView`；
- `WorkOrderExternalCollaborationView`；
- `ExternalCollaborationExceptionView`。

项目配置读模型必须直接提供：

- 外部协同方和连接业务名称；
- 连接授权与可用性摘要；
- 外部和内部业务字段名称；
- 业务状态名称；
- 映射说明；
- 敏感信息摘要；
- 校验问题；
- 允许动作；
- 配置并发版本。

运行记录读模型可以包含受控技术摘要，但普通运营用户不得看到认证秘密、完整原始报文、内部队列标识或异常堆栈。

## 18. 业务命令

项目配置命令至少包含：

- 关联已授权外部连接；
- 移除未被规则使用的外部连接引用；
- 新建和更新入站工单规则；
- 保存业务字段映射；
- 保存业务状态映射；
- 新建和更新回传规则；
- 保存敏感信息回传规则；
- 保存失败业务处理规则；
- 执行模块校验；
- 生成入站场景预览；
- 生成回传场景预览；
- 查看引用关系；
- 放弃当前模块未保存修改。

运行时命令至少包含：

- 接收外部工单；
- 处理允许的外部更新；
- 处理外部取消请求；
- 生成业务回传；
- 重新发送冻结回传；
- 标记外部已确认；
- 标记人工解决；
- 取消回传并记录原因；
- 转交集成异常处理；
- 查看协同操作记录。

所有命令必须校验项目、履约配置版本、权限、幂等键和并发版本，并返回明确中文业务错误。

## 19. 权限

至少区分：

- 查看生效集成规则；
- 查看配置草稿；
- 维护入站规则；
- 维护字段和状态映射；
- 维护回传规则；
- 配置敏感信息回传；
- 执行校验和场景预览；
- 查看连接可用性摘要；
- 查看工单外部协同记录；
- 处理业务级协同异常；
- 查看受控技术诊断；
- 发布项目履约配置。

查看或编辑认证凭据属于系统级连接权限，不包含在项目配置权限中。

无编辑权限时展示真正的只读页面，不用大量禁用控件模拟查看状态。

## 21. 当前阶段不包含

- 在项目页面创建或编辑认证秘密；
- 通用 API 网关管理；
- 通用消息队列管理；
- Inbox、Outbox、死信队列技术控制台；
- 任意脚本式字段转换；
- 普通运营人员直接编辑 JSON、XML 或协议报文；
- 公共集成模板市场；
- 根据项目名称或地址文本模糊猜测项目；
- 默认项目、默认服务类型或默认责任人静默兜底；
- 外部请求直接覆盖流程、派单、审核和整改结果；
- 发布新规则后自动重写历史入站和回传记录；
- 自动迁移进行中工单到新规则。

## 22. 验收标准

- 用户能够区分项目业务规则、系统级外部连接和实际运行记录；
- 页面使用入站工单、状态回传、结果回传等业务语言，而不是技术队列语言；
- 外部工单能够通过明确项目映射和幂等规则安全接收；
- 重复提交不会创建重复工单或覆盖历史业务数据；
- 外部更新和取消具有明确允许范围；
- 字段和状态映射使用可理解的业务名称；
- 敏感信息不会被默认全部回传；
- 回传内容在生成时冻结，新配置不会改变待重试内容；
- 项目岗位人员统一来自工单人员快照；
- 责任网点、师傅、审核和整改结果统一来自工单运行记录；
- 失败原因、重试和人工处理对运营人员可解释；
- 技术凭据和完整原始报文不暴露给普通运营人员；
- 所有项目规则随履约配置整体发布；
- 当前阶段不建设通用集成平台或公共模板。
