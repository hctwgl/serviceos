# IOS-002 任务详情

状态：Accepted
设计等级：A

## 1. 页面定位

“任务详情”是师傅理解并继续处理一项当前责任任务的权威移动工作区。

它必须让师傅在首屏回答：

1. 这是什么任务；
2. 何时、去哪里、为谁服务；
3. 当前进行到哪一步；
4. 下一步应该做什么；
5. 哪些内容已经在本机保存、正在上传或已经由平台确认；
6. 当前是否存在责任、版本、预约、工作包或同步风险。

任务详情不是 Admin 工单工作区的移动复制，也不是把所有现场功能堆在一个长页面。它提供最小必要上下文、步骤投影、状态解释和正式业务入口；联系预约、到场、表单资料、提交、整改和冲突分别进入后续专用页面。

## 2. 产品边界

IOS-002 负责：

- 展示当前责任任务的最小完整业务上下文；
- 展示客户、地址、预约、服务对象和安全要求；
- 展示当前步骤、完成条件、下一步和阻止原因；
- 合并服务器任务事实与本机工作包、草稿、上传和同步摘要；
- 展示最近业务进展和未完成事项；
- 在在线、离线、改派、撤权、版本变化和冲突时解释正确入口；
- 深链到 IOS-003 至 IOS-010 的专用工作区。

IOS-002 不负责：

- 修改责任网点或责任师傅；
- 编辑项目、流程、表单或资料规则；
- 在详情页直接完成预约确认、到场、表单提交、资料上传或工单完成；
- 根据任务状态在客户端自行推导业务命令；
- 展示其他师傅、其他网点或总部治理信息；
- 使用本机草稿覆盖服务器事实。

## 3. 访问与责任校验

在线访问必须同时校验：

```text
authenticated principal
+ trusted Technician Context
+ ACTIVE TechnicianProfile
+ ACTIVE NetworkTechnicianMembership
+ required capability
+ ACTIVE technician service or task responsibility
```

无法证明当前责任时统一失败关闭，不通过不同错误泄露任务是否存在。

离线访问必须同时满足：

- 当前账号和设备隔离有效；
- 本机存在属于当前账号的有效工作包；
- 工作包未过期；
- 工作包责任版本与本机已知 tombstone 不冲突；
- 本机未收到设备撤销、账号失效或责任终止结果。

普通缓存只允许展示非敏感摘要，不等同于可离线作业的工作包。

## 4. 页面结构

```text
任务头部 + SLA + 同步可信度
→ 当前步骤与下一动作
→ 客户、地址与预约
→ 服务对象与作业说明
→ 现场步骤完成度
→ 表单、资料与提交摘要
→ 风险、异常与整改
→ 最近业务进展
→ 固定底部主操作
```

首屏优先级固定为：任务风险、当前步骤、下一动作、预约和地址。历史记录、技术摘要和低频信息不应把主要动作挤出首屏。

## 5. 任务头部

至少展示：

- 任务标题；
- 工单号；
- 项目或服务名称；
- 当前业务状态；
- SLA 或业务截止时间；
- 预约时间窗口；
- 当前责任网点名称；
- 当前数据截至时间；
- 在线、离线、待同步或冲突状态。

任务头部不展示：

- 内部 Task ID、WorkOrder ID 或 UUID；
- 英文状态枚举；
- 项目内部价格；
- 其他责任网点或其他师傅详情；
- Capability、Scope、版本内部字段；
- 技术异常和原始 JSON。

资源版本、责任版本和工作包版本用于客户端校验，不作为普通师傅可见字段。

## 6. SLA 与时间语义

SLA、预约和作业时间使用服务端返回的业务时区和服务器时间基准。

页面必须区分：

- 正常；
- 即将超时；
- 已超时；
- 按业务规则暂停；
- 暂停等待确认；
- 数据截至时间过旧，无法可靠计算。

离线时可以根据服务器时间偏移展示估算倒计时，但必须标记“截至最近同步”，不能由设备时间单独产生超时结论。

师傅不能在详情页暂停、重置、豁免或关闭 SLA。

## 7. 当前步骤

当前步骤区域展示：

- 当前步骤名称；
- 为什么需要完成；
- 进入时间；
- 完成条件；
- 仍缺少的表单、资料或确认；
- 是否允许离线执行；
- 下一步；
- 异常入口。

步骤条只是服务端流程、任务、表单和资料状态的投影，不是客户端维护的第二套工作流。

客户端不得因为本地字段已经填写或文件已经拍摄，就把服务端步骤标记为完成。可以显示：

```text
本机已完成，等待同步
```

但不能显示：

```text
平台已完成
```

除非服务端已经确认对应业务资源和版本。

## 8. 下一动作

在线下一动作来自服务端 allowed-actions；离线下一动作来自仍有效的工作包 allowedOfflineActions。

常见下一动作：

- 联系客户；
- 确认或调整预约；
- 打开导航；
- 到场签到；
- 开始作业；
- 继续当前步骤；
- 填写表单；
- 拍摄资料；
- 处理待上传；
- 提交前检查；
- 处理整改；
- 处理同步冲突；
- 查看平台协调结果。

每个不可执行动作必须说明原因，例如：

- 尚未确认预约；
- 未到允许签到时间；
- 工作包未准备或已经过期；
- 定位或相机权限不满足任务要求；
- 责任已经变化；
- 表单或资料配置版本不匹配；
- 存在未处理同步冲突；
- 客户端版本不支持任务要求。

客户端不得提供“忽略限制并继续”的通用按钮。

## 9. 固定底部操作

底部操作最多突出一个主操作和一个直接相关的次操作。

示例：

```text
联系客户        确认预约
打开导航        到场签到
查看缺失项      继续填写
查看上传队列    提交前检查
查看退回原因    处理整改
```

当任务失权、工作包过期或存在阻断冲突时，主操作替换为“查看原因”或“处理冲突”，不得继续展示旧业务动作。

首页和详情页的下一动作必须使用同一服务端或工作包事实，不得出现相互矛盾的按钮。

## 10. 客户信息

客户区只展示完成当前任务所需的最少信息：

- 客户脱敏姓名；
- 经过字段策略授权的联系电话入口；
- 客户联系偏好；
- 当前任务相关的客户要求；
- 最近有效联系结果摘要。

默认不在页面长期明文展示完整手机号。点击“联系客户”时，由服务端字段策略和当前责任再次校验，再通过系统拨号能力发起联系。

返回 App 后进入 IOS-003 记录联系结果。拨号本身不等于已经联系成功，也不能自动生成联系结论。

禁止：

- 自动录音；
- 把完整号码写入客户端日志、剪贴板或普通偏好；
- 在任务撤权后继续显示号码；
- 将未确认的语音转文字直接保存为正式联系记录。

## 11. 地址与导航

地址区展示：

- 服务地址；
- 标准行政区域；
- 地址补充说明；
- 预约窗口；
- 可靠距离或预计路程摘要；
- “打开导航”入口。

导航使用系统地图能力，并明确传递的是当前任务地址快照。打开地图不等于到场签到，也不产生 Visit。

离线工作包可以包含完成任务所需的地址快照；普通任务缓存不得长期保存完整地址。任务改派、撤权或设备撤销后，地址和导航入口立即失效。

## 12. 预约摘要

展示：

- 当前有效预约 Revision；
- 预约状态；
- 时间窗口；
- 业务时区；
- 客户确认渠道和确认状态；
- 最近改约原因；
- 与师傅日程的冲突；
- 是否允许继续联系或改约。

本机预约草稿必须显示“已保存到本机”或“待同步”，不能替换服务器当前预约，也不能显示“客户已确认”。

通知发送失败不改变已经成立的预约事实，但页面必须显示通知结果和恢复入口。

## 13. 服务对象与作业说明

只展示当前任务需要的：

- 服务类型；
- 服务产品或设备摘要；
- 车辆、充电设施或现场对象的授权字段；
- 故障或作业说明；
- 项目特殊要求；
- 必要工具、技能或资质提醒；
- 安全注意事项。

不展示与当前任务无关的完整资产档案、合同、价格、后台备注或其他工单历史。

安全提示必须在相关步骤前清楚展示。紧急安全风险应优先提示停止作业和联系渠道，不能要求师傅先完成冗长表单。

## 14. 现场步骤投影

标准步骤投影可以包括：

```text
联系与预约
→ 到场签到
→ 现场作业
→ 填写表单
→ 拍摄资料
→ 提交前检查
→ 离场
```

实际步骤、名称、顺序和要求来自工单绑定的履约配置版本。

每一步展示：

- 状态；
- 服务端确认结果；
- 本机草稿或待同步摘要；
- 缺失项；
- 是否允许进入；
- 正确的专用页面入口。

已提交版本只读。需要修改时必须通过正式整改、补充或新 Revision，不能在详情页直接改写历史。

## 15. 表单摘要

表单区只展示：

- 应填写表单数量；
- 已完成草稿数量；
- 已提交并确认数量；
- 必填错误数量；
- 远程数据源或版本风险；
- 最近本机保存时间；
- “继续填写”入口。

不在任务详情中渲染完整动态表单。

条件显隐、字段校验和草稿冲突由 IOS-006 处理。详情页不得根据字段数量自行判断业务可提交。

## 16. 资料与上传摘要

资料区展示：

- 必需资料组数量；
- 已拍摄到本机数量；
- 待上传数量；
- 上传中数量；
- 平台校验中数量；
- 已确认可用数量；
- 失败、隔离和冲突数量；
- 最近状态时间；
- “继续拍摄”或“查看上传中心”入口。

必须严格区分：

```text
已保存到本机
待上传
上传中
平台校验中
已上传并确认
已隔离待处理
```

文件上传完成不等于资料已经通过审核，资料审核通过也不等于任务已经完成。

## 17. 提交摘要

提交区展示：

- 当前是否满足本地预检；
- 服务端仍需验证的事项；
- 仍在上传或校验中的资料；
- 预约、Visit、责任、权限和配置版本风险；
- 本机待发送提交命令；
- 最近服务端提交结果；
- “进入提交前检查”入口。

详情页不能直接提供“完成任务”按钮。只有 IOS-007 完成全部检查并由服务端接受必要业务对象后，才能显示权威提交或完成结果。

## 18. 整改摘要

存在有效整改时展示：

- 整改主题；
- 当前轮次；
- 被退回项目数量；
- 退回原因摘要；
- 截止时间和 SLA；
- 已补传和待补传数量；
- 复核状态；
- “处理整改”入口。

已通过项保持只读，不在任务详情中重新打开编辑。新资料必须形成新 Revision，师傅不能修改审核或复核结论。

## 19. 异常入口

任务详情提供业务化异常入口，例如：

- 无法联系客户；
- 无法按时到场；
- 地址或现场不符；
- 无法施工；
- 存在安全风险；
- 设备或材料不足；
- App、工作包或上传阻止继续；
- 需要网点或平台协调。

异常入口不能只有自由文本。用户选择标准原因后，系统根据业务规则要求说明、资料、是否重新预约和后续责任。

提交异常不会由 iOS 自行关闭工单，也不会直接修改责任网点或责任师傅。

## 20. 最近业务进展

详情底部可以展示与当前任务直接相关的最近记录：

- 最近预约和联系；
- 到场与离场摘要；
- 表单提交版本；
- 资料 Revision；
- 审核与整改轮次；
- 责任变化；
- 业务异常和协调结果。

只展示中文业务摘要、操作者角色、时间和结果，不展示内部事件、消息队列、数据库状态或完整审计对象。

完整历史进入专用记录页面，不在当前详情无限展开。

## 21. 服务器事实与本机状态合并

任务详情由以下事实合并：

```text
服务器任务详情
├── 当前责任与任务状态
├── 预约、SLA 和步骤投影
├── 表单、资料、整改和提交摘要
├── allowed-actions 和阻止原因
└── 资源、责任与配置版本

本机任务运行时
├── 工作包
├── 表单草稿
├── 本地文件与上传队列
├── 离线命令
└── 冲突与隔离状态
```

本机状态只叠加本机事实，不能覆盖服务器任务、预约、Visit、提交、审核或责任结果。

在线刷新必须保留仍未同步的本机成果。服务器新版本与本机成果冲突时进入 IOS-010，不使用最后写入覆盖。

## 22. 在线与离线模式

### 22.1 在线且数据新鲜

- 展示服务器最新详情；
- 合并本机未同步摘要；
- 所有写入口在进入专用页面和提交时再次鉴权；
- allowed-actions 发生变化时立即刷新主操作。

### 22.2 离线且工作包有效

- 展示工作包锁定的最小任务上下文；
- 展示服务器快照时间、工作包版本和有效期；
- 只允许工作包明确列出的离线动作；
- 本机结果显示为本地保存或待同步；
- 最终仍由服务端逐命令复核。

### 22.3 离线但只有普通缓存

- 只读展示非敏感摘要；
- 隐藏完整电话、详细地址和受限资料；
- 不提供现场写动作；
- 提示联网或准备工作包。

### 22.4 工作包过期或配置不匹配

- 阻止新的离线业务动作；
- 保留已经产生的本机成果；
- 显示具体原因和同步入口；
- 不自动迁移或删除草稿。

## 23. 改派、撤权和设备撤销

页面打开期间收到责任变化时：

1. 立即禁用所有旧业务动作；
2. 清除客户电话、详细地址和新资料下载权限；
3. 显示非敏感责任变化说明；
4. 没有本机成果时安全返回今日任务；
5. 有未同步成果时进入只读隔离和 IOS-010；
6. 服务端拒绝所有旧责任命令；
7. 不允许旧师傅刷新后继续执行。

设备撤销或账号失效时，不再展示任务详情。未同步成果进入受控恢复流程，不能静默删除，也不能继续以旧身份上传。

## 24. 并发与数据过期

以下变化必须拒绝旧操作并要求刷新或进入冲突处理：

- 任务或工单版本变化；
- 责任版本变化；
- 预约 Revision 变化；
- Visit 状态变化；
- 表单、资料槽位或配置版本变化；
- 整改轮次变化；
- allowed-action 失效；
- 工作包过期或被撤销；
- 客户端能力不再满足任务要求。

页面不得因为刷新失败而把本机草稿解释成服务器最新结果。

## 25. 页面读模型与本机投影

建议服务端读模型：

```text
TechnicianTaskWorkspaceView
TechnicianTaskHeaderView
TechnicianTaskCustomerAccessView
TechnicianTaskLocationView
TechnicianTaskAppointmentView
TechnicianTaskStepProjectionView
TechnicianTaskFormSummaryView
TechnicianTaskEvidenceSummaryView
TechnicianTaskSubmissionSummaryView
TechnicianTaskCorrectionSummaryView
TechnicianTaskAllowedActionView
```

建议本机投影：

```text
LocalTaskWorkspaceSnapshot
LocalTaskWorkPackageView
LocalTaskDraftSummary
LocalTaskUploadSummary
LocalTaskCommandSummary
LocalTaskConflictSummary
```

服务端直接返回中文业务名称、状态、缺失项、阻止原因、下一入口和数据时间。客户端不得调用多个底层接口自行重建业务含义。

## 26. 页面动作

任务详情自身只拥有导航和本机准备动作：

```text
RefreshTaskWorkspace
RequestWorkPackagePreparation
OpenContactAndAppointment
OpenVisitCheckIn
OpenTaskWorkloop
OpenFormAndEvidence
OpenSubmissionReview
OpenCorrection
OpenUploadCenter
OpenSyncConflict
OpenBusinessException
```

真正的写命令由对应 Feature 承担，并在执行时重新校验责任、版本、Capability、工作包和幂等键。

不提供：

```text
UpdateTaskDetail
ForceNextStep
MarkStepCompleted
CompleteWorkOrderFromDetail
IgnoreGuardAndContinue
```

## 27. 页面状态

必须覆盖：

- 在线且数据新鲜；
- 有本机成果并在线刷新；
- 离线且工作包有效；
- 离线但只有普通缓存；
- 工作包准备中；
- 工作包即将过期或已经过期；
- 本机有待上传或待同步；
- 上传或服务端校验中；
- 任务允许继续；
- 当前步骤被业务条件阻止；
- 整改中；
- 同步冲突；
- 责任被改派或撤销；
- 设备或账号被撤销；
- 部分来源不可用；
- 任务不存在或当前无权访问；
- 客户端需要升级；
- 加载失败和重试。

## 28. 空、错与无权限

任务详情不存在正常“空页面”。

- 服务端无法证明当前责任时统一显示“该任务当前不可访问”，不泄露任务是否存在；
- 任务已经完成时显示只读结果和返回入口，不继续展示旧动作；
- 任务已经取消时显示取消业务原因和授权摘要；
- 部分来源失败时保留可用区块并标明暂不可用；
- 核心责任或 allowed-actions 无法确认时整体只读；
- 错误提供中文说明、恢复建议和可复制 traceId。

不得用默认对象、空字符串、旧快照或内部 ID 填补关键业务字段。

## 29. 可访问性与现场可用性

- 支持 Dynamic Type，标题、SLA、预约窗口、当前步骤和主操作不得截断；
- VoiceOver 按“任务—风险—当前步骤—客户地址—下一动作—同步状态”朗读；
- 主操作触控区域不小于 44×44pt；
- 关键状态同时使用文字、图标和颜色；
- 地址、电话和主操作支持单手触达；
- 页面滚动后固定操作不能遮挡内容或系统安全区；
- 弱网刷新不能阻塞查看有效工作包和本机成果；
- 安全风险不依赖弹窗一次性表达，返回页面后仍可见。

## 30. 审计与可观测性

必须记录：

- 任务详情查询和当前责任校验结果；
- 使用的任务、责任、预约、配置和工作包版本；
- 敏感客户字段授权和拨号入口使用；
- 导航入口使用；
- allowed-actions 和阻止原因变化；
- 本机草稿、上传、命令和冲突摘要；
- tombstone、改派、撤权和本地隔离结果；
- 用户进入的专用业务页面；
- 加载失败、并发拒绝、强制升级和恢复结果；
- traceId、correlationId、clientKind 和 clientVersion。

日志使用中文业务说明和稳定英文结构字段，不记录 Token、完整电话、详细地址、VIN、表单值、照片路径或文件正文。

## 31. 验收标准

1. 首屏能够看到任务、SLA、当前步骤、下一动作、预约和地址；
2. 页面只展示完成当前任务所需的最小信息，不复制 Admin 工作区；
3. 当前责任、Capability 和网点关系无法证明时失败关闭；
4. 在线动作来自服务端 allowed-actions，离线动作来自有效工作包；
5. 当前步骤是服务端投影，不形成客户端第二套工作流；
6. 本机已填写或已拍摄不能伪造成平台已完成；
7. 客户电话按字段策略临时授权，改派后立即失效；
8. 导航不等于到场，不自动创建 Visit；
9. 本机预约草稿不覆盖服务器预约或显示客户已确认；
10. 表单、资料、上传、提交和整改状态分别表达；
11. 任务详情不直接执行到场、提交、完成或审核结论；
12. 离线且只有普通缓存时只读，不提供伪离线作业；
13. 工作包过期或配置不匹配时保留本机成果并阻止新动作；
14. 改派、撤权或设备撤销后旧师傅不能继续读取敏感信息或提交命令；
15. 服务器新版本与本机成果冲突时进入 IOS-010，不使用最后写入覆盖；
16. 页面不存在通用更新、强制下一步或忽略限制命令；
17. Dynamic Type、VoiceOver、弱网和单手操作满足移动端基线；
18. 错误使用中文业务说明、恢复建议和 traceId，不展示内部异常。

## 32. 与其他规格的关系

- IOS-001 今日任务：提供任务入口、优先级和同步摘要；
- IOS-003 联系与预约：记录联系事实和预约 Revision；
- IOS-004 到场与开始作业：执行定位、到场校验和 Visit 开始；
- IOS-005 作业步骤：承载当前现场步骤和异常处理；
- IOS-006 表单与资料采集：承载动态表单、相机和本地文件；
- IOS-007 提交前检查：执行完整性检查和正式提交；
- IOS-008 上传中心：管理后台上传、扫描和失败恢复；
- IOS-009 离线恢复：管理工作包、离线命令和杀进程恢复；
- IOS-010 同步冲突：处理责任、版本、配置和资料槽位冲突；
- DEC-005：定义责任师傅和当前任务处理人边界；
- DEC-006：定义网点、师傅、改派和失权边界。
