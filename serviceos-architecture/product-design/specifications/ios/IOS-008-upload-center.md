# IOS-008 上传中心

状态：Accepted
设计等级：A

## 1. 页面定位

“上传中心”是原生 iOS 师傅端管理本机现场文件传输、后台恢复和平台处理结果的统一工作区。

它必须让师傅清楚回答：

1. 哪些文件仍只存在本机；
2. 哪些文件正在等待网络或上传政策；
3. 哪些文件正在前台或后台传输；
4. 哪些文件已经完成字节传输但尚未 Finalize；
5. 哪些文件平台已经收件但仍在扫描或机器校验；
6. 哪些资料已经达到可提交状态；
7. 哪些文件可以安全重试、需要重新采集或必须处理冲突；
8. App 被挂起、杀进程、重启、退出账号或设备空间不足时，本机成果是否安全。

上传中心必须始终区分三层状态：

```text
本机传输状态
平台文件生命周期
资料业务校验与审核状态
```

后台传输完成不等于平台已收件，Finalize 成功不等于安全扫描通过，文件可用不等于资料审核通过。

## 2. 产品边界

IOS-008 负责：

- 按任务和状态展示本机上传队列；
- 管理单文件和分片上传；
- 使用 iOS Background URLSession 支持系统允许范围内的后台传输；
- 在 App 重启后恢复系统传输和本地队列关联；
- 执行 Begin、数据面上传、CompleteParts 和 Finalize；
- 展示网络、蜂窝、低数据模式、后台限制和磁盘空间影响；
- 支持暂停、继续、重试、取消和受控清理；
- 展示平台收件、安全扫描、机器校验和资料版本状态；
- 保证同一文件和命令重试不产生重复 EvidenceRevision；
- 将责任、槽位、版本、文件和会话冲突导向 IOS-010；
- 为 IOS-006 和 IOS-007 提供可信上传摘要。

IOS-008 不负责：

- 创建或修改 EvidenceSlot；
- 代替 IOS-006 拍摄、选择或编辑业务资料；
- 将上传进度写成资料审核状态；
- 绕过当前责任和控制面授权；
- 使用永久公共 URL 或长期上传凭据；
- 在数据面 PUT 中携带 Bearer Token 或 Technician Context；
- 修改已创建 EvidenceRevision；
- 自动删除未同步或冲突中的本机文件；
- 直接完成 Task、Visit、审核或工单；
- 承诺 iOS 会在任何系统状态下无限后台运行；
- 用 H5 上传能力替代原生后台可靠性。

## 3. 入口与导航

上传中心不默认占用一级导航。入口包括：

- IOS-001 今日任务的待上传摘要；
- IOS-002 任务详情的同步状态；
- IOS-005 作业步骤的资料摘要；
- IOS-006 表单与资料采集；
- IOS-007 提交前检查；
- App 内受控同步提醒；
- “我的”中的本机数据与同步入口。

通过任务入口进入时默认过滤当前 Task；通过全局入口进入时按任务分组，但只显示当前账号在当前设备上的本机队列及仍有访问权的安全业务摘要。

## 4. 访问与本机所有权

每个队列项必须绑定：

```text
localAccountPartition
deviceRegistration
tenant and project
task and evidence slot
localFileId
contentDigest
capture metadata reference
workPackage or online authority version
```

本机队列按账号和设备隔离。切换账号后，另一个账号不能看到文件名、缩略图、任务信息或上传状态。

在线 Begin、刷新会话、Finalize 和查询平台状态必须重新校验：

- 当前认证和受信 Technician Context；
- ACTIVE TechnicianProfile 和网点关系；
- 当前责任；
- Task、槽位、Item 和 Revision 状态；
- Project Scope 和 Capability；
- 工作包、责任和权限版本；
- 上传会话所有者和有效期。

数据面传输只使用服务端签发的短时、范围受限能力，不携带业务身份令牌。

## 5. 页面结构

```text
全局上传摘要 + 网络和磁盘状态
→ 需要立即处理
→ 按任务分组的队列
→ 本机待上传
→ 上传中与等待后台恢复
→ 已传输待 Finalize
→ 平台扫描和校验中
→ 可以用于提交
→ 可重试失败
→ 冲突、隔离和最终失败
→ 本机保留与安全清理
```

默认优先展示阻止任务提交、临近整改截止时间、需要用户操作和占用大量本机空间的队列项。排序原因由业务化规则解释，不能用不可理解的本地分数。

## 6. 页面级读模型

上传中心由本地持久化状态和服务端安全摘要共同组成：

```text
UploadCenterView
├── deviceAndAccountSummary
├── networkPolicySummary
├── storageSummary
├── queueGroups[]
│   ├── taskSummary
│   ├── blockingImpact
│   └── items[]
├── systemTransferSummary
├── serverProcessingSummary
├── conflictSummary
├── cleanupSummary
└── allowedActions
```

本地数据库是传输恢复状态的权威来源，服务端 Evidence/File 是业务收件和资料状态的权威来源。二者不能互相覆盖或合并成一张无来源状态表。

## 7. 队列项展示

每个队列项展示：

- 资料业务名称；
- 所属任务和步骤；
- 安全缩略图或文件类型图标；
- 文件大小和媒体时长；
- 当前传输进度；
- 当前网络策略；
- 本机保存时间；
- 最近尝试时间和次数；
- 平台收件、扫描和机器校验摘要；
- 是否阻止 IOS-007 提交；
- 失败或冲突原因；
- 允许动作。

普通界面不展示本机绝对路径、对象存储 key、预签名 URL、上传凭据、内部会话 ID、Revision UUID 或原始异常堆栈。

## 8. 三层状态模型

### 8.1 本机传输状态

- 本机已保存；
- 等待计算摘要；
- 等待创建上传会话；
- 等待符合网络政策；
- 等待系统后台调度；
- 上传中；
- 已暂停；
- 字节传输完成；
- 等待 Finalize；
- 可重试失败；
- 需要用户处理；
- 已取消但本机保留；
- 本机文件缺失。

### 8.2 平台文件状态

- 尚未收件；
- 平台已收件，等待安全扫描；
- 安全扫描中；
- 文件可用；
- 已隔离；
- 已失效。

### 8.3 资料业务状态

- 尚未创建 EvidenceRevision；
- 资料验证中；
- 资料已验证；
- 资料验证失败；
- 等待人工确认；
- 审核待处理；
- 审核通过；
- 需要整改；
- Revision 已作废。

一个队列项可以同时显示：

```text
本机传输：已完成
平台文件：安全扫描中
资料状态：尚不可提交
```

## 9. 本机文件准备

文件进入队列前必须：

- 已安全落盘到 App 私有目录；
- 获得稳定 localFileId；
- 完成大小、MIME 和基础结构检查；
- 计算待上传字节的 SHA-256；
- 保存采集来源和受控 metadata 引用；
- 校验本机文件可读且大小稳定；
- 绑定 Task、Slot、Item 或整改上下文；
- 记录本机数据库 Schema 版本。

任何压缩、转码、水印或派生处理必须在 BeginUpload 和最终 checksum 之前完成。上传会话建立后不得继续修改同一字节文件。

原始文件与派生文件的关系必须可追溯。生成新的上传字节内容时使用新的摘要和队列版本，不能沿用旧会话或伪装成同一文件。

## 10. 图片和视频处理

图片、视频的压缩、分辨率、码率、时长和蜂窝上传限制由项目资料政策、平台安全政策和用户允许设置共同决定，不在客户端硬编码一个全局值。

处理前页面必须说明：

- 是否保留原始文件；
- 将上传原始版还是受控派生版；
- 预计大小；
- 是否可能在蜂窝网络上传；
- 处理失败的恢复方式。

不能为节省流量静默降低到无法审核的质量，也不能在转码失败时偷偷上传另一个不符合槽位要求的文件。

## 11. BeginUpload 控制面

创建上传会话前，服务端重新校验当前责任、Task、EvidenceSlot、数量、文件政策、checksum、MIME、大小和会话所有者。

客户端只提交受控文件元数据和摘要，不能提交：

- tenant 或 uploader；
- object key；
- 永久文件地址；
- `locationVerified`；
- 权威 offline 标记；
- onBehalf 身份；
- 文件已通过扫描或审核的状态。

服务端返回：

- 短时上传会话；
- 单文件或 multipart 策略；
- 受限目标和必要请求头；
- 会话有效期；
- 分片大小和并发上限；
- 完成分片所需的受控引用；
- 是否允许后台传输。

客户端不将会话和凭据写入普通日志、通知或产品埋点。

## 12. 数据面传输

数据面请求只能访问服务端授权的精确对象或分片，且绑定：

- 字节范围或精确文件；
- 预期大小；
- MIME；
- checksum 或分片摘要；
- 短时有效期；
- 允许的方法和必要请求头。

数据面不携带：

- OIDC Bearer Token；
- Technician Context；
- Tenant、Project 或用户自报头；
- 业务 Idempotency-Key；
- 表单或客户信息；
- 本机绝对文件路径。

上传凭据泄漏时的影响必须限制在单个对象或分片和短有效期内。

## 13. 单文件和分片上传

服务端根据文件大小、存储能力和策略返回单文件或 multipart 方案，客户端不能自行把同一会话改成另一种协议。

分片上传必须持久化：

- 上传会话和 multipart 标识；
- 每个 part 编号、字节范围和摘要；
- 系统传输任务标识；
- 完成状态和服务端 ETag；
- 最近尝试、错误和退避时间；
- 会话有效期；
- 最终文件摘要。

只重试失败或未确认的分片。已由服务端确认的 part 不重复上传，除非服务端明确要求失效重传。

CompleteParts 成功只表示对象存储已经组装目标对象，仍需 Evidence Finalize 才能创建平台资料版本。

## 14. Background URLSession

原生 iOS 使用独立的 Background URLSession 执行系统允许的后台文件传输。

要求：

- 每个系统任务使用稳定、本机可恢复的 transfer identifier；
- 请求体来自持久化文件 URL，不依赖 ViewModel 内存；
- delegate 结果写入本地上传状态机；
- App 重启时重新关联系统任务和本地队列；
- 系统唤醒 App 处理后台事件时完成最小状态落盘和后续调度；
- UI 不直接拥有 URLSession delegate 生命周期；
- 多账号和多环境使用隔离的 session identifier。

iOS 是否立即执行、何时唤醒和何时暂停由系统、网络、电量和用户行为共同决定。页面不能承诺准确开始时间或完成时间。

用户从系统强制退出 App、系统限制后台活动或设备关机后，后台任务可能暂停，必须在下次启动时恢复和解释真实状态。

## 15. BGTaskScheduler 边界

BGTaskScheduler 可用于：

- 队列与系统任务对账；
- checksum 或轻量预处理续作；
- 刷新即将过期但仍有权限的会话；
- 处理已完成传输的 Finalize；
- 刷新平台扫描和校验摘要；
- 清理已满足安全条件的临时文件。

它不能用来宣称 App 拥有无限后台执行时间，也不能绕过 Background URLSession、系统能耗政策或当前业务授权。

超过系统时间预算时安全保存进度并等待下次机会，不把任务标记为失败或成功。

## 16. App 重启与系统任务对账

App 启动、登录恢复和系统后台回调时执行对账：

```text
读取本地队列
→ 枚举当前账号/环境的系统传输任务
→ 按稳定 identifier 匹配
→ 查询缺失的服务端会话或结果
→ 修复可确定状态
→ 将歧义状态转为需要确认或冲突
```

禁止：

- 找不到系统任务就直接标记上传完成；
- 找不到本地记录就把系统任务归给当前账号；
- App 重启后重新创建重复上传会话；
- 因 UI 进度丢失重新上传已确认分片；
- 删除无法匹配但可能含未同步成果的文件。

## 17. 网络政策

页面展示当前：

- Wi-Fi、蜂窝或离线；
- 网络是否昂贵或受限；
- Low Data Mode；
- 用户上传偏好；
- 项目或组织强制政策；
- 当前是否允许后台传输；
- 被政策暂停的文件数量。

政策优先级：

```text
平台安全和项目硬限制
→ 组织设备政策
→ 用户允许设置
→ 当前网络和系统约束
```

用户只有在上级政策允许时才能临时使用蜂窝网络。不能通过普通设置覆盖法律、安全、文件大小或项目限制。

蜂窝上传的默认策略、视频预算和是否允许用户覆盖必须在试点项目和设备政策中明确；IOS-008 只消费权威政策，不提前锁定业务值。

## 18. 暂停、继续和取消

### 暂停

暂停停止新分片调度，并尽力挂起可控传输；已经由系统或对象存储处理的请求可能继续完成。页面必须等待真实回调后更新状态。

### 继续

继续前重新检查文件、网络政策、会话有效期和当前责任。会话失效时不能直接复用旧凭据。

### 取消

取消分为：

- 取消当前传输但保留本机文件；
- 放弃尚未平台收件的本机资料；
- 对已经 Finalize 的资料发起受控作废或整改动作。

Finalize 后不能通过删除本机队列项撤销 EvidenceRevision。页面必须根据真实阶段提供不同动作。

## 19. 重试策略

错误分为：

### 可自动重试

- 短暂断网；
- 请求超时；
- 可重试对象存储错误；
- 服务端明确返回的暂时不可用；
- 系统后台任务中断；
- Finalize 租约暂时被占用。

### 需要刷新授权后重试

- 上传会话过期；
- 短时凭据过期；
- 服务端要求重新 Begin；
- multipart 状态需要对账。

### 不可自动重试

- 当前责任已变化；
- EvidenceSlot 已失效或 generation 变化；
- MIME、大小或 checksum 不匹配；
- 本机文件缺失或损坏；
- 文件被安全隔离；
- 权限或设备已撤销；
- 客户端能力不兼容；
- 明确的最终业务校验失败。

自动重试使用有上限的退避和抖动，不无限快速重试。达到上限后显示真实原因和下一步。

## 20. 幂等与重复响应

必须分别稳定保存：

- Begin 命令标识；
- 每个 part 的传输身份；
- CompleteParts 命令标识；
- Finalize 命令标识；
- EvidenceItem / Revision 目标；
- 文件 checksum。

不同业务命令不能共用一个幂等键。

相同文件、相同目标和相同 Finalize 命令重试应返回既有结果，不创建重复 StoredFile、EvidenceItem、EvidenceRevision 或扫描任务。

服务端已经接受相同操作时，客户端绑定既有权威结果并清理重复待办，而不是把 `idempotent accepted` 显示成失败。

## 21. Finalize

Finalize 前必须确认：

- 所有分片或单文件传输已由数据面确认；
- 本机文件摘要仍匹配；
- 会话和当前责任仍有效；
- Task、Slot、Item 和整改上下文仍匹配；
- 实际对象大小、摘要和 MIME 可以由服务端重新读取验证。

Finalize 由服务端：

- 抢占短期 finalization lease；
- 在数据库事务外检查大对象；
- 在短事务内建立不可变文件、扫描任务、审计和幂等结果；
- 通过 Evidence 编排创建或追加 Revision；
- 返回权威文件和资料状态。

响应丢失时使用相同 Finalize 命令查询或重试。不能重新上传或创建新 Item 猜测结果。

## 22. Finalize 后的平台处理

Finalize 成功后，页面依次展示：

```text
平台已收件
→ 安全扫描中
→ 文件可用 / 已隔离
→ 资料机器校验中
→ 资料已验证 / 验证失败 / 等待人工确认
→ 后续审核或整改
```

安全扫描、OCR、图片质量、SN/VIN 一致性和人工审核分别表达。扫描服务暂时不可用不能默认文件安全，机器校验不可用不能默认资料合格。

平台处理由服务端任务和事件驱动，上传中心只查询或订阅安全摘要，不在本地生成权威结论。

## 23. 磁盘空间

上传中心展示：

- App 当前本机业务文件占用；
- 未同步文件占用；
- 可安全清理文件占用；
- 预计下一次采集或转码空间；
- 设备剩余空间风险。

空间不足时：

- 在拍摄大文件前预警；
- 停止会产生损坏文件的处理；
- 保留已完成写入的本机成果；
- 优先建议清理已平台确认且满足保留政策的文件；
- 不删除待上传、处理中、冲突或整改文件；
- 不清理其他账号数据来为当前账号伪造空间。

系统清理、App 卸载或设备损坏的风险必须在试点运维和备份政策中明确，不能声称本机文件永久可靠。

## 24. 本地文件保留和清理

只有以下条件全部满足时，队列才可标记“可以安全清理”：

- 服务端已确认对应 EvidenceRevision；
- 文件达到项目允许的安全状态；
- 不再需要本机重试、整改或重新 Finalize；
- 当前没有未完成依赖命令；
- 本地 checksum 与服务器确认引用一致；
- 法务、项目和设备保留政策允许。

清理必须原子更新本地数据库和文件状态。文件删除失败不能把记录标为已清理；记录删除失败也不能留下无法解释的孤儿文件。

本地保留期限是项目和设备政策，当前阶段不硬编码统一天数。

## 25. 退出账号和切换账号

退出账号前必须展示：

- 待上传文件数量和大小；
- 正在进行的系统后台传输；
- 已传输待 Finalize 项；
- 冲突和最终失败项；
- 退出后将暂停或隔离的内容；
- 受控恢复入口。

退出后：

- 停止为该账号创建新的控制面授权；
- 暂停或取消仍可控的系统传输；
- 使本地队列进入账号隔离状态；
- 不把未同步文件展示给下一个账号；
- 不无提示删除业务成果；
- 短时数据面凭据到期后不可刷新；
- 同一账号重新认证后才允许恢复。

后台任务在退出边界附近已经完成时，回调只记录最小隔离结果，不将文件归给新账号；原账号重新认证后对账。

## 26. 改派、撤权和设备撤销

收到责任 tombstone、权限变化或设备撤销后：

- 立即停止创建或刷新控制面授权；
- 停止新的数据面传输调度；
- 尽力取消仍可控的短时上传；
- 隔离未同步本机文件和 metadata；
- 隐藏客户、地址和资料缩略图；
- 保留最小可审计恢复信息；
- 将队列项转入 IOS-010。

已经在对象存储完成传输但尚未 Finalize 的对象，不能由旧责任继续 Finalize。服务端临时对象清理策略负责最终清理，客户端不能冒用新责任接管。

已经 Finalize 的 EvidenceRevision 属于历史事实，不因改派而删除或转移给新师傅。

## 27. 冲突类型

进入 IOS-010 的冲突包括：

- 当前责任网点或师傅变化；
- WorkPackage 或 authorityVersion 失效；
- Task、配置或 Evidence resolution generation 变化；
- Slot 停用、重建或数量已满；
- 同一 Item 已出现新的 Revision；
- 上传会话与本机文件摘要不匹配；
- multipart 服务端 parts 与本机记录不一致；
- 本机文件缺失、损坏或被系统清理；
- 另一设备已经提交相同或不同资料；
- 当前账号或设备被撤销。

能通过权威查询确定为同一幂等结果的情况，不算业务冲突；客户端绑定既有结果即可。

## 28. 通知和后台反馈

允许的 App 内通知：

- 上传已暂停，需要打开 App；
- 文件上传失败，需要处理；
- 平台验证完成；
- 文件被隔离；
- 磁盘空间不足；
- 队列存在阻止任务提交的问题。

锁屏通知使用最小文案，不显示客户、地址、文件名、照片缩略图、项目敏感名称或错误正文。

通知到达不等于用户已经处理，也不能改变队列或资料业务状态。

## 29. 页面操作

上传中心允许：

- 暂停或继续允许的队列项；
- 重试可重试失败；
- 刷新过期授权；
- 查看平台处理状态；
- 打开原任务或资料项；
- 重新采集不可修复文件；
- 进入冲突处理；
- 清理满足安全条件的本机文件；
- 按策略批量继续或重试。

批量操作只能作用于状态和政策一致的队列项。不能通过“全部重试”绕过某个任务的责任、槽位或网络限制。

## 30. 命令边界

IOS-008 可以调用：

- BeginUpload；
- 获取或刷新受限分片上传授权；
- CompleteParts；
- FinalizeUpload；
- 查询平台文件和 Evidence 状态；
- 受控取消或清理未完成会话；
- 项目允许的 Evidence 重新采集或作废入口。

IOS-008 不提供：

- 修改 EvidenceSlot；
- 手工设置 StoredFile 或 Revision 状态；
- 绕过安全扫描；
- 生成永久下载 URL；
- 覆盖 EvidenceRevision；
- 直接审核通过；
- 直接完成 Task、Visit 或工单；
- 把本地删除当成平台作废。

## 31. 状态与恢复动作

| 状态 | 页面表现 | 允许动作 |
|---|---|---|
| 本机已保存 | 尚未开始上传 | 开始、等待政策 |
| 等待 Wi-Fi | 说明蜂窝政策 | 等待、按允许政策临时继续 |
| 等待系统后台 | 说明系统调度限制 | 保持队列、前台继续 |
| 上传中 | 展示真实字节进度 | 暂停、查看详情 |
| 已暂停 | 展示暂停来源 | 继续、取消并保留 |
| 会话过期 | 不复用旧凭据 | 刷新授权 |
| 字节传输完成 | 明确尚未平台收件 | 执行 Finalize |
| 平台已收件 | 明确扫描未完成 | 等待平台 |
| 平台验证中 | 展示处理阶段 | 刷新、返回任务 |
| 资料已验证 | 可用于提交 | 返回 IOS-007 |
| 可重试失败 | 显示原因和尝试次数 | 单项或受控批量重试 |
| 文件已隔离 | 不提供提交或下载 | 查看原因、重新采集 |
| 本机文件缺失 | 不伪造重试 | 重新采集、处理冲突 |
| 责任失效 | 隐藏敏感内容 | 进入 IOS-010 |
| 可以安全清理 | 展示平台确认依据 | 清理本机文件 |

## 32. 安全、隐私和审计

Begin、会话刷新、CompleteParts、Finalize、重试、取消和清理必须记录：

- 租户、Project、Task、Slot 和 Item；
- 当前责任网点和师傅；
- 操作者、设备注册和受信 Context；
- 本机稳定 transfer ID 和业务命令 ID；
- 文件 MIME、大小、checksum 和分片摘要；
- 会话、网络政策和失败分类；
- `capturedAt`、传输时间和服务器接收时间；
- 幂等、Finalize、扫描和 Evidence 结果；
- 清理依据和执行结果。

日志、埋点和崩溃报告不得包含：

- 客户或详细地址；
- VIN、SN 等敏感业务值；
- 文件正文、缩略图或绝对路径；
- 对象 key、预签名 URL 和 requiredHeaders；
- Bearer Token、Technician Context 或 Keychain 内容；
- 原始 EXIF、坐标或设备秘密。

## 33. 无障碍和移动交互

- 进度、暂停、失败和隔离不能只依赖颜色；
- VoiceOver 能读取资料名称、大小、进度、失败原因和可执行动作；
- 动态字体放大后保留任务分组、状态和主操作；
- 大列表使用稳定分组，不因后台回调跳动焦点；
- 暂停、取消和清理提供明确影响确认；
- 后台和网络限制使用自然中文解释；
- 进度变化不过度频繁播报，完成和需要用户处理时才重点通知；
- 弱网时操作按钮保持幂等并防止重复点击。

## 34. 与其他页面的关系

```text
IOS-006 表单与资料采集
    → 本机文件进入 IOS-008 上传中心
    → 平台 EvidenceRevision 状态回到 IOS-006

IOS-007 提交前检查
    → 上传或验证未完成
    → IOS-008
    → 满足后返回 IOS-007

IOS-008
    → IOS-009 离线恢复
    → IOS-010 同步冲突
```

上传中心不建立第二套 Evidence 状态。Admin、Network Portal 和 iOS 读取同一平台文件和资料事实；只有本机传输恢复状态属于当前设备。

## 35. 验收标准

### 状态真实性

- [ ] 本机传输、平台文件和资料业务状态分层展示；
- [ ] 字节传输完成不显示平台已收件；
- [ ] Finalize 成功不显示扫描或审核通过；
- [ ] 隔离、验证失败和审核驳回分别表达；
- [ ] 无法获取平台状态时不显示默认成功。

### 后台与恢复

- [ ] Background URLSession 使用持久化文件和稳定 transfer identifier；
- [ ] App 挂起、杀进程和重启后可以恢复队列及系统任务关联；
- [ ] 系统强制退出或后台受限时展示真实暂停状态；
- [ ] BGTaskScheduler 只做有限维护和续作，不承诺无限后台运行；
- [ ] 本地记录和系统任务无法匹配时不归给错误账号或伪造结果。

### 传输与幂等

- [ ] 单文件和 multipart 方案由服务端会话决定；
- [ ] 已确认分片不重复上传；
- [ ] 相同 Finalize 命令重试不创建重复文件、Revision 或扫描任务；
- [ ] checksum、大小和 MIME 由服务端重新校验；
- [ ] 大对象传输和检查不占用数据库长事务；
- [ ] 数据面请求不携带业务身份令牌。

### 网络、磁盘和清理

- [ ] Wi-Fi、蜂窝、受限网络和用户政策可解释；
- [ ] 不在客户端硬编码统一视频和蜂窝策略；
- [ ] 空间不足时不破坏已有本机成果；
- [ ] 未同步、冲突和整改文件不会被自动清理；
- [ ] 只有平台确认和保留政策满足后才能安全清理；
- [ ] 退出账号不会向新账号泄露或无提示删除队列。

### 责任与冲突

- [ ] 改派、撤权或设备撤销后停止创建和刷新上传授权；
- [ ] 旧师傅不能继续 Finalize；
- [ ] 已 Finalize 历史 Revision 不因改派被删除；
- [ ] 槽位、版本、分片或文件冲突进入 IOS-010；
- [ ] 幂等接受可以绑定既有结果，不误报业务冲突。

### 安全与隐私

- [ ] 对象 key、预签名 URL、上传头和绝对路径不进入日志；
- [ ] 锁屏通知不显示客户和文件敏感信息；
- [ ] 本机文件按账号和设备隔离；
- [ ] 临时上传凭据范围受限且短时有效；
- [ ] 已隔离文件不能提交或授权下载。

## 36. 禁止实现

禁止：

- 用一个“上传成功”合并传输、扫描、校验和审核；
- Background URLSession 请求依赖内存 Data；
- ViewModel 直接拥有后台传输状态机；
- App 重启后无条件重传全部文件；
- 重试已确认分片或重复创建 EvidenceRevision；
- 数据面携带 Bearer Token 或 Technician Context；
- 永久保存或记录预签名 URL；
- 会话过期后静默复用旧凭据；
- Finalize 失败后创建新 Item 绕过幂等和 maxCount；
- 为省流量静默降低资料质量；
- 用户退出账号时无提示删除未同步文件；
- 改派后把旧师傅文件自动转给新师傅；
- 磁盘不足时自动删除待上传或冲突成果；
- 捕获后台错误后显示默认完成；
- 为旧 H5、旧开发数据或假设客户端保留第二套上传模型。
