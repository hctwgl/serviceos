# IOS-006 表单与资料采集

状态：Accepted
设计等级：A

## 1. 页面定位

“表单与资料采集”用于当前责任师傅按照任务锁定的表单和资料要求，填写现场业务数据、拍摄或选择受控资料、保存本机草稿，并形成后续可提交和可审核的版本化业务成果。

页面必须清楚区分：

- 本机表单草稿；
- 本机表单校验结果；
- 平台已经接收的不可变 FormSubmission；
- 当前权威资料槽位；
- 本机已经采集的文件；
- 已进入上传队列的文件；
- 平台已经收件的 EvidenceRevision；
- 已完成安全扫描和机器校验的资料；
- 已冻结进入提交集合的资料；
- 审核通过、驳回和整改版本。

“已填写”“已拍摄”“已上传”“平台已验证”和“审核已通过”是不同事实，不能使用一个绿色完成状态代替。

## 2. 产品边界

IOS-006 负责：

- 使用任务锁定的 FormVersion 渲染动态表单；
- 展示字段说明、单位、精度、权限、预填来源和条件规则；
- 在设备私有存储中自动保存本机草稿；
- 使用同版本可移植规则提供即时校验；
- 显式提交不可变 FormSubmission；
- 展示由权威条件事实解析出的 EvidenceSlot；
- 使用相机、相册、文件、扫码或签名能力采集项目允许的资料；
- 计算 checksum 并保存受控 CaptureMetadata；
- 把本机文件加入 IOS-008 上传队列；
- 展示平台收件、扫描、机器校验和资料版本状态；
- 在整改时只补充被退回的表单或资料版本；
- 在离线工作包允许时保存草稿、文件和离线命令；
- 将版本、预填、责任和多设备冲突导向 IOS-010。

IOS-006 不负责：

- 修改 FormVersion、EvidenceTemplate 或资料槽位定义；
- 把动态表单当作客户、车辆、设备、预约等核心主数据的唯一模型；
- 在客户端运行任意 JavaScript、SQL 或远程脚本；
- 将文件对象地址或永久下载 URL 写入表单值；
- 直接完成 Task、工单、Visit 或审核；
- 覆盖已提交 FormSubmission、EvidenceRevision 或审核决定；
- 在未上传、未验证时显示资料可用；
- 通过忽略未知字段或规则继续提交；
- 代替 IOS-008 实现后台传输和队列恢复；
- 代替 IOS-007 完成跨资源提交前检查。

## 3. 访问与版本校验

在线读取、提交表单和上传资料必须同时满足：

```text
authenticated principal
+ trusted Technician Context
+ ACTIVE TechnicianProfile
+ ACTIVE NetworkTechnicianMembership
+ current ACTIVE technician responsibility
+ required capability
+ current task allowed-action
+ exact frozen configuration and form/evidence versions
```

每个写命令必须由服务端重新校验：

- 当前 Task、Visit 和责任；
- 工单绑定的履约配置；
- Task 锁定的 FormVersion；
- 当前 Evidence resolution generation 和活动槽位；
- 表单、槽位、资料项和文件会话状态；
- 并发版本和幂等键；
- 字段和资料访问政策；
- 客户端能力与 Schema 兼容性。

离线采集还必须具有有效工作包，且工作包包含精确表单、规则、资料要求、预填快照、支持能力和允许离线动作。

## 4. 页面结构

```text
任务与步骤摘要 + 同步可信度
→ 表单 / 资料页内导航
→ 表单分区和字段
→ 条件规则、错误和警告
→ 本机自动保存状态
→ FormSubmission 状态
→ 资料槽位和采集要求
→ 本机文件、上传和验证状态
→ 整改或冲突提示
→ 固定底部主操作
```

“表单”和“资料”可以作为同一工作区的两个页内区块或页签，但必须共享当前 Task、步骤、配置版本和同步状态。页面不能把二者拆成互不知道对方条件关系的独立事实。

## 5. 页面级读模型

建议使用：

```text
TechnicianCaptureWorkspaceView
├── taskSummary
├── stepSummary
├── responsibilitySummary
├── formDefinition
├── prefillSnapshot
├── latestFormSubmission
├── evidenceResolution
│   └── activeSlots[]
├── evidenceItems[]
├── localDraftSummary
├── localFileSummary
├── uploadSummary
├── correctionSummary
├── syncSummary
├── clientCapabilityRequirements
├── allowedActions

```

客户端不能分别读取最新表单、最新资料模板和当前任务后自行猜测版本组合。任务锁定版本和当前 resolution generation 必须由服务端明确返回。

## 6. FormVersion 渲染

页面只使用当前 Task 精确锁定的已发布 FormVersion。发布新版本不改变在途任务。

FormVersion 至少提供：

- 表单业务名称和用途；
- 分区和排序；
- 稳定字段键；
- 中文标签、帮助说明和占位；
- 字段类型、单位和精度；
- 必填、只读、隐藏和编辑政策；
- 枚举或受控选项；
- 默认值和预填策略；
- 条件显隐、条件必填和校验规则；
- 重复组定义；
- 资料联动；
- 敏感等级；
- 离线兼容性；
- Schema、规则和内容摘要。

普通界面不展示 fieldKey、FormVersion ID、表达式或 JSON Schema。技术版本只用于兼容校验和诊断。

## 7. 客户端能力门禁

客户端启动渲染前必须声明和校验支持的：

- 表单 Schema 版本；
- 字段类型；
- 规则解释器版本；
- 资料采集类型；
- 工作包版本；
- 安全能力和最低 App 版本。

出现未知字段类型、未知规则、未知采集策略或不兼容版本时，必须失败关闭并展示：

> 当前 App 版本无法安全处理本任务，请升级后重试或联系网点协调。

禁止忽略未知规则、把未知字段当文本框、把未知资料要求当普通照片或删除不支持字段后继续提交。

## 8. 字段体验

字段控件必须匹配业务类型：

- 文本和多行说明使用明确长度限制；
- 整数和小数使用正确键盘、单位和精度；
- 金额不得使用浮点近似展示或提交；
- 日期和时间使用项目业务时区；
- 单选和多选显示中文业务名称；
- 布尔值使用明确“是 / 否”，不使用含义不清的开关；
- 地址、设备、车辆等核心对象使用受控选择或扫描，不用自由文本代替权威引用；
- 扫码结果显示原始值、解析值和用户确认状态；
- 签名字段只保存受控资料引用，不把图片正文写入 values。

字段说明必须解释为什么采集、正确示例和常见错误。不能直接把数据库字段名或外部报文字段暴露给师傅。

## 9. 条件规则

显隐、条件必填、跨字段校验、计算和资料联动使用已发布的声明式规则及确定性解释器。

客户端与服务端必须运行同一规则规范和发布测试样例。客户端执行只提供即时反馈，服务端仍是最终权威。

规则处理必须满足：

- 缺失字段、类型漂移和表达式错误失败关闭；
- 不把未知值解释为 false；
- 不执行任意脚本；
- 不依赖未冻结的随机数、本机时间或外部实时数据；
- 隐藏字段不能在同一条件下仍要求必填；
- 条件变化时明确说明对资料和后续步骤的影响。

条件字段从显示变为隐藏时，不得静默删除已有值。必须按发布规则执行以下明确动作之一：

- 保留但不提交，并向用户说明；
- 请求用户确认后清除；
- 因安全或业务规则必须清除，并保留受控变更记录；
- 阻止继续并要求处理冲突。

## 10. 预填与只读值

预填值必须带来源和版本，例如：

- 已确认的工单或客户事实；
- 前序任务提交；
- 项目默认值；
- 设备或资产绑定；
- 当前任务人工输入。

页面使用业务化说明展示来源，例如“来自已确认的客户地址”，不显示内部对象引用。

权威只读字段不能通过调试入口或离线编辑器修改。预填来源在提交前发生变化时，服务端按字段政策：

- 自动刷新并提示差异；
- 阻止并要求确认；
- 允许保留旧值并记录原因；
- 转入人工处理。

客户端不得静默用最新值覆盖本机输入，也不得默认坚持旧值。

## 11. 重复组和明细

重复组每一行使用稳定 rowId，不按数组位置识别业务行。

增删改必须保留明确差异和本机保存状态。物料、费用或设备等具有独立生命周期的对象，表单只负责采集并通过受控领域命令映射，不能永远只存在 JSON 重复组中。

离线合并时以稳定 rowId 识别冲突，禁止按位置覆盖或把服务器新增行静默删除。

## 12. 本机草稿

本机草稿属于当前账号、设备、Task 和 FormVersion，保存在 App 私有加密存储中。

草稿状态明确为：

```text
正在保存
本机已保存 · 时间
保存失败
基于旧预填版本
等待同步
发生冲突
```

本机自动保存要求：

- 字段变化后可靠落盘；
- App 挂起和杀进程后可恢复；
- 保存失败立即提示，不能显示已保存；
- 不把敏感值写入普通偏好、日志或崩溃报告；
- 退出账号前检查未同步草稿；
- 账号或责任变化后隔离旧草稿；
- 平台确认提交前不自动删除草稿。

草稿不是业务事实，不触发流程、结算、审核或资料重解析。

## 13. 表单本地校验

本地校验覆盖：

- 必填；
- 类型和格式；
- 最小、最大和长度；
- 单位和精度；
- 枚举值；
- 条件必填；
- 已发布跨字段规则；
- 重复组结构；
- 当前客户端可执行的确定性规则。

错误阻止提交，警告要求用户理解但是否允许继续由项目规则决定。客户端不能把服务端错误降级为警告。

错误信息使用中文并定位到具体字段、原因和修复方式。不能只显示“参数错误”或表达式内部信息。

## 14. 不可变 FormSubmission

用户明确执行“提交表单”后，客户端提交精确 FormVersion、values、基础版本和稳定 `deviceCommandId`。

服务端在同一事务中完成：

- 当前责任和 Task guard 校验；
- 精确 FormVersion 校验；
- 服务端规则校验；
- 不可变 FormSubmission；
- SubmissionValidation；
- 单调 submissionVersion；
- 审计、幂等结果和 Outbox。

只有服务端返回权威提交版本和验证状态后，页面才能显示“平台已接收表单”。`VALIDATED` 只代表结构和业务校验通过，不等于审核通过或 Task 完成。

提交后不能原地修改。需要更正时必须执行明确的重新提交或整改动作，创建新的 submission version，关联被替代版本和原因。

## 15. 表单提交与条件资料解析

同一 Task 锁定 FormVersion 的最新 `VALIDATED FormSubmission` 是条件资料重解析的权威表单事实。

表单提交后，Evidence 模块按不可变 generation 重新解析资料要求：

- 新条件为 true：创建或沿用活动槽位；
- true 保持 true：沿用当前活动槽位及资料；
- false 变 true：创建新的槽位世代，不恢复历史槽位；
- true 变 false 且已有资料：保留历史并进入 `REVIEW_REQUIRED`；
- 缺值、类型漂移或摘要不一致：失败关闭。

页面不能在客户端直接删除或恢复 EvidenceSlot。存在 `REVIEW_REQUIRED` 时展示明确处理入口，不能自动作废历史文件或宣布资料要求已经消失。

## 16. 资料槽位

每个 EvidenceSlot 卡片展示：

- 资料中文名称；
- 为什么需要；
- 正确示例；
- 当前是否必需；
- 数量范围；
- 允许的文件类型、大小和时长；
- 现场拍摄、相册、文件、扫码或签名限制；
- 定位、水印和采集元数据要求；
- 机器校验要求；
- 当前资料项和 Revision 状态；
- 缺失、冲突或整改原因；
- 允许动作。

槽位来自当前权威 resolution generation。客户端不能因为旧工作包或旧页面仍有槽位卡片，就向已经停用的槽位创建新资料。

## 17. 资料项和 Revision

`EvidenceItem` 表示一个逻辑资料项，`EvidenceRevision` 表示该资料项的一次不可变版本。

规则：

- 新增一个独立资料项时，在槽位 maxCount 内创建新 EvidenceItem；
- 补拍、更正或整改已有资料项时，在同一 EvidenceItem 追加新 Revision；
- 不能为了规避 maxCount 在整改时创建额外 EvidenceItem；
- 新 Revision 不覆盖旧文件和旧审核决定；
- 已冻结 Snapshot 精确引用历史 Revision，不随补传变化；
- 作废必须使用专用命令、原因和授权；
- 审核决定只追加并精确引用版本。

页面使用“第 2 次补拍”等业务表达，不向师傅展示内部 Revision UUID。

## 18. 采集来源

资料要求决定可使用的采集来源：

### 18.1 现场拍摄

要求现场拍摄时，只提供相机入口，不提供相册或文件兜底。

拍摄前展示构图、光线、目标内容和安全提示。捕获原文件后计算 checksum，并记录受控采集时间、来源和必要设备信息。

### 18.2 相册选择

只有项目政策明确允许时才提供 PhotosPicker。页面必须提示该资料不是本次 App 内现场拍摄，服务端不能将其标记为可信现场拍摄。

### 18.3 文件选择

适用于项目允许的 PDF、文档或其他文件。必须检查 MIME、扩展名、大小和安全政策，不能只相信文件名。

### 18.4 扫码和 OCR

扫码保存原始结果、解析结果、用户确认和采集时间。OCR 是机器建议，必须展示置信度和模型版本，并把用户确认值与原始识别结果分开。

关键 SN、VIN 或设备编号由服务端再次校验。

### 18.5 签名

签名只在合同、合规和确认主体已经明确时使用。页面必须展示签署内容、主体、时间和用途。师傅不能代替客户签名，也不能把空白签名图片复用到其他任务。

客户端不支持所需签名能力时失败关闭，不能改成普通图片上传。

## 19. 水印、定位和原始文件

水印是基于受控元数据生成的派生版本，不能永久修改或替代原始捕获文件。

页面必须区分：

- 原始捕获时间；
- 服务器接收时间；
- 定位是否可用；
- 水印展示内容；
- 服务端验证结果。

客户端不能自报 `locationVerified=true`、可信设备或审核通过。定位、时间和设备结论由服务端根据证据政策验证。

水印不得暴露超出项目要求的完整客户地址、手机号、身份证或内部技术标识。

## 20. 本地资料预检

在不替代服务端验证的前提下，可以执行：

- 文件类型、大小和时长检查；
- 模糊、过暗、反光提示；
- 基础构图提示；
- 条码或 OCR 建议；
- 重复文件 checksum 提示；
- 槽位数量检查。

本地结果必须标记为“设备预检”。本地通过不等于平台验证通过；模型不可用或无法判断时显示真实状态，不能默认通过。

危险现场不应为了获得更好照片鼓励师傅靠近带电设备、攀爬或进入不安全区域。安全阻断优先于资料完整度。

## 21. 本机文件状态

本机采集后使用以下状态：

```text
本机已保存
等待创建上传会话
等待上传
上传中
等待平台收件
平台验证中
平台可用
需要重拍
已隔离
发生冲突
最终失败
```

“本机已保存”不代表已创建 EvidenceRevision；只有安全上传 Finalize 成功后，平台才创建或追加不可变 Revision。

本机文件存放在 App 私有受保护目录，不默认写入系统相册。只有业务明确要求用户保存副本且获得确认时，才允许额外写入相册。

## 22. 安全上传链路

IOS-006 负责创建采集成果和加入队列，实际可靠传输由 IOS-008 管理。

链路为：

```text
本机文件和 checksum
→ BeginUpload 控制面授权
→ 受限数据面上传
→ Finalize
→ EvidenceRevision STORED / VALIDATING
→ 安全扫描
→ 机器校验
→ VALIDATED / QUARANTINED / FAILED
```

约束：

- Begin 和 Finalize 实时或恢复同步时重新校验责任、槽位和会话所有者；
- 数据面上传不携带 Bearer Token、Technician Context 或业务幂等键；
- 上传地址短时有效且不能作为永久下载 URL；
- Finalize 使用稳定命令标识，响应丢失后重试不创建重复 Revision；
- 文件 I/O 不放在持有数据库锁的长事务中；
- Revision、投影、审计、Outbox 和幂等结果遵守既有事务边界。

## 23. 扫描、机器校验和审核

文件状态、机器校验和人工审核是三个不同维度。

例如：

```text
文件已收件
安全扫描通过
机器校验需要确认
尚未进入人工审核
```

禁止：

- Finalize 成功即显示审核通过；
- 安全扫描通过即显示资料合格；
- OCR 一致即显示业务审核通过；
- 机器服务不可用时默认通过；
- 隔离文件提供下载或提交入口。

平台处理延迟时页面展示数据截至时间和刷新入口，不生成固定倒计时或伪造预计完成时间。

## 24. 整改和补传

整改页面只要求处理被退回的对象：

- 原 FormSubmission 或 EvidenceRevision；
- 审核原因和补充说明；
- 正确示例；
- 截止时间；
- 当前整改轮次；
- 允许动作。

表单整改创建新的 FormSubmission version；资料整改在同一 EvidenceItem 追加新 Revision。已通过项目保持只读，不要求重新上传。

师傅不能修改 ReviewDecision、删除驳回历史或把新文件标记为已通过。新版本必须进入后续复核。

## 25. 离线表单与资料

有效工作包包含：

- 精确 FormVersion 和规则摘要；
- 预填快照和版本；
- 当前 Evidence resolution generation；
- 活动槽位和采集要求；
- 允许的离线字段、选项和采集能力；
- 工作包有效期；
- 命令依赖和 Schema 版本。

离线时允许：

- 填写并自动保存本机草稿；
- 执行已冻结的确定性本地校验；
- 采集允许的本机资料；
- 计算 checksum 和设备预检；
- 排队工作包明确允许的表单或资料命令。

离线时禁止：

- 使用未缓存远程选项；
- 自行扩充 EvidenceSlot；
- 使用过期上传会话；
- 显示平台提交或平台验证成功；
- 以最新配置替换任务锁定版本；
- 将未知规则或能力静默忽略。

表单条件导致资料要求变化时，客户端可以基于锁定规则提供预览，但服务端恢复网络后必须以最新权威 `VALIDATED FormSubmission` 重新解析。客户端不能删除历史槽位或资料。

## 26. 离线命令依赖

典型依赖关系：

```text
离线表单提交命令
    → 条件资料重解析确认
    → 资料 Begin / Upload / Finalize
    → EvidenceRevision 平台验证
    → IOS-007 提交前检查
```

每个命令保存稳定 `deviceCommandId`、基础版本、`capturedAt`、配置摘要和前置命令。

服务器逐条返回接受、拒绝或冲突结果。前置表单提交失败时，依赖其条件结果的资料命令停止，不能向旧槽位或猜测槽位继续上传。

## 27. 冲突处理

以下情况进入 IOS-010：

- 当前责任师傅或网点变化；
- Task、Visit 或工作包失效；
- 预填权威值发生变化；
- FormVersion 或客户端能力不兼容；
- 同一表单出现其他设备的新 submission；
- Evidence resolution generation 变化；
- 槽位被停用或重新生成；
- 同一 EvidenceItem 出现新 Revision；
- 本机文件缺失或 checksum 不一致；
- 审核或整改状态变化。

冲突界面并列展示本机成果、服务器事实、来源版本和允许动作。禁止整表最后写入覆盖、自动丢弃文件或把旧师傅成果转给新师傅。

## 28. 本地清理和退出账号

本机原始文件和草稿只有在以下条件同时满足时才能按政策清理：

- 平台已经确认对应 FormSubmission 或 EvidenceRevision；
- 文件达到允许清理的服务器状态；
- 不再需要本机重试或整改；
- 保留政策允许；
- 本机校验清理目标与平台引用一致。

退出账号前必须检查未同步草稿、文件和命令。不能无提示删除，也不能让下一个账号看到前一个账号资料。

设备空间不足时展示可处理的本机占用和安全清理建议，不得删除尚未同步或处于冲突中的业务成果。

## 29. 命令边界

IOS-006 可以调用：

- 提交不可变 FormSubmission；
- Begin Evidence 上传；
- Finalize Evidence 上传并创建或追加 Revision；
- 项目允许的资料作废或重新采集申请；
- 整改产生新 submission 或 Revision；
- 有效工作包允许的对应离线命令。

IOS-006 不提供：

- 通用更新 FormSubmission；
- 覆盖 EvidenceRevision；
- 客户端创建或修改 EvidenceSlot；
- 手工设置 VALIDATED、APPROVED 或 AVAILABLE；
- 修改 ReviewDecision；
- 永久文件 URL 写入表单；
- 绕过文件安全扫描；
- 直接完成 Task、Visit 或工单。

## 30. 页面状态与恢复动作

| 状态 | 页面表现 | 允许动作 |
|---|---|---|
| 新草稿 | 展示预填和来源 | 开始填写 |
| 本机已保存 | 显示保存时间 | 继续填写、校验 |
| 草稿保存失败 | 明确数据风险 | 重试保存、查看原因 |
| 表单校验错误 | 定位具体字段 | 修正字段 |
| 平台已接收表单 | 显示不可变版本 | 查看、按明确动作更正 |
| 资料未采集 | 显示要求和示例 | 使用允许来源采集 |
| 本机文件待上传 | 不显示平台可用 | 进入 IOS-008 |
| 平台验证中 | 区分扫描和机器校验 | 等待、刷新 |
| 需要重拍 | 展示具体原因 | 在同一 Item 追加 Revision |
| 条件资料待处置 | 保留历史资料 | 查看处置要求 |
| 责任失效 | 隐藏敏感入口并停止写入 | 进入 IOS-010 |
| 工作包过期 | 保留本机成果 | 联网校验、处理冲突 |
| 客户端不兼容 | 明确升级要求 | 升级、联系网点 |

## 31. 安全、隐私与审计

表单提交、资料 Begin/Finalize、Revision、整改和作废必须记录：

- 租户、项目、工单、Task 和步骤；
- 当前责任网点和师傅；
- 操作者和受信 Context；
- FormVersion、Evidence generation、Slot、Item 和 Revision；
- 业务动作、标准原因和稳定错误码；
- checksum、MIME、大小和受控采集元数据；
- `capturedAt`、`receivedAt` 和服务器时间基准；
- 在线、离线和设备命令来源；
- 幂等、事务、审计和 Outbox 结果。

日志、埋点和崩溃报告不得记录：

- 完整表单 values；
- 客户姓名、手机号、详细地址或证件；
- 文件正文、缩略图或永久 URL；
- 原始 EXIF 和精确位置；
- Authorization、上传凭据或完整设备标识。

应用切换器快照、锁屏通知和系统分享默认隐藏敏感表单和现场资料。

## 32. 无障碍和现场交互

- VoiceOver 能读取字段标签、必填状态、单位、错误和资料状态；
- 动态字体放大后不截断字段说明和安全提示；
- 错误不能只通过红色表达；
- 相机拍摄提供清晰语音和文字指导，但不鼓励危险姿势；
- 自动保存、上传和验证状态使用文字说明；
- 长表单按业务分区并保留当前位置；
- 输入键盘不会遮挡当前字段和主操作；
- 低网速时不阻塞本机草稿保存；
- 触控区域适合单手和手套场景。

## 33. 与其他页面的关系

```text
IOS-005 作业步骤
    → IOS-006 表单与资料采集
        → IOS-008 上传中心
        → IOS-009 离线恢复
        → IOS-010 同步冲突
    → IOS-007 提交前检查

审核退回
    → IOS-006 指定表单或 EvidenceItem 整改
    → 新 Submission / Revision
    → 重新进入复核
```

Admin、Network Portal 和 iOS 读取同一 FormSubmission、EvidenceSlot、EvidenceItem、EvidenceRevision、Snapshot、Review 和 Correction 事实，不建立端侧平行资料状态。

## 34. 验收标准

### 表单

- [ ] 只渲染 Task 锁定的精确 FormVersion；
- [ ] 未知字段、规则和版本失败关闭；
- [ ] 条件显隐不静默删除已有值；
- [ ] 重复组使用稳定 rowId；
- [ ] 本机草稿支持自动保存和杀进程恢复；
- [ ] 本地校验通过不等于平台已接收；
- [ ] FormSubmission 不可变，更正创建新版本；
- [ ] `VALIDATED` 不等于审核通过或 Task 完成。

### 资料

- [ ] EvidenceSlot 来自当前权威 resolution generation；
- [ ] 现场拍摄要求时不提供相册兜底；
- [ ] 本机文件、上传、扫描、机器校验和审核状态分别表达；
- [ ] Finalize 幂等重试不创建重复 Revision；
- [ ] 整改在同一 EvidenceItem 追加 Revision，不规避 maxCount；
- [ ] 已提交历史和 ReviewDecision 不被覆盖；
- [ ] true→false 的已有条件资料不会被自动删除或作废。

### 离线和冲突

- [ ] 普通缓存不授权离线表单或资料命令；
- [ ] 有效工作包冻结表单、规则、槽位和预填版本；
- [ ] 离线状态不显示平台提交或验证成功；
- [ ] 条件资料命令正确依赖表单提交和权威重解析；
- [ ] 责任、预填、槽位或多设备变化不使用最后写入覆盖；
- [ ] 本机成果在冲突时受控保留并进入 IOS-010。

### 文件安全和隐私

- [ ] 数据面上传不携带业务 Bearer Token 或 Context；
- [ ] 上传地址不是永久下载地址；
- [ ] 隔离或未扫描资料不能用于提交；
- [ ] 原文件和草稿存放在 App 私有受保护空间；
- [ ] 完整 values、文件正文和敏感元数据不进入日志；
- [ ] 退出账号不会泄露或无提示删除未同步成果。

### 业务闭环

- [ ] 表单提交能够触发权威条件资料重解析；
- [ ] 页面能够解释活动槽位、历史槽位和待处置资料；
- [ ] 已达到平台要求的表单和资料可进入 IOS-007；
- [ ] 页面不会直接完成 Task、Visit、工单或审核；
- [ ] Admin、Network Portal 和 iOS 对同一资料版本状态一致。

## 35. 禁止实现

禁止：

- 使用最新表单版本替换任务锁定版本；
- 把未知字段降级为文本框；
- 执行任意 JavaScript、SQL 或不受控表达式；
- 条件隐藏时静默清空用户输入；
- 草稿保存成功就创建业务提交；
- 修改或删除历史 FormSubmission；
- 把对象存储 URL 保存进表单 values；
- 现场拍摄要求使用相册兜底；
- 本机拍摄完成即显示平台资料可用；
- Finalize 完成即显示审核通过；
- 整改时创建新 EvidenceItem 绕过槽位 maxCount；
- 条件 false 时自动删除已有资料；
- 用最新上传覆盖旧 Revision 或审核决定；
- 上传失败后返回空资料或默认成功；
- 用普通缓存、旧工作包或开发模式授权离线采集；
- 为旧 H5、旧开发数据或假设客户端保留双轨表单和资料模型。
