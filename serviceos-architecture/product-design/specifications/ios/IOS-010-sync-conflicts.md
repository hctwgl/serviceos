# IOS-010 同步冲突

状态：Accepted
设计等级：A

## 1. 页面定位

“同步冲突”是原生 iOS 师傅端处理本机成果与平台当前事实不一致的统一工作区。

它必须让师傅清楚回答：

1. 哪个本机操作没有被平台接受；
2. 平台发生了什么变化；
3. 哪些本机表单、资料、预约或作业事实受到影响；
4. 哪些平台事实已经确认，不能被本机覆盖；
5. 当前允许采用平台值、保留本机后重新校验、重新填写、重新采集、放弃或转人工中的哪一种；
6. 处理一个冲突会如何影响下游命令和任务提交；
7. 本机成果是否仍被安全保留，何时才可以清理。

冲突中心不是通用错误列表。网络中断、暂时平台不可用、普通校验错误和幂等命中都必须在进入冲突中心前正确分类。

## 2. 产品边界

IOS-010 负责：

- 按任务和影响程度展示真正需要业务决策的同步冲突；
- 并列展示本机事实、平台事实、来源、版本和差异；
- 解释责任、业务状态、表单、预填、资料槽位、文件和多设备冲突；
- 展示受影响的 OfflineCommand、LocalDraft、LocalFile 和下游依赖；
- 只提供服务端允许的解决动作；
- 支持采用平台事实、基于最新版本重新确认、重新填写、重新采集、明确放弃或申请人工处理；
- 在离线时保存用户的处理意图，联网后重新校验并提交；
- 保留原命令、原草稿、原文件和平台收据的追溯关系；
- 只有平台确认解决后才解除依赖和清理本机重复待办；
- 为 IOS-009 返回可信的冲突处理结果。

IOS-010 不负责：

- 使用最后写入覆盖平台或另一设备的数据；
- 在客户端实现通用对象合并器；
- 允许旧师傅覆盖当前责任人的业务事实；
- 修改平台已经确认的历史 Visit、Submission、EvidenceRevision、审核或整改事实；
- 把网络错误、可重试失败或幂等接受伪装成冲突；
- 直接修改项目配置、表单版本、资料槽位或流程状态；
- 自动把旧师傅本机成果转交新师傅；
- 把“联系网点”当作冲突已经解决；
- 在平台确认前删除本机成果或恢复下游命令；
- 声称当前仓库已经实现原生 ConflictResolver 运行时。

## 3. 当前工程事实与目标边界

本规格定义原生 iOS ConflictResolver 的产品目标，不改变当前工程事实：

- 平台已有部分在线版本校验、责任变化拒绝、幂等和 feed/tombstone 能力；
- Technician H5 只展示有限冲突指引，并诚实登记 OfflineCommand 明细缺口；
- 原生冲突记录、差异模型、解决命令和本机迁移尚需后续机器契约与工程实现；
- IOS-009 负责发现、隔离和路由冲突，IOS-010 负责需要用户理解与选择的处理过程；
- 精确 OpenAPI、错误码、冲突 Schema、解决命令、保留期限和权限必须由后续工程设计形成唯一事实源；
- 本文中的模型名称是产品语义，不是已经接受的字段协议。

## 4. 什么才是同步冲突

只有同时满足以下条件的问题才进入 IOS-010：

```text
存在尚未被平台接受的本机业务成果
+
平台当前事实或规则已经与本机基线不同
+
客户端不能安全自动确定唯一结果
+
需要用户重新确认、重新采集、明确放弃或转人工
```

以下问题不是同步冲突：

| 情况 | 正确归类 |
|---|---|
| 无网络、请求超时 | IOS-009 等待网络或可重试失败 |
| 平台短暂不可用 | IOS-009 有上限重试 |
| 同一命令已经被平台接受 | 幂等绑定既有结果 |
| 必填字段尚未填写 | IOS-006 或 IOS-007 校验问题 |
| 文件仍在上传 | IOS-008 上传状态 |
| 文件安全扫描中 | IOS-008 平台处理状态 |
| 用户无权查看任何冲突内容 | 权限拒绝与安全隔离 |
| App 不支持 WorkPackage Schema | IOS-009 要求更新或刷新 |

## 5. 入口与导航

入口包括：

- IOS-009“需要处理”的冲突分组；
- IOS-001 今日任务的冲突摘要；
- IOS-002 任务详情的阻塞提示；
- IOS-003 预约冲突提示；
- IOS-006 表单与资料冲突提示；
- IOS-007 提交前检查；
- IOS-008 上传中心的槽位、文件或 Revision 冲突；
- 平台发送的受控深链通知。

通过任务入口进入时只展示当前 Task 的冲突；通过全局入口进入时按任务分组。已经失去任务访问权的师傅只能看到安全的失效说明和本机隔离摘要，不能继续读取平台敏感数据。

## 6. 页面结构

```text
冲突总览与阻塞任务数量
→ 需要立即处理
→ 按任务分组的冲突列表
→ 冲突详情
   ├── 发生了什么
   ├── 本机成果
   ├── 平台当前事实
   ├── 差异与影响
   ├── 受影响命令和文件
   └── 服务端允许的解决动作
→ 等待平台确认
→ 最近已解决
→ 已隔离或需要人工处理
```

默认优先级：

1. 责任撤销或设备撤销；
2. 阻止现场安全步骤或离场的冲突；
3. 阻止当前任务提交的冲突；
4. 临近预约、整改或 SLA 截止时间；
5. 影响多个下游命令的冲突；
6. 普通字段、资料和多设备冲突。

排序原因必须使用业务中文解释，不能显示不可理解的本地风险分数。

## 8. 冲突身份与生命周期

每个冲突必须具有稳定身份，并关联：

- 原始 `deviceCommandId`；
- Task 和目标业务对象；
- WorkPackage、责任、配置和业务对象基线版本；
- 本机草稿或文件引用；
- 平台检测到的当前版本；
- 冲突分类和结构化原因；
- 受影响的下游命令；
- 平台允许的解决动作；
- 创建、更新时间和解决结果。

冲突状态包括：

- 已发现，等待查看；
- 等待用户选择；
- 处理意图已保存，等待联网；
- 解决命令发送中；
- 等待平台确认；
- 平台已解决；
- 解决条件再次变化，需要重新处理；
- 已转人工；
- 因责任或权限撤销被隔离；
- 已明确放弃本机成果；
- 已按保留政策安全清理。

用户不能仅在本机把冲突标记为“已解决”。

## 9. 冲突分类

### 9.1 责任变化

任务已经改派、当前责任网点变化或当前师傅不再负责。

处理原则：

- 旧师傅立即停止提交和刷新授权；
- 服务端在幂等占位、业务写入、审计和 Outbox 之前拒绝旧责任写入；
- 本机成果进入隔离，不自动转给新师傅；
- 页面只展示安全摘要、失效原因和受控后续路径；
- 已被平台接受的历史事实不受影响。

### 9.2 权限或设备变化

账号失去 Capability、组织关系失效、设备被撤销或安全策略阻止继续。

这类冲突不提供“保留本机并强制提交”。用户只能停止操作、重新认证、联系管理员或进入受控恢复。

### 9.3 任务或工单状态变化

平台任务已经完成、取消、进入其他节点或不再允许原动作。

页面必须显示平台当前阶段、原动作为什么不再允许以及本机成果的保留状态。客户端不能把任务回退到旧状态。

### 9.4 WorkPackage 或配置变化

表单、资料、流程、SLA 或业务规则版本发生变化，本机草稿不再能按原结构直接提交。

只有存在经过版本化验证的迁移规则时，才允许生成迁移预览。没有明确规则时必须重新填写、重新采集或转人工，不能按字段名相同就自动复制。

### 9.5 预填或字段值变化

本机编辑基于旧预填值，而平台权威值已经变化。

冲突页按字段并列展示：

- 字段业务名称；
- 本机值及来源；
- 平台值及来源；
- 本机修改时间；
- 平台更新时间；
- 对校验、资料、价格或后续步骤的影响；
- 当前允许动作。

不得默认使用时间较晚的一方。

### 9.6 多设备编辑

同一师傅或授权用户在另一设备上修改了同一草稿或业务对象。

页面必须显示安全的设备来源、版本差异和影响。设备时间不能用于决定胜者；平台版本和显式用户选择才是解决依据。

### 9.7 预约变化

客户、网点或平台已经确认新的 AppointmentRevision，而本机仍持有旧预约动作。

处理必须回到 IOS-003 的预约语义：采用当前预约、基于最新版本重新改约或转人工。不能直接覆盖已确认预约。

### 9.8 资料槽位或 Revision 变化

资料槽位失效、数量要求变化、原 Item 已被整改替代、Revision 已作废或服务端已存在不同文件事实。

允许动作可以包括：

- 绑定平台已经幂等接受的同一结果；
- 选择新的有效槽位；
- 基于当前要求重新采集；
- 保留本机文件并转人工；
- 明确放弃本机文件。

不得覆盖已有 Revision 或创建新 Item 绕过 `maxCount`。

### 9.9 文件或上传会话变化

checksum、大小、MIME、分片、上传会话、StoredFile 或 Finalize 结果不一致。

数据面恢复和普通失败回 IOS-008；只有存在两个不兼容业务结果、会话所有者变化或文件无法安全绑定时才进入 IOS-010。

### 9.10 提交快照或审核变化

平台已经建立 EvidenceSetSnapshot、FormSubmission、审核或整改事实，而本机仍试图基于旧输入完成任务。

这些不可变事实不能被覆盖。用户必须查看平台当前版本，重新形成允许的新提交，或转人工处理。

## 10. 可自动确认的情况

以下情况不需要用户选择，可以自动收口：

- 相同 `deviceCommandId` 已经被平台接受，返回同一业务结果；
- 相同文件摘要和命令已经绑定既有 StoredFile 或 EvidenceRevision；
- 本机结果与平台事实业务等价，且机器契约明确允许绑定；
- 仅响应丢失，查询后得到明确接受收据；
- 暂时错误恢复后，原命令按同一 ID 成功。

自动收口必须保存平台收据并更新依赖，不能仅删除本机待办。等价判断必须由服务端或接受的机器契约提供，前端不能自行比较显示文本后认定相同。

## 11. 不允许自动解决的情况

以下情况必须要求用户或授权人员处理：

- 本机与平台字段值均有业务意义且不相同；
- 配置迁移可能改变字段含义、单位、精度或必填条件；
- 责任已经变化；
- 资料槽位或审核目标已经变化；
- 多设备修改无法证明业务等价；
- 本机文件与平台文件摘要不同；
- 任务状态已经前进或结束；
- 解决会改变预约、客户确认、签名、费用、安全或审核事实；
- 服务端没有返回唯一安全动作。

## 12. 解决动作模型

页面只能展示服务端针对当前冲突返回的动作。动作可以包括：

### 12.1 采用平台事实

放弃当前冲突中的本机修改，刷新到平台当前版本。

执行前必须：

- 展示将放弃的本机字段、文件和下游命令；
- 要求明确确认；
- 保存原冲突与处理结果；
- 平台确认后才解除依赖和进入本机清理。

### 12.2 保留本机内容并基于最新版本重新确认

只在服务端明确允许时出现。

它不是覆盖平台，而是：

```text
保留原本机内容
→ 获取平台最新基线
→ 重新校验和展示差异
→ 用户重新确认
→ 创建新的解决命令
→ 服务端再次校验当前状态
```

原命令保持不可变并标记为冲突；新的解决命令使用新的命令 ID，并引用原冲突和原命令，不能修改原命令载荷后继续使用旧 ID。

### 12.3 重新填写或重新采集

适用于配置、表单、资料槽位、文件或安全规则已经变化。

新草稿或文件必须基于当前 WorkPackage 和当前要求创建。旧成果继续按保留政策隔离，不能伪装成新版本采集结果。

### 12.4 明确放弃本机成果

只放弃当前设备上的未确认成果，不改变平台事实。

页面必须说明：

- 将放弃哪些字段、文件和命令；
- 是否影响下游任务；
- 是否仍有平台已确认事实；
- 本机文件何时清理；
- 操作是否不可恢复。

### 12.5 转人工处理

创建平台权威的人工处理任务或异常记录，并返回可追踪引用。仅显示“联系网点”或拨打电话不能算转人工成功。

转人工后本机成果保持只读和隔离，直到平台给出处理结论。

## 13. 字段级差异

字段级差异适用于动态表单和预填冲突，但不是任意 JSON Diff。

每个差异需要业务语义：

- 字段名称和所属分组；
- 数据类型、单位和精度；
- 本机值与来源；
- 平台值与来源；
- 是否由用户修改；
- 是否影响条件显隐、校验、资料或计算；
- 是否允许单字段选择；
- 选择后必须重新执行的规则。

涉及签名、客户确认、费用、安全测量、定位验证、审核结论或不可变提交时，不允许拆成普通字段逐项拼接。

## 14. 配置与表单迁移

草稿迁移必须遵守：

- 只有已接受、版本化且可测试的迁移规则才能自动生成预览；
- 字段标识、类型、单位、枚举和条件规则都必须兼容；
- 删除、拆分、合并或改变含义的字段必须人工确认；
- 条件隐藏不能静默删除已有值；
- 迁移后重新执行完整本地校验和必要平台校验；
- 原草稿作为只读来源保留到新结果被平台确认；
- 迁移结果创建新草稿版本，不覆盖原草稿。

没有迁移规则时，页面明确显示“当前草稿无法安全迁移”，只能重新填写或转人工。

## 15. 资料与文件处理

冲突中的本机文件必须保持：

- 原始 `localFileId`、checksum 和采集 metadata；
- 与原 Task、Slot、Item、Revision 和命令的关联；
- 当前账号和设备分区；
- 隔离状态和保留截止时间；
- 是否允许重新绑定、重新上传、重新采集或放弃；
- 平台是否已经持久化相同内容。

“采用平台事实”不能立即删除本机文件。只有平台确认、依赖解除、审计完成和保留策略满足后，IOS-008 才能执行安全清理。

## 16. 依赖传播

冲突会阻止所有强依赖当前命令的下游命令，但不会阻止无依赖的其他分支。

解决后：

- 采用平台事实时，下游命令必须基于平台结果重新校验；
- 创建新解决命令时，下游依赖改为等待新命令；
- 重新填写或采集时，下游继续阻塞；
- 放弃本机成果时，依赖该成果的下游命令不能自动恢复；
- 转人工时，下游等待平台人工结论；
- 任一基线再次变化时产生新的冲突状态，而不是复用旧结论。

## 17. 部分解决

同一任务可能同时存在多个独立冲突。解决其中一个不能把整张任务标记为已解决。

例如：

```text
预约冲突：已采用平台预约
表单字段冲突：等待用户选择
资料槽位冲突：已转人工
上传命令：无冲突，继续执行
```

页面必须分别显示状态，并根据依赖计算任务是否仍被阻止。不能用一个本地布尔值覆盖所有冲突。

## 18. 离线处理冲突

离线时用户可以：

- 阅读已经安全缓存的冲突说明；
- 查看本机成果和上次同步的平台摘要；
- 保存明确的处理意图；
- 继续不依赖冲突的其他允许步骤。

离线时不能：

- 声称平台事实仍然最新；
- 完成需要平台重新校验的解决动作；
- 获取已经失权的敏感数据；
- 把处理意图标记成冲突已解决。

联网后必须先刷新责任、授权、平台版本和允许动作。保存的处理意图如果已经不再适用，页面要求用户重新确认。

## 19. 解决命令与并发

每个解决动作使用新的稳定命令 ID，并关联：

- 原冲突 ID；
- 原 `deviceCommandId`；
- 用户选择的解决动作；
- 当前平台基线版本；
- 本机成果摘要；
- 用户确认时间；
- 当前设备和认证上下文。

服务端处理解决命令时重新校验当前责任、权限、冲突状态和平台版本。若处理期间状态再次变化，返回新的结构化冲突，不使用旧选择强制覆盖。

## 20. 改派与责任撤销

责任冲突的首要目标是停止越权，而不是保住原命令的可提交性。

页面规则：

- 隐藏客户、地址、文件缩略图等不再允许访问的内容；
- 展示安全的本机成果数量、采集时间和隔离状态；
- 不提供“仍然提交”“改成新师傅”“转移文件”按钮；
- 可以提供平台允许的人工处理申请；
- 已经平台接受的历史事实保持只读；
- 新师傅必须使用自己的 WorkPackage 和命令。

## 21. 多设备冲突

多设备冲突必须展示：

- 安全的设备名称或注册摘要；
- 各自基于的平台版本；
- 各自修改范围和时间；
- 平台已经接受的结果；
- 仍未确认的本机成果；
- 当前允许的选择。

设备时间较晚不代表业务优先。不能自动合并签名、照片、预约确认、完成命令或不可变提交。

## 22. 人工处理闭环

转人工必须创建平台事实，并展示：

- 处理编号；
- 负责角色或组织；
- 创建时间；
- 需要处理的问题；
- 本机成果是否已安全登记；
- 当前预计下一步；
- 平台处理结果。

人工结论可以要求采用平台、允许重录、恢复特定动作或终止任务，但客户端仍需通过正式允许动作执行，不能把人工备注解析成权限。

## 23. 冲突重新打开

已解决冲突不会被原地改写。如果解决后平台状态再次变化：

- 原冲突保留解决历史；
- 创建新的冲突记录或新的冲突轮次；
- 明确关联前一冲突；
- 用户可以看到为什么需要再次处理；
- 不覆盖旧选择和平台收据。

## 24. 保留与清理

本机成果只有满足以下条件才可清理：

- 平台已确认解决结果；
- 不再被任何未完成命令依赖；
- 不处于人工处理或审计保留中；
- 当前账号和设备政策允许；
- 用户已经确认不可恢复影响，或保留期限已经明确到期；
- IOS-008 已确认相关文件不再需要。

清理只删除允许删除的本机副本，不删除平台历史事实、审计、收据或不可变 Revision。

## 25. 用户可见状态与动作

| 场景 | 页面表达 | 主要动作 |
|---|---|---|
| 幂等接受 | 平台已接收相同操作 | 绑定结果、返回任务 |
| 责任已变化 | 任务已改派，本机成果已隔离 | 查看原因、申请人工处理 |
| 平台版本已更新 | 平台数据已变化 | 查看差异、重新确认 |
| 表单配置变化 | 当前草稿不能直接提交 | 迁移预览、重新填写、转人工 |
| 预填字段变化 | 本机值与平台值不一致 | 采用平台、保留后重新校验 |
| 多设备修改 | 另一设备存在不同修改 | 比较并选择受控动作 |
| 资料槽位失效 | 原资料要求已经变化 | 选择新槽位、重新采集、转人工 |
| 文件内容冲突 | 平台文件与本机文件不同 | 查看摘要、重新上传或转人工 |
| 任务已结束 | 原操作已不再允许 | 采用平台、保留本机记录 |
| 等待人工处理 | 已创建平台处理事项 | 查看进度 |
| 解决条件再次变化 | 原选择已经过期 | 刷新并重新确认 |
| 平台已解决 | 平台已确认处理结果 | 返回任务、安全清理 |

## 26. 错误与不可用状态

- 无法获取最新平台事实时，显示“暂时无法核对冲突”，不展示旧动作可直接提交；
- 本机草稿缺失时，显示“本机成果不完整”，不伪造差异；
- 服务端未返回允许动作时，只允许刷新或转受控支持；
- 冲突详情 Schema 不受支持时，要求更新 App，不显示原始 JSON；
- 人工处理服务不可用时，不显示“已转人工”；
- 提交解决动作响应丢失时，按命令 ID 查询结果，不重复创建解决记录；
- 权限不足时隐藏敏感差异，只展示业务化拒绝原因。

## 27. 通知

允许通知：

- 新冲突阻止当前任务；
- 人工处理已经受理；
- 冲突解决条件再次变化；
- 平台已经确认解决；
- 本机隔离成果即将达到保留期限。

锁屏通知不得显示客户、地址、字段值、照片、文件名、冲突载荷或其他设备敏感信息。

## 28. 命令与查询边界

后续机器契约至少需要覆盖以下语义：

- 查询当前账号和设备可见的冲突列表；
- 查询脱敏后的冲突详情、版本差异和影响；
- 查询当前允许的解决动作；
- 提交带幂等键和当前版本的解决命令；
- 查询解决命令结果；
- 创建并查询人工处理事项；
- 确认本机已经安全处理 tombstone；
- 查询本机成果是否允许清理。

服务端必须从受信身份解析 tenant、project、operator、设备、责任和权限。客户端提交的冲突类型、允许动作和平台版本都必须重新校验。

## 29. 安全、日志与审计

平台对冲突发现、查看、选择、解决、放弃、转人工和清理记录：

- Tenant、Project、Task 和目标业务对象；
- 当前责任网点和师傅；
- 操作者、设备注册和受信 Context；
- 冲突、原命令和解决命令标识；
- WorkPackage、配置、责任和业务对象版本；
- 解决动作、平台结果和失败原因；
- 受影响的草稿、文件和命令摘要；
- 审计、幂等、业务结果和 Outbox 引用。

日志、埋点和崩溃报告不得包含：

- 本机和平台字段原值；
- 表单正文、签名、照片、视频或客户信息；
- 本机绝对路径和数据库密钥；
- OfflineCommand 或解决命令载荷；
- Token、上传凭据和原始异常响应；
- 其他设备的可识别秘密。

## 30. 无障碍与移动交互

- 本机值、平台值和来源不能只用左右位置或颜色区分；
- VoiceOver 能读取冲突类型、差异来源、影响和允许动作；
- 动态字体放大后保持差异项和动作的逻辑顺序；
- 高风险动作提供明确标题、影响说明和二次确认；
- 列表更新不跳动焦点；
- 已解决、隔离、等待人工和再次变化分别提供文本状态；
- 字段差异支持逐项导航，不要求横向拖动宽表；
- 重要处理结果提供视觉、文本和适度触觉反馈。

## 31. 与其他页面的关系

```text
IOS-009 离线恢复
    → 识别、隔离并创建冲突入口

IOS-010 同步冲突
    → 比较本机与平台事实
    → 提交受控解决动作
    → 平台确认后返回 IOS-009

IOS-003 / IOS-006 / IOS-008
    → 承担预约、表单资料和上传领域的重新处理

IOS-007
    → 所有阻塞冲突解决后重新执行提交检查
```

IOS-010 不建立第二套工单、预约、表单、资料或审核状态。它只保存冲突、解决过程和本机成果关联。

## 32. 验收标准

### 分类真实性

- [ ] 网络错误、可重试失败、普通校验和真正冲突分别表达；
- [ ] 幂等接受自动绑定既有结果，不要求用户重复处理；
- [ ] 冲突说明使用业务中文，不显示异常类名或原始 JSON；
- [ ] 服务端返回分类和允许动作，前端不解析错误字符串推导；
- [ ] 无法取得最新平台事实时不展示过期动作可直接执行。

### 对比与解决

- [ ] 本机值、平台值、来源、版本和影响可明确比较；
- [ ] 不使用最后写入覆盖；
- [ ] 采用平台事实前明确展示将放弃的本机成果；
- [ ] 保留本机重新确认时创建新解决命令，不修改原命令；
- [ ] 配置迁移只使用已接受的版本化规则；
- [ ] 解决期间平台再次变化会要求重新处理。

### 责任与多设备

- [ ] 改派后旧师傅写入在幂等占位和业务事实前被拒绝；
- [ ] 旧师傅本机成果不会自动转交新师傅；
- [ ] 权限和设备撤销不提供强制提交动作；
- [ ] 多设备不按设备时间或最后写入决定胜者；
- [ ] 已经平台接受的历史事实不会因冲突处理被覆盖。

### 文件与依赖

- [ ] 本机冲突文件在平台确认和保留条件满足前不会清理；
- [ ] 资料槽位变化不能通过创建新 Item 绕过数量限制；
- [ ] 强依赖冲突的下游命令保持阻塞；
- [ ] 无依赖分支可以继续同步；
- [ ] 单个冲突解决不会错误清除同任务其他冲突。

### 人工处理与离线

- [ ] 转人工会创建可追踪平台事项，而不是只显示联系方式；
- [ ] 离线只能保存处理意图，不能显示平台已解决；
- [ ] 联网后重新校验责任、版本和允许动作；
- [ ] 平台确认前本机冲突和原始成果保持可恢复；
- [ ] 解决命令响应丢失时按幂等键查询真实结果。

### 安全与可访问性

- [ ] 已失权用户看不到平台敏感差异；
- [ ] 日志和通知不包含字段值、客户信息、文件或命令载荷；
- [ ] 不同账号和设备分区不共享冲突成果；
- [ ] 状态和差异不只依赖颜色；
- [ ] 高风险放弃和清理动作支持读屏并明确影响。

## 33. 禁止实现

禁止：

- 将所有同步失败统一标记成冲突；
- 使用最后写入、设备时间或请求到达顺序决定业务结果；
- 在客户端实现通用 JSON 自动合并；
- 解析异常文字推导冲突类型和动作；
- WorkPackage 版本变化后按字段名相同自动迁移；
- 修改原 OfflineCommand 载荷并继续使用旧命令 ID；
- 责任变化后提供“仍然提交”或自动转交按钮；
- 覆盖既有 FormSubmission、EvidenceRevision、AppointmentRevision 或审核事实；
- 创建新资料 Item 绕过槽位数量限制；
- 用户点击“采用平台”后立即删除本机文件；
- 只在本机把冲突标记为已解决；
- 把“联系网点”显示成平台人工事项已创建；
- 离线时声称平台版本最新或处理完成；
- 一个冲突解决后清空整张任务的冲突状态；
- 为旧 H5、旧开发数据或假设客户端保留第二套 ConflictResolver；
- 在机器契约未评审前，把本文示意模型当作已接受协议；
- 将本规格标记为当前原生冲突运行时已经 Implemented。
