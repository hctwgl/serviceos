# IOS-005 作业步骤

状态：Accepted
设计等级：A

## 1. 页面定位

“作业步骤”是当前责任师傅执行现场任务的主工作区。它把工单绑定的履约配置、当前任务、`Visit`、现场操作、表单、资料和异常事实组织成一条清晰的执行路径，让师傅知道：

1. 当前应该做什么；
2. 为什么要做；
3. 完成这一步需要哪些真实业务成果；
4. 哪些内容仅保存在本机；
5. 哪些内容已经由平台确认；
6. 什么问题正在阻止继续；
7. 发生现场异常时应该如何安全退出或协调。

步骤条只是权威业务事实的导航和完成度投影，不是客户端维护的第二套流程状态机。页面不能用“点击下一步”“本机填完”或“已经拍摄”替代正式表单、资料、现场操作、确认、审核或任务事实。

## 2. 产品边界

IOS-005 负责：

- 展示当前任务锁定的现场作业步骤；
- 说明步骤顺序、前置条件、完成条件和阻止原因；
- 聚合服务器事实与本机草稿、上传和离线命令状态；
- 打开 IOS-006 完成表单与资料采集；
- 执行已接受的检查、确认或现场操作命令；
- 在允许范围内返回修改尚未提交的草稿；
- 展示步骤级 SLA、安全提示、人员确认和异常入口；
- 支持杀进程恢复和有效工作包范围内的离线执行；
- 将提交阶段导向 IOS-007；
- 将上传、离线恢复和冲突分别导向 IOS-008、IOS-009 和 IOS-010。

IOS-005 不负责：

- 在客户端编排项目工作流；
- 修改步骤定义、顺序、必填规则或完成条件；
- 通过通用“完成步骤”命令绕过真实业务资源；
- 在列表中直接编辑复杂表单或上传大文件；
- 将本机草稿标记为平台已完成；
- 修改已提交或已审核的不可变版本；
- 直接审核、强制通过或关闭整改；
- 在没有真实现场操作引用时签退 Visit；
- 直接完成工单或跳过提交前检查。

## 3. 访问、责任和到场前置

在线读取和执行必须同时满足：

```text
authenticated principal
+ trusted Technician Context
+ ACTIVE TechnicianProfile
+ ACTIVE NetworkTechnicianMembership
+ current ACTIVE technician responsibility
+ required capability
+ confirmed or policy-accepted Visit state
+ current task allowed-action
```

服务端必须在每个写命令中重新校验：

- 当前责任网点和师傅；
- Task、Visit 和 FieldOperation 当前状态；
- 工单绑定的履约配置版本；
- 步骤所引用的表单和资料版本；
- 当前资源并发版本；
- 项目安全、预约和异常政策；
- 操作者权限和数据范围。

离线执行还必须具有有效工作包，并且工作包明确包含当前步骤及其允许的离线动作。普通缓存、截图、旧表单定义或历史任务详情不能授权离线作业。

到场签到仍等待平台确认时，只有工作包显式允许依赖离线签到继续，IOS-005 才能进入可执行状态；否则只能显示步骤预览和同步入口。

## 4. 页面结构

```text
任务、Visit 与同步摘要
→ 总体完成度和当前阻止项
→ 纵向作业步骤列表
→ 当前步骤工作卡
→ 操作说明和安全提示
→ 表单、资料、确认与上传摘要
→ 现场异常和中断入口
→ 最近保存与平台确认状态
→ 固定底部主操作
```

手机首屏优先展示当前步骤名称、关键安全提示、仍缺少的成果和下一动作。完整历史、技术版本和低频说明不能把主操作挤出首屏。

## 5. 作业计划的权威来源

步骤来自工单创建时绑定的项目履约配置版本，由服务端根据当前 Task、工作流节点、表单版本、资料要求和现场操作定义生成权威作业计划。

建议页面级结构：

```text
TechnicianWorkloopView
├── taskSummary
├── visitSummary
├── fieldOperationSummary
├── executionPlan
│   └── steps[]
├── localProgressSummary
├── uploadSummary
├── syncSummary
├── allowedActions
└── blockingReasons
```

每个步骤至少包含：

- 稳定步骤引用；
- 中文名称和目的；
- 业务顺序和分组；
- 是否必需；
- 前置条件；
- 完成条件；
- 所引用的表单、资料、确认或现场操作；
- 是否允许离线执行；
- 安全等级和提示；
- 当前权威状态；
- 本机成果摘要；
- 允许动作和阻止原因；
- 配置版本和兼容要求。

客户端不能根据步骤名称、数组位置或 UI 顺序猜测业务语义。

## 6. 步骤类型和业务成果

页面可以使用统一的步骤外观，但不同步骤必须由真实业务成果支撑。

### 6.1 操作说明

用于展示施工方法、安全说明或标准检查清单。仅阅读说明默认不等于完成；项目确实要求确认时，必须产生受控确认事实，不能只保存在 `UserDefaults`。

### 6.2 现场检查

要求师傅对受控检查项逐项确认。检查结果必须使用稳定选项、必要说明和异常入口，不用一段自由文本代替结构化结果。

### 6.3 动态表单

打开 IOS-006 使用锁定的 FormVersion 填写和校验。表单草稿、本地校验通过、FormSubmission 已提交和服务端已验证是不同状态。

### 6.4 资料采集

打开 IOS-006 按 EvidenceSlot 要求拍摄、选择或生成资料。文件已经拍摄、已经进入上传队列、已经完成上传、已经通过服务器验证和已经进入冻结集合必须分别表达。

### 6.5 客户或现场确认

使用项目定义的确认主体、确认内容和受控渠道。师傅不能代替客户确认，也不能用本人勾选伪造客户签字或授权。

### 6.6 现场操作

勘测、安装、维修、拆装或整改等业务操作由 `FieldOperation` 及其提交事实承载。一个 Visit 可以包含多个现场操作，客户端不能把整个 Task、Visit 或 Evidence Revision 冒充一个现场操作引用。

### 6.7 系统门禁

由服务器根据责任、预约、SLA、表单、资料、上传、审核或外部依赖计算。客户端只展示结论和恢复入口，不能手工勾选通过。

## 7. 步骤状态

普通用户界面使用以下中文状态：

- 待开始；
- 可以开始；
- 进行中；
- 本机已保存；
- 等待同步；
- 平台处理中；
- 平台已确认完成；
- 需要补充；
- 整改中；
- 已阻止；
- 发生冲突；
- 按项目规则不适用。

页面必须同时区分“权威状态”和“本机状态”。例如：

```text
平台状态：进行中
本机状态：表单已保存，3 张照片待上传
```

不得把“本机已保存”渲染成绿色完成对勾。只有服务器确认真实完成条件满足时，步骤才显示“平台已确认完成”。

“按项目规则不适用”必须由服务端基于锁定配置和业务输入计算，并保留依据；客户端不能提供任意跳过按钮。

## 8. 步骤顺序和并行关系

默认按照服务端执行计划展示顺序。项目规则可以允许部分步骤并行或任意顺序，但必须明确返回：

- 当前可以开始的步骤；
- 必须先完成的依赖；
- 并行步骤之间的提交关系；
- 条件步骤何时适用；
- 返回修改会影响哪些后续步骤；
- 哪些步骤一旦提交便不可编辑。

客户端禁止：

- 因前一步本地已填完就解锁下一步；
- 通过修改数组索引改变流程顺序；
- 在离线时自行判断条件步骤不适用；
- 用“全部展开”绕过步骤访问控制；
- 自动选择看起来最接近完成的步骤作为当前步骤。

存在多个可执行步骤时，服务端返回推荐下一动作及原因；用户可以选择其他同样允许的步骤，但界面必须明确其依赖和影响。

## 9. 当前步骤工作卡

当前步骤至少展示：

- 步骤名称和业务目的；
- 当前状态；
- 预计或规定完成时间；
- 安全说明；
- 操作指引；
- 必填表单摘要；
- 必需资料摘要；
- 需要的确认主体；
- 已有本机成果；
- 已有平台成果；
- 阻止原因；
- 主操作和异常入口。

固定底部主操作根据服务端或有效工作包 allowed-action 切换，例如：

```text
开始本步骤
继续填写
拍摄资料
查看待上传
完成本步骤检查
等待平台确认
处理缺失项
进入提交前检查
处理冲突
```

一次最多突出一个主操作和一个直接相关次操作。

## 10. 开始和继续现场操作

当步骤明确代表一个 `FieldOperation` 时，“开始本步骤”提交受控开始命令，并形成或恢复对应现场操作。

命令至少关联：

- 当前 Task 和 Visit；
- 当前责任师傅；
- 现场操作定义；
- 配置版本；
- 资源并发版本；
- 稳定 `deviceCommandId`；
- 业务发生时间和服务器接收时间。

重复点击或响应丢失后重试必须返回同一现场操作，不能重复创建。

已经存在可恢复的进行中操作时，页面显示“继续本步骤”；存在互斥或冲突操作时进入 IOS-010，不能由客户端选择一个覆盖另一个。

并非所有步骤都需要独立“开始”命令。纯表单、资料或确认步骤应直接进入其真实业务入口，禁止为了统一按钮而创建无意义的 FieldOperation。

## 11. 完成条件

步骤完成条件由服务端权威定义，可以组合：

- 指定检查项已经形成有效结果；
- 必填表单已经提交并满足当前版本校验；
- 必需 EvidenceSlot 已具有合格 Revision；
- 资料已经上传、Finalize 并通过必要验证；
- 指定确认主体已经完成确认；
- 现场操作已经形成不可变提交；
- 必要异常已经处理或获得批准；
- 前置步骤和系统门禁已经满足。

“本机校验通过”只是提前反馈，不等于服务端接受。服务器必须独立重验完成条件。

当前 Accepted 契约尚未提供从 Task、Snapshot 到真实已完成 FieldOperation `operationRefs` 的权威映射时，客户端不得生成占位引用，也不得开放依赖这些引用的 Visit 签退。

## 12. 草稿和返回修改

尚未提交的表单、检查和资料说明可以在项目政策允许时返回修改。

页面必须显示：

- 最近本机保存时间；
- 最近平台确认时间；
- 草稿基于的表单、资料和预填版本；
- 修改是否会使后续本机校验失效；
- 是否存在未同步依赖命令；
- 是否已经进入不可修改状态。

已提交、已冻结、已审核或已经被后续事实引用的版本不能原地修改。需要更正时必须通过 Correction、Revision 或明确的重新提交动作产生新版本。

禁止使用最后写入覆盖服务器事实，也禁止因条件字段隐藏就静默删除已有值。

## 13. 表单和资料摘要

IOS-005 只展示步骤级摘要，不在步骤列表内复制完整编辑器。

表单摘要展示：

- 已填写字段数和必填缺失数；
- 本地校验错误和警告；
- 本机保存或平台提交状态；
- 当前 FormVersion；
- 冲突或整改状态。

资料摘要展示：

- 必需槽位和已采集数量；
- 本机文件数量；
- 待上传、上传中、验证中和可用数量；
- 隔离、失败和冲突数量；
- 需要补拍或整改的项目。

点击摘要进入 IOS-006。上传和后台队列问题进入 IOS-008。

## 14. 安全步骤

涉及电气、登高、道路、地下空间或其他高风险作业时，安全步骤优先于普通业务进度。

安全步骤必须：

- 使用业务安全负责人批准的中文说明；
- 明确禁止条件和停止作业条件；
- 提供紧急联系或协调方式；
- 支持快速提交标准安全异常；
- 避免要求师傅在危险操作中持续注视或操作手机；
- 不以完成业务指标为由弱化安全阻断。

发现紧急危险时，页面应先提示停止作业并撤离风险区域，再提供最短异常上报路径；不能要求先完成冗长表单或拍摄危险照片。

## 15. 无法施工和中断

异常入口不能只是自由文本。

服务端根据当前步骤返回受控异常原因、必填说明、资料要求、责任对象、预约影响、SLA 影响和后续动作。当前已接受的在线 Visit 中断原因范围继续以服务端契约为准，例如 `SITE_UNSAFE`、`MATERIAL_MISSING`；客户端不能擅自新增或宽松接受未知原因码。

中断流程为：

```text
选择标准原因
→ 展示必须立即执行的安全动作
→ 补充必要说明和受控资料
→ 用户确认影响
→ 服务端校验当前责任、Visit 和版本
→ 创建权威中断或协调事实
→ 页面进入等待协调、改约或后续处理状态
```

中断不会自动完成 Task、工单或审核，也不会允许师傅自行改派网点、修改 SLA 或关闭异常。

离线中断只有工作包显式允许时才能排队；安全提示不依赖网络，但平台确认状态必须等服务端返回。

## 16. 离线执行

有效工作包中的执行计划必须冻结：

- Task、Visit 和责任版本；
- 履约配置版本；
- 步骤定义和依赖；
- 表单和资料定义及摘要；
- 预填快照；
- 允许的离线动作；
- 工作包有效期；
- 命令依赖规则。

每个离线成果使用稳定 `deviceCommandId`，保存 `capturedAt`、基础资源版本、配置摘要和依赖命令。

页面按以下层级表达：

```text
本机草稿
→ 本机命令已排队
→ 正在同步
→ 平台已接受
或
需要处理冲突
```

离线时客户端可以基于锁定规则提供即时校验，但不能宣布平台已完成。恢复网络后服务器逐条校验身份、责任、有效期、资源版本和依赖关系，不使用“最后上传覆盖服务器”。

离线期间后续步骤依赖前一步命令时，必须形成明确依赖图并按序提交。前置命令被拒绝时，后续命令停止，不能跳过失败依赖继续写入。

## 17. 杀进程和设备恢复

App 被挂起、杀进程或设备重启后，重新进入 IOS-005 必须恢复：

- 当前任务和步骤位置；
- 本机表单草稿；
- 资料本地引用；
- 上传队列；
- 离线命令及依赖；
- 最近服务器版本；
- 当前冲突和阻止状态。

恢复时先校验账号、设备隔离和工作包有效性，再展示敏感业务数据。不能因为恢复失败而删除未同步成果，也不能自动以新账号接管旧账号草稿。

本地库升级失败、文件缺失或 checksum 不一致时进入 IOS-009 或 IOS-010，并保留可审计错误；禁止创建空草稿伪装恢复成功。

## 18. 多设备和并发

同一师傅在另一设备、同一网点协同人员或平台可能同时改变任务事实。

页面必须检测：

- Task、Visit 或 FieldOperation 版本变化；
- 责任变化；
- 预约 Revision 变化；
- 表单预填或配置变化；
- 资料出现新 Revision；
- 审核或整改状态变化；
- 当前步骤已被其他权威动作推进。

存在本机未同步成果时，不得自动刷新后直接覆盖。IOS-010 并列展示本机成果、服务器事实、版本来源和允许动作。

已撤权师傅的旧命令不能以新责任或最近责任身份重放。

## 19. 提交前检查入口

当所有当前作业步骤达到可提交状态时，页面显示“进入提交前检查”。

进入 IOS-007 前仍需展示：

- 尚未平台确认的本机成果；
- 待上传或验证中的资料；
- 未解决警告和异常；
- 仍需客户或其他主体确认的内容；
- 当前责任、Visit 和工作包状态；
- 可能阻止提交的冲突。

IOS-005 不能因为列表全部显示绿色就直接完成 Task。IOS-007 必须重新执行跨表单、资料、Visit、责任和任务的完整检查。

## 20. 命令边界

IOS-005 可以调用：

- 当前步骤真实业务定义的开始或继续现场操作命令；
- 受控检查结果和确认命令；
- 服务端允许的中断、无法施工或协调命令；
- IOS-006、IOS-008 提供的正式表单、资料和上传命令；
- 有效工作包允许的对应离线命令。

IOS-005 不提供：

- 通用 `CompleteStep`；
- 通用 `SkipStep`；
- 通用 `ForceContinue`；
- 客户端自定义步骤；
- 手工修改流程状态；
- 修改配置版本；
- Visit 强制签退；
- 直接完成 Task 或工单。

每个业务写命令必须使用稳定幂等键、正确资源版本、受信身份和明确事务边界。

## 21. 页面状态和恢复动作

| 状态 | 页面表现 | 允许动作 |
|---|---|---|
| 可以开始 | 突出当前步骤目的和条件 | 开始本步骤 |
| 进行中 | 展示未完成成果 | 继续填写或采集 |
| 本机已保存 | 明确尚未平台确认 | 继续、查看同步 |
| 等待上传 | 展示文件和剩余量 | 进入 IOS-008 |
| 平台处理中 | 展示处理时间和来源 | 等待、刷新权威结果 |
| 平台已确认 | 展示完成时间 | 进入下一允许步骤 |
| 需要补充 | 列出精确缺失项 | 进入对应表单或资料 |
| 整改中 | 展示驳回对象和轮次 | 进入整改入口 |
| 安全阻断 | 优先显示停止作业要求 | 发起异常、联系协调 |
| 责任失效 | 隐藏敏感数据并停止写入 | 处理冲突 |
| 工作包过期 | 保留成果但禁止新离线动作 | 联网校验、处理冲突 |
| 数据不可用 | 不显示伪完成度 | 重试、查看原因 |

任何不可执行动作都必须解释原因和恢复方式，不能仅置灰。

## 22. 安全、隐私和审计

作业开始、检查、确认、中断和现场操作变更必须记录：

- 租户、项目、工单、Task、Visit 和现场操作；
- 当前责任网点和师傅；
- 操作者和受信 Context；
- 配置、步骤和资源版本；
- 业务动作、标准原因和稳定错误码；
- `capturedAt`、`receivedAt` 和服务器时间基准；
- 在线、离线和设备命令来源；
- 幂等结果、事务结果和后续事件。

日志使用中文事件说明和稳定英文字段，不记录客户完整信息、表单敏感值、资料正文、原始位置、Authorization 或设备秘密。

步骤页面遵守最小必要展示。与当前步骤无关的客户信息、其他师傅资料、内部审核意见和技术对象不进入普通界面。

## 23. 无障碍和现场交互

- 当前步骤、完成状态和阻止原因不能只依赖颜色；
- VoiceOver 能按业务顺序读取步骤名称、状态、缺失项和主操作；
- 动态字体放大后不隐藏安全说明和异常入口；
- 手套、强光和单手场景下主要按钮仍易于操作；
- 长步骤列表保持当前步骤定位，但不使用自动跳转造成误操作；
- 网络或保存处理中提供明确状态，避免重复点击；
- 危险操作中不要求持续触屏或观看动画；
- 触觉反馈只作为辅助，不能代替文字确认。

## 24. 与其他页面的关系

```text
IOS-002 任务详情
    → IOS-004 到场与开始作业
    → IOS-005 作业步骤
        → IOS-006 表单与资料采集
        → IOS-008 上传中心
        → IOS-009 离线恢复
        → IOS-010 同步冲突
    → IOS-007 提交前检查
```

IOS-005、Network Portal 和 Admin 读取同一 Task、Visit、FieldOperation、FormSubmission、Evidence 和异常事实，不建立端侧平行流程。

## 25. 验收标准

### 步骤与状态

- [ ] 步骤来自工单绑定的履约配置和页面级读模型；
- [ ] 步骤条不承担第二套流程状态机；
- [ ] 权威完成和本机保存具有不同文字与视觉语义；
- [ ] 条件步骤不适用由服务端计算，客户端不能任意跳过；
- [ ] 每个完成状态可以追溯到真实表单、资料、确认或现场操作事实。

### 执行和异常

- [ ] 当前步骤只展示服务端或有效工作包允许的动作；
- [ ] 重复开始命令不会创建重复 FieldOperation；
- [ ] 安全风险优先提示停止作业；
- [ ] 中断使用受控原因、说明和资料要求；
- [ ] 中断不会自动完成 Task 或工单；
- [ ] 没有真实 operationRefs 时不开放 Visit 签退。

### 离线和恢复

- [ ] 普通缓存不授权离线作业；
- [ ] 离线成果显示“本机”或“等待同步”，不显示平台完成；
- [ ] 命令依赖顺序明确，前置失败会阻止后续提交；
- [ ] 杀进程后草稿、资料、上传和命令队列可以恢复；
- [ ] 工作包过期或责任变化时保留成果并停止新动作；
- [ ] 多设备冲突不使用最后写入覆盖。

### 导航和提交

- [ ] 复杂表单和资料进入 IOS-006，不塞入步骤列表；
- [ ] 上传问题进入 IOS-008；
- [ ] 冲突进入 IOS-010；
- [ ] 所有步骤可提交后仍必须进入 IOS-007 完整检查；
- [ ] 页面不存在通用完成、跳过或强制继续命令。

### 安全与可用性

- [ ] 普通界面不显示内部 ID、英文枚举和原始技术异常；
- [ ] 敏感表单值和资料正文不进入日志；
- [ ] VoiceOver 和动态字体可读取当前步骤、缺失项和安全阻断；
- [ ] 应用被挂起、杀进程或弱网时不丢失已保存成果；
- [ ] 无法获得真实数据时明确失败，不伪造完成度。

## 26. 禁止实现

禁止：

- 客户端维护独立流程状态机；
- 点击“下一步”直接把步骤设为完成；
- 本机表单校验通过即显示平台完成；
- 文件拍摄完成即显示资料可用；
- 任意跳过必需步骤；
- 用自由文本替代结构化检查和异常原因；
- 通过 Task、Snapshot 或 Evidence URI 伪造 FieldOperation 引用；
- 为统一页面而给所有步骤创建空 FieldOperation；
- 离线命令失败后继续提交依赖命令；
- 责任变化后静默把本机成果转给新师傅；
- 捕获异常后显示默认成功或空步骤；
- 为旧 H5、旧开发数据或假设客户端保留双轨步骤模型。
