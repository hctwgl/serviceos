# IOS-009 离线恢复

状态：Accepted
设计等级：A

## 1. 页面定位

“离线恢复”是原生 iOS 师傅端管理离线工作包、本机草稿、离线命令、同步进度和设备恢复的统一工作区。

它必须让师傅清楚回答：

1. 哪些任务已经安全保存到本机，可以在无网环境继续；
2. 哪些现场成果仍只存在当前设备；
3. 哪些操作已经提交平台确认，哪些仍在等待同步；
4. App 被挂起、杀进程或升级后，草稿、文件和队列是否完整恢复；
5. 当前阻塞来自网络、命令依赖、权限变化、工作包过期还是业务冲突；
6. 责任撤销、账号退出或设备撤销后，本机敏感数据如何隔离；
7. 哪些问题可以自动恢复，哪些必须进入 IOS-010 由用户处理。

离线恢复必须始终区分五类事实：

```text
平台已确认业务事实
已校验的 WorkPackage 快照
本机草稿与文件
等待平台确认的 OfflineCommand
平台返回的收据、拒绝和冲突
```

本机保存成功不等于平台业务操作成功；命令已经发送不等于平台已经接受；网络恢复不等于所有内容已经同步完成。

## 2. 产品边界

IOS-009 负责：

- 下载、校验、保存、刷新和失效 WorkPackage；
- 使用加密本地数据库保存任务快照、草稿、文件引用和同步队列；
- 在断网、App 挂起、杀进程、设备重启和 App 升级后恢复本机状态；
- 创建带稳定 `deviceCommandId` 的 OfflineCommand；
- 按依赖顺序同步命令并保存逐条平台收据；
- 展示等待网络、依赖阻塞、发送中、平台处理中、已接受、被拒绝和冲突；
- 处理部分成功，不回滚已经被平台接受的真实业务事实；
- 优先同步责任撤销、授权变化和任务 tombstone；
- 在账号切换、退出、设备撤销或责任变化时隔离未同步成果；
- 将需要业务选择或人工合并的问题导向 IOS-010；
- 为 IOS-001 至 IOS-008 提供一致的本机同步摘要。

IOS-009 不负责：

- 在客户端重新实现平台状态机；
- 根据本机页面状态推导 `allowed-actions`；
- 用本机时间决定业务事实先后顺序；
- 在冲突时使用最后写入覆盖；
- 将旧师傅的本机成果自动转交给新师傅；
- 绕过当前责任、设备注册、Tenant、Project Scope 或 Capability；
- 用 H5 缓存模拟原生离线能力；
- 将上传字节进度与命令同步状态合并；
- 在退出账号或数据库异常时静默删除未同步成果；
- 猜测 WorkPackage、OfflineCommand 或签名的最终机器字段；
- 声称当前仓库已经实现本规格描述的原生离线运行时。

## 3. 当前工程事实与设计目标

本规格定义正式原生 iOS 客户端的产品目标，不改变当前工程事实：

- Technician feed 和 tombstone 已有独立设计与在线能力；
- H5 只提供在线受限体验，不承诺 WorkPackage 或 OfflineCommand；
- 原生 WorkPackage 下载、签名校验、加密本地数据库、离线命令执行器和冲突解析仍需后续工程里程碑实现；
- 精确 OpenAPI、Schema、字段名、签名算法、密钥分发和数据库迁移必须在工程设计和安全评审中确定；
- 未形成机器契约前，客户端与服务端不得各自根据本文示意字段实现不一致协议。

## 4. 入口与导航

离线恢复不默认占用一级导航。入口包括：

- IOS-001 今日任务顶部的网络与待同步摘要；
- IOS-002 任务详情的本机保存状态；
- IOS-005 作业步骤的离线提示；
- IOS-006 表单与资料采集的本机保存提示；
- IOS-007 提交前检查的等待同步项；
- IOS-008 上传中心的跨队列依赖提示；
- 网络恢复、工作包过期或责任撤销通知；
- “我的”中的“本机数据与同步”。

从任务入口进入时默认过滤当前 Task；从全局入口进入时按任务分组，但只能展示当前账号在当前设备分区中的本机记录。

## 5. 页面结构

```text
网络、账号、设备和最近同步摘要
→ 需要立即处理
→ 按任务分组的待同步命令
→ 等待依赖和等待网络
→ 正在同步与平台处理中
→ 最近平台确认
→ WorkPackage 状态
→ 冲突、拒绝和隔离成果
→ 本机数据库、存储和恢复状态
```

默认优先展示：

1. 责任撤销、设备撤销和账号隔离；
2. 阻止现场继续或阻止任务提交的问题；
3. WorkPackage 过期或完整性失败；
4. 需要进入 IOS-010 的冲突；
5. 依赖失败导致的命令链阻塞；
6. 等待网络和普通待同步项；
7. 最近完成的同步收据。

## 6. 页面级读模型

离线恢复页面由本机持久化状态和服务端安全摘要共同组成：

```text
OfflineRecoveryView
├── accountAndDeviceSummary
├── connectivitySummary
├── localDatabaseSummary
├── attentionItems[]
├── taskSyncGroups[]
│   ├── safeTaskSummary
│   ├── workPackageSummary
│   ├── pendingCommandSummary
│   ├── uploadDependencySummary
│   └── allowedLocalActions
├── recentReceipts[]
├── tombstoneAndRevocationSummary
├── conflictSummary
└── recoveryActions
```

这里的字段名称是产品语义示意，不是已经接受的机器契约。后续 OpenAPI、本地 Schema 和客户端模型必须引用唯一的版本化契约。

## 7. WorkPackage 的作用

WorkPackage 是平台为当前师傅、当前任务和当前设备签发的最小离线工作快照。它不是完整工单副本，也不是客户端可修改的业务对象。

WorkPackage 至少要表达以下语义：

- Tenant、Project、Task 和当前师傅责任绑定；
- 工作包版本和配置摘要；
- 当前可离线执行的动作；
- 表单、资料槽位、校验规则和必要预填快照；
- 任务执行所需的最小客户与现场信息；
- 责任、授权和配置基线版本；
- 业务时区、可信服务器时间基准和有效期；
- 客户端最低能力或 Schema 兼容要求；
- 内容摘要和完整性证明。

WorkPackage 不得默认包含完整工单历史、无关客户信息、平台内部 ID 集合、其他师傅信息、技术诊断对象或永久下载凭据。

## 8. WorkPackage 下载与校验

下载前服务端必须实时校验：

- 当前认证、设备注册和受信 Technician Context；
- ACTIVE TechnicianProfile 和组织关系；
- 当前 Task 责任；
- Project Scope 和 Capability；
- 当前配置及客户端支持范围；
- 设备撤销、账号风险和工作包政策。

客户端保存前必须校验：

- 内容完整性和签名；
- WorkPackage Schema 版本；
- 当前 App 是否支持该版本；
- 账号、设备和任务绑定；
- 配置摘要与资源摘要；
- 有效期和可信时间基准；
- 必要本机空间与数据库状态。

任何未知 Schema、摘要不一致、签名失败、账号不匹配或设备不支持都必须失败关闭。不能回退读取旧字段、忽略未知结构或偷偷使用上一次缓存。

## 9. WorkPackage 状态

用户可见状态包括：

- 正在准备；
- 可下载；
- 正在下载；
- 已下载并校验；
- 等待刷新；
- 已过期；
- 已被责任撤销；
- 当前 App 不支持；
- 完整性校验失败；
- 本机空间不足；
- 本机数据库不可用。

“已下载”只有在完整内容落盘、数据库事务提交和完整性校验全部成功后才能显示。

## 10. WorkPackage 更新与过期

一个已经校验的 WorkPackage 是不可变快照。平台发布新配置、任务发生变化或工作包即将过期时，只能创建或下载新版本，不能原地修改旧快照。

刷新必须先比较：

- 本机是否存在未同步草稿；
- 是否存在基于旧版本创建的 OfflineCommand；
- 是否存在未完成上传；
- 责任和授权是否仍然有效；
- 新旧表单、槽位和动作是否兼容。

WorkPackage 过期后：

- 不允许创建新的离线业务命令；
- 已保存草稿和本机文件继续安全保留；
- 已经创建的命令可以在联网后提交，由服务端依据采集时间、有效窗口、当前责任和当前业务状态决定是否接受；
- 客户端不能仅凭“操作发生在过期前”自行显示平台已接受；
- 无法刷新时明确提示“需要联网刷新工作包”。

## 11. 本机数据模型边界

本机离线运行时至少需要区分以下概念：

```text
WorkPackage
LocalDraft
LocalFile
UploadQueueItem
OfflineCommand
CommandDependency
ServerReceipt
TaskTombstone
ConflictRecord
```

这些名称表达领域职责，不预先决定 Core Data、SQLite 或其他数据库的表名。实现必须建立唯一 Schema，禁止为旧 H5、旧实验数据或早期草稿保留双轨表和回退读取。

## 12. 加密本地数据库

本机数据库必须：

- 使用系统安全能力保护加密密钥；
- 按账号、Tenant、Project 和设备建立明确隔离；
- 拥有显式 Schema 版本；
- 使用事务保存 WorkPackage、草稿、命令和依赖；
- 在崩溃或杀进程后恢复到最后一个完整事务；
- 不把敏感业务数据放入 UserDefaults、普通日志或未保护缓存；
- 对锁屏、设备备份、文件保护等级和系统数据保护策略进行安全评审。

数据库损坏、密钥不可用或迁移失败时，不得静默新建空库并删除原数据。页面必须进入受控恢复状态，说明本机未同步成果是否仍可恢复以及下一步支持路径。

## 13. App 重启恢复

App 启动后按以下顺序恢复：

```text
确认账号和设备安全分区
→ 打开并迁移本机数据库
→ 校验未完成 WorkPackage
→ 恢复 LocalDraft 和 OfflineCommand
→ 对账 IOS-008 后台传输任务
→ 恢复命令依赖图
→ 联网时拉取 tombstone 和权限变化
→ 查询发送结果未知的命令
→ 继续允许的同步
```

恢复过程中不能因为 UI 尚未加载就创建重复命令，也不能把内存中不存在的对象当作已经丢失。

## 14. 离线动作来源

客户端只允许执行 WorkPackage 明确列出的离线动作。可见按钮、表单步骤和提交入口不能根据本机状态自行推导。

创建 OfflineCommand 前必须确认：

- WorkPackage 仍在允许创建动作的有效期内；
- 动作属于 `allowedOfflineActions`；
- 当前本机草稿已成功保存；
- 必要命令依赖和文件引用完整；
- 当前账号、设备和任务分区一致；
- 本机没有已知 tombstone 或撤权事实。

## 15. OfflineCommand 语义

每个 OfflineCommand 至少要表达以下语义：

- 稳定且不可重建的 `deviceCommandId`；
- 当前设备注册和账号分区；
- 目标业务对象和动作类型；
- 用户实际操作时间和可信服务器时间偏移；
- 创建命令时的责任、业务对象和配置基线版本；
- 必要的最小业务载荷；
- 前置命令依赖；
- 当前尝试次数和最后结果；
- 内容摘要，防止同一命令 ID 被替换成不同载荷。

精确字段、编码和签名方式由后续机器契约定义。客户端不得把 tenant、operator、权限结果或“已验证”状态作为可信事实提交给服务端。

## 16. 命令状态

本机命令状态包括：

- 本机草稿已保存；
- 已准备，等待依赖；
- 等待网络；
- 可以发送；
- 正在发送；
- 等待平台处理结果；
- 平台已接受；
- 可重试失败；
- 被业务规则拒绝；
- 进入冲突处理；
- 因责任撤销被隔离；
- 已由用户明确放弃且保留审计依据。

只有收到可验证的平台收据后，才允许显示“平台已接受”。HTTP 请求成功、请求体已经写出或本地队列删除都不能替代平台确认。

## 17. 命令依赖图

离线命令使用明确的有向无环依赖关系，例如：

```text
开始到场
→ 保存现场操作
→ 保存表单草稿
→ 创建资料上传会话
→ 完成上传与 Finalize
→ 等待资料验证
→ 创建提交快照
→ 完成当前任务
```

依赖规则：

- 依赖图不得存在循环；
- 只有所有强依赖被平台接受后，下游命令才能发送；
- 某个依赖被拒绝时，下游命令进入“前置操作未完成”，不能继续重试；
- 相互独立的命令分支可以继续；
- IOS-008 文件传输可以并行，但需要平台文件结果的业务命令必须等待；
- 同一领域动作只保留一个稳定命令，不能为了重试创建新 ID。

## 18. 同步触发

同步可以由以下事件触发：

- 用户点击“立即同步”；
- App 进入前台；
- 网络从不可用恢复；
- iOS 在系统允许的后台时机唤醒 App；
- IOS-008 产生关键上传结果；
- 用户完成一个可离线动作；
- 工作包接近过期并且当前网络可用。

系统后台机会不是实时保证。页面不得承诺“退出 App 后一定立即完成全部同步”，也不得使用无上限轮询消耗电量和流量。

## 19. 同步顺序

每次同步遵循：

```text
校验认证、账号、设备和客户端能力
→ 校准服务器时间并确认同步游标
→ 优先拉取 tombstone、撤权和责任变化
→ 拉取当前 Task、版本和 WorkPackage 摘要
→ 查询发送结果未知的命令
→ 重新计算可发送命令
→ 按依赖顺序发送 OfflineCommand
→ 保存逐条平台收据
→ 拉取命令产生的最新平台事实
→ 原子更新本机状态和页面摘要
```

先处理撤权和责任变化，是为了避免旧责任继续提交业务命令。客户端不能先批量发送旧命令，再在最后检查 tombstone。

## 20. 同步批次与部分成功

一次网络请求或同步批次不代表一个跨命令业务事务。平台必须为每个命令返回独立结果。

例如：

```text
命令 A：平台已接受
命令 B：版本冲突
命令 C：依赖 B，尚未发送
命令 D：与 B 无依赖，平台已接受
```

客户端必须：

- 保留 A 和 D 的平台确认事实；
- 不回滚或用新命令重放 A、D；
- 将 B 导向 IOS-010；
- 将 C 标记为“等待前置冲突处理”；
- 展示批次部分完成，而不是整体成功或整体失败；
- 保存每条命令的平台收据和处理时间。

## 21. 幂等与未知结果恢复

`deviceCommandId` 在命令创建时生成，并在重试、App 重启、网络切换和查询结果时保持不变。

如果请求已经发送但响应丢失：

- 命令进入“平台结果未知”；
- 网络恢复后优先按原 `deviceCommandId` 查询或重试；
- 平台已经接受时返回同一业务结果；
- 不能创建新 ID 猜测重发；
- 同一 ID 但载荷摘要不同必须被拒绝并进入安全诊断。

## 22. 时间与顺序

客户端记录用户实际操作时间，但设备时间不能决定业务状态先后。

服务端处理命令时必须区分：

- `capturedAt`：用户实际执行时间；
- `receivedAt`：平台收到时间；
- 服务端当前时间；
- 工作包有效窗口；
- 责任和业务对象版本。

平台以状态机、版本和服务端规则决定是否接受。客户端不能通过修改设备时间绕过 WorkPackage 过期、预约窗口、SLA 或责任变化。

## 23. 自动重试

可自动重试的情况包括受控网络中断、短暂平台不可用和可恢复传输错误。

自动重试必须：

- 保持同一命令 ID 和载荷摘要；
- 使用有上限的退避；
- 记录尝试次数和最近失败；
- 尊重蜂窝、低数据和后台政策；
- 达到上限后进入需要用户处理。

责任变化、授权拒绝、业务状态冲突、配置不兼容、命令载荷非法和未知 Schema 不属于自动重试问题。

## 24. Tombstone 与责任撤销

Task feed 返回 tombstone、责任撤销或设备撤权时，客户端必须优先：

1. 停止创建新的业务命令；
2. 停止刷新 WorkPackage 和上传授权；
3. 将现有 WorkPackage 标记失效；
4. 隐藏客户、地址、照片缩略图等敏感内容；
5. 隔离未同步草稿、文件和命令；
6. 取消尚未开始的后台授权操作；
7. 向平台查询已发送命令的真实结果；
8. 将需要人工决定的本机成果导向 IOS-010 或受控支持流程。

feed tombstone 可以只携带任务标识和失效原因。客户端不能要求 tombstone 包含完整工单数据，也不能继续使用旧 WorkPackage 补齐敏感信息。

## 25. 改派后的本机成果

改派后：

- 旧师傅不能继续提交、Finalize 或刷新授权；
- 旧师傅已经被平台接受的历史事实继续保留；
- 旧师傅未同步成果不能自动成为新师傅的草稿；
- 新师傅必须下载自己的 WorkPackage；
- 旧师傅本机成果只能按安全政策隔离、受控导出诊断或在授权流程中处理；
- 服务端在任何写入、审计和 Outbox 之前重新校验当前责任。

客户端不得把“本机操作发生得更早”解释为拥有覆盖当前责任的权利。

## 26. 账号切换、退出与设备撤销

账号切换时：

- 新账号不能读取旧账号的任务、文件、缩略图、队列或错误详情；
- 旧账号分区保持锁定；
- 只有重新验证同一账号和设备策略后才能恢复；
- 未同步成果不能搬运到新账号。

用户退出时，页面必须说明未同步内容数量和后果。退出不等于静默删除；是否保留加密隔离数据、何时清理以及是否需要在线确认，由设备与安全政策决定。

设备被撤销时必须停止所有新同步和授权刷新。本机密钥轮换、分区销毁和受控恢复需要独立安全实现与测试，不能只清空 UI 状态。

## 27. App 升级与数据库迁移

App 升级前后的本机迁移必须：

- 识别现有 Schema 和未同步数据量；
- 在事务或可恢复迁移中升级；
- 保留 WorkPackage、草稿、文件引用、命令 ID、依赖和平台收据；
- 迁移失败时回到可诊断状态，不能打开空库继续运行；
- 对关键版本提供带未同步数据的自动化迁移测试；
- 不用永久兼容旧 Schema 替代一次性迁移；
- 不支持的 WorkPackage 版本明确要求更新 App 或重新下载。

## 28. 本机数据库或文件异常

异常分类包括：

- 数据库暂时不可打开；
- 加密密钥不可用；
- 数据库完整性失败；
- 文件引用存在但文件缺失；
- 文件存在但摘要不一致；
- 命令依赖记录损坏；
- 平台收据与本机命令无法关联；
- 本机空间不足。

处理原则：

- 停止产生新的不可恢复写入；
- 不把异常记录当作已同步；
- 不自动删除整个账号分区；
- 尽可能保留加密原始证据和诊断摘要；
- 提供明确的恢复、重新采集、进入冲突或联系支持动作；
- 日志不记录文件正文、客户信息、密钥或绝对路径。

## 29. IOS-008 上传队列协作

文件上传与业务命令同步是两个独立但有依赖的运行时：

```text
IOS-008 上传中心
    负责文件字节、Finalize、扫描和资料状态

IOS-009 离线恢复
    负责 WorkPackage、OfflineCommand、依赖和平台收据
```

OfflineCommand 可以依赖 IOS-008 的平台文件结果，但不能把后台传输任务直接当作业务命令成功。上传冲突进入 IOS-010；网络和后台传输恢复仍回到 IOS-008。

## 30. IOS-010 冲突边界

IOS-009 只负责识别、冻结和路由冲突，不负责自动合并业务内容。

以下情况进入 IOS-010：

- 责任网点或师傅已变化；
- WorkPackage、表单、槽位或配置版本不兼容；
- 同一字段在多设备产生不同修改；
- 预填版本已经变化；
- 文件、摘要、Revision 或上传分片冲突；
- 服务端业务对象已经前进到不允许当前命令的状态；
- 同一命令 ID 出现不同载荷；
- 用户必须选择保留、重录、放弃或申请人工处理。

可以通过幂等查询直接确认“平台已接受”的，不应误报为冲突。

## 31. 用户可见状态与动作

| 场景 | 页面表达 | 主要动作 |
|---|---|---|
| 已离线且工作包有效 | 已安全保存到本机 | 继续作业 |
| 本机有未同步成果 | 显示命令和文件数量 | 查看、等待网络 |
| 网络恢复 | 正在核对责任和平台状态 | 等待、查看详情 |
| 等待前置命令 | 说明具体依赖 | 查看前置操作 |
| 平台结果未知 | 已发送，正在确认结果 | 重新查询 |
| 部分成功 | 分别列出已接受和未完成项 | 处理剩余问题 |
| 工作包过期 | 不能创建新离线操作 | 联网刷新 |
| 当前 App 不支持 | 无法安全读取工作包 | 更新 App |
| 责任撤销 | 已停止同步并隔离本机成果 | 查看原因 |
| 数据冲突 | 本机成果未被覆盖 | 进入 IOS-010 |
| 数据库恢复失败 | 本机成果需要受控恢复 | 重试、联系支持 |
| 全部完成 | 平台已确认全部命令 | 返回任务 |

禁止只显示“同步失败”“网络异常”或内部异常类名。

## 32. 通知

允许通知：

- 网络恢复并开始同步；
- 同步全部完成；
- 存在需要处理的冲突；
- WorkPackage 即将过期；
- 本机空间不足影响保存；
- 责任撤销导致本机成果隔离。

锁屏通知不得显示客户姓名、电话、地址、任务详情、照片缩略图、文件名或冲突载荷。

## 33. 命令与查询边界

后续机器契约至少需要覆盖以下语义，不以本文名称作为最终 API 承诺：

- 下载或刷新 WorkPackage；
- 查询 Task feed、tombstone 和同步游标；
- 提交一组带独立结果的 OfflineCommand；
- 按 `deviceCommandId` 查询幂等结果；
- 查询任务当前版本、责任和授权摘要；
- 确认客户端已经处理 tombstone；
- 查询需要进入 IOS-010 的冲突详情；
- 受控撤销设备或账号分区。

服务端必须从受信身份重新解析 tenant、project、operator、设备和责任，不能接受客户端把这些值作为权威事实。

## 34. 安全、隐私、日志和审计

服务端对 WorkPackage 下载、命令接受、拒绝、幂等命中、责任变化和设备撤销记录：

- 租户、Project、Task 和业务对象；
- 当前责任网点和师傅；
- 设备注册和受信 Context；
- WorkPackage、配置、责任和业务对象版本；
- `deviceCommandId`、命令类型和载荷摘要；
- `capturedAt`、`receivedAt` 和处理时间；
- 接受、拒绝、冲突和幂等结果；
- 审计、Outbox 和业务结果引用。

客户端日志和崩溃报告不得包含：

- WorkPackage 正文；
- 客户信息、详细地址和现场资料；
- OfflineCommand 业务载荷；
- 本机绝对路径、数据库密钥和 Keychain 内容；
- Bearer Token、上传凭据或签名秘密；
- 照片摘要以外的文件正文。

## 35. 无障碍和移动交互

- 离线、等待、冲突、隔离和完成不能只依赖颜色；
- VoiceOver 能读取任务、同步数量、阻塞原因和主要动作；
- 动态字体放大后保留任务分组和状态来源；
- 同步进度变化不过度播报，完成和需要处理时重点通知；
- 大队列更新时保持焦点，不因后台回调跳到列表顶部；
- “放弃本机成果”“退出账号”等高风险动作必须明确确认；
- 页面使用自然中文说明网络、责任、版本和恢复问题。

## 36. 与其他页面的关系

```text
IOS-001 今日任务
    → 展示 WorkPackage 和待同步摘要

IOS-002 至 IOS-007
    → 产生本机草稿、文件和 OfflineCommand

IOS-008 上传中心
    → 恢复文件上传和平台文件结果

IOS-009 离线恢复
    → 恢复命令依赖和平台收据
    → 冲突进入 IOS-010
```

Admin、Network Portal 和 iOS 读取同一平台业务事实。只有 WorkPackage、本机草稿、文件队列和未确认命令属于当前设备。

## 37. 验收标准

### WorkPackage

- [ ] 只有当前有权师傅和受信设备可以下载；
- [ ] 内容最小化、版本锁定并具有完整性校验；
- [ ] 未知 Schema、签名失败和摘要不一致失败关闭；
- [ ] 工作包过期后不能创建新离线命令；
- [ ] 刷新不会静默覆盖未同步草稿和文件；
- [ ] tombstone 到达后旧工作包立即失效。

### 本机恢复

- [ ] 断网、挂起、杀进程和设备重启后草稿、文件和命令可恢复；
- [ ] 本机数据库使用系统安全密钥和明确账号分区；
- [ ] App 升级迁移保留未同步数据、命令 ID、依赖和收据；
- [ ] 数据库损坏或迁移失败时不静默建立空库；
- [ ] 账号切换不会泄露另一账号的任务和资料。

### 命令、依赖和幂等

- [ ] 每个 OfflineCommand 使用稳定 `deviceCommandId`；
- [ ] 重试、重启和未知结果查询保持同一 ID 和载荷摘要；
- [ ] 命令只在强依赖被平台接受后发送；
- [ ] 依赖失败不会无限重试下游命令；
- [ ] 相同命令重试返回同一业务结果，不重复产生事实；
- [ ] 同一命令 ID 的不同载荷被明确拒绝。

### 同步真实性

- [ ] 同步前优先处理 tombstone、撤权和责任变化；
- [ ] 每条命令拥有独立平台结果；
- [ ] 部分成功不会回滚已经接受的事实；
- [ ] 请求发出或 HTTP 成功不会显示业务已完成；
- [ ] 无网络、平台不可用和业务冲突分别表达；
- [ ] 自动重试有上限、退避和可解释状态。

### 责任与冲突

- [ ] 改派后旧师傅不能继续提交或刷新授权；
- [ ] 旧师傅未同步成果不会自动转交新师傅；
- [ ] 服务端在业务写入前重新校验当前责任；
- [ ] 冲突不使用最后写入覆盖；
- [ ] 幂等接受不误报为冲突；
- [ ] 需要用户选择的问题进入 IOS-010。

### 安全与隐私

- [ ] WorkPackage、命令载荷和本机文件不进入日志；
- [ ] 退出账号或设备撤销不会无提示删除未同步成果；
- [ ] 锁屏通知不泄露客户和现场信息；
- [ ] 本机密钥、数据库和文件保护经过安全测试；
- [ ] H5 不伪装支持 WorkPackage 和 OfflineCommand。

## 38. 禁止实现

禁止：

- 用网页缓存或 UserDefaults 代替加密本地数据库；
- 在 ViewModel 中实现离线命令状态机；
- 根据页面状态推导离线允许动作；
- WorkPackage 校验失败后使用旧缓存兜底；
- 未知 Schema 时忽略字段继续执行；
- 为重试生成新的 `deviceCommandId`；
- 请求发送后立即删除本机命令；
- 用一个“同步成功”覆盖逐条命令结果；
- 部分失败时回滚已经被平台接受的业务事实；
- 使用最后写入覆盖服务端或其他设备修改；
- 先发送全部旧命令，再检查责任撤销；
- 改派后自动把旧师傅草稿交给新师傅；
- 退出账号、数据库损坏或空间不足时静默删除成果；
- 修改设备时间绕过有效期、预约或 SLA；
- 将后台机会描述成无限可靠执行；
- 为旧 H5、旧开发数据或假设客户端保留第二套离线模型；
- 在机器契约未评审前，把本文示意字段当作已接受协议；
- 将本规格标记为当前运行时已经 Implemented。
