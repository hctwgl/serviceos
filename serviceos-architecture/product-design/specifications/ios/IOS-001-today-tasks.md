# IOS-001 今日任务

状态：Accepted
设计等级：A

## 1. 页面定位

“今日任务”是原生 iOS 师傅端的默认首页，用于让师傅在打开 App 后立即知道：

1. 现在最应该处理哪一项任务；
2. 今天有哪些已预约、待联系、进行中或待整改任务；
3. 哪些本地成果尚未上传或同步；
4. 当前看到的是服务器最新结果，还是截至某个时间的本机快照；
5. 下一步应进入哪个正式作业页面。

它不是后台工单列表的移动版，不展示完整工单字段，也不在首页直接跨越联系、预约、到场、表单、资料和提交等业务步骤。

## 2. 产品边界

IOS-001 只负责：

- 展示当前师傅有权处理的今日任务；
- 展示进行中任务、紧急风险、整改和待同步摘要；
- 合并服务器任务快照和本机运行状态；
- 根据服务端 allowed-actions 或有效工作包动作清单提供下一入口；
- 在改派、撤权、离线、数据过期和冲突时给出明确状态；
- 深链到任务详情、同步中心、整改或消息。

它不负责：

- 分配或改派责任师傅；
- 编辑项目流程或业务规则；
- 在首页直接完成到场、表单、拍摄、整改或提交；
- 用本机状态覆盖服务器责任和任务状态；
- 在没有有效工作包时伪造离线可执行能力；
- 展示其他师傅或其他网点任务。

## 3. 目标用户与上下文

目标用户是正式参与现场履约并已完成设备绑定的师傅。

页面数据必须同时满足：

```text
authenticated principal
+ ACTIVE TechnicianProfile
+ ACTIVE NetworkTechnicianMembership
+ trusted X-Technician-Context
+ current task or service responsibility
+ required capability
```

客户端不能通过请求参数自报网点、师傅或项目范围。用户存在多个有效网点关系时，只能通过服务端返回的可信上下文切换；切换上下文后必须重新加载任务、清理不再授权的敏感缓存，并隔离尚未同步的本地成果。

## 4. 今日口径

“今日”不能仅由设备本地时间判断。

服务端 Feed 必须返回：

- 业务日期；
- 业务时区；
- 快照生成时间；
- 任务进入今日列表的原因；
- 服务端优先级和优先级说明。

今日列表至少包含：

- 当前正在执行且尚未结束的任务；
- 今日预约任务；
- 今日需要联系或确认预约的任务；
- 今日到期或已经超时的任务；
- 今日需要处理的整改；
- 今日待提交或等待服务端确认的任务；
- 今日完成的近期任务摘要。

不属于今日但存在紧急 SLA、有效整改或服务端明确风险的任务，可以进入“紧急处理”区，但必须显示进入原因。

设备时区与任务业务时区不一致时，页面显示业务时区，不得静默按照设备时间重新分组。

## 5. 页面结构

```text
企业蓝品牌栏 + 消息入口
→ 身份、环境和同步可信度
→ 当前进行中任务
→ 今日概览
→ 紧急处理
→ 今日任务分组
→ 待上传与待同步提示
→ 底部导航
```

底部导航使用已经接受的四项：

1. 今日任务；
2. 工单；
3. 消息；
4. 我的。

上传队列和同步中心通过状态条、任务卡或“我的”进入，不默认占用一级导航。

## 6. 顶部品牌栏

顶部展示：

- “今日任务”；
- 当前环境标识，非生产环境必须醒目；
- 未读业务消息数量；
- 当前是否离线；
- 最近成功同步时间。

不展示：

- 租户 ID；
- 网点 ID；
- Capability 数量；
- 客户端内部版本字段；
- 原始 Scope 或 Context 字符串；
- 调试入口。

存在多个有效网点上下文时，显示可理解的网点名称。技术标识只允许进入受控诊断信息。

## 7. 同步可信度状态条

首页必须持续区分：

- 在线，任务已刷新；
- 正在刷新；
- 离线，显示本机快照；
- 有待同步命令；
- 有待上传资料；
- 存在同步冲突；
- 任务数据已过期；
- 部分来源不可用；
- 账号或设备授权失效。

示例文案：

```text
已同步 · 09:42
离线 · 显示截至 09:42 的任务
本机已保存 3 项 · 等待同步
2 个文件上传中 · 可继续填写
1 项责任冲突需要处理
```

不能使用一个“同步成功”同时表达本地保存、文件上传和服务端业务确认。

## 8. 当前进行中任务

存在已经开始但尚未结束的现场任务时，首页顶部只突出一张主任务卡。

主任务卡至少展示：

- 工单号；
- 任务标题；
- 当前步骤；
- 已执行时长；
- SLA；
- 本地保存、上传和同步状态；
- 当前风险；
- “继续作业”主操作。

如果异常数据导致同时存在多个“进行中”现场任务，客户端不能自行选一个隐藏其他任务。页面必须进入冲突状态，展示所有冲突任务并要求通过同步或平台协助解决。

## 9. 今日概览

概览展示：

- 今日任务；
- 紧急任务；
- 待联系或待预约；
- 待提交；
- 待上传或待同步；
- 待整改；
- 今日已完成。

数字来自服务端 Feed 与本机同步运行时的明确组合，并显示截至时间。

服务端任务来源不可用时，服务端相关数字显示“暂不可用”；本机队列仍可展示本机真实数量，但必须标记为本机数据。不能把来源失败显示为零。

“完成率”只有在服务端提供明确分母和口径时才展示，客户端不得使用当前页面卡片数量自行计算。

## 10. 今日任务分组

页面按业务任务分组，而不是按数据库状态分组：

1. 进行中；
2. 紧急处理；
3. 即将上门；
4. 待联系与预约；
5. 待继续作业；
6. 待提交；
7. 待整改；
8. 今日已完成。

同一任务只能在主列表中出现一次。任务同时命中多个风险时，在同一张卡片展示多个风险摘要，不能复制成多张卡片。

待上传和待同步是数据状态，可以与任务业务状态同时存在，不应把任务从原业务分组中移走。首页另提供同步汇总入口。

## 11. 排序

在线排序由服务端 Feed 返回，当前优先级为：

```text
进行中的现场任务
→ 临近预约且即将到场
→ 已超时或高 SLA 风险
→ 有效整改任务
→ 其他今日任务
```

同一优先级内，服务端结合预约开始时间、截止时间和等待时长返回稳定顺序及中文原因。

客户端不得根据卡片颜色、设备距离或本地更新时间重新计算任务优先级。离线时保持最近一次服务端顺序，并显示快照时间。

## 12. 任务卡片

每张卡片至少展示：

- 预约时间窗口或“待预约”；
- 当前业务状态；
- 工单号；
- 任务标题；
- 客户脱敏姓名；
- 地址摘要；
- 可靠距离或行政区域摘要；
- SLA 或业务截止时间；
- 工作包状态；
- 本地保存、上传和同步状态；
- 责任或版本风险；
- 下一动作；
- 数据截至时间。

卡片不得展示：

- 完整手机号；
- 与当前任务无关的完整地址；
- 内部 Task ID、UUID 和英文枚举；
- 项目内部价格、审核意见或其他师傅信息；
- 永久文件地址；
- 原始异常和技术诊断。

联系电话只在任务详情中按当前责任和字段策略临时展示。

## 13. 距离和位置边界

首页不持续采集师傅位置。

距离仅在存在可靠、授权且带更新时间的结果时展示，并注明：

- 距离类型，例如路线距离或直线距离；
- 数据截至时间；
- 结果不可用时的业务说明。

距离未知时显示区域摘要，不使用设备上一次位置或固定默认值伪造距离。任务排序不能仅由客户端距离决定。

## 14. 下一动作

卡片下一动作只能来自：

- 在线服务端 allowed-actions；
- 离线时仍在有效期内的工作包 allowedOfflineActions。

常见入口：

- 查看任务；
- 联系客户；
- 确认预约；
- 导航；
- 到场签到；
- 继续作业；
- 继续填写；
- 继续拍摄；
- 处理待上传；
- 提交前检查；
- 处理整改；
- 处理冲突。

首页主操作只负责进入正式页面，不直接执行到场、提交、完成或整改结论等高风险命令。

离线且没有有效工作包时，只允许查看缓存摘要和进入受限详情，不能显示可执行按钮。

## 15. 服务器快照与本机状态合并

首页由两类事实组成：

```text
服务器任务快照
├── 当前责任
├── 当前任务和业务状态
├── 预约、SLA、整改和优先级
├── allowed-actions
└── tombstone / authority version

本机运行状态
├── 工作包
├── 本地草稿
├── 上传队列
├── 离线命令
└── 冲突与隔离状态
```

本机状态只能叠加“已保存到本机、待上传、待同步、冲突、已隔离”等信息，不能把服务器任务改为已完成、已预约、已提交或已复核。

合并必须使用稳定任务标识、责任版本、工作包版本和服务器版本，不使用标题、工单号文本或列表位置猜测关联。

## 16. 本地缓存和首屏加载

冷启动时：

1. 先验证本机账号和设备数据隔离边界；
2. 读取已加密的最后任务快照；
3. 立即显示快照时间和离线可信度；
4. 在线时异步刷新服务端 Feed；
5. 增量处理新增、更新和 tombstone；
6. 将本机工作包和同步状态重新叠加；
7. 只在新快照完整可用后替换当前服务端快照。

不得在刷新开始时清空可用缓存，也不得在部分接口失败时用不完整结果覆盖完整旧快照。

缓存没有任何可用快照且当前离线时，页面必须明确显示“尚未下载任务，联网后才能获取”，不能显示“今日无任务”。

## 17. 工作包状态

每张任务卡按真实状态显示：

- 未准备离线；
- 准备中；
- 已准备，可离线查看；
- 已准备，可执行指定离线动作；
- 即将过期；
- 已过期；
- 责任待确认；
- 已撤销；
- 本地成果已隔离。

“已准备”必须来自完整工作包校验，不等于仅缓存了任务卡片。

工作包过期、配置摘要不一致或责任版本变化时，客户端不得继续依据旧动作清单执行新业务命令。

## 18. 改派、撤权和 tombstone

服务端返回任务责任结束、撤销或失权 tombstone 时：

### 18.1 没有本地未同步成果

- 从当前任务列表移除；
- 清理不再需要的敏感缓存和下载授权；
- 通过非敏感提示说明任务已调整；
- 不继续展示客户电话、详细地址和新资料。

### 18.2 存在本地未同步成果

- 立即禁止继续提交；
- 清除任务卡中的客户敏感信息；
- 将本地草稿、文件和命令放入只读隔离区；
- 展示“任务责任已变化，需要处理本地成果”；
- 进入同步冲突中心；
- 由受控转交、恢复或清理流程处理，不自动删除。

旧师傅不能用离线数据覆盖新责任人的服务器事实。

## 19. 刷新与后台更新

支持：

- 进入前台时轻量刷新；
- 用户下拉刷新；
- 收到业务通知后的受控增量刷新；
- 系统允许时后台预取临近预约的工作包；
- 网络恢复后的同步和 Feed 更新。

刷新失败时保留最后可信快照并说明失败原因和重试入口。

通知深链打开任务前必须重新校验认证、上下文、责任和 allowed-actions，不能凭通知内容直接展示敏感详情。

## 20. 页面查询与本机用例

建议服务端读模型：

```text
TechnicianTodayView
TechnicianTodaySummaryView
TechnicianTodayTaskCardView
TechnicianActiveTaskView
TechnicianTaskFeedDeltaView
TechnicianScheduleSummaryView
TechnicianSyncSummaryView
```

建议本机只读投影：

```text
LocalTaskSnapshot
LocalWorkPackageStatus
LocalDraftSummary
LocalUploadSummary
LocalOfflineCommandSummary
LocalConflictSummary
```

首页本身不拥有领域写命令。允许的本机用例仅包括：

```text
RefreshTodayTasks
LoadCachedTodaySnapshot
OpenTaskDetail
OpenSyncCenter
OpenCorrectionTask
RequestWorkPackagePreparation
```

“准备离线”只请求或下载受控工作包，不改变工单、任务、预约或现场状态。

## 21. 加载和错误状态

必须覆盖：

- 首次加载；
- 有缓存并在线刷新；
- 有缓存但离线；
- 无缓存且离线；
- 今日无任务；
- 部分来源不可用；
- Feed 数据过期；
- 正在同步；
- 有待上传或待同步；
- 同步冲突；
- 工作包过期；
- 任务被改派或撤销；
- 当前上下文失效；
- TechnicianProfile 或网点成员关系失效；
- Access Token 过期且刷新失败；
- 设备被撤销；
- 客户端版本或能力不满足任务要求；
- 服务端维护或暂时不可用。

错误必须使用中文业务说明、下一步建议和可复制 traceId。不能只展示网络错误、HTTP 状态码或异常类名。

## 22. 空状态

“今日无任务”仅在服务端成功返回权威空结果时展示。

空状态文案：

```text
今天暂时没有需要处理的任务
```

可以提供：

- 查看全部工单；
- 查看消息；
- 手动刷新。

不得提供初始化演示数据、伪造任务或开发诊断入口。

无缓存且离线、权限失败、上下文失效和数据源不可用都不是空状态。

## 23. 权限与敏感信息

至少校验：

- 读取本人当前任务；
- 读取预约摘要；
- 读取整改摘要；
- 读取最小客户摘要；
- 查看工作包状态；
- 请求准备工作包；
- 查看本机同步摘要。

敏感字段按当前任务、责任、时间和字段策略返回。任务改派、账号注销、上下文失效或设备撤销后，页面缓存、下载链接和通知深链必须失败关闭。

本地数据库和文件必须加密，多账号之间不能共享快照、工作包、草稿或上传文件。

## 24. 可访问性与现场可用性

- 支持 Dynamic Type，放大后任务标题、时间窗口、SLA 和主操作不得截断；
- VoiceOver 按“时间—任务—风险—同步状态—下一动作”顺序朗读；
- 关键状态同时使用文字、图标和颜色；
- 最小触控区域不小于 44×44pt；
- 紧急任务不用大面积红色背景制造持续压力；
- 单手操作时主入口位于容易触达区域；
- 弱网刷新不阻塞查看本机草稿和上传进度；
- 不依赖动画表达完成、冲突或失权。

## 25. 审计与可观测性

客户端和服务端至少记录：

- 当前 Technician Context 和网点关系版本；
- Feed 快照、增量游标和生成时间；
- 工作包版本和有效期；
- 本机快照读取和刷新结果；
- tombstone、撤权和本地隔离结果；
- 用户进入的任务和下一动作入口；
- 同步、冲突和恢复摘要；
- 认证、设备撤销和强制升级结果；
- traceId、correlationId、clientKind 和 clientVersion。

日志使用中文业务说明和稳定英文结构字段，不记录 Token、客户电话、详细地址、VIN、照片路径、表单敏感值或本地文件正文。

## 26. 验收标准

1. App 打开后首屏明确展示当前最重要任务和下一入口；
2. 今日口径、业务时区、快照时间和任务入选原因来自服务端；
3. 进行中任务、临近预约、SLA 风险、整改和其他任务排序符合服务端优先级；
4. 同一任务不会因为命中多个风险而重复展示；
5. 服务器任务状态与本机保存、上传、同步和冲突状态分别表达；
6. 离线时保留最后可信顺序并显示截至时间；
7. 无缓存且离线不会错误显示“今日无任务”；
8. 没有有效工作包时不提供伪离线执行动作；
9. 首页主操作只进入正式业务页面，不直接跨越业务步骤；
10. 改派或撤权后旧师傅不能继续读取敏感信息或提交命令；
11. 存在本地未同步成果时进入只读隔离区，不自动删除或覆盖；
12. 本机状态不能把任务伪造成已预约、已提交、已完成或已复核；
13. 部分来源失败显示不可用，不显示为零；
14. 任务卡不展示技术 ID、英文枚举、完整手机号或无关敏感数据；
15. Dynamic Type、VoiceOver、弱网和单手操作满足移动端基线；
16. 冷启动、断网、网络恢复、杀进程重启和增量 tombstone 均有明确状态；
17. 只有服务端确认的权威空结果才展示今日无任务。

## 27. 与其他规格的关系
