# NW-001 网点工作台

状态：Accepted

## 1. 页面定位

网点工作台是责任网点的履约运营首页，用于回答三个问题：

1. 本网点当前有哪些必须处理的工单和任务；
2. 哪些工单需要尽快分配师傅、处理预约、消除 SLA 风险或完成整改；
3. 用户应进入哪个正式业务页面继续处理。

网点工作台不是工单详情页、项目配置页、流程编辑器或第二套业务状态中心。它只展示由服务端读模型形成的事实、统计、风险和入口。

## 2. 目标角色

- 网点负责人：查看本网点全局履约情况、风险和人员调度结果；
- 获授权的网点运营人员：处理待派师傅、预约和调度异常；
- 获授权的网点质控或协同人员：处理审核整改、流程阻塞和外部协同异常；
- 平台人员：仅在具备跨组织支持权限时只读查看，不通过网点工作台修改责任网点。

师傅不使用该后台页面，师傅任务由原生 iOS 端承载。

当前阶段不建立独立“网点调度员”岗位。分配师傅、预约协同、整改和异常处理均由服务端 Capability 与有效网点成员关系授权，前端不得根据岗位名称自行推断权限。

## 3. 组织与项目上下文

一个网点可以服务多个项目。工作台以“当前责任网点”为第一数据边界，并允许按项目筛选或分组。

```text
当前责任网点
├── 项目 A
├── 项目 B
└── 项目 C
```

所有数据必须满足：

```text
work_order.responsible_site_id = current_site_id
```

历史上曾由本网点负责、但当前已经改派至其他网点的工单，不进入当前工作队列；其历史责任记录仍可在工单工作区查看。

## 4. 页面结构

```text
网点工作台
├── 当前网点与项目筛选
├── 今日履约概览
├── 我的工作队列
├── 今日与近期预约
├── SLA 风险
├── 审核整改
├── 流程阻塞与外部协同异常
└── 数据更新时间与局部不可用提示
```

首页不直接承载派单表单、工单详情、审核表单、整改提交或项目配置动作。点击卡片或队列后进入对应正式页面。

## 5. 今日履约概览

建议展示以下指标：

- 今日新接入本网点；
- 待分配师傅；
- 今日有预约；
- 当前处理中；
- 今日完成；
- 即将超时；
- 已超时；
- 待整改或待复核。

指标来自预先构建的读模型，不允许前端对工单、任务、SLA 和整改记录进行临时拼接统计。

每个指标必须包含：

- 数值；
- 统计口径说明；
- 数据更新时间；
- 对应业务列表入口；
- 来源不可用时的明确状态。

来源失败时显示“暂不可用”，不得显示为 0。

## 6. 工作队列

固定提供以下业务队列：

| 队列 | 进入条件 | 目标页面 |
|---|---|---|
| 待分配师傅 | 已有责任网点且当前无责任师傅，并允许继续派单 | NW-002 待分配工单 |
| 今日预约 | 预约开始时间位于当前网点今日范围 | NW-005 今日预约 |
| 当前处理中 | 当前责任网点为本网点且工单处于履约中 | NW-003 网点工单工作区列表入口 |
| 等待客户 | 当前活动流程任务或阻塞原因是等待客户 | NW-003 网点工单工作区 |
| 等待外部协同 | 当前活动流程任务或外部协同记录要求本网点跟进 | NW-003 网点工单工作区 |
| 审核整改 | 存在分配给本网点或本网点师傅的有效整改任务 | NW-006 整改处理 |
| SLA 风险 | 存在即将超时或已经超时的有效 SLA 计时器 | 风险工单列表或 NW-003 |
| 流程阻塞 | 当前流程无法推进且需要本网点处理 | NW-003 网点工单工作区 |
| 调度异常 | 师傅不可用、预约冲突、能力不匹配或派单命令失败 | NW-007 产能与异常 |

“处理中”和“审核整改”可以同时成立。整改是任务维度，不替代工单生命周期状态。

禁止使用含义模糊的“无人负责”。必须明确区分：

- 已派本网点但待分配责任师傅；
- 已有责任师傅但当前流程任务无人领取；
- 项目岗位人员快照缺失；
- 当前任务处理人无效或已离职。

## 7. 待分配师傅口径

进入“待分配师傅”队列必须同时满足：

```text
responsible_site_id = current_site_id
responsible_technician_id IS NULL
site_responsibility_status = ACTIVE
work_order_allows_assignment = true
```

以下工单不得进入普通待派队列：

- 尚未分配责任网点；
- 当前责任网点不是本网点；
- 已开始现场作业且不允许普通改派；
- 已关闭、取消或终止；
- 正在执行网点改派命令；
- 责任状态存在冲突，需要平台协调。

工作台仅提供入口，不在卡片中直接选择师傅。

## 8. 预约区

预约区按以下时间范围展示：

- 今日；
- 明日；
- 未来 7 天。

每条预约至少展示：

- 工单号和项目；
- 客户或服务地址摘要；
- 预约时间窗；
- 当前责任师傅；
- 预约确认状态；
- 冲突或即将迟到提示。

排序优先级：

1. 已发生冲突；
2. 即将开始；
3. 未确认；
4. 其余按预约开始时间升序。

## 9. SLA 风险区

风险分为：

- 即将超时；
- 已经超时；
- 暂停等待确认；
- 升级处理中。

页面展示必须来源于 SLA Projection，至少包含：

- SLA 名称；
- 当前计时阶段；
- 截止时间或超时时长；
- 当前责任层；
- 推荐进入的正式页面。

工作台不得自行修改 SLA 计时器，也不得在首页直接执行升级、豁免或重置。

## 10. 审核整改区

整改区展示有效整改任务，不把“整改中”建模成唯一工单状态。

至少区分：

- 待接收整改；
- 整改处理中；
- 待提交；
- 待复核；
- 复核退回。

点击后进入 NW-006 或工单工作区内对应整改任务。首页不直接提交整改结果。

## 11. 流程阻塞与外部协同

可展示的原因包括：

- 等待客户确认；
- 等待备件；
- 等待供应商；
- 等待外部系统确认；
- 回传失败需要人工处理；
- 当前任务缺少必填资料；
- 当前任务处理人不可用。

事实源必须是当前流程任务、外部协同记录或集成异常记录，不允许根据备注文本推断。

## 12. 筛选与分组

首页支持：

- 项目；
- 服务区域；
- 服务类型；
- 风险等级；
- 当前责任师傅；
- 时间范围。

默认展示当前网点全部项目。筛选只影响读模型展示，不改变责任关系。

## 13. 页面读模型

建议读模型：

```text
SiteWorkbenchProjection
- siteId
- generatedAt
- sourceHealth[]
- summaryMetrics[]
- queueSummaries[]
- upcomingAppointments[]
- slaRisks[]
- correctionTasks[]
- blockedTasks[]
- dispatchExceptions[]
```

每个队列项至少包含：

```text
- queueType
- count
- highestRiskLevel
- oldestWaitingSince
- targetRoute
- filterToken
- sourceStatus
```

读模型必须使用当前责任关系，而不是仅依赖工单历史事件或创建网点。

## 14. 查询契约

建议查询：

```text
GET /site-portal/workbench
GET /site-portal/workbench/queues
GET /site-portal/workbench/appointments
GET /site-portal/workbench/sla-risks
GET /site-portal/workbench/corrections
GET /site-portal/workbench/exceptions
```

查询必须自动限定当前用户可访问网点，不接受客户端传入任意 siteId 后越权读取。

## 15. 命令边界

工作台本身不直接执行高风险业务命令。用户进入正式页面后，网点侧可执行：

- 接收或拒绝网点责任任务（仅在业务规则允许时）；
- 首次分配责任师傅；
- 在允许阶段改派责任师傅；
- 处理预约；
- 处理当前流程任务；
- 提交整改；
- 发起平台协调。

网点侧禁止执行：

- 分配或改派责任网点；
- 将工单直接转交其他网点；
- 修改项目人员快照；
- 修改项目履约配置或发布版本；
- 绕过候选规则直接选择其他网点师傅；
- 在现场作业开始后执行普通改派；
- 修改平台审核结果或删除责任历史。

跨网点流转必须由平台通过 AD-004 的网点改派命令完成。

## 16. 权限

| 能力 | 网点负责人 | 获授权运营人员 | 获授权质控/协同人员 | 师傅 | 平台只读支持 |
|---|---:|---:|---:|---:|---:|
| 查看本网点全局指标 | 是 | 是 | 是 | 否 | 按授权 |
| 查看待派师傅队列 | 是 | 是 | 只读 | 否 | 按授权 |
| 分配师傅 | 是 | 是 | 否 | 否 | 否 |
| 查看整改队列 | 是 | 只读 | 是 | 否 | 按授权 |
| 查看 SLA 风险 | 是 | 是 | 是 | 否 | 按授权 |
| 改派责任网点 | 否 | 否 | 否 | 否 | 仅平台正式页面 |

权限检查必须同时验证用户角色、网点成员关系和工单当前责任网点。

## 17. 页面状态

必须覆盖：

- 正常；
- 全部为空；
- 单个来源为空；
- 单个来源不可用；
- 全部来源不可用；
- 无网点成员关系；
- 只读；
- 权限刚被撤销；
- 数据已过期；
- 工单责任刚发生变化；
- 命令执行后投影尚未刷新。

若用户点击后发现工单已改派，目标页面应提示“当前责任网点已变化”，并禁止继续执行本网点命令。

## 18. 验收标准

1. 一个网点可在同一工作台查看多个项目，并可按项目筛选；
2. 所有队列均基于当前责任网点，不混入已改派工单；
3. 待派网点与待派师傅不会混为一个状态；
4. 工作台不提供改派责任网点或跨网点转单入口；
5. 指标来源失败时显示不可用，不显示为零；
6. 工作台与正式业务页面不一致时，以正式页面最新状态为准；
7. 所有处理动作通过深链接进入正式业务页面；
8. 网点不能访问其他网点工单；
9. 审核整改与工单生命周期状态可同时展示；
10. 页面读模型、权限和命令边界与 DEC-005、DEC-006 一致。
