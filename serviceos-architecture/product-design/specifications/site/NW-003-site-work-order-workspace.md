# NW-003 网点工单工作区

状态：Accepted
设计等级：A

## 1. 页面定位

网点工单工作区是当前责任网点处理单张工单的权威业务页面。

它负责在一个明确上下文中组合本网点履约所需的：

- 当前流程阶段和任务；
- 责任网点与责任师傅；
- 客户联系和预约；
- 现场作业摘要；
- 表单、资料和整改；
- SLA、阻塞和异常；
- 与本网点有关的外部协同；
- 业务操作记录。

它不是 Admin 工单工作区的缩小复制，也不是接口对象详情页。页面只展示当前网点完成履约所需且经过授权的事实，不展示平台内部治理、其他网点、价格或技术诊断信息。

## 2. 目标用户

- 网点负责人；
- 获授权的网点运营人员；
- 获授权的网点质控或协同人员；
- 获授权的资料补充人员；
- 具备跨组织支持权限的平台只读人员。

师傅不使用该后台工作区。师傅现场任务由原生 iOS 客户端承载。

当前阶段不建立独立“网点调度员”岗位。页面可见区块和动作由服务端 Capability、有效网点成员关系、字段策略和工单当前责任共同决定。

## 3. 数据范围

页面访问必须同时满足：

- 当前用户属于当前有效网点上下文；
- 当前网点仍是工单的有效责任网点；
- 当前用户拥有工单读取权限；
- 请求的资料、联系人和操作记录满足字段级权限。

曾经负责过该工单不等于当前仍可访问。平台改派责任网点后：

- 旧网点刷新页面必须失败关闭；
- 旧深链接、缓存、下载地址和命令必须失效；
- 页面不得继续显示缓存数据并允许操作；
- 争议审计如确有需要，必须通过独立的最小字段授权入口提供。

## 4. 首屏必须回答的问题

用户进入页面后，应立即判断：

- 这是什么工单，属于哪个项目和服务类型；
- 当前进行到哪个业务阶段和任务；
- 当前责任网点和责任师傅是谁；
- 当前任务由谁处理；
- 客户预约和联系状态是什么；
- 是否存在 SLA、资料、整改、师傅或外部协同风险；
- 为什么被阻塞；
- 当前网点下一步可以执行什么动作；
- 哪些事项必须由平台、师傅、审核人或客户处理。

页面必须用业务语言回答：

> 本网点现在需要为这张工单做什么，谁负责，什么时候完成，存在什么风险。

## 5. 页面结构

```text
对象头部与主要动作
→ 当前责任、任务与风险摘要
→ 履约进度
→ 当前任务
→ 预约与联系
→ 师傅分配和现场作业
→ 表单、资料与整改
→ 异常与外部协同
→ 工单动态
→ 右侧业务上下文
```

页面默认突出当前任务和最高风险，不把全部模块等权平铺成超长字段页。

## 6. 对象头部

必须展示：

- 工单号；
- 项目名称；
- 服务类型；
- 客户授权摘要；
- 服务区域；
- 当前阶段；
- 当前任务；
- 当前任务处理人或处理对象；
- 当前责任网点；
- 当前责任师傅；
- 预约摘要；
- SLA 状态和截止时间；
- 当前最高优先级阻塞；
- 页面首要动作和更多操作。

不得展示：

- 内部流程实例 ID、任务 ID 或配置包 ID；
- 原始英文枚举；
- 其他网点候选和责任历史明细；
- 总部内部负责人和未授权备注；
- 对上价格、其他网点成本或平台毛利；
- 外部连接凭据、技术队列状态或完整异常堆栈。

## 7. 责任、任务与风险摘要

页面必须区分：

```text
履约组织责任：当前责任网点
现场执行责任：当前责任师傅
流程任务责任：当前任务处理人或处理对象
```

项目客服经理、项目经理和项目助理属于工单项目岗位人员快照，不是 Network Portal 的可编辑责任。只有在当前任务确需协同且字段策略允许时，页面才展示必要联系人，不默认展示完整平台项目团队。

明确状态包括：

- 责任网点有效，待分配师傅；
- 责任网点和师傅均已确定；
- 当前阶段不需要师傅；
- 当前任务待领取或待处理；
- 预约待确认或存在冲突；
- SLA 即将超时或已经超时；
- 资料不完整；
- 整改待处理或待复核；
- 等待平台、客户或外部协同；
- 当前网点责任正在被平台调整。

不得统一显示为“无人负责”或“异常”。

## 8. 履约进度

履约进度使用工单创建时绑定的工作流版本，以中文业务阶段展示：

- 已完成阶段；
- 当前阶段；
- 后续阶段；
- 退回、整改和异常分支；
- 每个阶段的主要业务责任；
- 关键完成时间；
- 当前阻塞原因。

阶段和 Task 必须分别表达。页面不得把 Task 状态伪装成工单生命周期状态，也不得直接展示技术节点名称。

网点只能查看与本网点履约相关的阶段详情。未授权的平台审核意见、其他网点历史和系统内部节点不进入普通工作区。

## 9. 当前任务

当前任务区至少展示：

- 业务名称和目的；
- 当前处理对象和实际处理人；
- 创建时间和截止时间；
- 当前状态；
- 完成条件；
- 必填表单和资料；
- 当前阻塞原因；
- 适用 SLA；
- 当前网点允许动作；
- 需要其他责任方处理的事项。

页面不通过通用“完成任务”按钮绕过表单、资料、预约、Visit、审核或整改门禁。任务动作必须进入对应业务流程。

## 10. 师傅分配与调度

区域展示：

- 当前责任师傅；
- 首次分配或最近改派时间；
- 分配人；
- 能力和资质匹配摘要；
- 当前任务量和预约冲突摘要；
- 最近责任变更记录；
- 候选数据更新时间；
- 当前允许动作。

操作边界：

- 无责任师傅时进入 NW-002 完成逐单首次分配；
- 已有责任师傅且当前阶段允许改派时进入 NW-004；
- 现场作业已经开始后，不显示普通改派；
- 网点不能改派责任网点或选择其他网点师傅；
- 无合格候选时只能发起平台协调、拒绝接单或进入异常处置。

本区不重复实现候选计算和确认逻辑。

## 11. 预约与联系

展示：

- 当前有效预约和修订次数；
- 预约时间窗和确认状态；
- 联系记录的时间、渠道、操作者和结果；
- 客户拒绝、无法联系、爽约或改约原因；
- 预约与师傅排班冲突；
- 下一次允许联系或改约动作。

联系电话、地址和联系人按当前任务所需及字段策略展示。网点代约必须显示真实操作者，不能伪装成师傅或客户操作。

联系记录只追加。并发改约必须显示最新修订并要求用户重新确认，不得覆盖其他人员已完成的预约变更。

## 12. 现场作业

Network Portal 只展示支持协同判断的现场摘要：

- 是否已联系、签到、开始、中断或签退；
- 关键时间；
- 当前师傅；
- 现场异常；
- 数据同步和资料上传摘要；
- 当前是否仍可由网点协调。

现场定位、离线表单、原始相机采集和大文件后台上传由原生 iOS 客户端负责。Network Portal 不提供伪离线、伪签到或代替师傅完成现场步骤的兜底。

## 13. 表单与资料

展示与本网点履约相关的：

- 当前任务必填表单和完成状态；
- 资料要求、数量和校验状态；
- 已提交版本摘要；
- 授权预览；
- 缺失、校验失败或被退回的具体项目；
- 当前允许的代补或整改入口。

网点默认不能修改师傅已经提交的表单字段。只有资料项允许代补、业务处于异常场景且用户拥有相应 Capability 时，才能进入明确的代补流程。

代补必须保存实际上传人、代办对象、原因、设备、来源和时间。页面不得直接暴露对象存储 URL 或未扫描文件。

## 14. 审核与整改

展示：

- 当前审核或整改任务；
- 退回原因和检查项；
- 整改责任对象；
- 截止时间和 SLA；
- 当前轮次；
- 原始资料快照摘要；
- 新增资料和提交记录；
- 复核结果；
- 当前网点允许动作。

整改是任务维度，可以与工单“履约中”同时存在。页面不得用一个工单状态覆盖整改生命周期。

网点不能修改原审核意见、原始证据或平台审核结论。处理整改时进入 NW-006，并调用正式整改命令。

## 15. 异常与外部协同

只展示需要当前网点参与的业务事项，例如：

- 师傅不可用或资质失效；
- 预约冲突或长期无法联系客户；
- 资料补充重试失败；
- 外部协同等待网点补充信息；
- 平台发起的履约协调；
- 网点接单资格或产能异常。

普通运营页面使用中文业务状态和恢复建议，不显示 HTTP 状态码、消息队列状态或异常类名。

异常必须通过原领域动作解决，例如重新预约、补资料、改派师傅或发起平台协调。页面不提供通用“标记已处理”。

## 16. 工单动态

时间线至少包含与当前网点相关的：

- 网点责任建立或结束；
- 师傅分配和改派；
- 联系和预约；
- 到场、开始、中断和签退；
- 表单与资料提交；
- 审核、整改和复核；
- 异常产生和解决；
- 外部协同业务结果；
- 工单完成或取消。

时间线显示业务事件、操作者、发生时间和结果，不显示事件类名、原始 payload 或内部 topic。

## 17. 页面动作

可能出现的动作：

- 分配责任师傅；
- 允许阶段内改派责任师傅；
- 记录联系；
- 提议、确认或调整预约；
- 标记爽约；
- 查看当前任务；
- 授权代补资料；
- 提交整改；
- 发起平台协调；
- 按规则拒绝接单；
- 查看本网点异常。

禁止出现：

- 分配或改派责任网点；
- 选择其他网点师傅；
- 修改项目岗位人员快照；
- 修改或发布项目履约配置；
- 修改平台审核结果；
- 删除责任历史；
- 绕过领域动作直接完成任务；
- 批量派师傅或批量改派。

所有动作来自服务端 `allowed-actions`。不允许执行的动作应解释当前状态、缺少条件、正确责任方和下一步，而不是只置灰。

## 18. 页面读模型

建议使用一个主工作区读模型：`SiteWorkOrderWorkspaceView`。

主读模型直接提供：

- 对象头部和当前责任；
- 当前阶段、任务、SLA、预约和最高风险；
- 各业务区块可见性、摘要和数据新鲜度；
- 当前网点允许动作；
- 业务深链接；
- 投影延迟或区块不可用说明。

详细区块可以按需加载：

- `TASKS`；
- `APPOINTMENTS_CONTACTS`；
- `VISITS`；
- `FORMS_EVIDENCE`；
- `REVIEWS_CORRECTIONS`；
- `EXCEPTIONS_INTEGRATION`；
- `TIMELINE`。

前端不得调用多个领域目录接口后自行重建责任、阶段、SLA、整改或允许动作。某个次级区块无权限时可以隐藏或显示受控说明，但主工作区权限失败必须失败关闭。

## 19. 命令与一致性

工作区不定义通用更新命令。每个写操作委托给对应业务命令，并必须包含适用的：

- 幂等键；
- 工单或子资源预期版本；
- 当前网点上下文；
- 当前允许动作；
- 业务原因；
- 操作者；
- 审计上下文。

命令成功后，页面可以显示“结果已提交，业务视图正在更新”，但不能伪造最终状态。异步操作只有在权威责任、任务或子资源状态实际生效后才显示完成。

并发冲突、责任网点变化或 allowed-action 失效时，页面必须刷新最新事实并保留用户未提交输入供其重新判断，不得重试旧命令覆盖新状态。

## 20. 页面状态

必须覆盖：

- 正常可操作；
- 正常只读；
- 当前无活动任务；
- 待分配师傅；
- 预约冲突；
- SLA 高风险；
- 资料缺失或整改中；
- 外部协同阻塞；
- 单个次级区块不可用；
- 投影延迟；
- 异步操作处理中；
- 并发冲突；
- 当前网点责任已变化；
- 用户成员关系或权限已撤销；
- 工单已完成或取消；
- 数据不完整且无法安全操作。

## 21. 验收标准

- 首屏能说明当前阶段、任务、责任师傅、预约、SLA、阻塞和下一步；
- 页面始终限定当前责任网点，不能读取其他网点数据；
- 项目岗位人员、责任网点、责任师傅和当前任务责任不会混淆；
- 只展示本网点履约所需且经过授权的客户、资料和协同信息；
- 页面不复制 Admin 的平台配置、内部价格或全局治理能力；
- 所有动作来自服务端并进入明确业务命令；
- 网点不能改派责任网点或选择其他网点师傅；
- 现场作业开始后不提供普通改派；
- 资料代补保存真实代办关系；
- 整改和工单生命周期状态可以同时表达；
- 改派或失权后旧页面、深链接、下载和命令立即失败关闭；
- 部分区块失败不伪装为空或零；
- 1280、1440 像素宽度、200% 缩放和键盘操作可用。
