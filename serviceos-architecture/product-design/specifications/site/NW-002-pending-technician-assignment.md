# NW-002 待分配工单

状态：Accepted

## 1. 页面定位

待分配工单是责任网点内部的师傅首次调度队列，不是全部工单列表，也不是跨网点调度页面。

页面只处理同时满足以下条件的工单：

```text
责任网点 = 当前登录网点
责任师傅 = 未分配
工单仍处于允许首次派师傅的生命周期阶段
```

核心目标：

- 让获授权的网点运营人员快速发现需要派师傅的工单；
- 解释候选师傅、排除原因和调度限制；
- 完成本网点师傅首次分配；
- 对无候选、SLA 风险和预约临近工单建立明确处置入口；
- 进入网点工单工作区查看完整事实。

页面不负责：

- 改派责任网点；
- 将工单直接转给其他网点；
- 跨网点借调师傅；
- 批量派师傅；
- 处理现场作业任务；
- 修改项目配置、项目人员快照或流程定义。

## 2. 角色与权限

| 角色 | 查看 | 分配师傅 | 查看候选解释 | 发起平台协调 | 导出 |
|---|---:|---:|---:|---:|---:|
| 网点管理员 | 是 | 是 | 是 | 是 | 是 |
| 获授权的网点运营人员 | 是 | 是 | 是 | 是 | 按权限 |
| 网点普通运营人员 | 是 | 否 | 是 | 否 | 否 |
| 师傅 | 否 | 否 | 否 | 否 | 否 |

权限判断必须同时验证：

1. 当前用户是否属于当前网点；
2. 当前网点是否仍是工单责任网点；
3. 工单是否仍未分配责任师傅；
4. 工单生命周期是否允许首次分配师傅；
5. 当前用户是否拥有调度命令权限。

任一条件变化后，页面中的旧操作入口立即失效。

## 3. 页面入口

主要入口：

- 网点工作台 → 待分配师傅；
- 网点工作台 → SLA 风险中的未派师傅工单；
- 网点工单工作区 → 分配师傅；
- 网点端主导航 → 待分配工单。

入口携带的筛选条件只用于初始化页面，不改变页面事实范围。

## 4. 页面布局

```text
页面标题与统计
筛选区
风险与异常提示
待分配工单列表
分页
```

页面采用单列任务列表模式，不在表格单元格中直接放置师傅下拉框。

分配师傅必须进入独立的候选与确认流程，避免在高密度列表中误操作。

## 5. 顶部统计

显示：

- 待分配总数；
- 今日新增；
- 今日预约；
- 即将超时；
- 已超时；
- 无可用候选。

统计来自服务端读模型。

部分指标来源失败时：

- 显示“暂不可用”；
- 不得显示为 0；
- 不影响其他可用列表数据展示。

## 6. 默认筛选与排序

默认只展示当前责任网点的待分配工单。

默认排序优先级：

1. 已超时；
2. 即将超时；
3. 已到预约时间或预约临近；
4. 候选人数为 0；
5. 工单创建时间从早到晚。

用户可切换排序：

- SLA 剩余时间；
- 预约时间；
- 创建时间；
- 候选人数。

排序只改变展示，不改变调度优先级事实。

## 7. 筛选条件

支持：

- 关键词：工单号、客户名称、联系人、联系电话、服务地址；
- 项目；
- 服务类型；
- 当前流程节点；
- SLA 状态：正常、即将超时、已超时；
- 预约时间：今天、明天、未来 7 天、自定义；
- 候选状态：有候选、无候选、候选数据暂不可用；
- 能力标签；
- 行政区域；
- 创建时间。

不支持按其他网点筛选。

## 8. 列表字段

| 字段 | 说明 |
|---|---|
| 工单号 | 可进入网点工单工作区 |
| 项目 | 一个网点可同时服务多个项目，必须明确项目上下文 |
| 客户与联系人 | 展示客户名称、联系人和脱敏联系电话 |
| 服务地址 | 展示行政区域和简要地址 |
| 服务类型 | 来自工单绑定的履约配置版本 |
| 当前流程节点 | 当前需要网点履约的业务阶段 |
| 预约时间 | 无预约时显示“未预约” |
| SLA | 展示正常、即将超时、已超时及剩余时间 |
| 能力要求 | 展示关键技能、资质或资源标签 |
| 候选人数 | 展示可分配候选数；来源失败时显示“暂不可用” |
| 等待时长 | 从进入待派师傅状态开始计算 |
| 操作 | 查看候选、分配师傅、查看工单、发起协调 |

列表不展示：

- 原始 UUID；
- 英文枚举；
- 内部评分明细；
- 其他网点师傅；
- 无权限的项目配置字段；
- 可直接修改责任网点的操作。

## 9. 行状态

每行可出现以下业务标签：

- 今日新增；
- 今日预约；
- 预约临近；
- 即将超时；
- 已超时；
- 无可用候选；
- 候选数据暂不可用；
- 信息待补充；
- 平台协调中。

同一工单可以同时具有多个标签，例如“今日预约 + 即将超时 + 无可用候选”。

## 10. 候选师傅来源

候选集合必须由以下事实共同计算：

```text
工单绑定的履约配置版本
+ 项目允许参与的责任网点规则
+ 项目师傅能力与资质要求
+ 当前责任网点的有效师傅
+ 当前排班、可用状态和服务区域
+ 当前负载与时间冲突
+ 工单生命周期限制
```

候选不是“本网点所有师傅”，也不能因为无候选而自动放宽项目规则。

候选读模型至少返回：

- 师傅姓名；
- 能力与资质匹配结果；
- 当前排班状态；
- 当日任务数；
- 预约时间冲突；
- 服务区域匹配；
- 预计到达或距离信息（存在可靠数据时）；
- 可分配 / 不可分配；
- 不可分配原因。

## 11. 查看候选

点击“查看候选”打开候选面板或独立页面。

候选按以下顺序组织：

1. 可分配且无预约冲突；
2. 可分配但负载较高；
3. 当前不可分配。

默认不隐藏不可分配候选，但必须明确不可分配原因，帮助网点运营人员理解能力缺口。

候选排序仅用于辅助人工判断，不等同自动派单。

## 12. 分配师傅流程

```text
选择工单
  ↓
打开候选师傅
  ↓
选择一名可分配师傅
  ↓
查看影响与冲突
  ↓
确认分配
  ↓
执行 AssignTechnician
  ↓
刷新工单责任与任务投影
```

确认页至少显示：

- 工单号、项目和服务地址；
- 当前责任网点；
- 待分配师傅；
- 预约时间和 SLA 风险；
- 选中师傅当前任务和时间冲突；
- 分配后产生的当前流程任务；
- 操作备注。

命令成功后，该工单必须从待分配列表移除。

## 13. 命令契约

### 13.1 AssignTechnician

前置条件：

- 工单存在；
- 当前责任网点等于当前登录网点；
- 当前责任师傅为空；
- 目标师傅属于当前责任网点；
- 目标师傅满足项目配置版本中的强制能力和资质要求；
- 工单仍处于允许首次派师傅的阶段；
- 用户拥有调度权限；
- 客户端提交的工单版本与服务端一致。

命令输入建议：

```text
workOrderId
technicianId
expectedWorkOrderVersion
assignmentReason
operatorNote
```

命令结果：

- 建立责任师傅当前责任；
- 记录责任历史和操作审计；
- 更新当前流程任务处理人（仅当流程规则要求责任师傅承接）；
- 更新网点工作台和队列投影；
- 触发正式通知规则；
- 不改变责任网点；
- 不改变工单绑定的履约配置版本。

## 14. 明确禁止的命令

本页面不得执行：

- AssignSite；
- ReassignSite；
- ReassignTechnician；
- BatchAssignTechnician；
- PublishWorkflow；
- ModifyProject；
- ModifyProjectPersonnelSnapshot；
- ChangeExternalCallbackPayload。

NW-002 只处理首次派师傅。已经存在责任师傅的工单应进入 NW-004 师傅调度执行允许阶段内的单单改派。

## 15. 批量操作

允许：

- 批量选择后导出；
- 批量添加只读标记（如本地关注，若后续确认需要）；
- 批量打开统计摘要。

当前阶段明确禁止：

- 批量派师傅；
- 批量改派师傅；
- 批量改变责任网点；
- 批量跳过候选校验。

原因：每张工单的项目规则、能力要求、预约冲突、SLA 风险和候选集合均可能不同，批量派师傅会弱化人工确认和责任审计。

## 16. 无候选处理

当候选人数为 0 时，页面显示：

```text
暂无符合项目规则和当前运营条件的可用师傅
```

同时展示主要原因，例如：

- 本网点无具备所需资质的师傅；
- 符合能力要求的师傅未排班；
- 全部候选存在预约冲突；
- 服务区域不匹配；
- 候选数据来源暂不可用。

允许动作：

- 查看全部排除原因；
- 补充工单信息（进入工作区并按权限处理）；
- 发起平台协调；
- 联系网点管理员。

不允许：

- 自动选择不合格师傅；
- 直接选择其他网点；
- 绕过强制资质；
- 将“候选数据不可用”解释为“无候选”。

## 17. 发起平台协调

当本网点无法履约时，网点只能发起平台协调，不得自行跨网点转单。

协调信息至少包括：

- 工单；
- 当前责任网点；
- 协调原因；
- 无候选或能力缺口证据；
- 预约与 SLA 风险；
- 建议处理时限；
- 操作人和时间。

协调记录属于工单运行数据。

平台后续可在 Admin 工单工作区执行正式责任网点改派。网点端不能预选新网点。

## 18. 页面状态

### 18.1 正常状态

展示统计、筛选、列表和可用操作。

### 18.2 空状态

```text
暂无待分配工单
当前网点没有责任师傅未分配的工单
```

提供返回网点工作台入口。

### 18.3 筛选无结果

```text
没有符合当前筛选条件的待分配工单
```

允许清除筛选。

### 18.4 部分数据不可用

例如候选服务不可用时：

- 工单列表继续展示；
- 候选人数显示“暂不可用”；
- 禁止把未知结果显示为 0；
- 分配按钮根据命令校验能力决定是否可用。

### 18.5 无权限

显示无权限说明，不泄露其他网点数据。

### 18.6 责任网点已变化

提交命令时若责任网点已被平台改派：

```text
该工单已不再由当前网点负责，无法继续分配师傅
```

刷新后从列表移除。

### 18.7 已被其他人员处理

若责任师傅已由其他人分配：

```text
该工单已完成师傅分配，请查看最新工单状态
```

不重复提交，不覆盖已有责任。

### 18.8 版本冲突

客户端版本落后时拒绝命令，并要求刷新后重新确认。

## 19. 页面读模型

建议读接口：

```text
GET /site/work-orders/pending-technician-assignment
GET /site/work-orders/pending-technician-assignment/summary
GET /site/work-orders/{workOrderId}/technician-candidates
GET /site/work-orders/{workOrderId}/assignment-context
```

列表读模型建议包含：

```text
workOrderId
workOrderNumber
projectId
projectName
customerSummary
contactSummary
serviceAddressSummary
serviceType
currentWorkflowNode
appointmentAt
slaStatus
slaRemaining
requiredCapabilityTags
candidateStatus
candidateCount
waitingSince
coordinationStatus
workOrderVersion

```

服务端必须完成当前责任网点的数据隔离，不能仅依赖前端传入 siteId。

## 20. 审计与可观测性

记录：

- 谁查看了候选详情（按审计策略）；
- 谁选择了哪名师傅；
- 分配前后的责任状态；
- 候选校验版本；
- 分配原因和备注；
- 命令失败原因；
- 平台协调发起与处理结果。

候选排序算法变化不能改写历史责任记录。

## 21. 验收标准

1. 页面只返回当前责任网点且责任师傅为空的工单；
2. 一个网点可在同一列表中处理多个项目，但每行明确项目；
3. 用户不能查看或操作其他网点待分配工单；
4. 页面不提供责任网点修改或跨网点转单；
5. 候选集合同时消费工单绑定配置版本和当前运营数据；
6. 无候选与候选数据不可用是两种不同状态；
7. 不支持批量派师傅；
8. 分配成功后工单立即离开待分配队列；
9. 并发分配、责任网点变化和工单版本冲突均会拒绝旧命令；
10. 页面不展示原始 UUID、英文枚举或内部调试信息；
11. 所有高风险操作进入候选与影响确认流程；
12. 无法履约时只能发起平台协调，不能自行选择其他网点。

## 22. 与其他规格的关系

- NW-001 网点工作台：提供待分配数量、风险摘要和本页面入口；
- NW-003 网点工单工作区：提供单张工单完整事实和后续履约操作；
- NW-004 师傅调度：处理已经存在责任师傅后的允许阶段内单单改派；
- AD-003 工单工作区：平台侧单张工单权威工作页面；
- AD-004 派单与改派：定义平台派网点、网点派师傅和改派原子边界；
- DEC-005：定义四层责任和派单命令边界；
- DEC-006：定义网点 Portal 的组织隔离与跨网点责任边界。
