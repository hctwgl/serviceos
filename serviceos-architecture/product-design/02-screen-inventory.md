# 页面清单与设计等级

状态：Accepted

## 设计等级

### A 级：完整产品规格

必须具备完整页面、关键交互、异常状态、页面读模型、业务命令和验收标准。高保真原型仅用于复杂交互验证，不要求为每个页面批量生成图片。

### B 级：标准页面模式加业务规格

复用已接受的页面模式，但必须明确字段、筛选、动作、权限和状态。

### C 级：规范覆盖

低频、只读或治理型页面，依据统一模式实现并完成基本验收。

## 已确认的信息架构

- 不设置 Admin 一级菜单“配置中心”；
- 项目列表是唯一项目权威列表；
- Admin 工作台只聚合任务投影，不维护第二套业务状态；
- 配置待办点击后进入对应项目详情的履约配置；
- Admin 工单中心负责发现、筛选和进入，不复制工单工作区；
- 工单工作区是单张工单的权威业务工作页面；
- 项目协同人员、责任网点、责任师傅和当前任务处理人是四层不同责任；
- 平台分配责任网点，责任网点分配责任师傅；
- “待派网点”和“待派师傅”是不同业务状态；
- 改派网点会结束旧网点和旧师傅当前责任，但不会自动选择新师傅；
- 一个网点可以服务多个项目；
- 网点工作台按当前责任网点聚合，并可按项目筛选；
- 网点不能改派责任网点或把工单直接转交其他网点；
- 跨网点流转由平台通过正式网点改派命令完成；
- 网点只能操作当前由自己负责的工单；
- 网点待分配页面只处理责任网点已确定、责任师傅为空且允许首次派师傅的工单；
- 网点端首次派师傅必须逐单确认，不支持批量派师傅；
- 无候选和候选数据不可用必须区分；
- 项目团队与区域分工位于项目详情中；
- 履约配置位于项目详情中；
- 项目团队与区域分工不属于履约配置版本；
- 一个项目当前只有一个履约工作流；
- 工单根据项目和地址行政区域匹配客服经理、项目经理和项目助理；
- 工单保存匹配时的人员快照；
- 流程、SLA、审核、整改、通知和回传统一消费工单人员快照；
- 区域分工变化不静默改写历史工单；
- 项目网点准入和覆盖规则进入履约配置版本；
- 网点、师傅、排班、负载和实时产能属于运营数据，不进入履约配置版本；
- 审核与整改规则进入履约配置版本，实际审核、整改和复核属于工单运行数据；
- 项目入站、映射和回传业务规则进入履约配置版本；
- 外部连接、认证凭据和技术健康状态属于系统管理；
- 实际入站、发送、确认、失败和重试属于工单运行数据；
- 七个业务规则模块只能作为项目完整版本原子发布；
- 新工单绑定新版本，进行中工单继续使用创建时版本；
- 公共模板延期到下一期。

## 已接受跨模块决策

### DEC-001 工单项目岗位人员快照

- 流程、SLA、审核、整改、通知、回传和工单页面不得各自重新按区域匹配项目人员；
- 人员缺失统一进入“项目人员待确认”；
- 已有工单换人必须通过明确操作并保留审计记录。

### DEC-002 项目集成三层边界

- 项目履约配置只维护外部协同的业务规则；
- 系统管理维护连接、认证、安全和技术协议；
- 工单保存实际入站、回传、确认、失败和人工处理记录；
- 新配置发布不得改变已经冻结并正在重试的回传内容。

### DEC-003 项目履约配置原子发布

- 每个项目最多一个当前生效版本和一个活动草稿；
- 所有业务规则全部生效或当前生效版本完全不变；
- 校验、对比和影响分析绑定草稿并发版本；
- 草稿变化后旧发布准备结果立即失效；
- 发布失败不产生部分生效版本；
- 历史规则恢复通过创建新草稿并发布新版本完成。

### DEC-004 Admin 工作台任务投影

- 工作台只聚合服务端任务读模型，不拥有独立业务状态；
- 配置待办的事实源始终是项目履约配置；
- 所有处理动作进入正式业务页面执行；
- 同一项目多个配置问题在工作台聚合为主要待办；
- 部分来源失败显示不可用，不得显示为零；
- 目标页面状态与工作台不一致时以目标页面为准。

### DEC-005 工单责任链与派单边界

- 工单区分项目协同人员、责任网点、责任师傅和当前任务处理人；
- 不使用一个通用“负责人”字段替代四层责任；
- 平台派责任网点，责任网点派责任师傅；
- 没有责任网点时不能分配师傅；
- 改派网点原子结束旧网点和旧师傅当前责任；
- 改派师傅不改变责任网点；
- 候选基于工单绑定配置版本和当前运营数据；
- 当前阶段只支持严格条件下的批量首次派网点，不支持平台侧批量派师傅和批量改派。

### DEC-006 网点 Portal 责任边界

- 一个网点可以服务多个项目；
- 网点工作台按当前责任网点聚合；
- 网点只能调度本网点师傅，不能改派责任网点；
- 网点不能把工单直接转给其他网点；
- 跨网点流转必须回到平台；
- 网点不修改项目配置、项目人员快照或平台审核结果；
- 责任网点变化后旧网点命令立即失效；
- 网点工作台只做任务投影，不维护第二套业务状态。

## 通用管理页面

客户、项目、用户、系统角色、网点、师傅、服务目录、外部连接、消息和审计等页面，在核心页面模式确认后按 B 级或 C 级设计。

项目详情必须承载：

- 项目概览；
- 服务与合同；
- 项目团队与区域分工；
- 履约配置；
- 项目工单；
- 操作记录。
