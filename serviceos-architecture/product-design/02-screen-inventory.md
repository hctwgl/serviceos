# 页面清单与设计等级

状态：Accepted

## 设计等级

### A 级：完整产品规格

必须具备完整页面、关键交互、异常状态、页面读模型、业务命令和验收标准。高保真原型仅用于复杂交互验证，不要求为每个页面批量生成图片。

### B 级：标准页面模式加业务规格

复用已接受的页面模式，但必须明确字段、筛选、动作、权限和状态。

### C 级：规范覆盖

低频、只读或治理型页面，依据统一模式实现并完成基本验收。

## 已确认的信息架构

- 不设置 Admin 一级菜单“配置中心”；
- 项目列表是唯一项目权威列表；
- 项目团队与区域分工位于项目详情中；
- 履约配置位于项目详情中；
- 项目团队与区域分工不属于履约配置版本；
- 一个项目当前只有一个履约工作流；
- 工单根据项目和地址行政区域匹配客服经理、项目经理、项目助理；
- 工单保存匹配时的人员快照；
- 区域分工变化不静默改写历史工单；
- 公共模板延期到下一期。

## 通用管理页面

客户、项目、用户、系统角色、网点、师傅、服务目录、消息和审计等页面，在核心页面模式确认后按 B 级或 C 级设计。

项目详情必须承载：

- 项目概览；
- 服务与合同；
- 项目团队与区域分工；
- 履约配置；
- 项目工单；
- 操作记录。

4. 审核与整改；

6. 版本与发布；
7. Admin 工作台配置待办；
8. Admin 工单中心和派单链路收口。
