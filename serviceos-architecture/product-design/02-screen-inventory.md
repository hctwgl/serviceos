# ServiceOS 产品地图与页面边界

状态：Accepted（产品结构）
说明：页面名称和职责已经确认；具体字段、布局比例和次要操作仍需在真实实现中验证。

## 1. 产品关系

```text
Admin
├── 全局运营、项目、配置、审核与治理
│
├── 分配责任网点
│       ↓
Network Portal
├── 本网点履约运营、预约、派师傅与整改协作
│
├── 分配责任师傅
│       ↓
原生 iOS 师傅端
└── 个人现场作业、资料采集、上传、离线与冲突恢复
```

三个产品共享服务端业务事实、权限和契约，不共享菜单、页面或带数据范围假设的状态容器。

## 2. Admin 产品地图

### 2.1 一级导航

| 一级入口 | 用户任务 | 核心页面 |
|---|---|---|
| 工作台 | 找到今天最需要处理的任务和风险 | 运营工作台、配置待办、异常入口 |
| 工单运营 | 检索、理解和处理全局工单 | 工单中心、工单工作区、派单与改派 |
| 审核与整改 | 完成质量审核、整改和复核 | 审核队列、审核工作区、整改工作区 |
| 客户与项目 | 管理客户、车企、品牌和项目 | 项目目录、项目详情、项目团队与区域分工 |
| 组织与资源 | 管理组织、网点、师傅和资源关系 | 组织目录、网点目录、师傅档案 |
| 系统管理 | 管理用户、角色、授权与外部连接 | 用户中心、角色授权、外部连接 |
| 审计与监控 | 查询操作事实和运行异常 | 审计记录、集成异常、运行监控 |

不设置一级“配置中心”。项目履约配置从“项目详情”进入；跨项目配置问题只在工作台形成待办和深链接。

### 2.2 项目详情

```text
项目详情
├── 项目概览
├── 服务与合同
├── 项目团队与区域分工
├── 履约方案
├── 项目工单
└── 操作记录
```

履约方案入口为方案列表；每个方案详情内部包含：

```text
方案列表
└── 方案详情
    ├── 基本信息与适用范围
    ├── 服务流程
    ├── 表单与资料
    ├── SLA 与预约
    ├── 网点与资源
    ├── 审核与整改
    ├── 集成与回传
    └── 版本管理
```

每个履约方案内维护一个履约工作流。方案内的流程画布、节点配置和流转规则属于同一个设计工作区，不建立独立的流程列表、节点设计首页或流转规则菜单。

## 3. Network Portal 产品地图

| 一级入口 | 用户任务 | 核心页面 |
|---|---|---|
| 工作台 | 查看本网点跨项目的优先任务和风险 | 网点工作台 |
| 工单 | 查找和处理当前由本网点负责的工单 | 网点工单目录、工单工作区 |
| 派单与调度 | 为工单分配或调整本网点师傅 | 待分配工单、师傅调度 |
| 预约 | 协调今日和近期服务安排 | 今日预约、预约详情 |
| 整改 | 完成本网点承担的整改任务 | 整改队列、整改详情 |
| 师傅与产能 | 理解本网点人员和运行能力 | 师傅列表、产能状态 |
| 异常处理 | 处理本网点可处置的业务异常 | 异常中心、异常详情 |

Network Portal 不出现跨网点目录、项目配置、平台审核规则或技术上下文。跨网点改派只能回到 Admin。

## 4. 原生 iOS 师傅端产品地图

### 4.1 主导航

| 入口 | 用户任务 |
|---|---|
| 今日任务 | 识别今天应优先完成的任务与风险 |
| 我的任务 | 查找本人当前和近期任务 |
| 上传 | 查看待上传、上传中和失败资料 |
| 我的 | 查看账号、设备、同步和安全状态 |

### 4.2 单任务作业闭环

```text
今日任务
→ 任务详情
→ 联系与预约
→ 到场与开始作业
→ 作业步骤
→ 表单与资料采集
→ 提交前检查
→ 提交任务
→ 后台上传与服务端确认
→ 完成，或进入整改/冲突处理
```

离线恢复、上传和同步冲突不是隐藏技术页，而是贯穿任务的可信状态；用户必须随时知道数据是否已保存、是否已同步、是否需要处理。

## 5. 实施优先级

### P0：先形成产品母版

1. Admin 工单中心与工单工作区；
2. 派责任网点和解释候选；
3. Network 待分配工单与派师傅；
4. iOS 今日任务到提交前检查的在线闭环；
5. iOS 上传、离线恢复和冲突处理。

### P1：补齐完整履约运营

- Admin 工作台、审核整改和项目履约配置；
- Network 预约、整改、产能和异常；
- iOS 多轮整改、后台恢复和设备安全。

### P2：治理与低频页面

- 用户、角色、组织、网点和师傅治理；
- 外部连接、审计、运行记录和消息中心；
- 公共模板（下一期，当前不实施）。

优先级是产品验证顺序，不等同于当前工程完成状态。具体已实现范围必须查询实施状态与直接工程证据。

## 6. 页面完成标准

一个页面只有同时满足以下条件才可作为产品母版：

- 目标角色和核心任务明确；
- 使用真实服务端权限与 `allowed-actions`；
- 不显示 UUID、内部字段或原始英文枚举；
- 正常、空、错、无权限、只读和冲突状态可验证；
- 关键责任、状态、SLA 和下一步首屏可见；
- 没有演示、Token、伪造上下文和技术说明入口；
- 真实运行页面已经过产品负责人走查；
- 对应代码、契约、测试和实施状态保持一致。

页面清单不是完成声明。没有工程证据时只能表示实施目标。
