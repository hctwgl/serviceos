# 页面清单与设计等级

状态：Accepted

## 设计等级

### A 级：完整产品规格

必须具备完整页面、关键交互、异常状态、页面读模型、业务命令和验收标准。高保真原型仅用于复杂交互验证，不再要求为每个页面批量生成图片。

### B 级：标准页面模式加业务规格

复用已接受的页面模式，但必须明确字段、筛选、动作、权限和状态。

### C 级：规范覆盖

低频、只读或治理型页面，依据统一模式实现并完成基本验收。

## 已确认的信息架构

- 不设置 Admin 一级菜单“配置中心”；
- 项目列表是唯一项目权威列表；
- 履约配置位于项目详情中；
- 一个项目当前只有一份履约配置和一个履约工作流；
- 工作流、表单和 SLA 不独立发布，随项目履约配置整体发布；
- 公共模板延期到下一期。

## 通用管理页面

客户、项目、用户、系统角色、网点、师傅、服务目录、消息和审计等页面，在核心页面模式确认后按 B 级或 C 级设计，不在第一批中逐页创造新视觉。

项目详情本身必须补充正式信息架构，以承载项目概览、服务与合同、项目成员、履约配置、项目工单和操作记录。

## 下一期候选

以下内容不进入当前阶段：

- 公共流程模板；
- 公共表单模板；
- 公共资料要求模板；
- 公共 SLA 模板；
- 公共通知模板；
- 公共业务字典。

下一期在确认跨项目复用需求后，统一评估“系统管理 / 公共模板”，不恢复含义模糊的“配置中心”。

## 当前设计顺序

1. 修正项目、履约配置和工作流的领域关系；
2. 项目详情与履约配置入口；
3. 履约配置概览；
4. 项目唯一流程设计工作区；
5. 项目表单与资料；
6. SLA 与预约；
7. 版本与发布；
8. Admin 工作台配置待办；
9. Admin 工单中心和派单链路收口。

先完成页面规格、状态、读模型和命令契约。只有复杂交互存在理解分歧时，再生成少量最终高保真原型。
